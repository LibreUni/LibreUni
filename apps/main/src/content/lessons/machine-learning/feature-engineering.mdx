# Feature Engineering

Preprocessing transforms raw data into numerical matrices optimized for training.

## Feature Transformation and Imputation

- **Ordinal Encoding**: maps categories to ordered integers.
- **One-Hot Encoding**: creates binary columns for categories.
- **Imputation**: replaces missing values using training statistics.

## Feature Scaling

Disparate scales bias distance-based models (KNN, SVM). We scale features via:
1. **Min-Max Scaling**: maps to $[0, 1]$: $x_{\text{scaled}} = \frac{x - x_{\text{min}}}{x_{\text{max}} - x_{\text{min}}}$.
2. **Standardization**: centers by mean and variance: $x_{\text{scaled}} = \frac{x - \mu}{\sigma}$.

## Feature Selection and Correlation

Linear relationships are analyzed using a **Correlation Matrix** of Pearson coefficients in the range $[-1, 1]$. Below is an example mapping features from a real estate dataset:

| | House Size | Num Bedrooms | Distance to Center |
|---|---|---|---|
| **House Size** | 1.00 | 0.85 | -0.60 |
| **Num Bedrooms** | 0.85 | 1.00 | -0.45 |
| **Distance to Center** | -0.60 | -0.45 | 1.00 |

The diagonal $1.00$ represents self-correlation. The coefficient $0.85$ indicates a strong positive relationship between size and bedrooms, while $-0.60$ shows a negative relationship with center distance (further houses tend to be smaller).

## Example: Scaling and Encoding

The following example demonstrates categorical one-hot encoding and standardization:

<CodeRunner client:load
  description="Scale continuous features using standardization and encode categorical features using one-hot encoding."
  code={`import pandas as pd
from sklearn.preprocessing import StandardScaler, OneHotEncoder

### Data Leakage

Preprocessing parameters must fit only on training data. Fitting on all data leaks test set statistics, inflating validation scores.
