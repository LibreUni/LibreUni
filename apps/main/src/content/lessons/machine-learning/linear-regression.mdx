# Linear Regression

Linear regression is a supervised learning algorithm used to model the relationship between a dependent variable $y$ and one or more independent variables $X$. At its core, the model assumes that the relationship is approximately linear. Despite its simplicity, it remains one of the most powerful and widely used tools in a data scientist's arsenal.

## The Model Representation

The hypothesis function $h_\theta(x)$ is defined as:

$$ h_\theta(x) = \theta_0 + \theta_1 x_1 + \theta_2 x_2 + \dots + \theta_n x_n $$

Here:
- $h_\theta(x)$ is the predicted output.
- $\theta_0$ is the **intercept** (or bias term).
- $\theta_1, \dots, \theta_n$ are the **weights** (parameters) for each feature.
- $x_1, \dots, x_n$ are the input features.

### Vectorization: The Power of Linear Algebra

In practice, we don't handle these variables individually. Using vector notation, where $\theta = [\theta_0, \theta_1, \dots, \theta_n]^T$ and $x = [1, x_1, \dots, x_n]^T$:

$$ h_\theta(x) = \theta^T x $$

Vectorization isn't just a notation convenience; it's computationally essential. Modern hardware (CPUs/GPUs) can perform matrix-vector multiplication much faster than iterative loops (FOR loops) thanks to Parallel Processing and SIMD (Single Instruction, Multiple Data).

## The Cost Function: Ordinary Least Squares (OLS)

To find the optimal parameters $\theta$, we need to measure how well our model fits the data. The most common metric is the **Mean Squared Error (MSE)**, also known as the $L2$ loss. The cost function $J(\theta)$ is defined as:

$$ J(\theta) = \frac{1}{2m} \sum_{i=1}^{m} (h_\theta(x^{(i)}) - y^{(i)})^2 $$

### Why "Squared" Error?
1. **Differentiability**: The squared function is smooth and differentiable everywhere, making it ideal for gradient-based optimization.
2. **Sensitivity**: Squaring the error penalizes larger deviations more heavily than absolute error ($|y - \hat{y}|$).
3. **Statistical Foundation**: Under the assumption that errors are normally distributed, OLS is the Maximum Likelihood Estimator.

## Optimization: Gradient Descent

Gradient descent is an iterative optimization algorithm used to minimize $J(\theta)$. Imagine standing on a hill in a thick fog; you can find the bottom by feeling the slope of the ground and walking in the direction that goes down.

The update rule for each parameter $\theta_j$ is:

$$ \theta_j := \theta_j - \alpha \frac{\partial}{\partial \theta_j} J(\theta) $$

Where $\alpha$ is the **Learning Rate**. Calculating the partial derivative gives:

$$ \theta_j := \theta_j - \alpha \frac{1}{m} \sum_{i=1}^{m} (h_\theta(x^{(i)}) - y^{(i)}) x_j^{(i)} $$

### The Learning Rate ($\alpha$)
- **Too Small**: Convergence will be extremely slow.
- **Too Large**: The algorithm can overshoot the minimum, potentially diverging or oscillating.

## Implementation from Scratch with NumPy

Let's implement a simple Linear Regression class that uses Gradient Descent.

<CodeRunner client:load
  code={`import numpy as np

class SimpleLinearRegression:
    def __init__(self, lr=0.01, n_iters=1000):
        self.lr = lr
        self.n_iters = n_iters
        self.weights = None
        self.bias = None

    def fit(self, X, y):
        n_samples, n_features = X.shape
        self.weights = np.zeros(n_features)
        self.bias = 0

        for _ in range(self.n_iters):
            model_prediction = np.dot(X, self.weights) + self.bias
            # Compute gradients
            dw = (1 / n_samples) * np.dot(X.T, (model_prediction - y))
            db = (1 / n_samples) * np.sum(model_prediction - y)

            # Update parameters
            self.weights -= self.lr * dw
            self.bias -= self.lr * db

    def predict(self, X):
        return np.dot(X, self.weights) + self.bias

# Test the implementation
X = np.array([[1], [2], [3], [4]])
y = np.array([2, 4, 6, 8])

model = SimpleLinearRegression()
model.fit(X, y)
print(f"Predicted weight: {model.weights[0]:.2f}")
print(f"Predicted bias: {model.bias:.2f}")`}

Predicted bias: 0.00"

## Advanced Case: The Normal Equation

For some linear regression problems, we can solve for $\theta$ analytically. This is done by setting the derivative of $J(\theta)$ with respect to $\theta$ equal to zero. The resulting formula is the **Normal Equation**:

$$ \theta = (X^T X)^{-1} X^T y $$

| Method | Gradient Descent | Normal Equation |
| :--- | :--- | :--- |
| **Learning Rate** | Need to choose $\alpha$ | No need |
| **Iterations** | Many iterations | One-step solution |
| **Complexity** | $O(k n^2)$ | $O(n^3)$ (matrix inversion) |
| **Scalability** | Works well for large $n$ | Slow if $n$ is large |

## Evaluating Regression Models

Accuracy isn't the right metric for regression because our predictions are continuous values. Instead, we use:

1. **Mean Squared Error (MSE)**: Average of squared residuals. $\frac{1}{m}\sum(y-\hat{y})^2$.
2. **Root Mean Squared Error (RMSE)**: Square root of MSE. It brings the error unit back to the scale of $y$.
3. **Mean Absolute Error (MAE)**: Average of absolute residuals. More robust to outliers than MSE.
4. **R-Squared ($R^2$)**: The "Coefficient of Determination". It measures the proportion of variance in $y$ explained by $X$. Values range from 0 to 1.

## Assumptions of Linear Regression

Linear regression is a parametric model, meaning it makes assumptions about the data:
1. **Linearity**: The relationship between $X$ and $y$ must be linear.
2. **Independence**: Observations should be independent of each other (no autocorrelation in time-series data).
3. **Homoscedasticity**: The variance of the error terms (residuals) should be constant across all levels of $X$.
4. **Normality of Residuals**: For hypothesis testing (p-values), the errors should be normally distributed.

If these assumptions are violated, your model's predictions and interpretability might suffer. In such cases, one might look toward **Polynomial Regression** or **Generalized Linear Models (GLM)**.
