# Logistic Regression

Despite its name, **Logistic Regression** is a classification algorithm. It is arguably the most important foundational algorithm in classification, used to predict the probability that a given input $x$ belongs to a specific category (e.g., "Yes" or "No", "Spam" or "Not Spam"). Instead of predicting a continuous value, it predicts a probability $p \in [0, 1]$.

## Why not Linear Regression for Classification?

If we try to use linear regression for classification by fitting a straight line to binary targets (0 and 1), we encounter two major problems:
1. **Outliers**: A single outlier far from the decision boundary can drastically shift the line, leading to incorrect classifications.
2. **Probability Bounds**: Linear models can output values greater than 1 or less than 0, which have no meaning in the context of probability.

Logistic regression solves this by passing the linear output through a **Sigmoid Activation**.

## The Sigmoid Function

To map any real-valued number to the interval $[0, 1]$, we use the **Logistic Function**, often called the **Sigmoid Function**:

$$ g(z) = \frac{1}{1 + e^{-z}} $$

Properties of the Sigmoid:
- It maps $(-\infty, \infty)$ to $(0, 1)$.
- At $z=0$, $g(z) = 0.5$.
- As $z \to \infty$, $g(z) \to 1$.
- As $z \to -\infty$, $g(z) \to 0$.

The hypothesis function for logistic regression is then:

$$ h_\theta(x) = g(\theta^T x) = \frac{1}{1 + e^{-\theta^T x}} $$

## Decision Boundaries

When we use the model for prediction, we typically set a threshold (usually 0.5):
- Predict $y=1$ if $h_\theta(x) \geq 0.5$ (which implies $\theta^T x \geq 0$)
- Predict $y=0$ if $h_\theta(x) < 0.5$ (which implies $\theta^T x < 0$)

The boundary where $\theta^T x = 0$ is the **Decision Boundary**. It can be linear OR non-linear (if we include polynomial features).

## The Cost Function: Log Loss (Binary Cross-Entropy)

In linear regression, we used Mean Squared Error (MSE). However, if we use MSE with the Sigmoid function, the cost function becomes non-convex, meaning gradient descent could get stuck in local minima. 

Instead, we use **Log Loss**:

$$ J(\theta) = -\frac{1}{m} \sum_{i=1}^{m} [y^{(i)} \log(h_\theta(x^{(i)})) + (1 - y^{(i)}) \log(1 - h_\theta(x^{(i)}))] $$

**Why this works:**
- If the true label $y=1$, the second term $(1-y)\log(1-h)$ disappears. We are left with $-\log(h)$. As our prediction $h$ stays closer to 1, the cost is near 0. As it drops toward 0, the cost penalizes the model exponentially.
- If the true label $y=0$, the first term disappears. We are left with $-\log(1-h)$.

## Optimization: Gradient Descent

Even though the cost function is different, the gradient of the Log Loss function with respect to the parameters $\theta$ results in the same update rule as linear regression!

$$ \theta_j := \theta_j - \alpha \frac{1}{m} \sum_{i=1}^{m} (h_\theta(x^{(i)}) - y^{(i)}) x_j^{(i)} $$

*Note: The mathematical derivation involves the chain rule on the sigmoid and the log functions.*

## Performance Metrics for Classification

Accuracy is simply $(\text{TP} + \text{TN}) / \text{Total Data}$. While intuitive, it is dangerous for imbalanced data (e.g., detecting rare diseases). Instead, we use:

1. **Confusion Matrix**: A table showing True Positives (TP), True Negatives (TN), False Positives (FP), and False Negatives (FN).
2. **Precision**: $\text{TP} / (\text{TP} + \text{FP})$. Measures accuracy of positive predictions.
3. **Recall (Sensitivity)**: $\text{TP} / (\text{TP} + \text{FN})$. Measures the coverage of positive instances.
4. **F1-Score**: $2 \times \frac{\text{Precision} \times \text{Recall}}{\text{Precision} + \text{Recall}}$. Harmonic mean of both.
5. **ROC Curve and AUC**: Plots the True Positive Rate against the False Positive Rate at various thresholds. AUC (Area Under the Curve) measures overall model quality.

## Multiclass Classification: Softmax and One-vs-Rest

Logistic regression is binary by default. To classify $K$ classes:
- **One-vs-Rest (OvR)**: Train $K$ binary classifiers. Pick the class with the highest probability.
- **Multinomial Logistic Regression (Softmax)**: Generalize the Sigmoid to the **Softmax function**, which turns a vector of $K$ scores into a probability distribution that sums to 1:
  
  $$ \sigma(z)_i = \frac{e^{z_i}}{\sum_{j=1}^{K} e^{z_j}} $$

## Regularization

Just like linear models, logistic regression can overfit if it has too many features. We can add a penalty term to $J(\theta)$ to keep parameters small:
- **L1 (Lasso)**: Adds $\lambda \sum |\theta_j|$. Can perform feature selection by making weights zero.
- **L2 (Ridge)**: Adds $\lambda \sum \theta_j^2$. Standard for preventing overfitting.

## Interpretability and the Odds Ratio

One of the great advantages of Logistic Regression over complex "black-box" models like Deep Learning is its interpretability. 

The linear part of the model, $z = \theta_0 + \theta_1 x_1 + \dots$, can be rewritten in terms of the **logit** function (the inverse of the sigmoid):

$$ \log\left(\frac{p}{1-p}\right) = \theta_0 + \theta_1 x_1 + \dots $$

The term $\frac{p}{1-p}$ is known as the **Odds**. By taking the exponent of a weight $\theta_i$, we get the **Odds Ratio**:
- If $e^{\theta_i} = 1.2$, it means that for every 1-unit increase in feature $x_i$, the odds of the positive class increase by 20%, holding all other features constant.
- This makes logistic regression a favorite in fields like medicine and sociology where understanding *why* a prediction was made is just as important as the prediction itself.

## Conclusion

Logistic Regression is the gateway to deep learning. A single neuron in a neural network is essentially a logistic regression unit (with a bias and weights). Mastering its optimization and evaluation logic is vital for transitioning to modern AI architectures.
