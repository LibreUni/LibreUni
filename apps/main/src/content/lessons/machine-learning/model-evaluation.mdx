# Why Accuracy is Not Enough

In machine learning, "accuracy" can be a deceptive metric, especially in unbalanced datasets. Imagine a dataset where 99% of people are healthy. A model that always predicts "Healthy" will have 99% accuracy but fails the 1% of sick individuals—the exact goal of the system. To build reliable systems, we must use metrics that reflect the cost of different types of errors.

## 1. Classification Metrics: The Confusion Matrix

The Confusion Matrix is a table that describes the performance of a classification model.

| | Predicted Positive | Predicted Negative |
| :--- | :--- | :--- |
| **Actual Positive** | True Positive (TP) | False Negative (FN) |
| **Actual Negative** | False Positive (FP) | True Negative (TN) |

- **Type I Error (False Positive)**: Rejection of a true null hypothesis ("False Alarm").
- **Type II Error (False Negative)**: Failure to reject a false null hypothesis ("Miss").

### Precision and Recall
- **Precision**: $\frac{TP}{TP + FP}$
  *Goal*: Minimize false positives. (e.g., in Spam detection, we don't want to mark a real email as spam).
- **Recall (Sensitivity)**: $\frac{TP}{TP + FN}$
  *Goal*: Minimize false negatives. (e.g., in Cancer detection, we don't want to miss a sick patient).

### F1-Score: The Harmonic Mean
The F1-score balances the trade-off between Precision and Recall. Unlike the arithmetic mean, the **harmonic mean** penalizes extreme values; if either precision or recall is near zero, the F1-score will be low.
$$ F_1 = 2 \cdot \frac{\text{Precision} \cdot \text{Recall}}{\text{Precision} + \text{Recall}} $$

## 2. Discrimination and Calibration (ROC/AUC)

Models often output a probability $P(y=1|x)$. A **threshold** (default 0.5) is applied to categorize it.
- **ROC Curve**: Plots the True Positive Rate vs. False Positive Rate for all possible thresholds.
- **AUC (Area Under the Curve)**: Represents the probability that a random positive example is ranked higher than a random negative example. An AUC of 0.5 is random; 1.0 is perfect.

## 3. Regression Metrics

Measuring the distance between prediction $\hat{y}$ and reality $y$.

1. **MAE (Mean Absolute Error)**: $\frac{1}{n} \sum |y - \hat{y}|$. Robust to outliers.
2. **MSE (Mean Squared Error)**: $\frac{1}{n} \sum (y - \hat{y})^2$. Penalizes large errors heavily; differentiable.
3. **RMSE**: $\sqrt{MSE}$. Same unit as the target variable.
4. **$R^2$ (Coefficient of Determination)**: $1 - \frac{RSS}{TSS}$. The percentage of variance explained by the model. $R^2=1$ is perfect; $R^2=0$ is equivalent to predicting the mean.

## 4. Robust Validation Strategies

Evaluation on the training set is a "survival bias." We need strategies to estimate generalization.

### The Hold-out Method
Splitting data into **Train**, **Validation**, and **Test** sets.
- **Train**: Adjust weights $\theta$.
- **Validation**: Tune hyperparameters (like $\lambda$ in regularization).
- **Test**: Final estimation of performance. **NEVER** tune the model based on test results.

### K-Fold Cross-Validation
To reduce the variance of the estimate, we split data into $k$ folds.
1. Hold out fold $i$.
2. Train on the remaining $k-1$ folds.
3. Evaluate on fold $i$.
4. Repeat for $i=1 \dots k$ and average.

### Stratified K-Fold
Essential for imbalanced data. It ensures each fold has the same percentage of samples of each class as the original dataset.

## 5. Pitfalls: Data Leakage

Data leakage makes a model seem much better than it is by inadvertently using "future" or "test" information during training.

- **Scaling Leakage**: Calculating the Mean/Std on the *entire* dataset before splitting. Correct way: Calculate on Train, apply to Test.
- **Temporal Leakage**: In time-series data, using data from $t+1$ to predict state at $t$.
- **Proxy Leakage**: Including a feature that contains the answer (e.g., using "Patient ID" if IDs are assigned alphabetically by diagnosis).

## Summary Checklist
- Use **Precision/Recall/F1** for imbalanced classification.
- Use **MAE** if your data has many outliers; use **MSE** if you need to penalize large errors heavily.
- Use **Stratified Cross-Validation** to ensure your results are statistically sound.
- Monitor the gap between **Training Loss** and **Validation Loss** to detect overfitting.
- Check **Correlation Matrices** to identify potential data leakage before training.
