# The Matching Problem

Recommender systems are algorithms designed to bridge the gap between a massive catalog of items and a specific user's intent. They transform a retrieval problem into a ranking problem. These systems are typically classified by whether they leverage item characteristics (**Content-Based**) or user behavior (**Collaborative Filtering**).

## 1. Content-Based Filtering

Content-based systems suggest items similar to those a user has liked in the past. It treats the problem as a high-dimensional similarity search.

1. **Item Profiling**: Items are represented as feature vectors (e.g., TF-IDF of movie descriptions, genre flags, or metadata).
2. **User Profiling**: A user vector is generated by averaging the vectors of items they have highly rated.
3. **Similarity Casting**: The system calculates the **Cosine Similarity** between the User Profile $U$ and candidate Item Profiles $I$:
   $$ \text{sim}(U, I) = \frac{U \cdot I}{\|U\| \|I\|} = \frac{\sum w_{u,k} w_{i,k}}{\sqrt{\sum w_{u,k}^2} \sqrt{\sum w_{i,k}^2}} $$

- **Pros**: Independent of other users; can recommend extremely niche items.
- **Cons**: "Overspecialization"—the system cannot recommend items outside the user's explicit past interests.

## 2. Collaborative Filtering (CF)

Collaborative filtering operates on the "wisdom of the crowd." It ignores item metadata and focuses entirely on the **User-Item Interaction Matrix** $R$.

### Neighborhood-Based Methods
- **User-User CF**: "Users who are similar to you also liked..." It calculates similarity between rows of $R$.
- **Item-Item CF**: "Users who liked this item also liked..." It calculates similarity between columns of $R$. Item-item is generally more stable and computationally efficient because the set of items changes more slowly than the set of users.

### The Problem of Sparsity
In a real-world system (e.g., Amazon), the interaction matrix $R$ is $99.9\%$ sparse. Most users only rate a handful of items. Traditional neighborhood methods struggle when there isn't enough overlap between users to calculate meaningful similarity.

## 3. Matrix Factorization (Latent Factor Models)

Matrix Factorization (MF) solves the sparsity problem by assuming that user-item interactions can be explained by a small number of **Latent Factors**.

We decompose the sparse matrix $R$ ($m \times n$) into two lower-rank matrices: $P$ (User Factors, $m \times k$) and $Q$ (Item Factors, $n \times k$).
$$ R \approx P Q^T $$

Each of the $k$ dimensions represents a hidden concept. For movies, a latent factor might automatically represent "Level of Action" or "Art-house aesthetic," even if those labels are never explicitly provided.

### Optimization Object
We learn $P$ and $Q$ by minimizing the squared error on **known** ratings, with L2 regularization to prevent overfitting to the sparse data:
$$ \min_{P, Q} \sum_{(u,i) \in \kappa} (r_{ui} - p_u \cdot q_i)^2 + \lambda(\|p_u\|^2 + \|q_i\|^2) $$

## 4. The Cold Start Challenge

Matrix Factorization and Collaborative Filtering fail for:
- **New Users**: No history to calculate similarity or latent factors.
- **New Items**: No interactions from users to map it into the latent space.

**Solutions**:
- **Exploration**: Show popular items or a diverse set of items to gather initial signal.
- **Metadata Hybridization**: Use Content-Based methods for new items (mapping them to the latent space based on descriptions).
- **Demographic Information**: Use age/location to group new users until behavior-based data is available.

## 5. Evaluation Metrics

How do we measure if a recommendation is good?
- **RMSE/MAE**: Measures accuracy of rating predictions.
- **Precision@K / Recall@K**: Out of the top $K$ items suggested, how many did the user actually interact with?
- **NDCG (Normalized Discounted Cumulative Gain)**: Rewards the model more for putting the "correct" items at the very top of the list rather than further down.
- **Diversity and Serendipity**: Measuring how much the system helps users discover items they wouldn't have found on their own.

## 6. Deep Learning in Recommenders

Modern production systems (like YouTube/TikTok) use deep learning:
- **Neural Collaborative Filtering (NCF)**: Replacing the dot product in Matrix Factorization with a multi-layer perceptron (MLP) to capture complex, non-linear interactions between user and item embeddings.
- **Sequence Models (RNNs/Transformers)**: Treating recommendation as a sequence problem—"Given the last 10 videos watched, what is the next most likely interaction?"

## Summary
Building a recommender system is an iterative process. Start with simple **Item-Item similarity** using Jaccard or Cosine metrics. As data grows, move to **Matrix Factorization** to solve sparsity. For state-of-the-art performance, integrate **Neural Embeddings** and hybridize with content metadata to solve the cold-start problem.
