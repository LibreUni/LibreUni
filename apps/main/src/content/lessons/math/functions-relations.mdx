Mappings allow us to transition from static sets to dynamic interactions. They allow us to compare sets, group elements by shared properties, and transform data while preserving structure.

### Binary Relations

A **Binary Relation** $R$ between sets $A$ and $B$ is a subset of the Cartesian product $A \times B$. If $(a, b) \in R$, we write $aRb$.

A relation $R$ on set $A$ can be:
- **Reflexive**: $aRa$.
- **Symmetric**: $aRb \implies bRa$.
- **Transitive**: $aRb \land bRc \implies aRc$.

### Equivalence Relations and Partitions

A relation that is reflexive, symmetric, and transitive is an **Equivalence Relation**. It partitions a set into disjoint **Equivalence Classes**.

<CodeRunner 
  client:load 
  code={`def get_equivalence_classes(elements, relation):
    classes = []
    seen = set()
    for e in elements:
        if e not in seen:
            # Class [e] = {x | xRe}
            cls = {x for x in elements if (x, e) in relation}
            classes.append(cls)
            seen.update(cls)
    return classes

### Classification of Mappings

A function $f: A \to B$ is a relation where every input has exactly one output.

1.  **Injection (One-to-One)**: $f(x) = f(y) \implies x = y$.
2.  **Surjection (Onto)**: Range equals Codomain.
3.  **Bijection**: Both injective and surjective (invertible).

### Morphisms: Structural Preservation

In abstract algebra, we study functions that preserve operations:
- **Homomorphism**: Preserves algebraic structure $(f(a \cdot b) = f(a) \cdot f(b))$.
- **Isomorphism**: A bijective homomorphism.
