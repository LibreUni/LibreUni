# Algorithmic Correctness

In 1985, the Therac-25 medical linear accelerator began delivering lethal radiation overdoses to patients. The root cause was a race condition in the control software: a concurrent task could overwrite the treatment parameters after the dosimeter check but before the beam fired. The machine had passed extensive testing because the bug only manifested under precise timing conditions that test suites rarely hit. Between 1985 and 1987, six patients died.

On June 4, 1996, the Ariane 5 rocket self-destructed 37 seconds after launch, destroying its payload of four Cluster satellites (a loss of $370M). The cause: an integer overflow. The inertial reference system converted a 64-bit floating-point horizontal-velocity value to a 16-bit signed integer. On Ariane 4, the velocity values never exceeded the 16-bit range. Ariane 5's faster trajectory produced a value that did not fit. Testing had covered Ariane 4 trajectories ‚ÄĒ it could not extrapolate to Ariane 5's flight envelope.

These failures share a fundamental lesson: **testing cannot prove correctness**. A passing test suite only shows that the observed bugs are absent from the tested inputs. Edge cases that require specific timing conditions, specific input values, or specific hardware states can hide for years. Complexity analysis (big-O, recurrences) says nothing about whether an algorithm produces the right answer ‚ÄĒ only about how fast it runs when it does.

The need for formal correctness proofs emerged from precisely this gap. Robert Floyd's 1967 paper "Assigning Meanings to Programs" established the principle of associating programs with logical assertions and introduced flowcharts annotated with invariants. Tony Hoare's 1969 "An Axiomatic Basis for Computer Programming" built on Floyd's work to create a deductive system ‚ÄĒ Hoare logic ‚ÄĒ for proving that a program satisfies its specification. The method works by reducing correctness to a set of predicate-logic obligations: a precondition, a loop invariant, a well-founded variant, and a postcondition compose into a proof that the algorithm halts and produces the correct output for **every** input satisfying the precondition.

Hoare logic for imperative programs, well-founded induction for recursive algorithms, exchange arguments for greedy methods, and certificate-based verification for combinatorial optimization are the tools that close this gap.

> **Prerequisites.** First-order predicate logic (propositional and predicate calculus with quantifiers), mathematical induction (simple and strong), arrays and basic data structures, and asymptotic notation ($\Theta$, $O$, $\Omega$). Prior exposure to sorting algorithms (insertion sort, merge sort) is helpful for the worked examples but not required ‚ÄĒ they are developed from first principles.

## Hoare Logic for Imperative Correctness

The standard framework for imperative program verification is Hoare logic. A *Hoare triple* $\{P\} \; C \; \{Q\}$ asserts that if precondition $P$ holds before command $C$ executes and $C$ terminates, then postcondition $Q$ holds after execution.

<MathStatement id="hoare-axiom" kind="definition" title="Hoare triple">
For program states $\sigma$ and command $C$, $\{P\}C\{Q\}$ means: for every state $\sigma$ satisfying $P$, if $\langle C, \sigma \rangle \Downarrow \sigma'$, then $\sigma'$ satisfies $Q$. Partial correctness drops the termination requirement.
</MathStatement>

### Axiomatic Semantics Rules

The proof system for Hoare logic comprises one axiom for assignment and inference rules for composition, conditionals, and loops.

**Assignment axiom.** For any expression $e$ and variable $x$,
$$
\{Q[e/x]\} \; x := e \; \{Q\}
$$
where $Q[e/x]$ denotes $Q$ with all free occurrences of $x$ replaced by $e$. This is the *backward* rule: to establish $Q$ after assignment, the precondition must hold with $e$ substituted for $x$. For example, to prove $\{y = 5\} \; x := y + 1 \; \{x > 5\}$, compute $(x > 5)[y+1/x] \equiv y + 1 > 5 \equiv y > 4$, and the assignment rule gives $\{y > 4\} \; x := y + 1 \; \{x > 5\}$, which is satisfied by $\{y = 5\}$ because $5 > 4$.

<MathStatement id="hoare-assign" kind="definition" title="Assignment axiom">
$$
\frac{}{\{Q[e/x]\} \; x := e \; \{Q\}}
$$
</MathStatement>

<MathStatement id="hoare-seq" kind="proposition" title="Sequential composition rule">
$$
\frac{\{P\}C_1\{R\} \quad \{R\}C_2\{Q\}}{\{P\}C_1;C_2\{Q\}}
$$
</MathStatement>

<MathStatement id="hoare-cond" kind="proposition" title="Conditional rule">
$$
\frac{\{P \land B\}C_1\{Q\} \quad \{P \land \lnot B\}C_2\{Q\}}{\{P\}\;\text{if }B\text{ then }C_1\text{ else }C_2\;\{Q\}}
$$
</MathStatement>

<MathStatement id="hoare-while" kind="proposition" title="While rule">
$$
\frac{\{I \land B\}C\{I\}}{\{I\}\;\text{while }B\text{ do }C\;\{I \land \lnot B\}}
$$
</MathStatement>

The **while rule** is the heart of iterative correctness. $I$ is the *loop invariant*: a predicate that holds before the loop, is preserved by each iteration, and together with the negation of the guard implies the postcondition. The rule establishes *partial* correctness; total correctness additionally requires a *variant* $V$ ‚ÄĒ a mapping from states to a well-founded set $(\mathcal{W}, \prec)$ that strictly decreases each iteration.

<MathStatement id="total-correctness" kind="theorem" title="Total correctness for while loops">
$$
\frac{\{I \land B \land V = v\}C\{I \land V \prec v\} \quad I \implies V \in \mathcal{W}}{\{I\}\;\text{while }B\text{ do }C\;\{I \land \lnot B\}}
$$
where $(\mathcal{W}, \prec)$ is well-founded (no infinite descending chains), and $v$ is a logical variable not appearing elsewhere.
</MathStatement>

### Complete Proof: Insertion Sort

Consider insertion sort on an array $A[0..n-1]$. We prove total correctness with respect to:
- **Precondition:** $n \ge 0$ and $A$ contains $n$ elements.
- **Postcondition:** $A$ is sorted in nondecreasing order and is a permutation of the original array.

**Outer loop invariant $I_{\text{outer}}$:** After $i$ iterations ($0 \le i \le n$), the subarray $A[0..i-1]$ is sorted and contains exactly the elements originally in $A[0..i-1]$ (i.e., the prefix is a sorted permutation of the original prefix). The suffix $A[i..n-1]$ is unchanged.

**Termination measure:** $V = n - i$, a nonnegative integer that strictly decreases each iteration.

**Proof of preservation.** Assume $I_{\text{outer}}$ holds at iteration $i$ with $i < n$. Let $key = A[i]$. The inner while loop shifts elements $A[i-1], A[i-2], \ldots$ rightward until finding the correct insertion point $j+1$. The inner invariant is: after $k$ shifts, $A[j+2..i]$ contains the original elements that were at $A[j+1..i-1]$, each shifted one position right, and $key$ is less than all elements in $A[j+2..i]$. The inner loop terminates because $j$ strictly decreases from $i-1$ down to at most $-1$, bounded below by induction. After the inner loop, $A[j+1] = key$ restores the outer invariant: the prefix grows by one sorted element.

## Well-Founded Induction for Recursive Correctness

Recursive algorithms require a different proof principle. Let $T$ be the set of valid inputs with a strict well-founded partial order $\prec$ (the *induction measure*). To prove correctness of algorithm $f$, we assume correctness for all $x' \prec x$ and prove it for $x$.

<MathStatement id="induction-principle" kind="definition" title="Well-founded induction for recursive algorithms">
Let $(D, \prec)$ be well-founded. For recursive function $f : D \to O$ with specification $\langle P, Q \rangle$, if for every $x \in D$ satisfying $P(x)$:
1. Every recursive call $f(y)$ made during execution satisfies $y \prec x$ and $P(y)$,
2. Assuming each such call returns a value satisfying $Q(y)$, the computation for $x$ returns a value satisfying $Q(x)$,
then $f$ is partially correct on all inputs.
</MathStatement>

### Complete Proof: Merge Sort

<MathStatement id="mergesort-spec" kind="definition" title="Merge sort specification">
**Precondition:** $A$ is an array of length $n \ge 0$ with elements from a totally ordered set.
**Postcondition:** $\text{merge\_sort}(A)$ returns a new array $B$ that is a sorted permutation of $A$.
</MathStatement>

**Proof by induction on $n$.**

*Base case ($n \le 1$).* An array of size 0 or 1 is trivially sorted and is its own permutation. The algorithm returns it directly. ‚úď

*Inductive step.* Assume merge sort is correct for all arrays of size $m < n$. For $n > 1$, partition $A$ into left $L = A[0..\lfloor n/2\rfloor)$ and right $R = A[\lfloor n/2\rfloor..n)$. Both $|L|, |R| < n$, so by the induction hypothesis, recursive calls return sorted permutations $L'$ and $R'$.

Now prove the **merge invariant**: after $t$ output elements have been written to result $B$, the prefix $B[0..t-1]$ contains the $t$ smallest elements of $L' \cup R'$ in sorted order, and the remaining unconsumed suffixes of $L'$ and $R'$ remain individually sorted. We prove this invariant by induction on $t$.

When $t = 0$, the invariant is vacuous. Assume it holds after $t$ writes. The merge step compares the front elements $a = L'[i]$ and $b = R'[j]$. If $a \le b$, it appends $a$ to $B$, increments $i$, and sets $t' = t+1$. The new prefix $B[0..t'-1]$ is $B[0..t-1]$ followed by $a$. Since $a \le b$ and $b$ is the minimum of the remaining right suffix, $a$ is the minimum of the combined remaining elements. Thus the invariant holds for $t'$. The symmetric argument applies when $b < a$. At termination, $t = n$ and $B$ is a sorted permutation of $L' \cup R'$, hence of $A$.

The recurrence $T(n) = 2T(n/2) + \Theta(n)$ with $T(1) = \Theta(1)$ gives $T(n) = \Theta(n \log n)$. The merge uses $O(n)$ auxiliary space. ‚Ė°

## Exchange Arguments for Greedy Algorithms

Greedy algorithms require a different correctness framework because there is no loop invariant over a single execution path. Instead, we prove that the locally optimal choice is globally optimal through an *exchange argument*.

<MathStatement id="exchange-framework" kind="theorem" title="Exchange argument framework">
Let $\mathcal{I}$ be the set of feasible solutions and $f : \mathcal{I} \to \mathbb{R}$ the objective (minimization). Let $g$ be the greedy choice. If for every optimal solution $O \in \mathcal{I}$:
1. **Exchange:** Replacing $O$'s first decision with $g$ yields a solution $O' \in \mathcal{I}$,
2. **Non-worsening:** $f(O') \le f(O)$,
3. **Decomposition:** The residual instance after fixing $g$ has the same structure, and any optimal solution to the residual extends to an optimal solution to the original,
then the greedy algorithm is optimal.
</MathStatement>

### Interval Scheduling: Complete Proof

**Problem.** Given intervals $[s_i, f_i)$ with $s_i < f_i$, select a maximum-cardinality subset of pairwise non-overlapping intervals.

**Greedy rule.** Always select the interval with the smallest finish time, discard all conflicting intervals, and repeat.

<MathProof id="interval-proof" for="exchange-framework" collapsible open>
Let $G = \{g_1, g_2, \ldots, g_k\}$ be the greedy solution with $g_1$ having minimum finish time. Let $O = \{o_1, o_2, \ldots, o_m\}$ be an optimal solution indexed by increasing finish time. We prove $|G| = |O|$.

**Lemma 1.** $f(g_1) \le f(o_1)$. This follows directly from the greedy choice rule: $g_1$ is the globally minimum-finish interval, and $o_1$ has some finish time.

**Lemma 2.** There exists an optimal solution containing $g_1$. Construct $O' = (O \setminus \{o_1\}) \cup \{g_1\}$. Since $f(g_1) \le f(o_1)$, every interval that was compatible with $o_1$ starts at or after $f(o_1) \ge f(g_1)$, hence remains compatible with $g_1$. Feasibility is preserved. The cardinality is unchanged, so $O'$ is optimal and contains $g_1$.

**Induction.** Remove $g_1$ and all intervals overlapping $g_1$ from the instance. The resulting instance has the same structure (intervals with start/finish times on a line). By the induction hypothesis, the greedy algorithm applied to this residual instance selects a maximum-cardinality subset. Together with $g_1$, this gives an optimal solution to the original instance. Therefore the greedy algorithm is optimal.
</MathProof>

### Counterexample: When Greedy Fails

Exchange arguments depend on the problem satisfying an optimal-substructure property. When the local choice does not lead to a globally optimal solution, the greedy algorithm is not correct.

**Coin change counterexample.** Given coin denominations $\{1, 3, 4\}$, find the minimum number of coins to make amount $6$. The greedy rule "take the largest coin first" selects $\{4, 1, 1\}$ (3 coins), but the optimal solution is $\{3, 3\}$ (2 coins). The exchange argument fails because replacing the $4$ with a $3$ changes the remaining capacity: $\{4\}$ leaves capacity $2$ (needing $\{1,1\}$), while $\{3\}$ leaves capacity $3$ (needing another $\{3\}$). The residual problem after fixing the greedy choice does not have the same structure as the original ‚ÄĒ the capacity constraint interacts with the discrete denomination set in a way that makes the local choice irreversible.

This illustrates a necessary condition for exchange-argument correctness: the problem must have the *greedy-choice property* (the first greedy choice appears in some optimal solution) and *optimal substructure* (the residual problem after fixing the greedy choice is an independent instance of the same type).

## Certificate-Based Verification

A correctness certificate is auxiliary data that allows an independent checker to verify an algorithm's output without re-executing the algorithm. The checker must be simpler (ideally, asymptotically faster) than the solver.

<MathStatement id="certificate-def" kind="definition" title="Certificate scheme">
A certificate scheme for problem $\Pi$ with specification $\langle P, Q \rangle$ consists of:
1. For every instance $x$ satisfying $P$, a set $\mathcal{C}(x)$ of certificates,
2. A polynomial-time verifier $V(x, y, c)$ that accepts iff $c \in \mathcal{C}(x)$ and output $y$ satisfies $Q(x, y)$.
The certificate must be *complete* (every correct output has some certificate) and *sound* (no incorrect output has any certificate).
</MathStatement>

### MST Certificate by Cycle Inequalities

Given a spanning tree $T$ of $G = (V, E, w)$, the tree is minimum iff for every non-tree edge $e = (u, v)$, the weight $w(e)$ is at least the maximum-weight edge on the unique $u$-$v$ path in $T$. The certificate is the tree $T$ itself plus the $n-1$ edge-path maxima.

**Proof.** If $T$ satisfies the cycle condition, consider any non-tree edge $e$ and let $f$ be the heaviest edge on the $u$-$v$ path in $T$. The cut argument: removing $f$ partitions $V$ into two components; $e$ reconnects them; since $w(e) \ge w(f)$, the tree $T - f + e$ has weight at least $w(T)$. By repeated exchange, any spanning tree can be transformed to $T$ without decreasing weight. Conversely, if some non-tree edge $e$ violates the condition, $T - f + e$ is a lighter spanning tree, proving $T$ is not minimum. ‚Ė°

## Termination for Nested Recursion

Termination for mutually recursive functions requires a *lexicographic* well-founded measure on tuples. Let $f_1, \ldots, f_k$ be mutually recursive. Define $\Phi(x) = (m_1(x), \ldots, m_k(x))$ where each $m_i : D \to \mathbb{N}$. Prove that every possible call from $f_i$ to $f_j$ strictly decreases $\Phi$ in the lexicographic order on $\mathbb{N}^k$: the first coordinate that differs must decrease.

### Ackermann Function Termination

The Ackermann function $A(m, n)$ is defined by nested recursion:
$$
A(0, n) = n+1, \quad A(m, 0) = A(m-1, 1), \quad A(m, n) = A(m-1, A(m, n-1)).
$$

The measure $\Phi(m, n) = (m, n)$ with the lexicographic order proves termination. In the first recursive case, $A(m, 0)$ calls $A(m-1, 1)$: $(m, 0) \succ (m-1, 1)$ because $m > m-1$. In the second case, $A(m, n)$ calls $A(m, n-1)$: $(m, n) \succ (m, n-1)$ because $n > n-1$ with the first coordinate equal. The outer call $A(m-1, \cdot)$ then decreases the first coordinate. The measure is well-founded because $\mathbb{N}^2$ under lexicographic order has no infinite descending chains.

## Hoare-Style Proof for Binary Search

<MathStatement id="binary-search-proof" kind="theorem" title="Binary search total correctness">
Let $A$ be a sorted array of length $n$ and $t$ a target value. The binary search algorithm terminates with index $i$ such that $A[i] = t$, or reports failure.
</MathStatement>

**Precondition:** $A$ is sorted in nondecreasing order: $\forall i, j. 0 \le i < j < n \implies A[i] \le A[j]$.

**Invariant $I$:** $\forall i. 0 \le i < n \land A[lo] \le t \le A[hi-1]$ where $[lo, hi)$ contains the range where $t$ may reside. Formally, if $t \in A$, then $t \in A[lo..hi)$.

**Variant $V$:** $hi - lo$, a nonnegative integer.

**Proof.** Initially $lo = 0$, $hi = n$. If $n = 0$, the loop body never executes and the algorithm correctly reports failure. Assume $n > 0$ and the invariant holds. At each iteration, $mid = \lfloor (lo+hi)/2\rfloor$ (using unsigned right shift to avoid overflow). If $A[mid] = t$, return $mid$ and the postcondition is satisfied. If $A[mid] < t$, then every element in $A[lo..mid]$ is $\le A[mid] < t$ by sortedness, so $t$ cannot lie in $A[lo..mid]$. Set $lo = mid+1$, preserving the invariant and strictly decreasing $hi-lo$ because $mid+1 > lo$. If $A[mid] > t$, symmetrically set $hi = mid$. The variant decreases because $mid < hi$ when $hi > lo+1$; when $hi = lo+1$, $mid = lo$ and either $A[lo] = t$ (return) or $A[lo] \ne t$ and $lo = lo+1$ or $hi = lo$, making $hi-lo = 0$ and the loop terminates. By the loop theorem, total correctness follows. ‚Ė°

Therefore $wp \equiv x > y$.

**T6.** Let count array $C[1..k]$ hold frequencies of each key. The invariant: after processing $A[1..i]$, $C$ contains the exact frequency of each key among the processed elements, and the output array $B$ is a total order on the processed elements with multiplicities matching their original counts. The variant is $n - i$.

**T7.** Let $x, y$ be the two least frequent symbols, $f_x \le f_y \le f_z$ for any $z \ne x, y$. In an optimal code tree, take the deepest pair of siblings $a, b$ (maximum depth). If $\{a, b\} = \{x, y\}$, done. Otherwise, swap $a$ with $x$ and $b$ with $y$. The change in weighted length is $f_a \cdot (d_x - d_a) + f_x \cdot (d_a - d_x) = (f_a - f_x)(d_x - d_a)$. Since $f_a \ge f_x$ and $d_x \le d_a$ (depth is maximal), this is $\le 0$. The symmetric argument for $b, y$ gives $\le 0$. The total weighted length does not increase, so a new optimal tree has $x, y$ as deepest siblings.

**T8.** Define the measure $\Phi(x) = x$ (nonnegative integer). For $x \le 1$, no recursive calls. For $x > 1$, the calls are on $\lfloor x/2\rfloor$ and $\lfloor x/3\rfloor$, both strictly $< x$. The order $\mathbb{N}$ with $<$ is well-founded. For an upper bound, note each call reduces the argument by at least a factor of 2/3, so depth is $O(\log_{3/2} x)$. The branching is 2, so at most $2^{O(\log x)} = O(x^{\log_{3/2} 2})$ calls.

**S9.** **Certificate:** A flow $f$ and an $s$-$t$ cut $(S, T)$. **Verifier:** Check capacity constraints $0 \le f(e) \le c(e)$ for all $e$, conservation at all internal vertices, and that $\sum_{u \in S, v \in T} f(u,v) - \sum_{u \in S, v \in T} f(v,u) = \text{value}(f) = \text{capacity}(S,T)$. **Completeness:** If $f$ is maximal, by max-flow min-cut theorem there exists a cut whose capacity equals the flow value. **Soundness:** If a feasible flow $f$ and cut $(S,T)$ satisfy the equality, weak duality ensures every other flow has value $\le \text{capacity}(S,T) = \text{value}(f)$, so $f$ is maximal.

</details>
