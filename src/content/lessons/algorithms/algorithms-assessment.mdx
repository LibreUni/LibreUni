## Part I: Foundations

### 1. Input size

Choose appropriate parameters for an array, a graph (directed and undirected), a binary integer, and a string. Explain why representation changes the analysis.

<ExerciseSolution hint="Think about how different data types are measured and why bit-length matters for integers vs element count for arrays.">

Array: $n$ length. Graph: $V, E$ separately. Binary integer: $\lfloor\log_2 x\rfloor+1$ bits. String: length $\ell$ or alphabet-dependent. Pseudopolynomial algorithms are polynomial in numeric value but exponential in bit length. The distinction matters: a graph with $n$ vertices may have $O(n^2)$ edges, so an $O(V+E)$ algorithm is linear while an $O(V^2)$ algorithm is quadratic, and reporting only $n$ obscures the difference [CLRS 4e, §2.1].
</ExerciseSolution>

<MathStatement id="comparison-lower-bound" kind="theorem" title="Comparison sorting lower bound">
Any comparison-based sorting algorithm requires $\Omega(n \log n)$ comparisons in the worst case.
</MathStatement>

<MathProof id="comparison-lower-bound-proof" for="comparison-lower-bound" collapsible open>
Every comparison sort corresponds to a decision tree. Each internal node is a comparison; each leaf is a permutation. For $n$ elements, there are $n!$ possible outcomes, so the tree must have at least $n!$ leaves. A binary tree of height $h$ has at most $2^h$ leaves. Hence $2^h \ge n!$, so $h \ge \lceil \log_2(n!) \rceil$. By Stirling's approximation $\log_2(n!) = n \log_2 n - n \log_2 e + O(\log n) = \Omega(n \log n)$.
</MathProof>

<ExerciseSolution>
Decision tree height $h$. Leaves $\le 2^h$. Need $2^h \ge n!$, so $h \ge \log(n!) = \Omega(n \log n)$.
</ExerciseSolution>

### 6. Analysis types

Distinguish worst-case, expected, average-case, amortized, and output-sensitive analysis with examples.

<ExerciseSolution>

| Type | Definition | Example |
| --- | --- | --- |
| **Worst-case** | Max over all inputs of size $n$ | Quicksort $\Theta(n^2)$ when pivot is always min/max |
| **Expected** | Expectation over algorithm's random choices, worst-case input | Randomized quicksort $\Theta(n \log n)$ expected |
| **Average-case** | Expectation over a distribution of inputs | Quicksort with random permutation $\Theta(n \log n)$ |
| **Amortized** | Worst-case average over a sequence of operations | Dynamic array: $n$ insertions cost $O(n)$ total, $O(1)$ per operation amortized |
| **Output-sensitive** | Complexity depends on output size | Reporting all $k$ intersecting pairs among $n$ rectangles: $O(n \log n + k)$ |

The distinction between expected and average-case is subtle but critical: expected analysis assumes an adversarial input but random algorithm decisions; average-case assumes a random input. Amortized analysis is not probabilistic — it bounds the worst-case sequence cost divided by operation count. Output-sensitive bounds prevent the algorithm from degrading when the output is large, e.g., enumeration problems [CLRS 4e, §17.1; Kleinberg--Tardos 2005, §13.2].
</ExerciseSolution>

### 7. Greedy exchange

<MathStatement id="interval-scheduling-optimal" kind="theorem" title="Earliest-finish-time optimality for interval scheduling">
Among all feasible schedules, the greedy algorithm that repeatedly selects the interval with the earliest finish time and removes overlapping intervals produces a maximum-cardinality schedule.
</MathStatement>

<MathProof id="interval-scheduling-optimal-proof" for="interval-scheduling-optimal" collapsible>
Let $G = \langle g_1, g_2, \dots, g_k \rangle$ be the greedy schedule (sorted by finish time) and $O = \langle o_1, o_2, \dots, o_m \rangle$ an optimal schedule (also sorted). Prove by induction that for each $i$, $f(g_i) \le f(o_i)$, where $f$ is the finish time. Base: $g_1$ has the earliest finish of all intervals, so $f(g_1) \le f(o_1)$. Inductive step: assume $f(g_{i-1}) \le f(o_{i-1})$. Since $o_i$ starts after $f(o_{i-1}) \ge f(g_{i-1})$, and $g_i$ is the earliest-finish interval that starts after $f(g_{i-1})$, we have $f(g_i) \le f(o_i)$. By induction $k \ge m$, so $k = m$ and the greedy schedule is optimal [CLRS 4e, §16.1].
</MathProof>

Prove earliest-finish interval scheduling optimality.

### 9. DP recurrence

<MathStatement id="knapsack-recurrence" kind="definition" title="0/1 knapsack recurrence">
Let $DP[i][c]$ be the maximum value achievable using a subset of items $\{1,\dots,i\}$ with capacity $c$. Then:
$$
DP[i][c] = \begin{cases}
DP[i-1][c] & \text{if } w_i > c \\
\max(DP[i-1][c],\; v_i + DP[i-1][c - w_i]) & \text{otherwise}
\end{cases}
$$
with base $DP[0][c] = 0$ for all $c$, $DP[i][0] = 0$.
</MathStatement>

### 12. Randomized analysis

<MathStatement id="quicksort-expected" kind="theorem" title="Expected comparisons of randomized quicksort">
Randomized quicksort makes $O(n \log n)$ expected comparisons.
</MathStatement>

<MathProof id="quicksort-expected-proof" for="quicksort-expected" collapsible>
Let $X_{ij}$ be the indicator that the $i$th and $j$th smallest elements are compared. They are compared iff one of them is chosen as pivot before any element between them. The probability is $2/(j-i+1)$. Expected comparisons = $\sum_{i=1}^n \sum_{j=i+1}^n 2/(j-i+1) = 2 \sum_{k=2}^n \sum_{i=1}^{n-k+1} 1/k \le 2n \sum_{k=1}^n 1/k = 2n H_n = O(n \log n)$ [CLRS 4e, §7.4].
</MathProof>

### 15. BFS invariant

<MathStatement id="bfs-correctness" kind="theorem" title="BFS correctness for unweighted shortest paths">
In an unweighted graph $G = (V, E)$, BFS from source $s$ computes the shortest-path distance $\delta(s, v)$ for every vertex $v \in V$.
</MathStatement>

<MathProof id="bfs-correctness-proof" for="bfs-correctness" collapsible>
Let $d[v]$ be the distance BFS assigns to $v$. BFS processes vertices in order of non-decreasing $d$ (queue property). Prove by induction on the number of edges in the shortest path that $d[v] = \delta(s, v)$. Base: $d[s] = 0 = \delta(s, s)$. Inductive step: let $u$ be the predecessor of $v$ on a shortest $s$-$v$ path. Then $\delta(s, v) = \delta(s, u) + 1$. By induction $d[u] = \delta(s, u)$. When BFS visits $u$, it discovers $v$ (if $v$ is not already discovered) and sets $d[v] = d[u] + 1 = \delta(s, u) + 1 = \delta(s, v)$. The queue property ensures $v$ is not discovered later with a smaller $d$ [CLRS 4e, §22.2].
</MathProof>

Prove BFS computes shortest paths in unweighted graphs.

### 16. Dijkstra proof

<MathStatement id="dijkstra-correctness" kind="theorem" title="Dijkstra's algorithm correctness">
Given a directed graph $G = (V, E)$ with nonnegative edge weights $w: E \to \mathbb{R}_{\ge 0}$ and a source $s$, Dijkstra's algorithm computes the shortest-path distance $\delta(s, v)$ for every $v \in V$.
</MathStatement>

<MathProof id="dijkstra-correctness-proof" for="dijkstra-correctness" collapsible>
The proof uses the invariant: for every vertex $v$ in the settled set $S$, $d[v] = \delta(s, v)$. When a vertex $u$ is extracted from the priority queue with minimum $d[u]$, we claim $d[u] = \delta(s, u)$. Suppose not; then there is a shorter path $p$ to $u$. Let $x$ be the first vertex on $p$ outside $S$, and $y$ its predecessor on $p$ (which is in $S$). Since $y \in S$, $d[y] = \delta(s, y)$. When $y$ was processed, it relaxed edge $(y, x)$, so $d[x] \le d[y] + w(y, x) = \delta(s, x)$. Since weights are nonnegative, the path segment from $x$ to $u$ has nonnegative weight, so $\delta(s, u) \ge \delta(s, x) \ge d[x]$. But $d[u] \le d[x]$ because $x$ is still in the queue and $u$ was extracted as the minimum. Hence $d[u] \le \delta(s, u)$, contradicting the assumption that $d[u] > \delta(s, u)$. The nonnegative-weight assumption is used exactly here: it guarantees that the path segment from $x$ to $u$ cannot decrease the total weight [CLRS 4e, §24.3].
</MathProof>

Identify exactly where nonnegative weights are used in Dijkstra's correctness proof.

### 18. MST cut property

<MathStatement id="cut-property" kind="theorem" title="MST cut property">
Let $G = (V, E)$ be a connected graph with edge weights $w: E \to \mathbb{R}$. For any cut $(S, V \setminus S)$, a minimum-weight edge crossing the cut belongs to every minimum spanning tree.
</MathStatement>

<MathProof id="cut-property-proof" for="cut-property" collapsible>
Let $e = (u, v)$ be a minimum-weight edge crossing cut $(S, V \setminus S)$ with $u \in S$, $v \notin S$. Suppose $T$ is an MST not containing $e$. Adding $e$ to $T$ creates a cycle. This cycle must cross the cut at least twice: once at $e$ and once at some other edge $e'$ crossing the cut. Since $e$ is minimum-weight, $w(e) \le w(e')$. Removing $e'$ from $T \cup \{e\}$ yields a spanning tree with weight $w(T) - w(e') + w(e) \le w(T)$, so it is also an MST. This proves the cut property and directly implies Kruskal's correctness: each edge Kruskal selects is the minimum-weight edge crossing some cut that separates the current forest components [CLRS 4e, §23.1].
</MathProof>

Prove the cut property and demonstrate Kruskal correctness.

### 20. Max-flow min-cut

<MathStatement id="max-flow-min-cut" kind="theorem" title="Max-flow min-cut theorem">
In a flow network, the maximum value of a feasible flow equals the minimum capacity of an $s$-$t$ cut.
</MathStatement>

<MathProof id="max-flow-min-cut-proof" for="max-flow-min-cut" collapsible>
Let $f$ be a maximum flow. Define $S$ as the set of vertices reachable from $s$ in the residual graph $G_f$. Since $f$ is maximum, $t \notin S$ (otherwise there would be an augmenting path). Then $(S, V \setminus S)$ is an $s$-$t$ cut. For every edge $(u, v)$ with $u \in S$, $v \notin S$, the residual capacity $c_f(u, v) = 0$, meaning $f(u, v) = c(u, v)$. For every edge $(u, v)$ with $u \notin S$, $v \in S$, the residual capacity $c_f(u, v) = f(v, u)$ (since $c_f(u, v) = f(v, u)$ when $(u, v)$ is not in the original graph), so $f(v, u) = 0$. Hence the net flow across the cut equals $\sum_{u \in S, v \notin S} c(u, v) = c(S, V \setminus S)$. By flow conservation, the flow value equals the net flow across any cut, so $|f| = c(S, V \setminus S)$ [CLRS 4e, §26.2].
</MathProof>

Prove using residual reachability when no augmenting path exists.

### 23. Hashing quantifiers

Explain the quantifier order for universal hashing and what it implies about adversarial keys.

<MathStatement id="universal-hashing" kind="definition" title="Universal family of hash functions">
A family $\mathcal{H}$ of hash functions $h: U \to [m]$ is *universal* if for any distinct $x, y \in U$, $\Pr_{h \sim \mathcal{H}}[h(x) = h(y)] \le 1/m$. The quantifier order is crucial: the adversary chooses $x, y$ *before* the random choice of $h$, so no pair is guaranteed to collide with probability greater than $1/m$ [CLRS 4e, §11.3].
</MathStatement>

## Counterexamples and Failure Modes

<MathStatement id="greedy-fail" kind="lemma" title="Greedy failure on matroid-independent problems">
A greedy algorithm that selects items by decreasing value-to-weight ratio fails to solve the 0/1 knapsack problem because the optimal solution may require a different subset than the one maximizing the ratio at each step.
</MathStatement>

**Concrete counterexample.** Items: $(w=6, v=10)$, $(w=5, v=8)$, $(w=5, v=8)$. Capacity $W=10$. Greedy by density picks item 1 (density 1.67), remaining capacity 4, cannot add any other item. Total value: 10. Optimal: items 2 and 3 (value 16). The greedy choice eliminates the better combination.

**Dijkstra with negative edges.** Dijkstra settles vertex $B$ with distance 5 via $A\to B$, then later discovers $A\to C\to B$ with distance 3. The settled label was wrong because a negative edge exists. The nonnegative-weight assumption is necessary: it guarantees that settled labels are final.

**Max-flow integrality with half-integer capacities.** If capacities are integers, the Ford-Fulkerson algorithm terminates with an integer flow. If capacities are irrational, the algorithm may not terminate. The integrality theorem depends on the capacity domain.

**Merge sort on degenerate input.** Merge sort always performs $\Theta(n \log n)$ comparisons regardless of input order, unlike quicksort. This is both a strength (no worst-case quadratic behavior) and a weakness: it cannot exploit existing order. On nearly-sorted data, insertion sort runs in $O(n)$ while merge sort still requires $\Theta(n \log n)$.

**Linear probing with high load factor.** As the load factor $\alpha$ approaches 1, linear probing's expected search time grows as $O(1/(1-\alpha)^2)$. At $\alpha > 0.9$, performance degrades catastrophically. Double hashing or cuckoo hashing avoids primary clustering.

**Pseudopolynomial illusion.** The $O(nW)$ DP for 0/1 knapsack is polynomial in the *numeric value* $W$ but exponential in the *bit length* $\log W$. For $W = 2^n$, the DP takes $O(n \cdot 2^n)$ time — no better than brute force.

3. **Time-dependent routing.** Suppose edge travel times are piecewise-linear functions of departure time $t$. Prove that if the FIFO property holds (earlier departure cannot produce later arrival), Dijkstra's algorithm remains correct with a modified relaxation: $d[v] = \min(d[v], \text{travel}(d[u], u, v))$, where $\text{travel}(t, u, v)$ is the arrival time at $v$ when departing $u$ at time $t$ via edge $(u,v)$. Construct a counterexample where FIFO is violated and Dijkstra fails.

4. **Synthesis: multimodal route planner.** Design a system combining driving, transit, and walking. Each mode has its own graph, edge costs, and transfer penalties at interchange vertices. Formulate a unified graph model, state which algorithm(s) apply, and specify the certificates each query must return. Analyze the total complexity in terms of $V_{\text{drive}}, V_{\text{transit}}, V_{\text{walk}}$ and transfer edges. Prove that the unified graph maintains the same optimality guarantees as the single-mode case.

5. **Approximation vs heuristic: the gap.** The maximal-matching 2-approximation for vertex cover is guaranteed to be within a factor of 2 of optimal. Design a heuristic that may outperform the approximation on typical instances but has no worst-case guarantee. Construct an instance where the heuristic is worse than factor 2, and another where the heuristic is optimal but the 2-approximation is not.

6. **Online greedy: competitive analysis.** The greedy algorithm for online interval scheduling (accept the first interval, reject overlaps) is 1-competitive (optimal) for unit-length intervals with release times, but has unbounded competitive ratio for arbitrary lengths. Prove both claims. For the unbounded case, design a variant with a finite competitive ratio.

## Worked Example 1: DP Recurrence Derivation and Proof (0/1 Knapsack)

**Problem.** Given items $1..n$ with weights $w_i$ and values $v_i$, and capacity $W$, select a subset maximizing total value subject to $\sum w_i \le W$.

**Solution approach.** Define $DP[i][c]$ as in §9. The recurrence is stated in the [knapsack recurrence](#knapsack-recurrence) MathStatement above.

**Correctness proof.** Let $S^*$ be an optimal subset of $\{1,\dots,i\}$ with capacity $c$. Two cases: $i \notin S^*$ or $i \in S^*$. If $i \notin S^*$, then $S^*$ is an optimal subset of $\{1,\dots,i-1\}$ with capacity $c$, so $DP[i][c] = DP[i-1][c]$ suffices. If $i \in S^*$, then $S^* \setminus \{i\}$ is a feasible subset of $\{1,\dots,i-1\}$ with capacity $c - w_i$. By the cut-and-paste lemma, $S^* \setminus \{i\}$ must be optimal for its subproblem; otherwise replacing it with the optimal $(i-1, c-w_i)$ subset would improve $S^*$. Thus $DP[i][c] = v_i + DP[i-1][c-w_i]$. The recurrence takes the maximum of the two exclusive cases.

## Worked Example 2: Graph Algorithm Design (Shortest Path with Customization)

**Problem.** Given a directed graph $G = (V, E)$ with edge weights $w(e) \in \mathbb{Z}$, a source $s$, and a set $F \subseteq E$ of "faulty" edges, find the shortest path from $s$ to each $v \in V$ that uses at most $2$ faulty edges.

**Solution approach.** State augmentation: define a new graph $G'$ where each vertex is a pair $(v, k)$ with $k \in \{0,1,2\}$ counting faulty edges used. Edges in $G'$: for each original edge $(u,v) \in E$, create edge $(u, k) \to (v, k)$ with weight $w(u,v)$ if $(u,v) \notin F$, and edge $(u, k) \to (v, k+1)$ with weight $w(u,v)$ if $(u,v) \in F$ and $k < 2$. Run Dijkstra from $(s, 0)$ in $G'$.

**Correctness proof.** Every path in $G$ using at most 2 faulty edges corresponds bijectively to a path in $G'$ of equal weight from $(s, 0)$ to $(v, k)$ for $k \le 2$. The state $(v, k)$ tracks the number of faulty edges traversed; the transition rules increment $k$ exactly when a faulty edge is traversed and forbid $k > 2$. Dijkstra's correctness in $G'$ follows because all augmented edge weights are unchanged (so nonnegativity is preserved). The shortest $s$-$v$ path in the original graph with the fault constraint is $\min_{k \le 2} \text{dist}_{G'}[(v, k)]$.

**Complexity.** $|V'| = 3|V|$, $|E'| = 3|E|$, Dijkstra runs in $O(|E'| \log |V'|) = O(E \log V)$.
