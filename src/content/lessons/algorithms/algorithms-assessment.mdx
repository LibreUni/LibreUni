# Algorithms Comprehensive Assessment

This assessment evaluates whether a student can design and defend algorithms rather than recite names. Every solution should state the input model, output specification, assumptions, pseudocode or precise procedure, correctness proof, termination argument, and resource analysis. If the solution is randomized, approximate, amortized, or pseudo-polynomial, state the relevant quantifiers and parameters.

## Part I: analysis and proof

1. Choose appropriate input-size parameters for an array, a graph, a string dictionary, and a binary integer. Explain why representation changes the analysis.
2. Analyze a nested loop whose inner variable doubles from 1 through i for i=1..n, proving a tight bound.
3. Solve `T(n)=3T(n/2)+n^2` and `T(n)=T(n/3)+T(2n/3)+n`, justifying method hypotheses.
4. State a loop invariant and termination measure for binary search.
5. Give a decision-tree proof of the comparison sorting lower bound.
6. Distinguish worst-case, expected, average-case, amortized, and output-sensitive analysis with examples.

## Part II: paradigms

7. Prove earliest-finish interval scheduling by exchange.
8. Give a counterexample to density-first 0/1 knapsack and design the DP state that repairs it.
9. Derive 0/1 knapsack recurrence, table order, and reconstruction.
10. Design LCS recurrence and explain space compression versus reconstruction.
11. Prove merge sort correctness and complexity.
12. Analyze randomized quicksort comparisons with indicator variables.
13. Prove a 2-approximation for vertex cover from maximal matching.
14. Explain PTAS versus FPTAS with a scaling example.

## Part III: graph algorithms

15. State BFS invariant and prove shortest unweighted distances.
16. Run Dijkstra and identify exactly where nonnegative weights are used.
17. Prove Bellman-Ford’s path-length induction and negative-cycle test.
18. Prove MST cut property and Kruskal correctness.
19. Compare Prim and Kruskal under sparse/dense representations.
20. Prove max-flow min-cut using residual reachability.
21. Reduce bipartite matching to max flow and prove integrality.
22. Design graph certificates for shortest paths, MST, and max flow.

## Part IV: advanced reasoning

23. Explain randomized hashing quantifiers and adversarial keys.
24. Design a state graph for an optimization problem and analyze its size.
25. Give an inapproximability gap-reduction outline.
26. Explain work/span for parallel divide-and-conquer.
27. Diagnose an algorithm that returns correct values but violates its memory contract.
28. Design a benchmark that distinguishes asymptotic behavior from cache effects.

## Integrated design case

You must schedule jobs on two identical machines. Each job has release time, processing time, deadline, and a precedence relation. The objective is to maximize total value of completed jobs, with a hard memory budget for the scheduler. Some instances contain a DAG of precedence; others contain cycles and must be rejected. The service receives jobs online but can defer commitment for 100 ms to batch arrivals.

29. Formalize the feasible schedule and objective. State whether an exact polynomial algorithm is plausible and which special cases are tractable.
30. Give an exact algorithm for a restricted case, such as one machine with no release times and weighted interval compatibility, and prove it.
31. Give a heuristic or approximation for the general case and state what it does not guarantee.
32. Design data structures for ready jobs, deadlines, and selected schedules.
33. Define tests for precedence cycles, deadline boundaries, simultaneous release, and memory exhaustion.
34. Design measurements for throughput, missed deadlines, objective gap on small exact instances, and tail latency.

## Diagnostic solutions and grading guidance

1. Use n for array/string length, V/E for graph, and bit length for binary integer. Numeric value versus encoding length can turn polynomial numeric work into exponential encoded work; retain V,E when graph density matters.
2. Inner loop costs `Theta(log i)`; sum `sum log i = Theta(n log n)` using upper bound `log n` and lower bound on `i  at least  n/2`.
3. First has a=n? `n^(log_2 3)` versus n², so case 3 gives `Theta(n²)` under regularity. The second unequal recurrence has critical exponent 1 and linear combine work, giving `Theta(n log n)`, not direct standard Master form.
4. If target exists it lies in [lo,hi); each comparison retains a valid half; hi-lo strictly decreases. Empty interval implies absence and bounds iterations by O(log n).
5. n! input permutations need distinct sorted leaves; binary decision tree height h has at most 2^h leaves, so `h  at least  log(n!) = Omega(n log n)`.
6. Worst-case is max input; expected averages algorithmic randomness; average-case assumes an input distribution; amortized bounds every operation sequence; output-sensitive includes output size k, e.g. O(log n+k) reporting.
7. Let greedy interval g finish earliest; replace first interval of any optimum with g, preserving compatibility and count. Recurse on intervals starting after g.
8. Density can select a large item that blocks two lower-density items with greater total value. DP state (i,w) records prefix and capacity, partitioning by include/exclude.
9. `D[i,w]=max(D[i-1,w], v_i+D[i-1,w-w_i])` when feasible. Iterate i upward; one-row capacity downward. Trace equal values backward for reconstruction.
10. Match last characters for diagonal plus one; otherwise max of deleting one suffix character. Two rows compute length; backpointers or divide-and-conquer are needed for the sequence.
11. Induct on halves and prove the merge-prefix invariant; recurrence `2T(n/2)+O(n)` gives `Theta(n log n)`, with an O(n) buffer.
12. Pair i,j are compared only if the first pivot in their interval is an endpoint, probability 2/(j-i+1); sum indicators to O(n log n) expected comparisons.
13. Endpoints of a maximal matching form a cover of size 2|M|; every cover needs at least one endpoint per disjoint matching edge, so `OPT  at least  |M|`.
14. PTAS polynomial n for fixed epsilon; FPTAS polynomial n and 1/epsilon. Value scaling loses at most epsilon OPT and bounds rounded total value polynomially.
15. BFS discovers vertices in nondecreasing distance; when dequeued, parent path has shortest distance because any earlier undiscovered path would have entered the queue earlier.
16. Choose smallest tentative label and relax edges. Nonnegative weights ensure a path through an unsettled vertex cannot improve a settled label; negative edges invalidate the argument.
17. After k passes, all paths using at most k edges have correct labels. A simple shortest path uses at most V-1 edges; an additional relaxation reachable from source implies a negative cycle.
18. A lightest cut edge can replace another crossing edge on an MST cycle without increasing weight. Kruskal’s forest remains extendable to an MST; union-find rejects cycles and V-1 accepted edges form a tree.
19. Kruskal sorts E edges and uses union-find; Prim with heap is O(E log V) on lists and array Prim O(V²), often suitable for dense matrices. State representation costs.
20. Any flow is bounded by every cut. If residual search cannot reach t, reachable S has saturated S→T and zero T→S edges, so flow equals cut capacity and is optimal.
21. Unit source/left and right/sink capacities force each vertex once; integral capacities yield integral augmentations, mapping flow edges to matching pairs and vice versa.
22. Shortest path: valid predecessor paths and no violated edge inequality. MST: tree shape plus non-tree cycle inequalities. Flow: feasible conservation/capacity plus equal cut certificate.
23. For every fixed distinct key pair, collision probability over a random family/seed is bounded. This does not give worst-case constant lookup for a known fixed seed; keyed hashing protects against seed-unaware adversaries.
24. Define state so equal states have identical future choices and objective context. Count state dimensions and transitions; pseudo-polynomial numeric dimensions must be reported.
25. Transform a hard decision instance into an optimization instance with optimum separated by a multiplicative gap. An approximation better than the gap distinguishes cases, contradicting hardness.
26. Work is total operations; span is longest dependency path; P processors take at least max(W/P,S). A serial combine can dominate speedup.
27. Check allocation, recursion depth, retained references, output storage, and key length. Correct values do not excuse exceeding a space or tail-latency contract; refine the algorithm or contract.
28. Pre-generate inputs, consume outputs, vary n/density/layout, record cache counters and latency distributions, and compare ratios against predicted n, n log n, or log n scaling.
29. Precedence constraints form a DAG requirement; cycles make any schedule infeasible. General weighted two-machine scheduling with release/deadline constraints is hard; restricted interval/DAG cases may admit DP or flow.
30. Sort intervals by finish and use weighted interval DP with predecessor p(j), `OPT(j)=max(OPT(j-1), v_j+OPT(p(j)))`. Exchange/partition by taking or skipping j proves optimality.
31. Earliest-deadline or value-density heuristics can be useful but need not optimize the full constrained objective. State feasibility repair, approximation/competitive status, and counterexamples or empirical limits.
32. Use a priority queue keyed by deadline/value for available jobs, indegree counters and adjacency lists for precedence, and a schedule array with memory accounting. Updating ready status must preserve indegree invariants.
33. Run topological cycle detection, test zero-length and boundary deadlines, simultaneous events, duplicate IDs, and failed allocations. Define whether a rejected job is dropped, retried, or reported.
34. Measure jobs/sec, deadline miss rate, objective value, memory peak, queue age, p99 latency, and gap from exact solutions on small instances. Record workload seeds and arrival patterns.
