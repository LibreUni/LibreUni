# Algorithms Capstone: Design a Reliable Route Planner

A transportation company needs a route-planning engine handling directed and undirected links, travel times, tolls, capacity constraints, temporary closures, and user preferences. The engine must answer shortest-time queries, identify unreachable destinations, produce a route certificate, and support batch mode for minimum-cost infrastructure planning.

<MathStatement
  id="capstone-problem"
  kind="definition"
  title="Route-Planning Problem"
  proofId="dijkstra-correctness"
  dependsOn={[{ id: "shortest-path-subproblem", label: "Shortest-path subproblem" }]}
>
Given a directed graph $G = (V, E)$ with edge attributes $\text{time}(e)$, $\text{toll}(e)$, $\text{capacity}(e)$, and $\text{active}(e)$ (a time interval), a source $s$, and a query mode $m \in \{\text{time}, \text{toll}, \text{combined}\}$, find a path $P$ from $s$ to $t$ minimizing $f_m(P)$ subject to $P$ being feasible under $\text{active}(e)$ constraints. Return a certificate $(d, \text{pred})$ where $d[v]$ is the optimal cost and $\text{pred}[v]$ is the predecessor on the optimal path.
</MathStatement>

## Algorithm Selection Framework

The choice of shortest-path algorithm depends on edge-weight properties, graph structure, and query volume.

<PlantUML
  code={`@startuml

if "Edge weights nonnegative?" then (yes)

  if "Graph is a DAG?" then (yes)

    endif

  note right: O(VE + V^2 log V)
endif
:Return certificate (d, pred);
@enduml`}

**Scenario 1: Shortest-time query from $A$ to $E$.** All weights nonnegative → Dijkstra. Execution: $d[A]=0$, relax $A\to B=10$, $A\to C=3$. Settle $C$ ($d=3$), relax $C\to B$: $d[B]=\min(10, 3+2=5)=5$, $C\to D$: $d[D]=3+8=11$. Settle $B$ ($d=5$), relax $B\to D$: $d[D]=\min(11,5+7=12)=11$, $B\to E$: $d[E]=5+15=20$. Settle $D$ ($d=11$), relax $D\to E$: $d[E]=\min(20,11+5=16)=16$. Shortest path: $A\to C\to D\to E$ with time 16 min. Certificate: $\text{pred}[C]=A$, $\text{pred}[D]=C$, $\text{pred}[E]=D$, $d=[0,5,3,11,16]$.

**Scenario 2: Minimum-toll path from $A$ to $E$ under budget 20 min.** Use toll as weight. Dijkstra on tolls: $A\to B$ (\$2), $A\to C$ (\$5). Settle $B$ (\$2), relax $B\to D$ (\$3), $B\to E$ (\$3). Settle $D$ (\$3), relax $D\to E$ (\$5). Minimum toll: \$3 via $A\to B\to E$. Time = 15 min ≤ 20 ✓.

**Scenario 3: Batch infrastructure planning.** Connect all intersections minimizing total construction cost. Edge set undirected with weights: AB=10, AC=3, CB=2, BD=7, CD=8, DE=5, BE=15. Kruskal: sort (2,3,5,7,8,10,15), add AC (2), CB (2+3=5), DE (5+5=10), BD (10+7=17). MST weight 17. Certificate: each non-tree edge is not lighter than the max-weight edge on the tree path it connects.

## Worked Example 2: Certificate Verification

Given distances $d=[0,5,3,11,16]$ and predecessors $\text{pred}=[\text{none}, C, A, C, D]$, verify the shortest-path certificate.

**Edge-inequality check:** $\forall (u,v)\in E$, $d[v] \le d[u] + w(u,v)$.
- $A\to B$: $5 \le 0+10 = 10$ ✓
- $A\to C$: $3 \le 0+3 = 3$ ✓
- $B\to D$: $11 \le 5+7 = 12$ ✓
- $B\to E$: $16 \le 5+15 = 20$ ✓
- $C\to B$: $5 \le 3+2 = 5$ ✓
- $C\to D$: $11 \le 3+8 = 11$ ✓
- $D\to E$: $16 \le 11+5 = 16$ ✓

**Path-consistency check:** $\forall v\ne s$, $d[v] = d[\text{pred}[v]] + w(\text{pred}[v], v)$.
- $C$: $3 = 0 + 3$ with $\text{pred}[C]=A$ ✓
- $B$: $5 = 3 + 2$ with $\text{pred}[B]=C$ ✓
- $D$: $11 = 3 + 8$ with $\text{pred}[D]=C$ ✓
- $E$: $16 = 11 + 5$ with $\text{pred}[E]=D$ ✓

<MathStatement
  id="certificate-soundness"
  kind="theorem"
  title="Certificate Soundness"
  dependsOn={[{ id: "triangle-inequality", label: "Triangle inequality" }]}
>
If a certificate $(d, \text{pred})$ satisfies edge inequalities $d[v] \le d[u] + w(u,v)$ for all $(u,v) \in E$ and path consistency $d[v] = d[\text{pred}[v]] + w(\text{pred}[v], v)$ for all $v \ne s$, then $d[t]$ is the true shortest-path distance from $s$ to $t$.
</MathStatement>

<MathProof id="certificate-proof" for="certificate-soundness" collapsible>
Let $P$ be any path $s = v_0, v_1, \dots, v_k = t$. By repeated application of the edge inequality:

$$d[t] = d[v_k] \le d[v_{k-1}] + w(v_{k-1}, v_k) \le \dots \le d[s] + \sum_{i=0}^{k-1} w(v_i, v_{i+1}) = \text{weight}(P).$$

Thus $d[t] \le \text{weight}(P)$ for all $P$. The path traced by $\text{pred}$ achieves equality, so $d[t]$ is optimal.
</MathProof>

  title="Certificate Checker"

    """Return (is_valid, path) or (False, error_msg)."""
    # Edge inequalities
    for u, v, w in E:
        if d[v] > d[u] + w + 1e-9:  # tolerance for floating point
            return False, f"Violation: {u}->{v}: d[{v}]={d[v]} > d[{u}]+w={d[u]}+{w}"

    path = []
    v = t

        v = p
    path.append(s)
    path.reverse()

## Counterexample: When Dijkstra Fails

<MathStatement
  id="negative-edge-failure"
  kind="example"
  title="Dijkstra Failure on Negative Edges"
>
Consider $G$ with $V = \{s, a, t\}$, $s \to a$ weight 5, $s \to t$ weight 10, $a \to t$ weight $-3$. Dijkstra settles $a$ ($d=5$), then $t$ ($d=10$), never relaxing $a \to t$ because $t$ is already settled. Correct shortest path: $s \to a \to t$ with weight $5 + (-3) = 2$.
</MathStatement>

**Negative cycle detection:** Bellman-Ford detects cycles by running one extra relaxation after $V-1$ passes. If any edge still relaxes, a negative cycle is reachable from $s$.

<CodeExercise client:load
  title="Negative Cycle Detection"

    dist[s] = 0
    for _ in range(V - 1):
        for u, v, w in graph:
            if dist[u] != float('inf') and dist[u] + w [!blank!] dist[v]:
                dist[v] = dist[u] + w
    for u, v, w in graph:
        if dist[u] != float('inf') and dist[u] + w [!blank!] dist[v]:

## Capacity and Throughput

Model edge capacities as a flow network. Answer "Can $k$ units be sent from $s$ to $t$?" via max flow. The integrality theorem ensures integral flow corresponds to integral paths when capacities are integral.

<MathStatement
  id="max-flow-min-cut"
  kind="theorem"
  title="Max-Flow Min-Cut Theorem"
  proofId="max-flow-min-cut-proof"
>
For any flow network $(G, c)$ with source $s$ and sink $t$, the maximum flow value equals the minimum capacity of any $s$-$t$ cut.
</MathStatement>

<MathProof id="max-flow-min-cut-proof" for="max-flow-min-cut" collapsible open>
Let $f$ be a maximum flow. In the residual network $G_f$, let $S$ be the set of vertices reachable from $s$. Since $f$ is maximum, $t \notin S$; otherwise an augmenting path exists. The cut $(S, V \setminus S)$ has capacity $c(S, V \setminus S) = |f|$ because every edge crossing from $S$ to $V \setminus S$ is saturated. Thus $|f|$ equals the cut capacity, and no cut can have smaller capacity than any flow value.
</MathProof>

## Capacity Constraint: A Failure Mode

Consider a network where a single edge $e$ has capacity 1 and all other edges have capacity 100. The max flow from $s$ to $t$ is bounded by 1. Failure mode: a naive greedy algorithm that sends flow without considering bottleneck edges may route flow into a dead-end subgraph, requiring reverse-flow augmentation in the residual network to correct.

6. **Dynamic edge failures.** Suppose edges fail probabilistically. Given failure probabilities $p_e$, compute the most reliable path (maximizing $\prod (1-p_e)$). Transform to a shortest-path problem using logarithms. Prove correctness.

### Synthesis

7. **Time-dependent routing.** Suppose edge travel times are piecewise-linear functions of departure time $t$. Prove that if the FIFO property holds (earlier departure cannot produce later arrival), Dijkstra's algorithm remains correct with a modified relaxation: $d[v] = \min(d[v], \text{travel}(d[u], u, v))$, where $\text{travel}(t, u, v)$ is the arrival time at $v$ when departing $u$ at time $t$ via edge $(u,v)$.

8. **Synthesis: multimodal route planner.** Design a system combining driving, transit, and walking. Each mode has its own graph, edge costs, and transfer penalties at interchange vertices. Formulate a unified graph model, state which algorithm(s) apply, and specify the certificates each query must return. Analyze the total complexity in terms of $V_{\text{drive}}, V_{\text{transit}}, V_{\text{walk}}$ and transfer edges.

<ExerciseSolution hint="For exercise 5, define a state graph where each vertex is (original vertex, remaining budget). Edges consume both time and budget.">

**Exercise 5 solution:** Construct $G' = (V', E')$ where $V' = V \times \{0, 1, \dots, B\}$ and $B$ is the maximum budget. For each $(u,v) \in E$ with toll $c_{uv}$ and time $t_{uv}$, add an edge from $(u, b)$ to $(v, b - c_{uv})$ with weight $t_{uv}$ for all $b \ge c_{uv}$. The shortest path from $(s, B)$ to any $(t, b)$ with $b \ge 0$ gives the minimum time for a given budget. Complexity: $O(V \cdot B \cdot \log(V \cdot B))$ with Dijkstra.

**Exercise 6 solution:** Let $p_e$ be failure probability. Path reliability = $\prod_{e \in P} (1-p_e)$. Maximizing this is equivalent to minimizing $-\log(\prod (1-p_e)) = \sum -\log(1-p_e)$. Since $-\log(1-p_e) \ge 0$ for $p_e \in [0,1)$, the transformed weights are nonnegative, and Dijkstra applies. Correctness: log is monotonic, so minimizing sum of logs is equivalent to maximizing product of probabilities.

</ExerciseSolution>

## Deliberately Omitted

- **Contraction hierarchies and ALT preprocessing** — Speedup techniques for real-time road networks (CH, landmark A*) are implementation-intensive and depend on specific preprocessing; the basic Dijkstra and Johnson machinery covers the algorithmic core.
- **Stochastic shortest paths with dynamic programming** — Markov decision process formulations for travel time minimization under stochastic edge costs rely on value iteration, covered in the dynamic programming module.
- **Multi-commodity flow for network design** — Simultaneously routing multiple commodities with coupling capacity constraints generalizes single-commodity max-flow to NP-hard territory; approximation schemes appear in the approximation algorithms lesson.
- **Route planning with turn restrictions and penalties** — Modeling turn costs requires expanding each vertex into a set of directed edges representing incoming-outgoing pairs; this is a straightforward graph transformation but adds notational overhead without new algorithmic insight.
- **Real-time traffic data integration** — Live data feeds, prediction models, and re-planning heuristics are engineering concerns orthogonal to the core algorithmic correctness proofs.
