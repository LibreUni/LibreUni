# Approximation Review

Bin packing: items of various sizes, bins of fixed capacity, minimize bins used. NP-hard (Garey and Johnson 1979). Four tools exist, each with a different cost--guarantee trade-off.

| Tool | Runtime | Guarantee | When to use |
|------|---------|-----------|-------------|
| Exact ILP solver | Exponential worst-case | Optimal | Small instances ($n \le 30$), offline |
| Constant-factor approx | Polynomial (e.g., $O(n \log n)$) | Fixed ratio ($1.5$, $2$, etc.) | Large instances, need certification |
| PTAS/FPTAS | $O(n^{f(1/\varepsilon)})$ or $O(n^c / \varepsilon)$ | $(1+\varepsilon)$ for any $\varepsilon > 0$ | High accuracy needed, small $\varepsilon$ |
| Heuristic | Fast polynomial | None (empirical only) | Emergency, prototyping, lower bound |

The decision framework depends on instance size, desired accuracy, available time, and problem structure. NP-hardness is a fork in the road, not a dead end.

**Prerequisites:** NP-completeness (decision problems, reductions, e.g. SAT $\le_p$ Vertex Cover), linear programming (LP duality provides lower bounds for minimization IPs), basic algorithm design (DP for knapsack, greedy for set cover, MST for TSP).

This decision tree depends on three prerequisite ideas: (1) **LP duality** provides the lower-bound engine for approximation certification — every dual feasible solution gives a bound on OPT, as used throughout Vazirani (2001). (2) **Dynamic programming** underlies the FPTAS pattern of scaling + DP, seen in knapsack (Ibarra and Kim 1975). (3) **Maximum matching** and **minimum spanning tree** algorithms supply the polynomial-time building blocks for vertex cover and TSP approximations.

## Guarantee Types

<MathStatement id="approx-types" kind="definition" title="Approximation taxonomy">
For minimization problem $\Pi$, algorithm $A$ is an $\alpha$-**approximation** if $A(I) \le \alpha \cdot \text{OPT}(I)$ for all instances $I$, with $\alpha \ge 1$. For maximization, $A(I) \ge \text{OPT}(I) / \alpha$.

- **PTAS:** For each $\varepsilon > 0$, $(1+\varepsilon)$-approximation in time $O(n^{f(1/\varepsilon)})$.
- **FPTAS:** Time $O(n^c \cdot g(1/\varepsilon))$, polynomial in both $n$ and $1/\varepsilon$.
- **EPTAS:** Time $O(n^c + h(1/\varepsilon))$ for some $c$ independent of $\varepsilon$.
- **Heuristic:** No worst-case guarantee; empirical evaluation only.
</MathStatement>

The hierarchy matters: an FPTAS allows $\varepsilon$ to shrink without exploding the exponent in $n$. A PTAS with $f(1/\varepsilon) = 2^{1/\varepsilon}$ makes runtime $n^{1024}$ at $\varepsilon = 0.1$ — polynomial but useless.

**Example.** A $2$-approximation for vertex cover via maximal matching (Vazirani 2001, §1.3). A $(7/8 + \varepsilon)$-inapproximability for MAX-3SAT under P $\neq$ NP (Håstad 2001).

## Canonical Proofs

### Vertex cover via maximal matching

$2$-approximation. Find a maximal matching $M$ (no edge can be added). Return both endpoints of each edge in $M$.

<PlantUML
  code={`@startuml
skinparam backgroundColor transparent
title Vertex Cover 2-Approximation via Maximal Matching
start
:Graph G = (V, E);
:M ← empty set;
repeat
  :Pick any uncovered edge e = (u, v);
  :Add e to M;
  :Remove all edges incident to u or v;
repeat while (uncovered edges remain?)
:Return C = {u, v : (u,v) ∈ M};
:Guarantee: |C| ≤ 2·OPT;
stop
@enduml`}
/>

<MathStatement id="vc-2approx" kind="theorem" title="Vertex cover 2-approximation" proofId="vc-proof">
The set $C = \{u, v : (u,v) \in M\}$ is a vertex cover with $|C| \le 2 \cdot \text{OPT}$.
</MathStatement>

<MathProof id="vc-proof" for="vc-2approx" title="Proof" collapsible>
Every edge $e \in E$ shares an endpoint with some edge in $M$ (otherwise $M$ would not be maximal). Hence $C$ covers all edges. Let $M$ have $k$ edges. Any vertex cover must contain at least one endpoint per matching edge, and matching edges are disjoint, so $\text{OPT} \ge k$. Thus $|C| = 2k \le 2\text{OPT}$.
</MathProof>

**Tight example:** $k$ disjoint edges. OPT $= k$, algorithm returns $2k$. Ratio $= 2$.

**Inline citation:** This algorithm first appears in Gavril (1974) and is exposited in Vazirani (2001, §1.3). The maximal matching can be found greedily in $O(|E|)$ time.

### Metric TSP: Double-tree and Christofides

<MathStatement id="double-tree" kind="theorem" title="Double-tree algorithm" proofId="dt-proof">
For metric TSP (complete graph, triangle inequality), double-tree is a $2$-approximation.
</MathStatement>

<MathProof id="dt-proof" for="double-tree" title="Proof" collapsible>
Compute MST $T$. $w(T) \le \text{OPT}$ (deleting one edge from the optimal tour yields a spanning tree). Double $T$: each edge appears twice, creating an Eulerian multigraph of weight $2w(T)$. Take an Euler tour; shortcut repeated vertices using triangle inequality, which cannot increase weight. Result: $\le 2w(T) \le 2\text{OPT}$.
</MathProof>

<MathStatement id="christofides" kind="theorem" title="Christofides' algorithm" proofId="christofides-proof">
For metric TSP, Christofides' algorithm achieves a $3/2$-approximation.
</MathStatement>

<MathProof id="christofides-proof" for="christofides" collapsible>
Let $O$ be the odd-degree vertices in MST $T$ (even count). Compute min-weight perfect matching $M$ on $O$. Claim: $w(M) \le \text{OPT}/2$.

Start from the optimal Hamiltonian tour. Shortcut it to vertices in $O$ only; triangle inequality keeps weight $\le \text{OPT}$. This tour on $O$ alternates between two perfect matchings. $M$ has weight $\le$ the lighter matching, hence $\le \text{OPT}/2$.

Then $T \cup M$ has all even degrees, so an Euler tour exists. Shortcut repeated vertices: weight $\le w(T) + w(M) \le 1.5\text{OPT}$.
</MathProof>

Christofides (1976) remains the best-known ratio for metric TSP. The general TSP has no $O(2^{n^{1-\varepsilon}})$ approximation unless P = NP (Sahni and Gonzalez 1976).

### Knapsack FPTAS

<MathStatement id="knapsack-fptas" kind="theorem" title="0/1 knapsack FPTAS" proofId="knap-proof">
For any $\varepsilon > 0$, a $(1-\varepsilon)$-approximation exists running in $O(n^2 / \varepsilon)$.
</MathStatement>

<MathProof id="knap-proof" for="knapsack-fptas" title="Proof" collapsible>
Let $V_{\max} = \max_i v_i$. Scale factor $K = \varepsilon V_{\max} / n$. Scaled values $v'_i = \lfloor v_i / K \rfloor$. Run value-indexed DP: $DP[j]$ = min weight achieving scaled value $j$. Max scaled value $\le \sum v'_i \le n \cdot V_{\max} / K = n^2 / \varepsilon$, so table $O(n^2 / \varepsilon)$.

For optimal $S^*$, $v(S^*) = \sum_{i \in S^*} v_i$. Scaled total: $\sum_{i \in S^*} v'_i \ge v(S^*)/K - n$. DP finds $S$:

$$
v(S) \ge K \cdot \sum_{i \in S} v'_i \ge K \cdot \sum_{i \in S^*} v'_i \ge v(S^*) - nK = v(S^*) - \varepsilon V_{\max}.
$$

Since $V_{\max} \le v(S^*)$, we get $v(S) \ge (1-\varepsilon) v(S^*)$. (Ibarra and Kim 1975)
</MathProof>

    n = len(values)
    Vmax = max(values)
    K = eps * Vmax / n
    scaled = [int(v // K) for v in values]
    max_scaled = sum(scaled)
    DP = [float('inf')] * (max_scaled + 1)
    DP[0] = 0

## LP Relaxation and Rounding

The LP relaxation of an integer program provides a lower bound (minimization). Dropping integrality expands the feasible region: every integer solution is LP-feasible, so $\text{OPT}_\text{LP} \le \text{OPT}_\text{IP}$. This connection to LP duality (see prerequisite) is central: the dual of the LP relaxation provides a feasible certificate for the lower bound.

**Randomized rounding** (Raghavan and Tompson 1987): solve LP to get fractional $x^* \in [0,1]^n$, then independently set $x_i = 1$ with probability $x_i^*$. Expected cost = $\sum c_i x_i^* = \text{OPT}_\text{LP} \le \text{OPT}$. Feasibility may fail; Chernoff bounds bound violation probability. Derandomization via conditional expectations yields a deterministic algorithm.

> **Warning.** Randomized rounding does not always preserve feasibility. For set cover, independent rounding leaves an element uncovered with probability $\prod_{i: e \in S_i} (1 - x_i^*) \approx e^{-1}$, requiring $O(\log n)$ repetitions (Vazirani 2001, §5.2).

<MathStatement id="lp-bound" kind="theorem" title="LP rounding approximation factor">
If a problem has LP relaxation with integrality gap $\gamma$ and rounding preserves feasibility with cost increase factor $\rho$, the overall approximation factor is at most $\rho \cdot \gamma$.
</MathStatement>

**Example: Weighted vertex cover.** LP rounding gives a $2$-approximation (set $x_v = 1$ iff $x_v^* \ge 1/2$). This matches the maximal matching ratio, and the integrality gap is exactly $2$. The LP is:

$$
\begin{aligned}
\min \quad & \sum_{v \in V} w_v x_v \\
\text{s.t.} \quad & x_u + x_v \ge 1 \quad \forall (u,v) \in E \\
& 0 \le x_v \le 1 \quad \forall v \in V
\end{aligned}
$$

Rounding $x_v^* \ge 1/2 \to 1$ preserves constraint feasibility: for each edge $(u,v)$, $x_u^* + x_v^* \ge 1$, so at least one endpoint rounds to $1$. Cost doubles: $w(C) \le 2 \sum w_v x_v^* = 2\text{OPT}_\text{LP} \le 2\text{OPT}_\text{IP}$.

## Inapproximability (PCP Theorem)

NP $\subseteq$ PCP[O(log n), O(1)] (Arora et al. 1998) implies that for some problems, achieving ratio better than a threshold is NP-hard.

| CLIQUE | No $n^{1-\varepsilon}$ unless P=NP | Håstad 1999 |
| VERTEX COVER | No $(2-\varepsilon)$ under UGC | Khot and Regev 2008 |

## Counterexamples and Failure Modes

> **Warning: Missing triangle inequality.** The double-tree and Christofides algorithms rely on the metric property. Without it, shortcutting can add arbitrary cost. Consider $A, B, C$ with $w(A,B) = w(B,C) = 1$, $w(A,C) = M$. MST weight $= 2$, Euler tour weight $= 4$, shortcut yields $A \to B \to C \to A$ with weight $M+2$. As $M \to \infty$, the approximation ratio $\to \infty$.

> **Warning: PTAS runtime blowup.** A PTAS with $f(1/\varepsilon) = 2^{1/\varepsilon}$ at $\varepsilon = 0.1$ gives exponent $2^{10} = 1024$, making $n^{1024}$ infeasible for any practical $n$. The FPTAS avoids this with polynomial dependence on $1/\varepsilon$.

> **Warning: Heuristic overconfidence.** A greedy heuristic that picks the highest-degree vertex for vertex cover may perform well on sparse random graphs but return $O(\log n)$ times OPT on adversarially constructed instances. Worst-case analysis prevents overconfidence.

> **Warning: Local search traps.** Local search for MAX-CUT (flip a single vertex to improve the cut) can get stuck in a local optimum with value half the global optimum. Consider a complete bipartite graph $K_{n/2, n/2}$: any balanced partition is a local optimum under single-vertex flips, but the maximum cut is the whole graph. This limits local search to constant-factor approximations unless the neighborhood size grows (Khanna et al. 1998).

> **Warning: LP rounding feasibility.** For set cover, the natural LP relaxation has $x_S$ variables for each set $S$. Randomized rounding sets $S$ independently with probability $x_S^*$. An element $e$ may be uncovered with probability $\prod_{S: e \in S} (1 - x_S^*) \le e^{-1}$. Guaranteeing complete coverage requires $O(\log n)$ repetitions (Vazirani 2001, §5.2), blowing up runtime by a logarithmic factor. Deterministic rounding via the greedy algorithm avoids this issue and achieves $H_n$-approximation (Chvátal 1979).

## Exercises

1. **Maximal matching tightness.** Prove that $k$ disjoint edges achieve the ratio $2$ for vertex cover.

   <ExerciseSolution hint="What is OPT for $k$ disjoint edges? What does maximal matching return?">

   For $k$ disjoint edges, optimal cover picks one endpoint per edge ($k$ vertices). Maximal matching picks all $k$ edges, returning $2k$ vertices. Ratio $= 2k/k = 2$.
   </ExerciseSolution>

2. **Triangle inequality necessity.** Construct a non-metric TSP instance where the double-tree algorithm produces a tour of weight $> c \cdot \text{OPT}$ for any desired $c$.

   <ExerciseSolution hint="Break triangle inequality on one edge of the triangle.">

   Consider vertices $A, B, C$ with $w(A,B) = w(B,C) = 1$, $w(A,C) = M$ (large). The triangle inequality fails: $w(A,C) \gg w(A,B) + w(B,C)$. MST = $\{AB, BC\}$ (weight 2). Double-tree Euler tour weight = 4. Shortcutting: $A \to B \to C \to A$ with weight $1 + 1 + M = M+2$. OPT = $M+2$ (any Hamiltonian tour must use edge $AC$). Ratio = $(M+2)/(M+2) = 1$ here because the only possible tour is forced. Construct a 4-vertex example: $A, B, C, D$ with $w(A,B)=w(B,C)=w(C,D)=1$, all other edges $= M$. MST = $\{AB, BC, CD\}$ (weight 3). Double-tree: weight 6. Shortcutting yields a tour with weight $3 + 2M - 1$ (depending on shortcut order), while OPT $= M + 3$ (the path $A\!-\!B\!-\!C\!-\!D$ plus edge $D\!-\!A$). As $M \to \infty$, ratio $\to 2$ — the double-tree guarantee is tight.
   </ExerciseSolution>

3. **Knapsack FPTAS proof details.** Show that the DP table size is $O(n^2 / \varepsilon)$ and that the loss per item is bounded by $K$.

   <ExerciseSolution>
   Scaled value $v'_i = \lfloor v_i / K \rfloor$ where $K = \varepsilon V_{\max} / n$. Error per item: $v_i - K v'_i < K$. Total error $< nK = \varepsilon V_{\max} \le \varepsilon \text{OPT}$. Table size: maximum scaled value $\sum v'_i \le \sum v_i / K \le n \cdot V_{\max} / K = n^2 / \varepsilon$.
   </ExerciseSolution>

4. **Derandomize Max-Cut.** Given the randomized $0.5$-approximation (each vertex assigned uniformly), derandomize via conditional expectations.

   <ExerciseSolution>
   Process vertices sequentially. For vertex $i$, compute the expected cut size if $i$ goes left vs. right, conditioned on prior assignments. Choose the side with larger conditional expectation. Since this expectation is a sum over edges (each edge crossing probability is known from assignments), the choice is computable in polynomial time. The final cut has size $\ge |E|/2 \ge \text{OPT}/2$.
   </ExerciseSolution>

5. **Transfer: Steiner forest connection.** Steiner forest: given pairs $(s_i, t_i)$, connect each pair (not all vertices). Show that the metric Steiner forest problem reduces to Steiner tree (all pairs connected) only when there is a single pair. Why does primal-dual for Steiner forest need a different analysis? (Hint: Consider overlapping components.)

   <ExerciseSolution>
   For Steiner tree, the MST-based 2-approximation works because the optimal Steiner tree costs at least the MST cost on terminals. For Steiner forest, multiple components can merge independently, and the optimal solution may connect some pairs indirectly. The primal-dual algorithm (Agrawal et al. 1991) uses a dual growing process that raises dual variables on active components simultaneously. The analysis requires a synchronization lemma: components freeze when their dual exceeds the cost to connect them.
   </ExerciseSolution>

6. **Synthesis: Multi-objective decision.** You are designing a cloud scheduler. Instances arrive online (unknown future), bin packing models server consolidation, and violations incur a penalty. Design a decision framework: which tool (heuristic, approximation, exact) for which regime? Defend with instance size, guarantee needs, and runtime constraints.

   <ExerciseSolution>
   **Regime 1 — small cluster ($\le 20$ servers):** exact ILP, since worst-case exponential is fine and optimal packing saves hardware cost. **Regime 2 — large cluster, peak load:** constant-factor approximation (first-fit decreasing, 11/9-approx for bin packing), since worst-case guarantee matters for SLA certification. **Regime 3 — online arrival, rapid decisions:** heuristic (best-fit decreasing) with periodic reoptimization by the exact solver. The lower bound comes from the LP relaxation of the bin packing formulation. Regime boundaries are determined by the threshold where exact solver runtime exceeds the scheduling interval.
   </ExerciseSolution>

7. **Transfer: LP rounding for set cover.** The set cover LP relaxation is $\min \sum_S w_S x_S$ subject to $\sum_{S: e \in S} x_S \ge 1$ for all elements $e$, $0 \le x_S \le 1$. If you round by selecting all sets with $x_S^* \ge 1/f$ where $f$ is the maximum frequency of any element, what approximation ratio do you get? Why is this approach insufficient when $f$ is large?

   <ExerciseSolution>
   Each element $e$ appears in at most $f$ sets with fractional value at least $1/f$ (otherwise the sum would exceed $1$). So rounding at threshold $1/f$ selects at most $f$ sets per element, and each selected set costs at most $f \cdot w_S x_S^*$. Total cost $\le f \cdot \text{OPT}_\text{LP} \le f \cdot \text{OPT}$. This is an $f$-approximation. When $f$ is large (e.g., set cover with unrestricted frequency), the bound is weak — this motivates the $O(\log n)$-approximation via greedy or randomized rounding with repetition.
   </ExerciseSolution>

8. **Synthesis: Adversarial instance design.** For each algorithm below, design a family of instances where the algorithm performs at its worst-case guarantee (or provably cannot do better):
   - (a) Double-tree for metric TSP
   - (b) Maximal-matching for vertex cover
   - (c) Scaled DP for knapsack
   - (d) Greedy high-degree for vertex cover

   <ExerciseSolution>
   (a) A line of $n$ vertices with all other edge weights equal to $2 \cdot w(\text{path})$. MST follows the line (weight $L$). Double-tree weight $2L$. Shortcutting skips intermediate vertices, producing a direct edge of weight $2L$ — the optimal tour is the line back-and-forth with weight $2L$, so ratio $=1$. The worst case is a graph where shortcutting uses a nearly-MST-weight edge: a star with MST weight $S$, optimal TSP weight $\approx S/2$, producing ratio $\to 2$ as shown in Exercise 2.

   (b) $k$ disjoint edges — optimal picks $k$, algorithm picks $2k$, ratio $=2$.

   (c) Two items: $(v_1, w_1) = (V_{\max}, C)$, $(v_2, w_2) = (V_{\max}-\delta, C)$. Only one fits. Scaled values may become equal if $\delta < K$, causing DP to arbitrarily choose one. The lost value is $\le \varepsilon V_{\max}$.

   (d) Star with $n-1$ leaves plus a matching among $n/2$ leaves (as in the CodeRunner above). Greedy picks the center then $n/2$ matching vertices, total $n/2+1$. Optimal picks just the center (size $1$). Ratio $\Theta(n)$.
   </ExerciseSolution>

## Deliberately Omitted

**Polynomial-time approximation schemes for geometric problems** — The shifting technique for Euclidean TSP and geometric packing (Arora 1998) uses a different paradigm; appears in the advanced algorithms module.

**Approximation algorithms for polynomial-solvable problems** — This review targets NP-hard optimization; maximum matching and shortest path use exact methods.

**PCP theorem proof** — One of the deepest results in theoretical CS; this lesson states consequences without proving the theorem. See Arora and Barak (2009, ch. 11, 22) for the full proof.

**Semidefinite programming relaxations** — The Goemans–Williamson (1995) Max-Cut algorithm uses SDP; treated in the advanced approximation course.

**Unique games conjecture and optimal inapproximability** — UGC implies tight thresholds for VERTEX COVER, MAX-CUT, and other problems; covered in the graduate complexity module.

**Local search approximation algorithms** — Local search provides constant-factor approximations for MAX-CUT ($\frac12$), facility location, and $k$-median, but requires careful neighborhood design to escape local optima (Khanna et al. 1998; Williamson and Shmoys 2011, ch. 9). Omitted because the analysis techniques differ substantially from the LP-rounding and primal-dual patterns in this review.
