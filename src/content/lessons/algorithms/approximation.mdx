# Approximation Algorithms

Some optimization problems are NP-hard, so an exact algorithm may require exponential time unless a major complexity collapse occurs. Approximation algorithms trade exactness for a provable guarantee: they run efficiently and return a solution whose value is within a factor or additive error of optimum. The guarantee must specify minimization versus maximization, worst-case inputs, and whether randomness or resource parameters are involved.

## Approximation definitions

For a maximization problem, an alpha-approximation returns value A(I) with (OPT(I)/alpha at most  A(I) at most  OPT(I)), alpha at least 1. For minimization, (OPT(I) at most  A(I) at most alpha OPT(I)). A PTAS takes an accuracy parameter epsilon greater than 0 and runs in polynomial time in input size for every fixed epsilon, but the exponent or dependence on epsilon may be large. An FPTAS is polynomial in both input size and 1/epsilon.

Do not compare ratios across objectives without normalization. If OPT can be zero, a multiplicative ratio may be undefined; use additive guarantees or a special case. For maximization, a solution worth zero may have an unbounded ratio on an instance with positive optimum. State whether the algorithm is allowed to output infeasible solutions—it generally is not.

## Vertex cover by maximal matching

A maximal matching M is a matching to which no edge can be added. Return both endpoints of every edge in M. This is a vertex cover: any uncovered edge could be added to M, contradicting maximality. Every vertex cover must include at least one endpoint of each matching edge, so (OPT_()VC() at least |M|). The returned cover has size 2|M| at most 2OPT, giving a 2-approximation.

The proof separates feasibility and ratio. Maximality—not maximum matching—is enough for feasibility, and the matching lower bound supplies the comparison. A maximal matching can be found greedily in O(V+E). The factor 2 is tight for this algorithm on graphs consisting of disjoint edges if the optimum cover chooses one endpoint from each while the algorithm returns both.

## Set cover greedy

In set cover, choose a minimum-cost collection of sets covering all elements. The greedy algorithm repeatedly chooses the set with minimum cost per newly covered element. Its approximation ratio is (H_d at most 1+ln d), where d is the maximum number of elements in any set (or related logarithmic bounds in universe size).

The charging proof assigns each newly covered element a fraction of the chosen set’s cost proportional to its marginal coverage. At a stage with r uncovered elements, the best fractional share argument shows greedy cost per new element is bounded by the average cost needed by an optimum cover. Summing harmonic contributions over decreasing r yields the logarithmic factor. Greedy may not be exact because early cheap coverage can block a better global combination.

## Metric traveling salesman

For metric TSP, distances satisfy triangle inequality. A minimum spanning tree has weight at most OPT because deleting one edge from an optimal tour yields a spanning tree. Double every MST edge to obtain an Eulerian multigraph of weight 2M, traverse an Euler tour, and shortcut repeated vertices using triangle inequality. The resulting tour has weight at most 2OPT, a 2-approximation.

Christofides’ algorithm improves to 3/2: find an MST, compute the odd-degree vertices, find a minimum-weight perfect matching among them, add it to make all degrees even, take an Euler tour, and shortcut. The matching has weight at most half the optimal tour’s edges restricted appropriately, yielding MST + matching  at most 3/2 OPT. The proof relies on metric symmetry and triangle inequality; arbitrary distances invalidate shortcutting.

## Knapsack approximation

0/1 knapsack has an FPTAS based on scaling item values. Let Vmax be the largest value and choose scale `K = epsilon Vmax/n`. Replace each value by `floor(vi/K)`, run value-indexed DP, and return a solution maximizing scaled value under capacity. Rounding loses less than K per selected item, so total loss is less than `nK = epsilon Vmax  at most  epsilon OPT`. The scaled total value is polynomial in n and 1/epsilon, giving an FPTAS.

The analysis must account for tiny values and the case OPT=0. The algorithm’s time is polynomial in n and 1/epsilon, not necessarily strongly polynomial in numeric capacity or value. Scaling changes the objective used by the DP but the unscaled returned solution remains feasible.

## Lower bounds and inapproximability

An approximation guarantee can be ruled out by gap reductions: transform instances so yes cases have optimum at most a and no cases have optimum at least b, with b/a beyond the claimed ratio. A hypothetical approximation would distinguish the hard cases. This method connects optimization guarantees to decision hardness.

A trivial feasible solution may already give a ratio, but measure it carefully. For maximization, any nonnegative solution compared with an upper bound may yield a weak ratio; for minimization, a spanning tree or greedy construction may provide an upper bound but needs a lower bound on OPT. A proof that compares against the wrong bound is not an approximation proof.

## Local search and metric structure

Approximation can arise from local improvement. Define a feasible neighborhood and repeatedly replace the current solution when a move improves its objective. Termination follows from a strictly improving bounded objective or a finite state space, but quality requires a local-optimality lemma. A solution that is locally optimal under one neighborhood can be arbitrarily poor under another; the neighborhood is part of the algorithm.

Metric structure enables shortcut arguments because a direct edge is no more expensive than a path through intermediate vertices. Similar inequalities appear in clustering, facility location, and Steiner approximations, but each problem needs its own lower bound and feasible construction. Do not apply triangle inequality to a domain where distances are asymmetric or violate it due to penalties, time windows, or directed travel.

## Randomized approximation

Random sampling can estimate a large objective or select a representative subset. If each trial is an unbiased indicator with probability p, the sample average estimates p; Chernoff or Hoeffding bounds determine sample size for additive error epsilon and failure probability delta. A multiplicative estimate needs p bounded away from zero or importance sampling. State whether the randomness affects feasibility, objective error, or both.

Randomized rounding solves a fractional relaxation by independently rounding variables, often scaling probabilities to preserve expected cost/coverage. Expected feasibility is not feasibility; repair or alteration may be needed. For a set of constraints, union bounds or concentration inequalities show all are satisfied with high probability, while dependent constraints may require conditional or dependent rounding.

## Linear-programming relaxations

Relax an integer program to a linear program, solve or approximate the fractional optimum, and round. The fractional optimum is a lower bound for minimization and an upper bound for maximization, so it supplies a comparison baseline. Rounding must preserve or control feasibility and objective loss. The integrality gap—the worst ratio between integral optimum and fractional optimum—limits what this template can prove.

For vertex cover, the LP constraints (x_u+x_v at least 1), (x_u at least 0) have a half-integral optimum; rounding x_u at least 1/2 to 1 yields a 2-approximation. The lower-bound direction is explicit: LP optimum at most integer optimum, and rounding costs at most twice the fractional objective. Similar reasoning applies to facility and cut relaxations with more specialized rounding.

## Approximation engineering

An algorithm should expose epsilon, running time, memory, and whether the ratio is exact or high probability. A PTAS with a huge epsilon-dependent constant may be unsuitable despite its asymptotic class. Scaling can overflow or lose precision; use checked arithmetic and document numeric representation. If the input has zero/negative costs, multiplicative ratios may require preprocessing or a different statement.

Compare approximation output to exact solutions on small instances and construct known worst-case families. A benchmark on random cases may show a much better ratio than the theorem; that is useful empirical evidence but not a stronger guarantee. Record random seeds and verify returned solutions are feasible before measuring objective quality.

## Choosing exact versus approximate

Use an exact polynomial algorithm when the problem has hidden structure such as a matroid, flow formulation, or convexity. Use pseudo-polynomial or FPTAS methods when numeric parameters are moderate and accuracy matters. Use a constant/logarithmic approximation when inputs are large and a proven bound is more valuable than exactness. If no meaningful approximation is possible under the assumed hardness, use heuristics but label them honestly and measure them against baselines.

## Approximation proof checklist

Define feasible solution and objective; state min/max convention; construct a feasible output; identify a lower or upper bound on OPT; prove the objective comparison; account for runtime and numerical precision; state randomness and failure probability; test edge cases such as OPT=0, empty instances, ties, and infeasible relaxations. This checklist prevents a “near-optimal” claim from being an unverified intuition.

## Exercises

1. Prove the 2-approximation for vertex cover from maximal matching.
2. Derive set-cover greedy’s harmonic charging intuition.
3. Prove metric double-tree TSP is a 2-approximation.
4. Explain which assumptions Christofides needs.
5. Derive value scaling loss for knapsack FPTAS.
6. Distinguish approximation ratio, PTAS, and FPTAS.
7. Construct a gap-reduction inapproximability argument.

### Diagnostic solutions

1. Endpoint cover is feasible by maximality. Matching edges are disjoint, and every cover includes one endpoint per edge, so `OPT  at least  |M|`. The returned size is `2|M|  at most  2OPT`.

2. Charge each newly covered element its proportional share of the selected set. At r uncovered elements, an optimum’s remaining sets cover them, so one set has average cost per new element no greater than the optimum residual cost divided by r. Greedy pays no more; summing `1/r` gives `H_d`.

3. `M  at most  OPT`. Doubling produces an Eulerian multigraph of weight `2M`; shortcut repeated vertices and use triangle inequality to avoid increasing weight. The Hamiltonian tour is at most `2OPT`.

4. Christofides needs complete metric symmetric distances, triangle inequality, and a minimum perfect matching on odd MST vertices. Without triangle inequality, shortcutting can increase cost arbitrarily; without a perfect matching, the Eulerian construction is not guaranteed.

5. Each chosen item loses less than K scaled value, so total loss is at most `nK = epsilon Vmax  at most  epsilon OPT`. The rounded optimum is bounded by `O(n^2/epsilon)`, so value-indexed DP is polynomial in n and 1/epsilon.

6. A ratio compares returned objective to OPT; a PTAS achieves `1+epsilon` for fixed epsilon in polynomial n; an FPTAS is polynomial also in `1/epsilon`. A PTAS may have exponential dependence on `1/epsilon`.

7. Create instances with optimum values separated by a gap. If an approximation ratio were smaller than the gap, its output would reveal which case occurred, deciding an NP-hard source problem. Verify both gap directions and reduction polynomiality.
