# Approximation Algorithms

TSP with 100 stops: no known algorithm finds the optimal route in reasonable time. Richard Karp proved it NP-complete in 1972 [Karp 1972]. But a logistics company needs routes by 8 AM, not a proof of intractability.

The question shifts from "optimal?" to "provably close to optimal, quickly?" This is the domain of approximation algorithms: polynomial-time algorithms with worst-case performance guarantees.

An $\alpha$-approximation returns a solution always within factor $\alpha$ of optimal ($\alpha \ge 1$ for minimization). For metric TSP, Christofides' algorithm (1976) [Christofides 1976] guarantees routes at most 50% longer than optimal — a 1.5-approximation.

## Approximation Ratio

<MathStatement id="approx-def" kind="definition" title="Approximation ratio">
For minimization problem $\Pi$, algorithm $A$ is an $\alpha$-approximation if for all instances $I$, $A(I) \le \alpha \cdot \text{OPT}(I)$ with $\alpha \ge 1$. For maximization, $A(I) \ge \text{OPT}(I)/\alpha$.
</MathStatement>

The ratio must hold for **every** instance, not average-case. A single bad instance that exceeds the bound breaks the guarantee.

<MathStatement id="ptas-def" kind="definition" title="PTAS and FPTAS">
- **PTAS:** For any fixed $\varepsilon > 0$, runs in $O(n^{f(1/\varepsilon)})$ and achieves $(1+\varepsilon)$-approximation.
- **FPTAS:** Runs in $O(n^c \cdot g(1/\varepsilon))$, polynomial in $n$ and $1/\varepsilon$.
- **EPTAS:** Runs in $O(n^c + h(1/\varepsilon))$ with $c$ independent of $\varepsilon$.
</MathStatement>

A PTAS may be useless in practice: if $f(1/\varepsilon) = 2^{1/\varepsilon}$, setting $\varepsilon = 0.1$ yields exponent $2^{10}=1024$, making $n^{1024}$ infeasible. FPTAS avoids this by polynomial dependence on $1/\varepsilon$.

## Vertex Cover: 2-Approximation via Maximal Matching

**Algorithm:** Find a maximal matching $M$. Return both endpoints of each edge in $M$.

**Feasibility.** Every edge is incident to some edge of $M$ (otherwise $M$ could be extended). Hence the matched vertices form a vertex cover.

**Ratio.** Any vertex cover must contain at least one endpoint of each matching edge, so $\text{OPT} \ge |M|$. Algorithm returns $2|M| \le 2\cdot\text{OPT}$.

**Tight example:** $k$ disjoint edges has OPT $= k$, algorithm returns $2k$.

## Counterexample: When 2-Approximation Misleads

The 2-approximation guarantee is tight, but the ratio alone can hide catastrophic absolute error. Consider a star graph $K_{1,n}$: center $c$ connected to $n$ leaves. A maximal matching picks a single edge $(c, \ell_1)$, returning $\{c, \ell_1\}$ — only 2 vertices. OPT is 1 (pick just $c$). Ratio is 2, fine. But now consider a sparse graph with $n$ isolated edges: the algorithm returns $2n$ vertices, OPT is $n$. The ratio is exactly 2, yet the absolute waste grows linearly with $n$.

The **failure mode**: the algorithm has no way to distinguish one endpoint per edge from both. In weighted vertex cover, this naive approach fails badly — the ratio guarantee no longer holds, and one must use LP rounding or primal-dual methods instead [Vazirani 2001].

## Metric TSP

<MathStatement id="metric-tsp-double" kind="theorem" title="Double-tree algorithm">
For metric TSP (complete graph, triangle inequality), the double-tree algorithm is a 2-approximation.
</MathStatement>

**Proof.** Compute an MST $T$. $w(T) \le \text{OPT}$ (deleting one edge from the optimal tour gives a spanning tree). Double $T$: each edge appears twice, creating an Eulerian multigraph of weight $2w(T)$. Take an Euler tour; shortcut repeated vertices using triangle inequality, which cannot increase weight. The resulting Hamiltonian tour has weight $\le 2w(T) \le 2\text{OPT}$. □

<MathStatement id="christofides" kind="theorem" title="Christofides' algorithm (1976)">
For metric TSP, Christofides' algorithm achieves a $3/2$-approximation.
</MathStatement>

<MathProof id="christofides-proof" for="christofides" collapsible open>
Compute MST $T$ ($w(T) \le \text{OPT}$). Let $O$ be odd-degree vertices in $T$ (even count). Compute a minimum-weight perfect matching $M$ on $O$. Claim: $w(M) \le \text{OPT}/2$.

**Proof of claim.** Shortcut the optimal tour to visit only $O$ vertices; weight $\le \text{OPT}$. This tour alternates between two perfect matchings on $O$. The minimum-weight perfect matching is at most the lighter of these two, hence at most $\text{OPT}/2$.

$T \cup M$ has all even degrees, so an Euler tour exists. Shortcut repeated vertices; weight $\le w(T) + w(M) \le 1.5\,\text{OPT}$.
</MathProof>

This remains the best-known ratio for metric TSP since 1976. General TSP has no bounded approximation unless P = NP.

## Knapsack FPTAS

<MathStatement id="knapsack-fptas" kind="theorem" title="0/1 knapsack FPTAS">
For any $\varepsilon > 0$, there is a $(1-\varepsilon)$-approximation for 0/1 knapsack running in $O(n^2 / \varepsilon)$ time.
</MathStatement>

**Proof sketch.** Let $V_{\max} = \max_i v_i$. Set $K = \varepsilon V_{\max} / n$. Define scaled values $v'_i = \lfloor v_i / K \rfloor$. Run value-indexed DP: $DP[j]$ = minimum weight to achieve scaled value $j$. Table size: $\sum v'_i \le n \cdot V_{\max} / K = n^2 / \varepsilon$.

For optimal $S^*$, the scaled total $\ge v(S^*)/K - n$. The DP solution $S$ satisfies $v(S) \ge v(S^*) - nK = v(S^*) - \varepsilon V_{\max} \ge (1-\varepsilon)v(S^*)$. □

### Counterexample: Precision Trap

The FPTAS guarantee is $(1-\varepsilon)v(S^*)$, but the **absolute error** can be large when all item values are small. Suppose $n=100$, $V_{\max}=1$, $\varepsilon=0.1$. Then $K = 0.1 \cdot 1 / 100 = 0.001$, and $v'_i = \lfloor v_i / 0.001 \rfloor$. If all $v_i = 1$, then $v_i' = 1000$, table size $\approx 100 \cdot 1000 = 10^5$, manageable. But if $v_i = 0.01$, then $v_i' = 10$, and the scaled DP loses precision: the rounding may discard items whose total value is below $K$, introducing an absolute error proportional to $nK = \varepsilon V_{\max}$.

**Failure mode:** The FPTAS guarantees a **relative** error bound, but when the optimal value itself is small, the absolute error $\varepsilon V_{\max}$ may be comparable to the optimum. This is inherent: no FPTAS can avoid this when instance values span many orders of magnitude [Vazirani 2001, Ch. 8].

## Set Cover: Greedy $H_n$ Approximation

**Algorithm:** While uncovered elements remain, pick the set covering the most uncovered elements.

**Analysis (charging).** For each element, assign price = cost of covering set / newly covered elements. Total price = greedy cost.

Let $e_1, \ldots, e_n$ be coverage order. When $e_i$ is covered, $\ge n-i+1$ elements remain. Optimal cover $S^*$ has $k$ sets covering all remaining. Some set in $S^*$ covers $\ge (n-i+1)/k$ remaining elements, so greedy's price for $e_i$ $\le k/(n-i+1)$. Sum: $\sum_{i=1}^n k/(n-i+1) = k \cdot H_n = O(k \log n)$.

<MathStatement id="set-cover-hardness" kind="theorem" title="Set cover inapproximability">
Unless P = NP, set cover cannot be approximated better than $(1-o(1))\ln n$ (unweighted) and $c \log n$ for some $c > 0$ (weighted) [Feige 1998].
</MathStatement>

### Failure Mode: Greedy Underestimates Overlap

The greedy algorithm's $H_n$ bound is tight, but the worst-case instances are contrived. In practice, the algorithm often performs well. However, consider a universe of $n$ elements where the optimal cover has $k$ sets, each of size $n/k$, and the greedy algorithm picks sets that overlap heavily. The ratio can approach $\ln n$ even when a much smaller cover exists.

> **Warning.** The $H_n$ bound is worst-case, not average. On real instances, the greedy algorithm often beats its guarantee, but it can fail catastrophically on adversarial inputs. The charging argument is tight: there exist instances where the greedy algorithm achieves exactly $H_n \cdot \text{OPT}$ [Vazirani 2001, Ch. 2].

## Max-Cut: Randomized 0.5-Approximation

**Algorithm:** Assign each vertex independently uniformly to one side.

**Expected value:** For each edge, crossing probability $= 1/2$. Expected cut $= |E|/2 \ge \text{OPT}/2$. Derandomization via conditional expectations yields a deterministic $1/2$-approximation.

Best known: $\approx 0.878$ (Goemans–Williamson 1995) [Goemans & Williamson 1995] using semidefinite programming, optimal under the unique games conjecture.

### Counterexample: The Random Algorithm Can Be Stuck

Consider a complete graph $K_n$ on $n$ vertices. The maximum cut is $(n^2/4)$ edges (approximately $n^2/4$). The random algorithm achieves expected cut $|E|/2 = n(n-1)/4 \approx n^2/4$, which matches OPT. But consider a sparse cycle $C_n$: OPT $= n-1$ (cut all edges by alternating sides), and the random algorithm achieves expected cut $n/2$ — a ratio of $2$, not $1/2$ (since for maximization, ratio is $\text{OPT}/A(I) = (n-1)/(n/2) \approx 2$). The $1/2$-approximation guarantee still holds ($A(I) \ge \text{OPT}/2$), but the ratio is tight on bipartite graphs.

## The PCP Theorem and Inapproximability

<MathStatement id="pcp" kind="theorem" title="PCP theorem (Arora–Safra 1998, Arora et al. 1998)">
NP = PCP[O(log n), O(1)]. Every NP decision problem has a probabilistically checkable proof where the verifier reads $O(\log n)$ random bits and $O(1)$ proof bits.
</MathStatement>

**Consequence.** MAX-3SAT is NP-hard to approximate within $7/8 + \epsilon$ for any $\epsilon > 0$. The trivial random algorithm (satisfying $7/8$ of clauses) is essentially optimal.

<MathStatement id="inapprox" kind="theorem" title="Inapproximability examples">
Under standard assumptions:
- **CLIQUE:** No $n^{1-\epsilon}$-approximation (unless P = NP).
- **SET COVER:** No $(1-\epsilon)\ln n$-approximation (unless P = NP).
- **VERTEX COVER:** 2-approximation exists, but no $(2-\epsilon)$-approximation (under UGC).
- **MAX-3SAT:** No $(7/8 + \epsilon)$-approximation (PCP theorem).
</MathStatement>

## Primal-Dual Method

The primal-dual method constructs an integer solution and a feasible dual solution simultaneously, comparing values to bound the ratio.

<MathStatement id="pd-vertex-cover" kind="theorem" title="Primal-dual 2-approximation for vertex cover">
The LP-rounding-equivalent primal-dual algorithm yields a 2-approximation for unweighted vertex cover.
</MathStatement>

<PlantUML
  code={`@startuml
title Primal-dual iteration for vertex cover
start
:Initialize x = 0, y = 0;
repeat
  :Pick uncovered edge (u, v);
  :Increase y_e until
  sum_{e incident to u} y_e = 1
  or sum_{e incident to v} y_e = 1;
  if (Which endpoint's dual sum reached 1?) then (u)
    :Set x_u = 1;
    :Mark all edges incident to u covered;
  else (v)
    :Set x_v = 1;
    :Mark all edges incident to v covered;
  endif
repeat while (Uncovered edges remain?)
stop
@enduml`}
/>

**Example: Vertex cover via LP duality.** LP relaxation: $\min \sum x_v$ s.t. $x_u + x_v \ge 1$ for each edge, $x_v \ge 0$. Dual: $\max \sum y_e$ s.t. $\sum_{e \ni v} y_e \le 1$, $y_e \ge 0$. Algorithm: initialize $x_v = 0$, $y_e = 0$. While some edge is uncovered, increase $y_e$ until one endpoint's incident dual sum reaches 1; set that $x_v = 1$. Equivalent to maximal matching — a 2-approximation.

2. **Metric TSP failure.** Construct a non-metric instance where double-tree produces a tour arbitrarily worse than OPT.

   <ExerciseSolution hint="Break the triangle inequality with one extremely long edge.">

3. **Transfer: Knapsack variant.** The *unbounded* knapsack problem allows unlimited copies of each item type. Modify the FPTAS to handle this case. Does the same value-scaling approach work? What changes in the DP?

   <ExerciseSolution hint="The DP structure changes from 0/1 per item to unbounded counts. The scaled-value DP now uses unbounded knapsack recurrence: DP[j] = min over items of DP[j - v'_i] + w_i.">
   The same scaling $K = \varepsilon V_{\max}/n$ works. The DP table size remains $O(n^2/\varepsilon)$. The recurrence changes from 0/1 to unbounded: for each scaled value $j$, $DP[j] = \min_i (DP[j - v'_i] + w_i)$. The approximation guarantee $(1-\varepsilon)$ follows identically because the scaling argument is independent of the recurrence form.
   </ExerciseSolution>

4. **Synthesis: Combining approximations.** You have a 2-approximation for problem $A$ and a 3-approximation for problem $B$. Problem $C$ can be solved by running $A$ then $B$ as a pipeline, where the output of $A$ becomes the input to $B$. The optimal solution to $C$ has cost $\text{OPT}_C = \text{OPT}_A + \text{OPT}_B$. What is the approximation ratio for this pipeline? Can you improve it by reordering the pipeline?

   <ExerciseSolution hint="Compose ratios: the output of A is at most 2·OPT_A, which feeds into B producing at most 3·(OPT_B + possibly error from A).">
   Pipeline cost $\le 2\text{OPT}_A + 3(2\text{OPT}_A + \text{OPT}_B)$ — no, careful. Running A gives solution $S_A$ with cost $\le 2\text{OPT}_A$. Running B on the modified instance: the instance for B depends on A's output, so the comparison is to $\text{OPT}_B$ on the original, but B's input is perturbed. The pipeline's cost is at most $2\text{OPT}_A + 3\text{OPT}_B$, so ratio $\le \max(2,3) = 3$. Reordering does not improve the worst case.
   </ExerciseSolution>

5. **Synthesis: Inapproximability and algorithm design.** The PCP theorem implies that MAX-3SAT cannot be approximated better than $7/8$. Why does this not rule out constant-factor approximations for metric TSP? What structural property of metric TSP makes it approximable despite being NP-hard?

   <ExerciseSolution hint="Metric TSP is a special case with additional structure (triangle inequality). The PCP reductions for general TSP exploit arbitrary edge weights.">
   The inapproximability results for general TSP rely on constructing instances with arbitrary edge weights that violate the triangle inequality. The metric condition (triangle inequality) provides enough structure for MST-based bounds. NP-hardness of exact metric TSP does not imply hardness of approximation for metric TSP — the same problem restricted to a special case can become approximable. This highlights the distinction between exact and approximation hardness.
   </ExerciseSolution>

6. **Transfer: Designing a PTAS for subset sum.** Subset sum: given integers $a_1, \ldots, a_n$ and target $T$, find a subset summing to at most $T$ with maximum total. Design a PTAS using the same value-scaling idea as knapsack. What is the running time?

   <ExerciseSolution hint="Scale by K = εT/n, round down, and run exact DP on scaled values.">
   Scale each $a_i$ to $a'_i = \lfloor a_i / K \rfloor$ with $K = \varepsilon T / n$. Run DP: $DP[j]$ = minimum sum to achieve scaled value $j$, table size $O(n^2/\varepsilon)$. The solution achieves sum $\ge (1-\varepsilon)T$ when OPT $\ge T$. The running time is $O(n^2/\varepsilon)$, which is an FPTAS (both PTAS and FPTAS apply here since the DP is polynomial in $n$ and $1/\varepsilon$).
   </ExerciseSolution>

7. **Synthesis: When does the primal-dual method fail?** The primal-dual vertex cover algorithm produces a 2-approximation. Consider the feedback vertex set problem (remove minimum vertices to eliminate all cycles). Can you adapt the primal-dual approach? What obstacle prevents the same ratio?

   <ExerciseSolution hint="The dual constraints for feedback vertex set are more complex; a single vertex can participate in many cycles, making the dual analysis less direct.">
   Feedback vertex set has an LP relaxation, but the dual constraints involve cycles rather than edges. The simple endpoint-selection rule from vertex cover does not carry over. The best known ratio for feedback vertex set is 2 (via a different method), but the primal-dual adaptation requires careful handling of the dual variables associated with cycles rather than edges. This illustrates that primal-dual success depends on the structure of the dual LP.
   </ExerciseSolution>

## Deliberately Omitted

The topics below are important in the broader approximation algorithms landscape but fall outside this lesson's scope for structural or prerequisite reasons. Each is listed with the rationale for its omission and, where applicable, a pointer to where it appears in the curriculum.

- **LP-based approximation for Steiner forest and facility location** — The primal-dual method for these problems demonstrates the schema's full power; included in the advanced approximation module.
- **Semidefinite programming relaxations beyond MAX-CUT** — The Goemans–Williamson SDP framework extends to MAX-2SAT and coloring; omitted because this lesson focuses on combinatorial and LP-rounding techniques accessible without SDP background.
- **Hardness of approximation via the unique games conjecture** — UGC implies tight inapproximability for VERTEX COVER, MAX-CUT, and many others; this is an active research area treated in graduate complexity.
- **Approximation schemes for geometric problems (Euclidean TSP, geometric packing)** — The shifting technique uses a different paradigm; covered in the advanced algorithms module.
- **Proof of the PCP theorem** — One of the deepest results in theoretical CS; this lesson states its consequences without proving the theorem itself.
