# Asymptotic Analysis and Cost Models

**Prerequisites.** Limit definitions, induction, basic combinatorics, and the definition of $O$, $\Omega$, $\Theta$, $o$, $\omega$ notation. Prior exposure to recurrence relations is helpful but not required — the substitution method is developed from first principles.

Suppose you are given two sorting programs. Program A runs in $0.1n^2$ seconds on an input of size $n$; program B runs in $5n \log_2 n$ seconds. Which is faster?

For $n = 10$, A finishes in 10 seconds and B in about 166 seconds. For $n = 100$, A takes 1000 seconds and B about 3320 seconds. A wins at both sizes. At $n = 1000$, A needs 100,000 seconds (over a day); B needs about 49,850 seconds (under 14 hours). Now B is faster. The crossover occurs near $n \approx 440$, where $0.1n^2 = 5n \log_2 n$.

No single number can capture which algorithm is better. The answer depends on $n$, and the gap grows without bound — at $n = 10^6$, A would take over 3000 years; B finishes in a little over 3 years. A language that describes how resource consumption *scales* with input size, abstracting away constant factors and machine details, is needed.

Asymptotic notation provides that language. Paul Bachmann introduced the $O$ symbol in 1894 to bound the error term of asymptotic series in number theory; Edmund Landau formalized its use in 1909. The notation entered computer science through Donald Knuth's 1976 article "Big Omicron and Big Omega and Big Theta" [Knuth, 1976].

The formal definition: $f(n) = O(g(n))$ if there exist constants $c > 0$ and $n_0$ such that $f(n) \le c \cdot g(n)$ for all $n \ge n_0$. The $O$ hides constants and lower-order terms; only the growth rate survives. In our example, $0.1n^2 = \Theta(n^2)$ and $5n\log_2 n = \Theta(n \log n)$. Because $n \log n = o(n^2)$, the asymptotic analysis correctly predicts that B is eventually faster — which no constant-factor comparison could.

Algorithm designers classify algorithms by their asymptotic growth rates, knowing that an $O(n \log n)$ algorithm will eventually outpace an $O(n^2)$ algorithm on all sufficiently large inputs, regardless of hardware or implementation language. The threshold "sufficiently large" may be $n = 10$ or $n = 10^6$ depending on constants, but the asymptotic ordering is invariant.

## When Asymptotics Mislead

Asymptotic analysis predicts long-term growth, but it can mislead in several practically important situations:

> **Counterexample 1: Large constants.** An algorithm with $T(n) = 10^{-6} \cdot 2^n$ is $O(2^n)$, while $T'(n) = 10^{12} \cdot n^2$ is $O(n^2)$. Asymptotically $2^n \gg n^2$, but the crossover is astronomically large: $10^{-6} \cdot 2^n > 10^{12} \cdot n^2$ only when $n > 110$. For all practical input sizes below this threshold, the "exponential" algorithm is faster than the "quadratic" one. Asymptotic analysis alone cannot predict real-world performance on finite inputs.

> **Counterexample 2: Hidden additive terms.** A function $f(n) = 10^{100}n$ is $O(n)$, and $g(n) = n \log n$ is asymptotically larger. Yet for any machine-representable $n$, $n \log n$ is smaller than $10^{100}n$. The asymptotic notation swallows the constant into the existential quantifier, but the constant's magnitude may dominate all feasible inputs.

> **Counterexample 3: Non-uniform growth.** The function $f(n) = n^{1 + \sin n}$ is $O(n^2)$ and $\Omega(n^0)$, but it oscillates unpredictably: at $n = 2\pi k$, $f(n) = n$; at $n = 2\pi k + \pi/2$, $f(n) = n^2$. A single asymptotic bound masks this oscillation, and worst-case analysis that uses the $O$ upper bound may be overly pessimistic for typical inputs.

These failure modes do not invalidate asymptotic analysis — they define its domain of applicability. Asymptotics describe *scaling behavior*, not absolute performance. When comparing algorithms for practical deployment, combine asymptotic insight with constant-factor measurement and input-distribution profiling.

## Multiple-Variable Asymptotics

When an algorithm depends on multiple parameters (e.g., $V$ and $E$ for graphs, $n$ and $d$ for high-dimensional data), the Landau definition must be extended. For $f : \mathbb{N}^k \to \mathbb{R}^{\ge 0}$, we say $f \in O(g)$ if there exist $c > 0$ and $n_1, \ldots, n_k$ such that for all $n_i \ge n_{i,0}$, $f(n_1, \ldots, n_k) \le c \cdot g(n_1, \ldots, n_k)$.

This definition is sensitive to the order of quantification. A bound $O(V+E)$ on a graph algorithm means the cost is linear in the total input size; collapsing to $O(n)$ loses the distinction between sparse ($E = O(V)$) and dense ($E = \omega(V)$) regimes. A correct multiple-variable bound like $O(V \log V + E)$ for Dijkstra with a binary heap captures both regimes: $O(V \log V)$ when $E = O(V)$, and $O(V \log V + E) = O(E)$ when $E = \Omega(V \log V)$.

## Recurrence Solving: Substitution Method

The substitution method proves an asymptotic bound by guessing the closed form and proving it by induction. The induction must account for floors, ceilings, and base cases.

<MathStatement id="substitution-example" kind="theorem" title="Substitution method example">
If $T(n) = 2T(\lfloor n/2\rfloor) + n$ with $T(1) = 1$, then $T(n) = O(n \log n)$.
</MathStatement>

**Proof.** Guess $T(n) \le c n \log n$ for some $c > 0$. Assume the inequality holds for all $m < n$. For $n \ge 2$:

$$
\begin{aligned}
T(n) &\le 2 \cdot c \lfloor n/2\rfloor \log \lfloor n/2\rfloor + n \\
&\le 2 \cdot c (n/2) \log (n/2) + n \\
&= c n (\log n - \log 2) + n = c n \log n - c n + n.
\end{aligned}
$$

We need $T(n) \le c n \log n$, which holds if $-c n + n \le 0$, i.e., $c \ge 1$. Choose $c = 1$; verify base case $T(1) = 1 \le 1 \cdot 1 \cdot \log 1 = 0$ fails because $\log 1 = 0$. Adjust by adding a lower-order term: guess $T(n) \le c n \log n + d$. Then:

$$
T(n) \le 2(c (n/2) \log (n/2) + d) + n = c n \log n - c n + 2d + n = c n \log n + (1-c)n + 2d.
$$

Choose $c = 2$, $d = 1$. Then $(1-2)n + 2 = -n + 2 \le 0$ for $n \ge 2$, and $T(1) = 1 \le 2\cdot 1 \cdot 0 + 1 = 1$ holds. □

This demonstrates the importance of the base case in the substitution method: a guess that works inductively may fail at the boundary, requiring a slack term or a larger constant.

## Master Theorem

The master theorem solves recurrences of the form $T(n) = a T(n/b) + f(n)$ where $a \ge 1$, $b > 1$, and $f$ is asymptotically positive.

<MathStatement id="master-theorem" kind="theorem" title="Master theorem" proofId="master-theorem-proof">
Let $T(n) = a T(n/b) + f(n)$ where $n/b$ is interpreted as $\lfloor n/b\rfloor$ or $\lceil n/b\rceil$. Then:

1. **Leaf-dominated:** If $f(n) = O(n^{\log_b a - \epsilon})$ for some $\epsilon > 0$, then $T(n) = \Theta(n^{\log_b a})$.
2. **Equal work per level:** If $f(n) = \Theta(n^{\log_b a})$, then $T(n) = \Theta(n^{\log_b a} \log n)$.
3. **Root-dominated:** If $f(n) = \Omega(n^{\log_b a + \epsilon})$ for some $\epsilon > 0$, and $a f(n/b) \le c f(n)$ for some $c < 1$ and all sufficiently large $n$ (regularity condition), then $T(n) = \Theta(f(n))$.
</MathStatement>

<MathProof id="master-theorem-proof" for="master-theorem" title="Proof sketch" collapsible>
Expand the recurrence to depth $k = \log_b n$:

$$
T(n) = a^k T(n/b^k) + \sum_{j=0}^{k-1} a^j f(n/b^j).
$$

The leaf term is $n^{\log_b a} T(1)$. The sum represents work across levels. Compare $f(n/b^j)$ to the branching factor $a^j$:

**Case 1.** $f$ grows slower than the branching factor: $a^j f(n/b^j) = O(a^j (n/b^j)^{\log_b a - \epsilon}) = O(n^{\log_b a} (b^{-\epsilon})^j)$. The geometric series $\sum (b^{-\epsilon})^j$ converges, so the sum is dominated by the leaf term.

**Case 2.** $f(n) = \Theta(n^{\log_b a})$, so $a^j f(n/b^j) = \Theta(a^j (n/b^j)^{\log_b a}) = \Theta(n^{\log_b a})$. All $\log_b n$ levels contribute equally, giving $\Theta(n^{\log_b a} \log n)$.

**Case 3.** The regularity condition ensures $a^j f(n/b^j) \le c^j f(n)$ for $c < 1$, so the root term dominates: $\sum_{j=0}^{k-1} a^j f(n/b^j) \le f(n) \sum_{j=0}^{k-1} c^j = O(f(n))$. ∎
</MathProof>

The regularity condition $a f(n/b) \le c f(n)$ is essential. Without it, Case 3 can fail even when $f(n) = \Omega(n^{\log_b a + \epsilon})$. For example, $T(n) = 2T(n/2) + n \cos n$ does not satisfy the regularity condition because $f(n/2) \approx (n/2) \cos(n/2)$ may not dominate $f(n)$.

### Master Theorem Limitations

The master theorem does not cover:
- **Unequal subproblem sizes:** $T(n) = T(n/3) + T(2n/3) + n$.
- **Non-polynomial gap:** $T(n) = 2T(n/2) + n \log n$ (Case 2 applies, but the gap between $n^{\log_b a}$ and $f(n)$ is polylogarithmic, not $n^\epsilon$).
- **Subproblem with negative or non-constant $a$:** $T(n) = \sqrt{n}\, T(\sqrt{n}) + n$.
- **Floors and ceilings that change the branching structure:** Already handled by bounding arguments if $T$ is eventually nondecreasing.

For these cases, use the Akra–Bazzi theorem or direct substitution.

## Akra–Bazzi Theorem

<MathStatement id="akra-bazzi" kind="theorem" title="Akra–Bazzi theorem">
Let $T(n) = \sum_{i=1}^k a_i T(n/b_i) + f(n)$ where $a_i > 0$, $b_i \in (0, 1)$, and $f$ is nonnegative and satisfies a polynomial growth condition. Then:

$$
T(n) = \Theta\left( n^p + n^p \int_1^n \frac{f(u)}{u^{p+1}} \, du \right)
$$

where $p$ is the unique real satisfying $\sum_{i=1}^k a_i b_i^p = 1$.
</MathStatement>

**Example.** $T(n) = T(n/3) + T(2n/3) + n$. Here $a_1 = a_2 = 1$, $b_1 = 1/3$, $b_2 = 2/3$. Solve $(1/3)^p + (2/3)^p = 1$. The solution is $p = 1$ because $1/3 + 2/3 = 1$. Then:

$$
T(n) = \Theta\left( n + n \int_1^n \frac{u}{u^{2}} \, du \right) = \Theta\left( n + n \int_1^n \frac{1}{u} \, du \right) = \Theta(n \log n).
$$

**Example.** $T(n) = 2T(n/4) + \sqrt{n}$. Solve $2 \cdot (1/4)^p = 1$, i.e., $(1/4)^p = 1/2$, so $p = 1/2$. Compute:

$$
T(n) = \Theta\left( \sqrt{n} + \sqrt{n} \int_1^n \frac{\sqrt{u}}{u^{3/2}} \, du \right) = \Theta\left( \sqrt{n} + \sqrt{n} \int_1^n \frac{du}{u} \right) = \Theta(\sqrt{n} \log n).
$$

## Comparison Model and Decision Trees

A comparison-based algorithm receives information only through pairwise comparisons of elements. The model captures sorting, selection, and certain search problems.

<MathStatement id="decision-tree" kind="definition" title="Decision tree model">
A decision tree for input size $n$ is a binary tree where each internal node is labeled by a comparison $a_i : a_j$ and each leaf is labeled by an output. The running time on input $x$ is the depth of the leaf reached. The worst-case running time is the tree height.
</MathStatement>

<MathStatement id="sorting-lower-bound" kind="theorem" title="Comparison sorting lower bound">
Every comparison-based sorting algorithm requires $\Omega(n \log n)$ comparisons in the worst case.
</MathStatement>

**Proof.** The output is a permutation of $n$ elements, and there are $n!$ possible permutations. A binary decision tree of height $h$ has at most $2^h$ leaves. To distinguish $n!$ distinct outputs, we need $2^h \ge n!$, so $h \ge \log_2 (n!) = \sum_{i=1}^n \log_2 i \ge \int_{1}^n \log_2 x \, dx = n \log_2 n - n/\ln 2 + O(1) = \Omega(n \log n)$. □

This lower bound is tight because merge sort and heapsort achieve $O(n \log n)$ comparisons.

An **adversary argument** strengthens lower bounds: the adversary maintains a set of possible inputs consistent with comparisons seen so far. For comparison sorting, the adversary can always answer comparisons to keep the largest possible set of permutations alive, forcing $\log_2(n!)$ comparisons. The adversary's strategy: for a comparison $x : y$, answer according to the outcome that eliminates fewer remaining permutations. Since the initial set has $n!$ elements and each comparison eliminates at most half, at least $\lceil \log_2 (n!) \rceil$ comparisons are needed.

## Input Size and Representation Pitfalls

Choosing the wrong size measure leads to pseudopolynomial algorithms. A primary-testing algorithm that is $O(\sqrt{x})$ in numeric value $x$ is exponential in the input bit length $b = \lfloor \log_2 x \rfloor + 1$, because $\sqrt{x} \approx 2^{b/2}$. An algorithm polynomial in $b$ is truly polynomial; an algorithm polynomial in $x$ is exponential in the input size and is called *pseudopolynomial*.

<MathStatement id="pseudopolynomial" kind="definition" title="Pseudopolynomial algorithm">
An algorithm whose running time is polynomial in the numeric value of the input (or in the largest integer appearing) but exponential in the input length (number of bits) is called pseudopolynomial. Example: dynamic programming for knapsack runs in $O(nW)$ where $W$ is the capacity; this is exponential in $\log W$, making it weakly NP-hard.
</MathStatement>

## Amortized Analysis

Amortized analysis bounds the total cost of a sequence of operations, distributing expensive operations across cheap ones. Unlike worst-case per-operation analysis, amortized analysis accounts for the fact that an expensive operation may create favorable state that makes future operations cheap.

**Example: dynamic array append.** Consider an array that doubles its capacity when full. A single append that triggers a resize costs $O(n)$ to copy $n$ elements. Over a sequence of $n$ appends starting from an empty array, the total cost is $1 + 2 + 4 + \cdots + 2^{\lfloor \log_2 (n-1) \rfloor + 1} < 2n = O(n)$ (each element is copied at most once per resize, and resize events happen at power-of-two boundaries). The amortized cost per append is $O(1)$.

Three methods exist:
- **Aggregate:** Sum the total cost over $n$ operations and divide by $n$.
- **Accounting:** Credit cheap operations with extra "charges" that fund expensive ones later.
- **Potential:** Define a potential function $\Phi$ over the data structure's state; amortized cost = actual cost $+ \Delta\Phi$.

These are developed in full in the data structures course on amortized analysis.

## Recurrence Tables and Recursion Trees

The recursion-tree method expands the recurrence level by level, summing work at each level and then over all levels.

/>

**Worked example: $T(n) = 3T(n/4) + n^2$.** The root does $n^2$ work. At depth $j$, there are $3^j$ subproblems, each of size $n/4^j$, each contributing $(n/4^j)^2 = n^2 / 16^j$ work. Total at depth $j$: $3^j \cdot n^2 / 16^j = n^2 (3/16)^j$. The geometric series $\sum_{j=0}^\infty (3/16)^j = 1/(1-3/16) = 16/13$, so $T(n) = \Theta(n^2)$.

**Worked example: $T(n) = 2T(n/2) + n/\log n$.** At depth $j$, there are $2^j$ subproblems of size $n/2^j$, each costing $(n/2^j) / \log(n/2^j) = n / (2^j (\log n - j \log 2))$. Level total: $2^j \cdot n / (2^j (\log n - j)) = n/(\log n - j)$. Summing over $j = 0$ to $\log n - 1$:

$$
T(n) = \sum_{j=0}^{\log n - 1} \frac{n}{\log n - j} = n \sum_{k=1}^{\log n} \frac{1}{k} = n H_{\log n} = \Theta(n \log \log n).
$$

The master theorem does not apply directly because $n/\log n$ is not $\Theta(n)$ and not $O(n^{1-\epsilon})$. The recursion tree resolves it.

## Correctness and Complexity Together

A loop invariant supports both correctness and complexity analysis. For binary search, the invariant is "if target $t$ exists in $A$, then $t \in A[lo..hi)$." The termination measure $hi - lo$ halves each iteration, providing an $O(\log n)$ bound on iterations. But the invariant also proves correctness: at termination ($lo \ge hi$), if the invariant held, then $t \notin A$ because the active interval is empty.

   (b) $a=4$, $b=2$, $\log_b a = 2$. $f(n) = n^2 \log n = \Theta(n^2 \log^{0+} n)$, but the gap is polylogarithmic. The master theorem's Case 2 extends: $T(n) = \Theta(n^2 \log^2 n)$.
   (c) Falls in the polylogarithmic gap between Case 1 and Case 2. Use recursion tree: $T(n) = \Theta(n \log \log n)$.
   (d) $f(n) = n(2 - \cos n)$ oscillates between $n$ and $3n$; the master theorem does not require monotonicity. $a=1$, $b=2$, $\log_b a = 0$, $f(n) = \Omega(n^\epsilon)$ for any $\epsilon \le 1$. The regularity condition fails because $f(n/2)$ may not dominate $f(n)$. Use Akra–Bazzi: $p=0$, $T(n) = \Theta(\int_1^n f(u)/u \, du) = \Theta(\int_1^n (2-\cos u) \, du) = \Theta(n)$.
   </ExerciseSolution>

4. **Akra–Bazzi.** Solve $T(n) = T(n/2) + T(n/3) + T(n/6) + n$.

   <ExerciseSolution>
   Solve $1/2^p + 1/3^p + 1/6^p = 1$. Try $p=1$: $1/2 + 1/3 + 1/6 = 1$. Then $T(n) = \Theta(n + n \int_1^n u / u^2 \, du) = \Theta(n \log n)$.
   </ExerciseSolution>

5. **Decision tree height lower bound.** Let $\Pi$ be a problem with $m$ distinct possible outputs on inputs of size $n$. Prove that any comparison-based algorithm for $\Pi$ requires $\Omega(\log m)$ comparisons in the worst case. Apply this to: (a) sorting ($m = n!$), (b) finding the minimum of $n$ elements ($m = n$), (c) determining whether all $n$ elements are distinct ($m = 2$).

   <ExerciseSolution>
   A binary decision tree of height $h$ has at most $2^h$ leaves. Each leaf corresponds to a distinct output. To accommodate $m$ outputs, $2^h \ge m$, so $h \ge \lceil \log_2 m \rceil$.
   (a) $m = n!$, so $h \ge \log_2(n!) = \Omega(n \log n)$.
   (b) $m = n$, so $h \ge \lceil \log_2 n \rceil$. This is tight: tournament-style comparison finds the minimum in exactly $n-1$ comparisons, but the decision-tree height bound only gives $\Omega(\log n)$. The $n-1$ lower bound requires an adversary argument, not a leaf-counting one.
   (c) $m = 2$ (all distinct vs. at least one duplicate), so $h \ge 1$. Again the bound is weak; a tighter lower bound of $\Omega(n \log n)$ for element distinctness requires a more sophisticated algebraic decision tree model.
   </ExerciseSolution>

6. **Amortized multi-pop.** A stack supports $\text{push}(x)$, $\text{pop}()$, and $\text{multipop}(k)$ which pops up to $k$ elements. Using the potential method with $\Phi = \text{size}$, prove that the amortized cost of each operation is $O(1)$.

   <ExerciseSolution>
   Let $\Phi = \text{size}$. For push: actual cost 1, $\Delta\Phi = 1$, amortized = $1+1 = 2$. For pop: actual cost 1, $\Delta\Phi = -1$, amortized = $1-1 = 0$. For multipop(k): actual cost $\min(k, \text{size})$, $\Delta\Phi = -\min(k, \text{size})$, amortized = $0$. All are $O(1)$.
   </ExerciseSolution>

7. **Substitution method.** Solve $T(n) = 3T(n/2) + n \log n$ by the substitution method. Show the full induction with base cases.

   <ExerciseSolution>
   Guess $T(n) \le c n^{\log_2 3} - d n$. Verify by induction: $T(n) \le 3(c (n/2)^{\log_2 3} - d (n/2)) + n \log n = 3c n^{\log_2 3} / 2^{\log_2 3} - (3d/2)n + n \log n = c n^{\log_2 3} - (3d/2)n + n \log n$. We need this $\le c n^{\log_2 3} - d n$, i.e., $-(3d/2)n + n \log n \le -d n$, or $n\log n \le (d/2)n$, i.e., $d \ge 2 \log n$, which fails for all $n$. The master theorem gives $T(n) = \Theta(n^{\log_2 3})$ because $f(n) = o(n^{\log_2 3})$.
   </ExerciseSolution>

8. **Synthesis: design and analyze.** You are given an array of $n$ distinct integers and a target threshold $t$. Design an algorithm that counts the number of pairs $(i,j)$ with $i < j$ such that $A[i] + A[j] \ge t$. Analyze its worst-case running time using the appropriate tool (master theorem, Akra–Bazzi, or direct summation). Provide a $\Theta$ bound. Then construct an input family that forces your algorithm to reach its worst-case bound.

   <ExerciseSolution hint="Consider sorting the array first, then using two pointers scanning from both ends.">

   Sort the array ($O(n \log n)$ using merge sort). Then set $lo = 0$, $hi = n-1$, and while $lo < hi$: if $A[lo] + A[hi] \ge t$, then all pairs $(lo, lo+1), \ldots, (lo, hi)$ are valid, so add $hi - lo$ to the count and decrement $hi$; else increment $lo$. Each iteration advances one pointer, giving $O(n)$. Total: $\Theta(n \log n)$. The worst case occurs for any input — sorting dominates. For a $\Theta(n^2)$ alternative, consider the naive double loop without sorting.
   </ExerciseSolution>

## Deliberately Omitted

- **Soft-O notation and polylogarithmic factors.** The $\tilde{O}$ notation that hides $\text{poly}(\log n)$ factors is used in the literature but is not central here; explicit polylog terms are written out throughout.
- **Concrete complexity and instruction-level analysis.** Counting individual machine instructions (RAM model constant factors) is architecture-dependent and in tension with the asymptotic philosophy. Remarks on when constant factors matter appear, but concrete complexity theory is not developed.
- **Worst-case optimal join algorithms and AGM bound.** Information-theoretic lower bounds for database joins use a different analytical framework (fractional edge cover, AGM inequality) covered in the database systems module.
- **Average-case and smoothed analysis.** These refinements are important for algorithms whose worst-case bound is rarely attained (e.g., quicksort, simplex method), but the present lesson focuses on worst-case asymptotic analysis as the foundation.
- **Lower bounds via algebraic decision trees.** The element distinctness problem requires $\Omega(n \log n)$ comparisons in the algebraic decision tree model, but the proof uses degree arguments beyond the combinatorial counting approach here.
