# Dynamic Programming

Dynamic programming (DP) solves problems whose recursive descriptions contain overlapping subproblems and optimal substructure. Instead of recomputing the same state, it evaluates each state once and stores its value. The hard part is not writing a table; it is choosing a state that contains exactly the information needed for future decisions, proving the recurrence, ordering dependencies, and accounting for reconstruction and space.

## From recursion to states

Start with a recursive formulation that exposes choices. Identify a state descriptor s, a value OPT(s), base cases, and transitions from s to smaller states. If two recursive calls have the same descriptor and always require the same answer, they are the same subproblem and can share a memo entry. If the descriptor omits relevant history, sharing states can combine incompatible contexts and produce a wrong answer.

**Worked Fibonacci.** The recurrence (F(n)=F(n-1)+F(n-2)) branches exponentially when implemented directly because the same arguments repeat. A memo table stores F(i) once, giving O(n) time and O(n) space. A bottom-up array computes i=0 through n; if only F(n) is needed, two rolling variables reduce space to O(1). The state is just i because future value depends only on the index.

Optimal substructure requires a theorem, not a visual recurrence. If an optimal solution to state s chooses a transition to t but the suffix solution for t is not optimal, replacing it with a better t-solution improves s, contradiction. This “cut and paste” proof is valid only when the replacement remains feasible under the state’s boundary conditions.

## 0/1 knapsack

Items have weights wi and values vi; choose a subset with total weight at most capacity W maximizing value. Define DP[i][w] as the maximum value using the first i items and capacity w. The recurrence is (max(DP[i-1][w],v_i+DP[i-1][w-w_i])) when wi at most w, otherwise DP[i-1][w]. Base row i=0 or capacity 0 is zero.

The proof partitions optimal subsets by whether item i is absent or present. In the present case, the remaining items solve the first i-1 problem with capacity w-wi; otherwise they solve capacity w. The table has O(nW) time and space, pseudo-polynomial because W may be exponential in its binary encoding. A one-dimensional table must iterate capacity downward so item i is not reused in the same iteration.

To reconstruct a subset, store choice bits or walk backward: if DP[i][w]=DP[i-1][w], omit item i; otherwise include it and set w-=wi. A rolling-value table saves space but loses direct reconstruction unless decisions are stored or recomputed.

## Longest common subsequence

For strings X[1..m] and Y[1..n], LCS[i][j] is the longest common subsequence length of prefixes X[1..i],Y[1..j]. If last characters match, one optimal solution can include them and LCS[i][j]=LCS[i-1][j-1]+1. If they differ, an optimal subsequence omits at least one of them, so take max(LCS[i-1][j],LCS[i][j-1]). The proof considers the last chosen symbols and uses prefix optimality.

Reconstruction follows equal-character diagonal steps and otherwise moves to a cell with the same value. Ties produce different valid LCSs. A two-row computation gives length in O(n) space, while recovering a sequence needs backpointers, divide-and-conquer reconstruction, or recomputation.

## Interval and sequence states

Some problems require two endpoints, such as optimal parenthesization or weighted interval selection. Define DP[i][j] for a contiguous interval and consider a final split k; transitions may be O(n) per state, giving O(n^3). The state must include all boundaries that affect legal combinations. A state that stores only length can lose the symbols or constraints needed to evaluate a split.

Weighted interval scheduling sorts jobs by finish time and defines p(j) as the last job compatible with j. (OPT(j)=max(OPT(j-1),v_j+OPT(p(j)))). The choice is not a greedy commitment: DP compares taking and skipping. Binary search computes p(j) efficiently, giving O(n log n) preprocessing plus O(n) DP.

## DAG shortest paths

In a directed acyclic graph, topologically order vertices. Set distance(source)=0 and others infinity; process vertices in order and relax outgoing edges. Every path to v ends with an edge u→v where u appears earlier, so when v is processed all predecessor distances are final. Negative weights are allowed because no edge returns to an earlier dependency.

The same structure is DP over paths. A predecessor array reconstructs a shortest path. Unreachable vertices remain infinity; arithmetic must avoid adding weights to infinity in fixed-width implementations. If equal distances need deterministic paths, specify tie-breaking and update rules.

## Knapsack-like state design

State dimensions encode constraints: item index, capacity, last choice, remaining budget, or position. Adding a dimension increases time/space; omitting one can make the recurrence invalid. A useful test is “can two histories with the same proposed state have different legal futures or different accumulated objective?” If yes, refine the state; if no, the histories can be merged.

Dominance can reduce states. If one partial solution uses no more resource and has at least as much value as another, the dominated one need not be retained. Pareto frontiers are exact but can be large; approximation schemes discretize values or resources and trade precision for complexity.

## Top-down versus bottom-up

Top-down memoization follows only states reached from the query and is often easiest to derive from a recursive proof. It needs recursion or an explicit stack and a memo status that distinguishes “not computed,” “currently computing,” and “computed,” especially if dependencies can cycle accidentally. Bottom-up tabulation evaluates a known acyclic dependency order and avoids recursion depth, but may fill states irrelevant to the requested answer.

If dependencies form a DAG, any topological ordering of the state graph is valid. For a grid recurrence depending on left and upper cells, row-major order works; for an interval recurrence depending on shorter intervals, increasing length works. The order is a correctness condition, not just an optimization. Reading a cell before its dependencies have been computed silently substitutes an uninitialized or stale value.

## Space compression

Space can be reduced when a state depends only on a bounded frontier. Fibonacci needs two previous values; LCS length needs two rows; 0/1 knapsack can use one row with descending capacity. Compression must preserve dependency direction. If reconstruction or later queries need discarded rows, store decisions, checkpoint rows, or use a divide-and-conquer reconstruction algorithm.

Do not compress away a dimension that is part of the proof. In 0/1 knapsack, ascending capacity reuses the current item and changes the problem. In a grid path problem, overwriting a cell before its right/down consumers read it can mix old and new layers. Write the dependency graph first, then derive the minimal safe storage.

## DP as shortest path in a state graph

Many DP tables can be interpreted as shortest or longest paths in a directed acyclic state graph. Each state is a vertex; each legal transition is an edge with a cost or reward. A topological DP is then path relaxation. This perspective clarifies negative rewards, reconstruction via predecessor edges, and cycle detection. If the state graph has cycles, ordinary acyclic DP is invalid; use a fixed-point algorithm, a different state, or a proof that cycles cannot improve the objective.

The state-graph view also exposes exponential state spaces. A DP is efficient only when the number of distinct states and transitions is manageable. “Use DP” is not a complexity result; count states, transition cost, memory, and the numeric encoding of parameters. Pseudo-polynomial tables may be excellent for small capacities and infeasible for binary-large capacities.

## Approximation and pruning

Exact DP can maintain a Pareto frontier of nondominated partial solutions. If a state has resource r and value v, another with r' at most r and v' at least v dominates it for monotone future choices. Removing dominated states preserves optimum because any extension feasible from the dominated state is no better than the corresponding extension from the dominator. In practice the frontier can still be large; approximation rounds values or resources to bound it.

Pruning needs a proof of dominance, not a heuristic threshold. If future value depends on identity, sequence, or remaining choices, two equal resource/value pairs may not be interchangeable. Include all future-relevant information in the state before applying dominance.

## DP debugging checklist

Test base cases, empty input, one-item input, ties, impossible states, negative values, and reconstruction. Compare against brute force on small instances. Check that every table entry has a witness or an explicit unreachable sentinel, that transitions never read an invalid state, and that the reconstructed solution’s objective equals the table optimum. A correct score with an incorrect reconstruction is an incomplete algorithm.

## Exercises

1. Convert naive Fibonacci to memoized and bottom-up DP with proofs.
2. Derive and prove 0/1 knapsack recurrence and one-dimensional loop direction.
3. Reconstruct an LCS from a table.
4. Design weighted interval scheduling DP.
5. Explain DAG shortest paths with negative edges.
6. Diagnose an incomplete state for a constrained sequence problem.
7. Compare value-table, choice-table, and recomputation strategies.

### Diagnostic solutions


1. State i, base F(0),F(1), transition from i-1/i-2. Memoization evaluates each i once; bottom-up satisfies dependencies. Induction on i proves each stored value equals the recurrence, hence the desired Fibonacci value.

2. Partition by item i absent/present; present leaves capacity w-wi. The 2D recurrence is correct by optimal substructure. In 1D iterate w downward so the previous row’s value is read; upward iteration would allow item i multiple times and solve unbounded knapsack instead.

3. At (i,j), if characters match, append the character and move diagonal; otherwise move to a neighboring cell with equal value. Stop at row or column zero. Reverse collected characters; ties yield alternate optimal subsequences.

4. Sort by finish, compute p(j) by binary search, and set OPT(0)=0, OPT(j)=max(OPT(j-1),v_j+OPT(p(j))). The optimal set either omits j or includes it plus an optimal compatible prefix.

5. Topological order ensures every predecessor of v is processed before v. Relaxation considers every path’s final edge, so the minimum is final. Negative edges are safe because acyclicity prevents repeated improvement cycles; Dijkstra’s nonnegative assumption is unnecessary here.

6. Find two histories with equal proposed state but different remaining legal choices or objective. Their difference proves the descriptor omits information; add the relevant boundary, last item, resource, or automaton state until future behavior is identical.

7. A value table supports optimal score and often reconstruction via recomputation; choice table adds O(number of states) space but makes reconstruction direct; recomputation saves memory but may increase time. State which output is required.
