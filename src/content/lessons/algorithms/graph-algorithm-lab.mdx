# Graph Algorithm Laboratory

Storing the adjacency matrix of a 10-million-vertex graph requires $10^{14}$ entries; at one byte per entry, that is 100 terabytes. An adjacency list storing only the $O(E)$ edges that actually exist fits in gigabytes for sparse graphs. This tension between representation and algorithm performance is the central theme of this laboratory.

Graph algorithms cannot be understood in isolation from the data structures that implement them. A theoretically fast algorithm on an unsuitable representation can be slower than a theoretically slower algorithm that exploits cache-efficient access patterns.

The laboratory connects theoretical invariants to concrete behavior. BFS's queue invariant guarantees shortest paths in unweighted graphs. Dijkstra's settlement invariant justifies greedy extraction. Ford's relaxation primitive unifies all shortest-path algorithms. The cut property guarantees that Kruskal and Prim produce optimal trees. The residual graph certifies optimality of a max flow. In each case, the representation choice (adjacency list vs. matrix, edge list vs. adjacency array) determines whether the algorithm runs in seconds or hours.

## Graph Model

<MathStatement id="graph-model" kind="definition" title="Graph interface">
A graph $G = (V, E)$ exposes:
- $\text{vertices}() \to \mathcal{P}(V)$ — the vertex set
- $\text{neighbors}(v) \to \{(u, w) \mid (v,u) \in E\}$ — adjacency with edge weight $w$
- $\text{weight}(u, v) \to \mathbb{R} \cup \{\infty\}$ — direct edge weight or sentinel
- $\text{has\_edge}(u, v) \to \{\text{true}, \text{false}\}$
</MathStatement>

Support directed and undirected modes. Edge weights may be integer, float, or unit. Document whether parallel edges, self-loops, and negative weights are permitted.

## Experiment 1: Traversal

<MathStatement id="bfs-invariant" kind="theorem" title="BFS distance invariant">
During BFS on an unweighted graph from source $s$, let $Q$ be the queue. If $Q$ contains vertices with distances $d_1 \le d_2 \le \cdots \le d_k$, then $d_k - d_1 \le 1$ and all vertices at distance $d_1$ appear before any at distance $d_1 + 1$. When $v$ is dequeued, $dist[v]$ is final.
</MathStatement>

<MathProof id="bfs-invariant-proof" for="bfs-invariant" collapsible open>
Proof by induction on the number of dequeued vertices. Base: $Q = [s]$, $d(s)=0$, property holds vacuously. Inductive step: dequeue $v$ with distance $d$. Its undiscovered neighbors have distance $d+1$ and are enqueued after all existing $d$-distance vertices and before any $d+2$ vertices (which cannot exist yet). The invariant propagates because the distance gap remains $\le 1$.
</MathProof>

**DFS edge classification.** For undirected graphs: tree edges and back edges only. For directed: tree, back, forward, and cross edges. The classification depends on discovery and finish times from a depth-first search.

def bfs(graph, start):
    visited = set()
    dist = {start: 0}
    q = deque([start])
    visited.add(start)
    while q:
        v = q.popleft()
        for u in graph[v]:
            if u not in visited:
                visited.add(u)
                dist[u] = dist[v] + 1
                q.append(u)

def dfs(graph, start, visited=None):

    visited.add(start)

    return visited

## Experiment 2: Shortest Paths

<MathStatement id="relaxation" kind="definition" title="Edge relaxation">
For edge $(u,v)$ with weight $w(u,v)$:
$$
\text{if } d[v] > d[u] + w(u,v) \text{ then } d[v] \leftarrow d[u] + w(u,v)
$$
</MathStatement>

Implement Dijkstra (nonnegative weights), Bellman-Ford (arbitrary weights, negative cycle detection), and DAG relaxation (topological order). Each must state its preconditions and return explicit status for unreachable vertices and negative cycles.

<MathStatement id="dijkstra-correctness" kind="theorem" title="Dijkstra correctness">
If all edge weights are nonnegative, after settling vertex $v$ (extracting it from the priority queue), $dist[v] = \delta(s,v)$, the true shortest-path distance from source $s$.
</MathStatement>

<MathProof id="dijkstra-proof" for="dijkstra-correctness" collapsible open>
Proof by induction on settlement order. Base: $s$ is settled first with $dist[s] = 0 = \delta(s,s)$. Inductive step: assume all settled vertices have final distances. Let $v$ be the minimum unsettled label. Any alternative $s$-$v$ path must cross from settled to unsettled vertices via some edge $(x,y)$ with $x$ settled, $y$ unsettled. By nonnegativity, that path has length $\ge dist[x] + w(x,y) \ge dist[y] \ge dist[v]$. Thus $dist[v]$ is minimal.
</MathProof>

<CodeRunner client:load

  language="python"
  code={`import heapq

def dijkstra(graph, start):
    dist = {v: float('inf') for v in graph}
    dist[start] = 0
    pred = {}
    pq = [(0, start)]
    while pq:
        d, u = heapq.heappop(pq)
        if d != dist[u]:
            continue
        for v, w in graph[u]:
            if dist[v] > d + w:
                dist[v] = d + w
                pred[v] = u
                heapq.heappush(pq, (dist[v], v))
    return dist, pred

    's': [('a', 4), ('b', 2)],
    'a': [('b', 1), ('t', 5)],
    'b': [('a', 3), ('t', 8)],
    't': []
}

print("Predecessors:", pred)
print("Certificate edges satisfy dist[v] <= dist[u] + w(u,v):")
for u in graph:
    for v, w in graph[u]:
        ok = dist[v] <= dist[u] + w
        print(f"  {u}->{v}: {dist[v]} <= {dist[u]} + {w} = {dist[u]+w} {'OK' if ok else 'FAIL'}")`}

<MathStatement id="bellman-ford-correctness" kind="theorem" title="Bellman-Ford correctness">
After $k$ passes of relaxing every edge, $dist[v] \le \delta^{(k)}(s,v)$ (shortest path using at most $k$ edges). After $|V|-1$ passes, $dist[v] = \delta(s,v)$ if no negative cycle is reachable.
</MathStatement>

## Experiment 3: Minimum Spanning Tree

<MathStatement id="mst-cut" kind="theorem" title="Cut property">
For any cut $(S, V \setminus S)$, the minimum-weight edge crossing the cut belongs to every minimum spanning tree.
</MathStatement>

Implement Kruskal (union-find, $O(E \log V)$) and Prim (priority queue, $O((V+E)\log V)$). Validate MST output using the cycle certificate: for each non-tree edge $e = (u,v)$, $w(e) \ge \max_{f \in P_T(u,v)} w(f)$, where $P_T(u,v)$ is the unique tree path between $u$ and $v$.

<CodeRunner client:load
  title="Kruskal MST with certificate"
  language="python"
  code={`class UnionFind:
    def __init__(self, n):
        self.parent = list(range(n))
        self.rank = [0] * n
    def find(self, x):
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])
        return self.parent[x]
    def union(self, x, y):
        rx, ry = self.find(x), self.find(y)
        if rx == ry: return False
        if self.rank[rx] < self.rank[ry]:
            self.parent[rx] = ry
        elif self.rank[rx] > self.rank[ry]:
            self.parent[ry] = rx
        else:
            self.parent[ry] = rx
            self.rank[rx] += 1
        return True

def kruskal(vertices, edges):
    edges = sorted(edges, key=lambda e: e[2])
    uf = UnionFind(len(vertices))
    mst = []
    for u, v, w in edges:
        if uf.union(u, v):
            mst.append((u, v, w))
    return mst

vertices = [0, 1, 2, 3]
edges = [(0, 1, 1), (0, 2, 4), (1, 2, 2), (1, 3, 5), (2, 3, 3)]
mst = kruskal(vertices, edges)
print("MST edges:", mst)
print("Total weight:", sum(w for _, _, w in mst))`}
  output="MST edges: [(0, 1, 1), (1, 2, 2), (2, 3, 3)]\nTotal weight: 6"

## Experiment 4: Maximum Flow

<MathStatement id="maxflow" kind="theorem" title="Max-flow min-cut theorem (Ford-Fulkerson 1956)">
For any flow network, the maximum flow value equals the minimum $s$-$t$ cut capacity.
</MathStatement>

Implement Edmonds-Karp (BFS-based augmentation, $O(VE^2)$). Return flow value and the residual-reachable cut. Validate: $0 \le f \le c$, flow conservation, residual symmetry, and $|f| = c(S,T)$.

<CodeRunner client:load
  title="Edmonds-Karp max flow"
  language="python"
  code={`from collections import deque

def edmonds_karp(n, cap, s, t):
    flow = 0
    parent = [0] * n
    while True:
        q = deque([s])
        parent = [-1] * n
        parent[s] = s
        while q and parent[t] == -1:
            u = q.popleft()
            for v in range(n):
                if parent[v] == -1 and cap[u][v] > 0:
                    parent[v] = u
                    q.append(v)
        if parent[t] == -1:
            break
        v = t
        bottleneck = float('inf')
        while v != s:
            u = parent[v]
            bottleneck = min(bottleneck, cap[u][v])
            v = u
        v = t
        while v != s:
            u = parent[v]
            cap[u][v] -= bottleneck
            cap[v][u] += bottleneck
            v = u
        flow += bottleneck
    return flow

n = 4
cap = [[0]*n for _ in range(n)]
cap[0][1] = 10  # s -> a
cap[0][2] = 10  # s -> b
cap[1][2] = 5   # a -> b
cap[1][3] = 15  # a -> t
cap[2][3] = 10  # b -> t
print("Max flow:", edmonds_karp(n, cap, 0, 3))`}
  output="Max flow: 20"

**Initialization.** $dist[a] = 0$, $queue = [a]$. All other distances $\infty$.

**Step 1.** Dequeue $a$ (distance 0). Discover neighbors $b$ ($dist[b]=1$), $c$ ($dist[c]=1$). $queue = [b, c]$.

**Step 2.** Dequeue $b$ ($dist[b]=1$, finalized). Discover $d$ ($dist[d]=2$). $queue = [c, d]$.

**Step 3.** Dequeue $c$ ($dist[c]=1$, finalized). Discover $e$ ($dist[e]=2$). $queue = [d, e]$.

**Step 4.** Dequeue $d$ ($dist[d]=2$, finalized). Discover $f$ ($dist[f]=3$). $queue = [e, f]$.

**Step 5.** Dequeue $e$ ($dist[e]=2$, finalized). $queue = [f]$.

**Step 6.** Dequeue $f$ ($dist[f]=3$, finalized). $queue = []$. Algorithm terminates.

**Correctness verification.** Distances: $a=0$, $b=1$, $c=1$, $d=2$, $e=2$, $f=3$. The BFS invariant held throughout: the queue always contained vertices with nondecreasing distances, and each vertex was finalized upon dequeue because any alternative path would have length at least the current level.

## Counterexample: Dijkstra Fails on Negative Edges

<MathStatement id="dijkstra-negative" kind="warning" title="Dijkstra failure mode">
Dijkstra's algorithm requires nonnegative edge weights. A single negative edge can create a path that becomes cheaper after the target vertex is already settled.
</MathStatement>

Consider graph $G$ with $s\to a$ weight 2, $s\to b$ weight 5, $b\to a$ weight $-10$. Dijkstra settles $a$ at distance 2 (step 1, minimum unsettled label). But the true shortest path to $a$ is $s \to b \to a$ with weight $5 + (-10) = -5$. The induction breaks because the shortest path to $a$ goes through $b$, which has a larger label at settlement time.

## Exercises

### Recall

1. State the BFS queue invariant precisely.

<ExerciseSolution hint="What distances appear in the queue at any given time, and when is a vertex's distance finalized?">
After processing level $d$, all vertices at distance $\le d$ are discovered, and the queue contains exactly vertices at distance $d$ (after processing all at $d-1$). Finalization occurs at dequeue because any path through undiscovered vertices has length $\ge$ current level.
</ExerciseSolution>

2. What are the preconditions and running time of Dijkstra vs. Bellman-Ford?

<ExerciseSolution>
Dijkstra: nonnegative weights, $O((V+E)\log V)$. Bellman-Ford: arbitrary weights, detects negative cycles, $O(VE)$. Dijkstra is faster but less general; Bellman-Ford is slower but handles all cases.
</ExerciseSolution>

### Application

3. Show a certificate for Dijkstra correctness (satisfied edge inequalities).

<ExerciseSolution>
Return distances and predecessor tree. Check: for each edge $(u,v)$, $dist[v] \le dist[u] + w(u,v)$ (no violated inequality). For each $v \ne s$, $dist[v] = dist[pred[v]] + w(pred[v], v)$ (predecessor path matches distance).
</ExerciseSolution>

4. Show an MST certificate and explain why it suffices.

<ExerciseSolution>
Return tree edges. Check: exactly $V-1$ edges, no cycle (via union-find), connected (via DFS from any vertex). For each non-tree edge $e=(u,v)$, compute max edge on tree path $u$-$v$; verify $w(e) \ge$ that max. This is the cycle property: if a non-tree edge were lighter than the heaviest tree edge on its cycle, swapping would produce a lighter tree.
</ExerciseSolution>

### Transfer

5. Construct a graph where adjacency list outperforms matrix by a factor $V^2/E$.

<ExerciseSolution>
Sparse graph with $V=10^5$, $E=2V$ (a cycle with some chords). Matrix: $V^2 = 10^{10}$ entries. Adjacency list: $V+E \approx 3\times 10^5$ entries. Factor $\approx V^2/(V+E) \approx 33{,}000\times$.
</ExerciseSolution>

6. Design a shared graph interface supporting all four experiments. Consider which operations each experiment needs — traversal, weight lookup, neighbor iteration.

<ExerciseSolution>
Interface: `vertices()` returns iterable of vertex IDs; `neighbors(v)` returns iterable of `(neighbor, weight)`; optional `edge_weight(u,v)` for fast lookup. Keep algorithm state separate from graph representation.
</ExerciseSolution>

### Synthesis

7. Suppose a graph has both positive and negative edge weights but no negative cycles. Can you combine Dijkstra with a preprocessing step to use it? If so, describe the method and its limitations.

<ExerciseSolution>
Yes — Johnson's algorithm: run Bellman-Ford once to compute a vertex potential $h(v)$, reweight edges to $\hat{w}(u,v) = w(u,v) + h(u) - h(v) \ge 0$, then run Dijkstra from each source. Limitation: the Bellman-Ford preprocessing costs $O(VE)$, so this is only beneficial when computing all-pairs shortest paths on sparse graphs.
</ExerciseSolution>
