# Greedy Algorithms and Exchange Arguments

Suppose you must connect $n$ cities with a road network of minimum total cost. The naive approach — list all subsets of the $n(n-1)/2$ candidate edges and check which are spanning trees, then pick the cheapest — enumerates $2^{O(n^2)}$ possibilities. Hopeless for even modest $n$.

A friend suggests: "Sort edges by cost. Start with an empty set. For each edge in increasing order, add it if it doesn't create a cycle." This greedy procedure — Kruskal's algorithm (1956) — produces an optimal minimum spanning tree. The proof follows from the **cut property**: for any cut, the cheapest crossing edge belongs to some MST. Local optimality aligns with global optimality.

Now consider the knapsack problem: capacity $W = 50$, items with $(v_i, w_i)$ = (60, 10), (100, 20), (120, 30). The greedy analogue — "pick the item with best value-to-weight ratio, commit, repeat" — fails. Greedy picks in density order: item 1 (6.0), item 2 (5.0), then cannot fit item 3, yielding \$160. The optimal solution is \$220 (\$100 + \$120). The locally optimal choice is globally suboptimal.

This tension — greedy works beautifully for MST but fails for knapsack — motivates the formal frameworks in this lesson. When does local optimality imply global optimality? The answer requires exchange arguments, matroid theory (Whitney 1935, Edmonds 1971), and, for problems where exact optimality is impossible, approximation guarantees via submodularity.

## The Exchange Argument Framework

<MathStatement id="exchange-framework" kind="theorem" title="Exchange argument theorem" proofId="exchange-proof" dependsOn={[]}>
Let $P$ be a combinatorial optimization problem with feasible set $\mathcal{F}$, objective $f: \mathcal{F} \to \mathbb{R}$ (maximization). Greedy makes choice $g$ and then solves residual $P'$. Suppose:

1. **Exchange:** For every optimal solution $O$ to $P$, there exists a solution $O'$ such that $g \in O'$, $|O'| = |O|$ (or $f(O') \ge f(O)$), and $O'$ is feasible.
2. **Residual structure:** $P'$ has the same form as $P$ (self-reducibility).

Then the greedy algorithm produces an optimal solution.
</MathStatement>

<MathProof id="exchange-proof" for="exchange-framework" collapsible open>
Let $G$ be greedy's solution. By (1), some optimal $O^*$ contains $g$. The residual instance $P'$ after fixing $g$ inherits the same structure by (2). By the induction hypothesis, greedy solves $P'$ optimally. Thus $G = \{g\} \cup \text{greedy}(P')$ has total value $\ge \text{value}(O^*)$. Since $G$ is feasible, $G$ is optimal. The exchange must preserve feasibility, not just objective — replacing a choice in $O$ with $g$ must keep the solution within $\mathcal{F}$.
</MathProof>

## Interval Scheduling: Exchange by Earliest Finish

**Problem.** Given intervals $[s_i, f_i)$ with $s_i < f_i$, select a maximum-cardinality subset of pairwise non-overlapping intervals.

**Greedy rule.** Choose the interval with the smallest finish time, discard conflicts, repeat.

<MathStatement id="interval-scheduling" kind="theorem" title="Interval scheduling optimality" proofId="interval-proof">
The earliest-finish-time greedy algorithm produces a maximum-cardinality set of non-overlapping intervals.
</MathStatement>

<MathProof id="interval-proof" for="interval-scheduling" collapsible open>
Let $G = \{g_1, g_2, \ldots, g_k\}$ with $g_1$ having the globally minimum finish time. Let $O = \{o_1, \ldots, o_m\}$ be any optimal solution, indexed by increasing finish time.

**Lemma 1.** $f(g_1) \le f(o_1)$ by greedy's choice rule.

**Lemma 2.** There exists an optimal solution containing $g_1$. Construct $O' = (O \setminus \{o_1\}) \cup \{g_1\}$. Since $f(g_1) \le f(o_1)$, all intervals compatible with $o_1$ (starting at or after $f(o_1)$) start at or after $f(g_1)$, hence remain compatible with $g_1$. Feasibility is preserved and $|O'| = |O| = m$, so $O'$ is optimal and contains $g_1$.

**Induction.** Remove $g_1$ and all intervals overlapping $g_1$. The residual instance has the same structure. By the induction hypothesis, greedy solves it optimally. Combined with $g_1$, this yields an optimal solution.
</MathProof>

## Matroid Theory

Matroids provide the broadest known class of problems where the greedy algorithm is optimal for all weight functions.

<MathStatement id="matroid-def" kind="definition" title="Matroid">
A matroid $\mathcal{M} = (E, \mathcal{I})$ consists of a finite ground set $E$ and a family $\mathcal{I} \subseteq 2^E$ of *independent sets* satisfying:

1. **Heredity:** $\emptyset \in \mathcal{I}$; if $A \in \mathcal{I}$ and $B \subseteq A$, then $B \in \mathcal{I}$.
2. **Exchange:** If $A, B \in \mathcal{I}$ and $|A| < |B|$, then $\exists e \in B \setminus A$ such that $A \cup \{e\} \in \mathcal{I}$.
</MathStatement>

<MathStatement id="greedy-matroid" kind="theorem" title="Greedy optimality on matroids" proofId="greedy-matroid-proof">
For a matroid $\mathcal{M} = (E, \mathcal{I})$ with nonnegative weight function $w: E \to \mathbb{R}^{\ge 0}$, the algorithm that sorts $E$ by decreasing weight and adds an element if independence is preserved produces a maximum-weight independent set.
</MathStatement>

<MathProof id="greedy-matroid-proof" for="greedy-matroid" collapsible>
Let $G = \{g_1, \ldots, g_k\}$ be the greedy set in weight order, and $O = \{o_1, \ldots, o_m\}$ be an optimal set (maximum-weight independent set). We prove $|G| = |O|$ and $w(g_i) \ge w(o_i)$ for all $i$.

**Lemma 1:** $|G| = |O|$. If $|G| < |O|$, the exchange axiom applied to $G$ and $O$ yields some $e \in O \setminus G$ with $G \cup \{e\} \in \mathcal{I}$. But greedy rejected $e$ only if adding $e$ would violate independence, a contradiction.

**Lemma 2:** $w(g_i) \ge w(o_i)$. Sort both sets by weight. Let $G_i = \{g_1, \ldots, g_i\}$ and $O_i = \{o_1, \ldots, o_i\}$. $G_i$ is independent and $O_i$ is independent with $|O_i| = i$. By exchange, $G_{i-1} \cup \{o_j\}$ is independent for some $o_j \in O_i \setminus G_{i-1}$. Greedy chose $g_i$ when this element was available, so $w(g_i) \ge w(o_j) \ge w(o_i)$ (since $o_i$ is the smallest in $O_i$). □
</MathProof>

### Examples of Matroids

**Graphic matroid.** $E =$ edges of $G$, $\mathcal{I} =$ acyclic edge subsets (forests). Exchange: if $F$ and $F'$ are forests with $|F| < |F'|$, some edge of $F'$ connects two components of $F$, preserving acyclicity. Greedy with weight $w(e)$ produces a maximum-weight forest (or minimum-weight spanning tree with negated weights).

**Uniform matroid.** $E$ arbitrary, $\mathcal{I} = \{S \subseteq E : |S| \le k\}$. Greedy selects the $k$ heaviest elements. Exchange: if $|A| < |B| \le k$, pick any element from $B \setminus A$.

**Partition matroid.** $E$ partitioned into $E_1, \ldots, E_m$ with capacities $c_i$. $\mathcal{I} = \{S : |S \cap E_i| \le c_i\}$. Greedy sorts all elements globally and adds if capacity remains in the partition.

## Fractional Knapsack: Continuous Exchange

**Greedy rule.** Sort items by density $v_i / w_i$, take items in decreasing density, taking a fraction of the last item if capacity is limited.

<MathStatement id="fractional-knapsack" kind="theorem" title="Fractional knapsack optimality">
The density-sorted greedy algorithm produces an optimal solution to the fractional knapsack problem.
</MathStatement>

**Proof.** Let $x_i$ be the amount taken of item $i$ in any feasible solution. Let $d_i = v_i / w_i$. Suppose an optimal solution has $x_j > 0$ and $x_i < w_i$ for some $i$ with $d_j < d_i$. Transfer $\epsilon = \min(x_j, w_i - x_i)$ weight from $j$ to $i$. The change in value is $\epsilon(d_i - d_j) > 0$, contradicting optimality. Therefore any optimal solution takes items in decreasing density with at most one fractional item. The greedy algorithm produces exactly this structure. □

<CodeRunner client:load
  title="Fractional knapsack"
  language="python"
  code={`def fractional_knapsack(items, capacity):
    items = sorted(items, key=lambda x: x[0]/x[1], reverse=True)
    total = 0.0
    for v, w in items:
        if capacity >= w:
            total += v
            capacity -= w
        else:
            total += v * (capacity / w)
            break
    return total

**Why it fails for 0/1 knapsack.** The exchange requires transferring a *fraction* of an item, which is not allowed under integrality constraints. A high-density small item may consume capacity that, together with other items, would yield more value.

## Huffman Coding: Full Optimality Proof

<MathStatement id="huffman-lemma" kind="lemma" title="Huffman sibling lemma" proofId="huffman-lemma-proof">
Let $a$ and $b$ be the two symbols with smallest frequencies $f_a \le f_b$ in alphabet $\Sigma$. There exists an optimal prefix code where $a$ and $b$ are siblings at the maximum depth (or tied for deepest).
</MathStatement>

<MathProof id="huffman-lemma-proof" for="huffman-lemma" collapsible>
Let $T$ be an optimal binary prefix tree. Let $x$ and $y$ be a pair of deepest siblings in $T$ (maximum depth). If $\{x, y\} = \{a, b\}$, we are done. Otherwise, assume without loss that $a \ne x$. We have $f_a \le f_x$ (by definition of $a$ as minimum) and $\text{depth}(a) \le \text{depth}(x)$ (since $x$ is deepest). Construct $T'$ by swapping the subtrees of $a$ and $x$. The weighted path length changes by:

$$
\Delta = f_a(\text{depth}(x) - \text{depth}(a)) + f_x(\text{depth}(a) - \text{depth}(x)) = (f_a - f_x)(\text{depth}(x) - \text{depth}(a)).
$$

Since $f_a - f_x \le 0$ and $\text{depth}(x) - \text{depth}(a) \ge 0$, we have $\Delta \le 0$. Thus $T'$ is also optimal. After the swap, $a$ is at depth $\text{depth}(x)$. If $b$ is also not a sibling of $a$, repeat with $b$ and $y$. After at most two swaps, $a$ and $b$ are siblings in an optimal tree. □
</MathProof>

<MathStatement id="huffman-optimal" kind="theorem" title="Huffman optimality" proofId="huffman-optimal-proof">
Huffman's algorithm produces an optimal prefix code.
</MathStatement>

<MathProof id="huffman-optimal-proof" for="huffman-optimal" collapsible>
Proof by induction on $|\Sigma|$. Base: $|\Sigma| = 2$, the optimal code assigns 0 and 1 to the two symbols, and Huffman does the same.

Inductive step: By the lemma, there exists an optimal code where the two least frequent symbols $a, b$ are siblings. Contract them into a meta-symbol $c$ with frequency $f_c = f_a + f_b$, and give $c$ the bit string assigned to the pair's parent in the original code. Solve the smaller instance $(\Sigma \setminus \{a,b\}) \cup \{c\}$ optimally by the induction hypothesis. Expanding $c$ back to $a,b$ with appended 0/1 preserves optimality because the contraction adds bit length 1 to $a$ and $b$, contributing $f_a + f_b$ to the total weighted length, which matches the cost of having $c$ at depth $d$ in the contracted tree. □
</MathProof>

## Greedy Approximation: Set Cover

When greedy is not optimal, it can still provide a provable approximation.

**Set cover problem.** Given universe $U$ and a family of subsets $\mathcal{S}$, find a minimum-cardinality subfamily covering $U$.

**Greedy rule.** At each step, choose the subset covering the maximum number of uncovered elements.

<MathStatement id="set-cover-ratio" kind="theorem" title="Greedy set cover approximation ratio">
The greedy algorithm for set cover has approximation ratio $H_n = \sum_{i=1}^n 1/i = \Theta(\log n)$, where $n = |U|$.
</MathStatement>

**Proof.** Let $OPT$ be the optimal cover size. Let $c_t$ be the number of elements covered for the first time by the $t$th greedy choice. At the start of step $t$, at most $n_{t-1}$ elements remain uncovered. Since $OPT$ sets cover all remaining elements, by the pigeonhole principle, some set covers at least $n_{t-1} / OPT$ elements. Greedy picks the best, so $c_t \ge n_{t-1} / OPT$. Therefore $n_t \le n_{t-1} (1 - 1/OPT)$. Unrolling: $n_T < n \cdot e^{-T / OPT}$. Setting $n_T < 1$, we need $T > OPT \cdot \ln n$. Thus greedy uses at most $OPT \cdot \ln n$ sets, giving ratio $\ln n = O(\log n)$. □

This bound is tight: there exist instances where greedy achieves $\Omega(\log n)$ times optimal.

## Submodularity: The General Structure

A function $f: 2^E \to \mathbb{R}$ is *submodular* if for all $A \subseteq B \subseteq E$ and $e \notin B$:

$$
f(A \cup \{e\}) - f(A) \ge f(B \cup \{e\}) - f(B).
$$

Submodularity captures diminishing returns: adding an element to a smaller set yields at least as much gain as adding it to a superset. When greedy maximizes a monotone submodular function under a cardinality constraint, it achieves a $(1 - 1/e)$ approximation ratio.

## Failure Modes and Limitations

Greedy algorithms fail when local optimality does not align with global optimality. Several distinct failure patterns exist:

### Pattern 1: Capacity-constrained selection (0/1 knapsack)

As shown above, fractional exchange is blocked by integrality. The greedy algorithm has no unbounded approximation ratio — it can be arbitrarily bad relative to the optimal solution.

### Pattern 2: Commit-and-suffer (traveling salesman)

The nearest-neighbor heuristic for TSP chooses the closest unvisited city at each step. The resulting tour can be arbitrarily worse than optimal, even in metric spaces.

### Pattern 3: Matroid structure violation

When the feasible sets do not satisfy the exchange property, greedy can fail. For example, finding a maximum-weight vertex cover is NP-hard, even though finding a maximum-weight independent set (matroid-compatible) is easy.

3. **Exchange proof for MST.** Formalize Kruskal's algorithm: prove by exchange that the lightest edge crossing a cut is safe using the cut property.

   <ExerciseSolution>
   Let $e$ be the lightest edge crossing cut $(S, V \setminus S)$. In any MST $T$, either $e \in T$ or there is a path in $T$ between $e$'s endpoints, which crosses the cut at some edge $f$ with $w(f) \ge w(e)$. Replace $f$ with $e$ to get a spanning tree $T'$ with weight $\le w(T)$. Thus every MST is at least as heavy as $T'$, and $T'$ is minimal.
   </ExerciseSolution>

   <ExerciseSolution hint="Consider two partition matroids where the exchange property is not simultaneously satisfiable.">

   Let $E = \{a, b\}$ with $w(a) = 10, w(b) = 1$. $\mathcal{M}_1$: partition matroid with capacity 1 on $\{a\}$. $\mathcal{M}_2$: partition matroid with capacity 1 on $\{b\}$. Simple greedy picks $a$ (highest weight), then checks $b$ — but $\{a, b\}$ is not independent in either matroid. Greedy returns $\{a\}$ with weight 10. Optimal common independent set of size 1? Actually $\{a\}$ is optimal here. Better: $E = \{a, b, c\}$, $w(a)=10$, $w(b)=9$, $w(c)=8$. $\mathcal{M}_1$: partition with $c_1=1$ on $\{a,b\}$, $c_2=1$ on $\{c\}$. $\mathcal{M}_2$: partition with capacity 1 on $\{a,c\}$, capacity 1 on $\{b\}$. Greedy picks $a$, then checks $b$ (fails independence in $\mathcal{M}_2$), checks $c$ (fails independence in $\mathcal{M}_1$). Returns $\{a\}$, weight 10. Optimal: $\{b,c\}$, weight 17. The simple exchange argument fails because swapping must respect both matroid structures simultaneously.

## Deliberately Omitted

- **Greedy embedding and routing in metric spaces** — Using greedy embeddings for geographic routing in networks is a distributed-computing application, not a core greedy algorithm design technique.
- **Polymatroid optimization and submodular flow** — The generalization of matroids to polymatroids and submodular function minimization extends greedy to continuous settings; this is a combinatorial optimization topic beyond the scope.
- **Greedy approximation for maximum coverage beyond set cover** — The greedy $(1 - 1/e)$-approximation for maximum coverage is briefly noted; the full analysis of submodular maximization under matroid and knapsack constraints is covered in the approximation algorithms module.
- **Matroid intersection algorithm (Edmonds)** — Finding a maximum-weight common independent set of two matroids is polynomial via augmenting paths, but the algorithm is substantially more involved than the single-matroid greedy.
- **Online greedy algorithms and regret bounds** — Greedy strategies in adversarial or stochastic online settings (multi-armed bandits) form a separate body of theory.
