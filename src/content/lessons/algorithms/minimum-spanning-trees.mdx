# Minimum Spanning Trees

In 1926, Otakar Borůvka faced a problem while working for the power company in the Bohemian countryside: how do you connect a set of villages to a new electrical grid at minimum total cost? Building a power line between two villages has a known cost; the grid must reach every village, but redundant connections waste money. Borůvka solved this in a paper published the same year (Borůvka 1926, *Práce Mor. Přírodověd. Spol. v Brně* 3: 37–58), giving the world the first algorithm for what we now call the minimum spanning tree (MST).

<MathStatement id="mst-def" kind="definition" title="Spanning tree and MST">
Given a connected, undirected graph $G = (V, E)$ with edge weights $w: E \to \mathbb{R}$, a **spanning tree** is an acyclic subset $T \subseteq E$ such that $(V, T)$ is connected. A **minimum spanning tree** (MST) is a spanning tree minimizing $w(T) = \sum_{e \in T} w(e)$. If $G$ is disconnected, the same concept applied per component yields a **minimum spanning forest**.
</MathStatement>

The central structural insight powering all MST algorithms is the **cut property**: for any partition of the vertices into two nonempty sets, the lightest edge crossing the partition belongs to some MST. This property was independently discovered multiple times (Borůvka 1926; Jarník 1930; Kruskal 1956, *Proc. Amer. Math. Soc.* 7: 48–50; Prim 1957, *Bell System Technical Journal* 36: 1389–1401). Each algorithm takes a different approach — component-merging, tree-growing, and edge-scanning — but all exploit the same underlying cut property.

The dual **cycle property** completes the picture: the heaviest edge on any cycle belongs to no MST. Together, they make greedy algorithms provably correct for MST — a rare phenomenon in combinatorial optimization.

## Cut and Cycle Properties

<MathStatement id="cut-property" kind="theorem" title="Cut property" proofId="cut-property-proof">
Let $S \subset V$ be any proper subset. The lightest edge crossing the cut $(S, V\setminus S)$ belongs to some MST.
</MathStatement>

<MathProof id="cut-property-proof" for="cut-property">
Let $e = (u,v)$ be a lightest crossing edge with $u \in S$, $v \notin S$. Let $T$ be any MST. If $e \in T$, done. Otherwise, adding $e$ to $T$ creates a cycle $C$. The cycle $C$ must cross the cut an even number of times. Since $(u,v)$ is one crossing, there exists another edge $f \in C$ crossing $(S, V\setminus S)$. By minimality of $w(e)$, we have $w(e) \le w(f)$. Consider $T' = T \setminus \{f\} \cup \{e\}$. $T'$ is connected (removing $f$ disconnects $T$ into two components, which $e$ reconnects) and acyclic (the only cycle of $T \cup \{e\}$ is broken by removing $f$). Thus $T'$ is a spanning tree, and $w(T') = w(T) - w(f) + w(e) \le w(T)$. Since $T$ was minimum, $T'$ is also minimum. Therefore $e$ belongs to some MST.
</MathProof>

<MathStatement id="cycle-property" kind="theorem" title="Cycle property" proofId="cycle-property-proof">
Let $C$ be a cycle. The heaviest edge on $C$ belongs to no MST (if strictly heaviest).
</MathStatement>

<MathProof id="cycle-property-proof" for="cycle-property">
Let $e$ be a strict heaviest edge on cycle $C$. Suppose $T$ is an MST containing $e$. Removing $e$ splits $T$ into two components. Since $C$ is a cycle, some other edge $f$ of $C$ also connects these two components (otherwise $e$ would be the only cycle edge connecting the components, contradicting that $C$ is a cycle). Thus adding $f$ to $T \setminus \{e\}$ reconnects it, forming a spanning tree $T' = T \setminus \{e\} \cup \{f\}$. Since $w(f) < w(e)$, $w(T') < w(T)$, contradicting $T$'s minimality.
</MathProof>

## Kruskal's Algorithm

**Algorithm:** Sort edges by weight; scan; add edge $(u,v)$ if $u$ and $v$ are in different components (union-find).

**Proof of correctness.** Let $T_k$ be the set of edges chosen after processing $k$ edges in sorted order. We prove the invariant: $T_k$ can be extended to an MST.

Base: $T_0 = \emptyset$, which can be extended to any MST.

Inductive step: assume $T_{k-1}$ can be extended to some MST $M$. Let $e_k = (u,v)$ be the $k$th edge. If $e_k$ connects two different components of the forest $(V, T_{k-1})$, then by the cut property applied to the cut $(C_u, V \setminus C_u)$ where $C_u$ is the component containing $u$, $e_k$ is the lightest edge crossing this cut (all lighter edges have already been considered and either accepted into $T_{k-1}$ or rejected for creating cycles within components). Thus $e_k$ is safe: some MST $M$ contains $T_{k-1} \cup \{e_k\}$. Set $T_k = T_{k-1} \cup \{e_k\}$.

If $e_k$ connects vertices in the same component, adding it would create a cycle. By the cycle property, $e_k$ is the heaviest edge on that cycle (all other cycle edges were already considered and are in $T_{k-1}$ or were rejected as heavier), so $e_k$ belongs to no MST. Reject $e_k$; $T_k = T_{k-1}$ retains the invariant.

**Complexity:** $O(E \log E)$ for sorting, $O(E \alpha(V))$ for union-find operations (Tarjan 1975, *J. ACM* 22: 215–225).

**Worked example.** Consider the graph with vertices $0,\dots,4$ and edges $(0,1,1)$, $(2,3,1)$, $(1,2,2)$, $(0,2,3)$, $(3,4,4)$, $(1,4,5)$. Sorted edges: $(0,1,1)$, $(2,3,1)$, $(1,2,2)$, $(0,2,3)$, $(3,4,4)$, $(1,4,5)$. Kruskal accepts $(0,1)$, $(2,3)$, $(1,2)$ (all acyclic), then rejects $(0,2)$ (creates cycle $0\!-\!1\!-\!2\!-\!0$), accepts $(3,4)$ (connects component $\{3\}$ to $\{0,1,2\}$), and rejects $(1,4)$ (creates cycle). The MST is $\{(0,1), (2,3), (1,2), (3,4)\}$ with total weight $1+1+2+4 = 8$.

    def __init__(self, n):
        self.parent = list(range(n))
        self.rank = [0] * n
    def find(self, x):
        while self.parent[x] != x:
            self.parent[x] = self.parent[self.parent[x]]
            x = self.parent[x]
        return x
    def union(self, x, y):
        rx, ry = self.find(x), self.find(y)
        if rx == ry: return False
        if self.rank[rx] < self.rank[ry]: rx, ry = ry, rx
        self.parent[ry] = rx
        if self.rank[rx] == self.rank[ry]: self.rank[rx] += 1
        return True

def kruskal(edges, n):
    edges = sorted(edges, key=lambda e: e[2])
    uf = UnionFind(n)
    mst = []
    for u, v, w in edges:
        if uf.union(u, v):
            mst.append((u, v, w))
    return mst

## Prim's Algorithm

**Algorithm:** Grow a single tree from an arbitrary root. Maintain $\text{key}[v]$ = min-weight edge connecting $v$ to the current tree. Extract minimum, add to tree, update neighbors.

**Proof of correctness.** Invariant: the current tree $T$ can be extended to an MST. Base: $T = \emptyset$ (or a single vertex). Inductive step: Let $U$ be the set of vertices already in $T$. The edge $e$ selected by Prim is the lightest edge crossing the cut $(U, V\setminus U)$. By the cut property, some MST contains $e$ together with $T$. Extend $T$ with $e$.

**Complexity:** $O(E \log V)$ with binary heap, $O(V^2)$ with array.

**Worked example.** On the same graph, start with vertex $0$. The frontier edges are $(0,1,1)$, $(0,2,3)$. Pick $(0,1)$. Frontier adds edges from $1$: $(1,2,2)$, $(1,4,5)$. Now frontier: $(0,2,3)$, $(1,2,2)$, $(1,4,5)$. Pick $(1,2)$. New frontier from $2$: $(2,3,1)$ (the $(2,0)$ edge would create a cycle). Continue — the result is the same MST.

## Borůvka's Algorithm

**Algorithm:** For each component $C$, find the lightest edge connecting $C$ to another component. Add all such edges (avoiding duplicates). Each phase reduces the number of components by at least half.

**Proof.** For each component $C$, apply the cut property to cut $(C, V\setminus C)$. The lightest outgoing edge $e_C$ is safe for some MST. Since different components' choices may coincide (two components pick the same edge), accept at most $V/2$ edges per phase, all safe. After $\log V$ phases, a single component remains — the MST.

**Complexity:** $O(E \log V)$ phases, each scanning all edges: $O(E \log V)$.

## MST Uniqueness

<MathStatement id="mst-uniqueness" kind="theorem" title="MST uniqueness" proofId="mst-uniqueness-proof">
If all edge weights are distinct, the MST is unique.
</MathStatement>

<MathProof id="mst-uniqueness-proof" for="mst-uniqueness">
Suppose $T$ and $T'$ are two distinct MSTs. Let $e$ be the minimum-weight edge in the symmetric difference $(T \setminus T') \cup (T' \setminus T)$. Assume without loss $e \in T \setminus T'$. Adding $e$ to $T'$ creates a cycle $C$. All other edges of $C$ are in $T'$, and since $e$ is the minimum edge in the symmetric difference, every edge $f$ in the cycle with $f \notin T'$ (i.e., $f \in T \setminus T'$) satisfies $w(f) \ge w(e)$. At least one cycle edge $g$ is in $T' \setminus T$ (otherwise the cycle would be entirely within $T$, impossible). This $g$ belongs to the symmetric difference, so $w(e) < w(g)$. Thus $T'' = T' \setminus \{g\} \cup \{e\}$ has smaller weight, contradicting $T'$'s optimality. Therefore $T = T'$.
</MathProof>

## Counterexamples and Failure Modes

### Ties break uniqueness

If some edge weights are equal, multiple MSTs can exist. Consider a triangle with edges $(a,b,1)$, $(b,c,1)$, $(a,c,2)$. Both $\{(a,b), (b,c)\}$ (total 2) and $\{(a,c), (b,c)\}$ (total 3) are spanning trees, but only the first is an MST. The second MST candidate fails because swapping $(a,c)$ (weight 2) for $(a,b)$ (weight 1) yields a lighter tree. The two MSTs are actually $\{(a,b), (b,c)\}$ and $\{(a,b), (a,c)\}$ — both have total weight $1+2 = 3$? No — $\{(a,b), (a,c)\}$ has weight $1+2 = 3$, which is heavier than $\{(a,b), (b,c)\}$ (weight $1+1 = 2$). So with weights 1, 1, 2, the only MST is $\{(a,b), (b,c)\}$. For a genuine non-uniqueness example, consider a square $a\!-\!b\!-\!c\!-\!d\!-\!a$ with all edges weight 1. Any three edges forming a tree are MSTs — there are four distinct MSTs, all with total weight 3.

### Negative weights are fine

MST algorithms handle negative edge weights without modification. The cut and cycle properties do not require non-negativity. Contrast this with shortest paths, where negative cycles cause Dijkstra to fail. The MST problem is strictly easier in this respect.

### Disconnected graphs

The algorithms produce a **minimum spanning forest** covering all vertices reachable within each component. No algorithm can connect disconnected components without additional edges.

### Directed graphs (minimum arborescence)

The MST problem on directed graphs is the **minimum arborescence** problem (rooted directed spanning tree), which does **not** satisfy the cut property. A simple greedy algorithm fails. Chu–Liu/Edmonds' algorithm (Edmonds 1967, *J. Res. Nat. Bur. Standards* 71B: 233–240) solves it in $O(EV)$ time using cycle contraction — an altogether different technique.

## The Matroid Structure

Spanning forests form a **graphic matroid** (Whitney 1935, *Amer. J. Math.* 57: 509–533). The ground set is $E$; independent sets are acyclic subgraphs. The matroid greedy theorem (Edmonds 1971, *Mathematical Programming* 1: 127–136) explains Kruskal's correctness: sorting by weight and adding if independence is preserved produces a minimum-weight basis.

Adding degree constraints, hop limits, or Steiner nodes destroys the matroid property. The resulting problems (degree-constrained MST, shallow-light tree) are NP-hard or require different techniques.

## MST Verification

Given a spanning tree $T$, how do we verify it is minimum? The **cycle property certificate**: for every non-tree edge $e$, $w(e) \ge \max_{f \in P_T(u,v)} w(f)$ where $P_T$ is the unique $u$-$v$ path in $T$.

<MathStatement id="verification-cert" kind="theorem" title="MST verification certificate" proofId="verification-proof">
A spanning tree $T$ is an MST iff for every non-tree edge $e = (u,v)$, $w(e) \ge$ the maximum edge weight on the unique $u$-$v$ path in $T$.
</MathStatement>

<MathProof id="verification-proof" for="verification-cert">
If some non-tree edge $e$ violates the inequality ($w(e) < \max_{f \in P} w(f)$), let $f$ be the heaviest edge on $P$. Then $T \setminus \{f\} \cup \{e\}$ is a lighter spanning tree, so $T$ is not minimum. Conversely, if the inequality holds for all non-tree edges, the exchange argument (via the cut property) shows $T$ is minimum.
</MathProof>

Verification can be done in $O(n)$ time via Tarjan's off-line LCA algorithm (Tarjan 1979, Tech. Rep. STAN-CS-79-729) or in $O(n \log n)$ with binary lifting.

## Exercises

1. **Cut property converse.** Prove: if an edge $e$ belongs to some MST, then there exists a cut for which $e$ is the unique lightest edge.

   <ExerciseSolution hint="Remove $e$ from an MST that contains it and examine the resulting cut. Why must every other crossing edge be strictly heavier?">

   Let $T$ be an MST containing $e$. Removing $e$ from $T$ partitions $V$ into two components $S$ and $V\setminus S$. For this cut, $e$ is the unique lightest crossing edge because any other crossing edge $f$ would be heavier (otherwise $f$ could replace $e$ in $T$ to obtain a lighter tree).
   </ExerciseSolution>
2. **Cycle property converse.** Prove: if edge $e$ belongs to no MST, then there exists a cycle where $e$ is the unique heaviest edge.

   <ExerciseSolution hint="Add $e$ to any MST and analyze the cycle it creates. Why must $e$ be the unique heaviest edge on that cycle?">

   Take the minimum spanning tree $T$ and add $e$, creating a cycle $C$. On $C$, $e$ must be the unique heaviest edge (otherwise $e$ could replace the heaviest edge on $C$ and remain an MST). This cycle certifies that $e$ belongs to no MST.
   </ExerciseSolution>
3. **Maximum spanning tree.** Modify Kruskal and Prim to find a maximum-weight spanning tree. Prove correctness by negating weights or by modifying the cut/cycle properties.

   <ExerciseSolution>
   Negate all weights, or equivalently sort edges in decreasing order and apply Kruskal (add if acyclic). The cut property for maximum weight: the heaviest crossing edge of any cut belongs to some maximum spanning tree.
   </ExerciseSolution>
4. **Minimum bottleneck spanning tree.** A spanning tree minimizing the maximum edge weight is called a minimum bottleneck spanning tree. Prove that every MST is a minimum bottleneck spanning tree. Show that the converse is false.

   <ExerciseSolution>
   Let $T$ be an MST and $b(T) = \max_{e \in T} w(e)$. For any other spanning tree $T'$, consider the cut determined by removing the edge achieving $b(T)$ from $T$. The crossing edge in $T'$ has weight at least $b(T)$ (cut property), so $b(T') \ge b(T)$. Thus MST is minimum bottleneck spanning tree. Converse false: consider a triangle with weights 1, 2, 2; the tree with edges 2,2 is a minimum bottleneck (bottleneck 2) but not minimum total weight (which would be 1+2=3 vs 2+2=4).
   </ExerciseSolution>
5. **Reverse-delete algorithm.** Start with all edges; delete edges in decreasing weight order if removal does not disconnect the graph. Prove correctness using the cycle property.

   <ExerciseSolution>
   Sorted edges descending. When considering $e$, if removal disconnects the graph, $e$ must be in every spanning tree, so keep it. If removal preserves connectivity, $e$ can be omitted from some spanning tree; by the cycle property (there exists a cycle with $e$ as heaviest), $e$ is not needed. The remaining edges form an MST.
   </ExerciseSolution>

### Transfer and Synthesis

6. **MST on a line.** Given $n$ points on the real line with Euclidean distances as weights, describe the MST without computation. What is the maximum degree of any vertex in this MST?

   <ExerciseSolution hint="Consider the geometric structure of points on a line. Which edges must every MST include?">

   The MST connects each point to its nearest neighbor(s). On a line, the MST is simply the path connecting consecutive points after sorting. Every interior vertex has degree 2; endpoints have degree 1. No vertex can exceed degree 2 because any edge that skips a point would be heavier than connecting that point directly.
   </ExerciseSolution>
7. **Synthesize: MST from scratch.** Design an MST algorithm that does not sort edges globally and does not use a priority queue. Prove its correctness. (Hint: consider partitioning edges by weight range and processing each bucket.)

   <ExerciseSolution hint="Bin edges by logarithmic weight ranges. Process each bin with a union-find structure. What invariant must hold between bins?">

   Partition edges into $k$ bins $B_1,\dots,B_k$ where $B_i$ contains edges with weight in $(2^{i-1}, 2^i]$. Process bins in increasing order. For each bin, run Kruskal-like scanning within the bin, using union-find to detect cycles. Correctness: when processing bin $i$, any edge in a later bin is heavier than all edges in bin $i$, so the relative ordering within each bin is irrelevant — only the interleaving of weights matters. Within a bin, edges are cheap enough that any MST edge in a later bin that could replace a bin-$i$ edge would be heavier, violating the cut property. This achieves $O(E \log W)$ where $W$ is the max weight ratio.
   </ExerciseSolution>
8. **Critical edges.** An edge $e$ is *critical* if it belongs to every MST. Design an $O(E \log V)$ algorithm to find all critical edges. Prove its correctness.

   <ExerciseSolution hint="Consider edges of equal weight. Under what condition can an edge be absent from some MST? Use the cycle property.">

   Sort edges by weight. Process equal-weight groups together. For each weight group $G$, consider the forest formed by all lighter edges. An edge $e \in G$ is critical iff $e$ is the unique edge of $G$ crossing some cut in this forest. Equivalently, build the forest using all lighter edges; then $e \in G$ is critical iff its endpoints lie in different components of that forest **and** no other $G$-edge connects those two components. Run union-find on lighter edges, then for each weight group, check connectivity within the group. Complexity: $O(E \log V)$ for sorting and $O(E \alpha(V))$ for union-find.
   </ExerciseSolution>
9. **Arborescence counterexample.** Construct a directed graph with non-negative edge weights where selecting the cheapest incoming edge for each vertex (the directed analogue of Prim's cut argument) fails to produce a minimum-cost arborescence.

   <ExerciseSolution hint="Consider a root at vertex 0. Make vertex 1 have two incoming edges: one cheap from 0, one slightly more expensive from 2. Arrange weights so the cheap edge choice leads to overall higher total cost.">

   Vertices: 0 (root), 1, 2. Edges: $0 \to 1$ (cost 1), $0 \to 2$ (cost 10), $2 \to 1$ (cost 2). The cheapest incoming edge for vertex 1 is $0 \to 1$ (cost 1). Selecting it yields arborescence $\{0\to1, 0\to2\}$ (cost 11). But $\{0\to2, 2\to1\}$ is an arborescence (cost 12) — actually, 12 > 11, so this isn't a counterexample. Let's refine: $0\to1$ (cost 100), $0\to2$ (cost 1), $2\to1$ (cost 1). Greedy picks $(0\to2, 1)$ and $(2\to1, 1)$, arborescence cost 2 with edges $\{0\to2, 2\to1\}$. This is actually optimal. The failure mode requires a cycle trap: $0\to1$ (cost 1), $1\to2$ (cost 1), $2\to1$ (cost 1), $0\to3$ (cost 10), $3\to2$ (cost 1). Greedy picks $0\to1$, $2\to1$, $1\to2$, $3\to2$ — but $\{0\to1, 1\to2, 0\to3\}$ has a cycle $1\to2\to1$. The greedy incoming-edge rule creates cycles; breaking them requires the Chu–Liu/Edmonds contraction method.
   </ExerciseSolution>
10. **Application: clustering via MST.** Given $n$ points in the plane, removing the $k-1$ heaviest edges from the Euclidean MST partitions the points into $k$ clusters. This is the single-linkage clustering algorithm. Prove that for any $k$, this produces a $k$-clustering minimizing the maximum inter-cluster distance (the **minimum spanning tree clustering** property).

    <ExerciseSolution hint="Use the cycle property. Suppose a better clustering existed — what edge would violate the MST property?">

    Let $T$ be the MST. Removing $k-1$ heaviest edges yields $k$ clusters $C_1,\dots,C_k$. Suppose some $k$-clustering $C'$ achieves a smaller maximum inter-cluster distance $d' < d$ where $d$ is the weight of the $(k-1)$th heaviest edge in $T$. Then there must exist two points in different $C'$ clusters that are connected by an edge $e$ of weight $\le d' < d$ in $T$ (otherwise the $k$ clusters would be disconnected in $T$, contradicting that $T$ has only $k-1$ edges heavier than $d$). But then $e$ is lighter than the heaviest inter-cluster edge in the MST clustering, meaning $C'$ separates two points connected by a lighter edge — which the cycle property would forbid the MST from including. Contradiction.
    </ExerciseSolution>

## Deliberately Omitted

**Expected-time linear MST algorithms (Karger–Klein–Tarjan).** A randomized algorithm achieving $O(V+E)$ expected time uses random sampling and Borůvka phases; the deterministic near-linear MST via Chazelle's soft heap is an advanced data-structure result. These build on the same cut/cycle properties but require probabilistic analysis and soft heaps respectively — a full treatment belongs in an advanced algorithms module.

**Dynamic MST (maintaining MST under edge insertions/deletions).** The link-cut tree data structure (Sleator–Tarjan 1983, *J. Comput. System Sci.* 26: 362–391) supports dynamic connectivity and MST maintenance in $O(\log V)$ per operation. This is covered in the advanced data structures module.

**Minimum Steiner tree and its approximation.** The Steiner tree problem (connecting a subset of vertices) is NP-hard with a 2-approximation via MST on the metric closure. This appears in the approximation algorithms lesson.

**Parallel and distributed MST algorithms.** Borůvka's algorithm parallelizes naturally (each phase is trivially parallel), and there exist $O(\log V)$-time PRAM algorithms for MST. The distributed MST algorithms (Gallagher–Humblet–Spira 1983, *IEEE Trans. Comput.* 32: 362–371) are covered in the distributed systems module.
