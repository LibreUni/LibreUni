# Network Flow and Cuts

A flow network models divisible movement through directed edges with capacities. A source s supplies flow, a sink t receives it, and intermediate vertices conserve flow. The maximum-flow problem asks for the greatest value that can be sent from s to t. Its algorithms are a rich example of invariant-driven improvement: residual edges encode how an existing solution can be repaired, and cuts provide a matching optimality certificate.

## Flow definition

For capacity c(u,v) at least 0, a flow f satisfies capacity (0 at most  f(u,v) at most  c(u,v)) and conservation (sum_u f(u,v)=sum_w f(v,w)) for every intermediate v. The value is net flow leaving s, equal to net flow entering t under conservation. Parallel edges can be represented separately or aggregated; reverse arcs in a residual graph are algorithmic records, not necessarily original network edges.

An s-t cut partitions vertices into S containing s and T containing t. Its capacity is the sum of capacities from S to T. Any flow value is at most every cut capacity: conservation cancels internal flows, and net source flow crosses the boundary, bounded edgewise by capacities. This weak duality is the upper-bound half of max-flow min-cut.

## Residual networks

Given flow f, the residual capacity from u to v is (c_f(u,v)=c(u,v)-f(u,v)) on a forward edge, plus f(u,v) on a reverse edge. A residual path from s to t permits increasing flow along forward edges and canceling flow along reverse edges by the bottleneck amount. Updating f preserves capacity and conservation at internal vertices because path increments enter and leave each internal vertex equally.

**Worked augmentation.** Suppose paths s→a capacity 3, a→t 2, s→b 2, b→t 3, and a→b 1. Augment s-a-t by 2, then s-b-t by 2. A residual path may use reverse a→s or b→a to reroute flow if a later path needs capacity. Greedy path choices affect iteration count, but residual reversals preserve the ability to correct earlier choices.

Ford-Fulkerson repeatedly finds an s-t residual path and augments by its bottleneck. If capacities are integers, each augmentation increases integral flow by at least one and terminates, though the number of augmentations can be large. With irrational capacities, arbitrary path choices can lead to nontermination; with rational capacities, scaling and precise arithmetic matter.

## Edmonds-Karp and complexity

Edmonds-Karp chooses a shortest residual path in edge count using BFS. Each augmentation saturates an edge whose distance from s can increase only O(V) times before it leaves critical status; total augmentations are O(VE), each BFS O(E), yielding O(VE^2). The proof tracks distance labels and a critical-edge argument, not just the fact that BFS finds a path.

Capacity scaling finds augmenting paths using residual capacity thresholds, reducing the number of relevant augmentations for large integer capacities. Dinic’s algorithm builds a level graph by BFS and sends a blocking flow; its general bound is O(V^2E), with stronger bounds for special networks. The right algorithm depends on V,E, capacities, and whether the network is unit, bipartite, or sparse.

## Max-flow min-cut theorem

If no s-t path exists in the residual graph, let S be vertices reachable from s. Every edge from S to T is saturated, or else its residual forward capacity would make T reachable. Every edge from T to S carries zero flow, or its residual reverse capacity would make S reachable. Thus flow value equals the capacity of the cut (S,T), proving optimality. Together with weak duality, this establishes max-flow=min-cut.

The reachable set is an optimality certificate. A solver can return both flow and cut, and a validator checks capacities, conservation, flow value, and equality with cut capacity. This certificate catches implementation errors more effectively than checking only that no path was found by one particular search.

## Integrality and modeling

If capacities are integers, augmenting algorithms starting from zero produce integral flows because bottlenecks are integers. This supports assignment and matching reductions. In a bipartite graph, create source-to-left capacity 1, left-to-right edges capacity 1, and right-to-sink capacity 1. An integral max flow corresponds to a matching; max-flow min-cut yields Hall-style bottleneck conditions.

Many problems become flow through a careful network: edge-disjoint paths, project selection with closure constraints, circulation with demands, and scheduling under capacity. A modeling mistake—omitting conservation, allowing an invalid edge direction, or choosing the wrong capacity—can produce a mathematically optimal answer to the wrong problem. State the correspondence between feasible objects and flows in both directions.

## Circulation and lower bounds

A circulation has no distinguished source/sink and may have lower bounds ell(u,v) at most f(u,v) at most c(u,v). Convert lower bounds by assigning baseline ell, computing vertex demands, and adding a super-source/sink to satisfy imbalances; a feasible circulation exists iff all super-source demand can be sent. For demands and costs, min-cost flow adds an objective and requires different optimality arguments.

## Matching and vertex capacities

Vertex capacities can be modeled by splitting vertex v into v_in→v_out with capacity b(v). Directed edge capacities then constrain flow through the vertex. A source/sink choice and edge orientation must reflect whether paths may reuse vertices. The split construction is a reusable reduction, but explain how paths map back and how artificial edges affect capacity.

## Implementation invariants

Represent each original directed edge with a residual forward arc and a reverse arc linked by indices. The invariant is that residual capacities are nonnegative, reverse updates are symmetric, and the original flow can be reconstructed from the paired capacities. When parallel original edges exist, do not identify them merely by endpoints unless aggregation is semantically valid; each edge may have a different identity or lower bound.

An augmentation routine must find a path whose every residual capacity is positive, compute the minimum along that path, update every arc and its reverse, and never update an edge twice under an inconsistent direction. A stale BFS parent array can create a non-path or reuse a previous route. Reset visited/parent state for every residual search and stop reconstruction at s; validate that t was reached before dereferencing parents.

## Numerical and capacity issues

With integer capacities, use an integer type wide enough for the total possible flow; summing many capacities can overflow even if each edge fits. Floating capacities create equality and termination concerns; use tolerances only with a proof or policy for residual near-zero values, because an incorrect tolerance can invent or remove capacity. Rational scaling can restore integer arithmetic when denominators are bounded.

If an edge capacity is effectively infinite, choose a sentinel larger than any feasible source supply but still safe for arithmetic. Do not use the maximum machine integer and add to it. A capacity parser should reject negative values, malformed endpoints, and values outside the documented domain before constructing the network.

## Modeling checklist

Draw one flow unit as a feasible object and trace it through the reduction. Check source/sink balance, intermediate conservation, every capacity meaning, and whether splitting or merging paths changes the original object. Then map an arbitrary feasible flow back to a solution, possibly decomposing it into paths and cycles. Cycles may carry flow without changing value; remove them when the target object does not include cycles.

## Testing and certificates

For small integer networks, enumerate feasible integral flows or compare against a trusted linear-programming result. Include zero-capacity edges, disconnected source/sink, parallel arcs, antiparallel arcs, bottlenecks, multiple equal paths, and capacities near numeric limits. After solving, verify conservation at every intermediate vertex, capacity bounds, residual symmetry, and max-flow/min-cut equality using the residual-reachable set.

If the returned flow value is not equal to the cut capacity, either the flow is not maximal, the cut set was computed from the wrong residual state, or an update invariant is broken. Keep the final residual graph and certificate for debugging; a single scalar flow value is not enough evidence.

## Exercises

1. State flow feasibility and prove path augmentation preserves it.
2. Prove every flow is bounded by every s-t cut.
3. Trace residual reverse-edge rerouting.
4. Explain Edmonds-Karp’s critical-edge complexity idea.
5. Prove max-flow min-cut from a saturated reachable cut.
6. Reduce bipartite matching to max flow and prove integrality correspondence.
7. Model vertex capacities by splitting.

### Diagnostic solutions


1. Forward augmentation increases flow on path edges and reverse augmentation cancels existing flow; capacities remain within bounds by bottleneck choice. Each internal path vertex receives and sends the same delta, preserving conservation.

2. Sum conservation over vertices in S excluding s. Internal edges cancel, leaving flow value equal to net flow crossing the cut. Forward crossing flow is at most cut capacity and reverse contribution cannot increase it, so value at most capacity.

3. If a prior path sends flow a→b, residual b→a can cancel it. An augmentation using b→a reduces f(a,b) and changes adjacent conservation consistently; the new flow may route the same total value along a different original path.

4. BFS distances never decrease after augmentation. When a shortest-path edge saturates, it cannot remain critical at the same distance; its distance must rise before it can be critical again. Summing O(E) possible critical events over O(V) rises gives O(VE) augmentations, each O(E).

5. Let S be residual-reachable from s when no augmenting path exists. Every S→T edge is saturated and every T→S edge carries zero flow. Flow value therefore equals capacity(S,T), and weak duality makes both optimal.

6. Unit source/left and right/sink capacities enforce each vertex used at most once; middle edges represent allowed pairs. Integral flows choose whole edges and correspond to matchings; a matching produces a flow of equal value. Thus maximum matching size equals max flow.

7. Replace v by v_in→v_out with capacity b(v), redirect incoming edges to v_in and outgoing from v_out. Any flow uses at most b(v) through the split edge; any vertex-capacitated path/flow maps back by contracting it.
