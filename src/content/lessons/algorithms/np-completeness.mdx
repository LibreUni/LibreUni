# NP-Completeness and Algorithmic Limits

A logistics company must route delivery vehicles through 500 cities. A chip designer must place a million transistors on a die. A cryptographer must prove a protocol secure against any adversary. All three face the same barrier: a class of problems for which no efficient algorithm is known, yet no one has proved that none exists.

In 1971, Stephen Cook formalized this barrier. He asked: can theorem-proving be automated efficiently? He encoded this as Boolean satisfiability (SAT) — given a propositional formula, does there exist an assignment of true/false to its variables that makes the whole formula true? Cook proved that SAT is as hard as any problem in a class he called NP (nondeterministic polynomial time) [Cook 1971]. This was the Cook–Levin theorem, the first NP-completeness result.

Richard Karp (1972) recognized the broader implications. He constructed polynomial-time reductions from SAT to 21 fundamental combinatorial problems — including the traveling salesman problem, knapsack, clique, vertex cover, and Hamiltonian cycle — proving each is equally hard [Karp 1972]. If any one of them admits a polynomial-time algorithm, then all of them do; conversely, if any is provably exponential, then all are.

The question of whether P equals NP is one of the seven Clay Millennium Problems, carrying a $1M prize [Clay 2000]. A proof that P = NP would revolutionize optimization, cryptography, and artificial intelligence. A proof that P $\neq$ NP would confirm what most researchers suspect: that thousands of practically important problems are fundamentally intractable in the worst case.

## Decision Problems and Certificates

Complexity classes are defined for *decision problems* (languages) — sets of strings encoding yes-instances. Every algorithm question is reframed as: does string $x$ belong to language $L$?

<MathStatement id="p-def" kind="definition" title="P">
A language $L \subseteq \{0,1\}^*$ is in P if there exists a deterministic Turing machine $M$ and polynomial $p$ such that for all $x$, $M$ accepts $x$ iff $x \in L$, and $M$ halts within $p(|x|)$ steps.
</MathStatement>

<MathStatement id="np-def" kind="definition" title="NP">
A language $L$ is in NP if there exists a polynomial-time verifier $V$ and polynomial $p$ such that for all $x$:
- If $x \in L$, then $\exists y$ with $|y| \le p(|x|)$ and $V(x,y)$ accepts.
- If $x \notin L$, then $\forall y$ with $|y| \le p(|x|)$, $V(x,y)$ rejects.

The string $y$ is called a *certificate* or *witness*.
</MathStatement>

The key insight: NP is defined by **verification**, not generation. A problem is in NP if a proposed solution can be checked quickly, even if finding one requires exhaustive search.

## Polynomial Reductions

<MathStatement id="reduction" kind="definition" title="Polynomial-time many-one reduction">
$A \le_p B$ if there exists a polynomial-time computable function $f: \{0,1\}^* \to \{0,1\}^*$ such that $x \in A \iff f(x) \in B$.
</MathStatement>

If $A \le_p B$ and $B \in P$, then $A \in P$ (compose reduction with $B$'s algorithm). If $A \le_p B$ and $A$ is NP-hard, then $B$ is NP-hard.

**Direction is critical.** To prove $B$ is NP-hard, reduce FROM a known NP-hard problem $A$ TO $B$: $A \le_p B$. If $B$ were easy, $A$ would be easy, contradiction.

## NP-Completeness

<MathStatement id="np-complete" kind="definition" title="NP-completeness">
A language $L$ is NP-complete if:
1. $L \in NP$ (membership).
2. $L$ is NP-hard: for every $A \in NP$, $A \le_p L$.
</MathStatement>

An NP-complete problem is the "hardest" in NP: if any NP-complete problem is in P, then P = NP.

### Cook–Levin Theorem

<MathStatement id="cook-levin" kind="theorem" title="Cook–Levin theorem (1971)" proofId="cook-levin-proof">
SAT is NP-complete.
</MathStatement>

<MathProof id="cook-levin-proof" for="cook-levin" collapsible open>

**Membership.** SAT $\in$ NP: certificate is a satisfying assignment; verifier evaluates the formula in polynomial time.

**Hardness.** Let $L \in NP$ be decided by nondeterministic Turing machine $M$ in time $p(|x|)$. Construct a Boolean formula $\phi_{M,x}$ in polynomial time from $|x|$ such that $M$ accepts $x$ iff $\phi_{M,x}$ is satisfiable.

The formula encodes the computation tableau of $M$ on $x$ for $p(|x|)$ steps, using $O(p(|x|)^2)$ Boolean variables $T[i,j,k]$ meaning "cell $i$ at time $j$ contains symbol $k$." Clauses enforce:

1. **Initialization:** The first row of the tableau matches the input $x$ in state $q_0$.
2. **Transition:** Each row follows from the previous via $M$'s transition relation.
3. **Acceptance:** Some cell at the final time contains the accept state.
4. **Consistency:** Each cell contains exactly one symbol per time step.

Each constraint is a conjunction of local clauses (each involving $O(1)$ variables), so the total formula size is $O(p(|x|)^2)$. A satisfying assignment corresponds to an accepting computation of $M$ on $x$. □

</MathProof>

### NP-Completeness of 3SAT

**3SAT** is SAT restricted to clauses of exactly 3 literals. Reduce SAT to 3SAT by converting each clause: a clause $(l_1 \lor l_2 \lor \cdots \lor l_k)$ becomes a conjunction of 3-literal clauses using auxiliary variables $z_1, \ldots, z_{k-3}$:

$$
(l_1 \lor l_2 \lor z_1) \land (l_3 \lor \overline{z_1} \lor z_2) \land (l_4 \lor \overline{z_2} \lor z_3) \land \cdots
$$

The auxiliary variables ensure the original clause is satisfiable iff the 3CNF is satisfiable.

### Karp's 21 NP-Complete Problems

Karp (1972) extended Cook's result by giving polynomial reductions showing that 21 fundamental problems are NP-complete [Karp 1972], including:

- **CLIQUE:** Does $G$ contain $K_k$?
- **VERTEX COVER:** Does $G$ have a vertex cover of size $k$?
- **HAMILTONIAN CYCLE:** Does $G$ have a cycle visiting every vertex exactly once?
- **SUBSET SUM:** Does a subset of numbers sum to target $t$?
- **KNAPSACK (decision):** Can a subset achieve value $\ge V$ within capacity $W$?

**SAT → 3SAT:** Add auxiliary variables, splitting long clauses into 3-literal clauses.

**3SAT → CLIQUE:** For a 3CNF with $k$ clauses, create a vertex per literal occurrence. Connect vertices from different clauses unless they are complementary literals. A $k$-clique selects one compatible literal per clause, giving a satisfying assignment.

## A Counterexample and a Failure Mode

Not every problem with a short certificate turns out to be in NP, and not every NP-hard problem needs exact solution.

**Counterexample: UNSAT.** Given a Boolean formula $\phi$, is $\phi$ unsatisfiable? This is the complement of SAT. A short certificate for unsatisfiability would require proving that no assignment works — which means checking all $2^n$ possibilities. Unless NP = coNP, there is no short certificate. UNSAT is coNP-complete, and believed to lack polynomial-size certificates.

<MathStatement id="conp-def" kind="definition" title="coNP">
A language $L$ is in coNP if its complement $\overline{L} = \{0,1\}^* \setminus L$ is in NP. Equivalently, $L \in$ coNP if there exists a polynomial-time verifier $V$ and polynomial $p$ such that for all $x$:
- If $x \in L$, then $\forall y$ with $|y| \le p(|x|)$, $V(x,y)$ accepts.
- If $x \notin L$, then $\exists y$ with $|y| \le p(|x|)$ and $V(x,y)$ rejects.
</MathStatement>

The relationship between NP and coNP is open: if NP $\neq$ coNP, then P $\neq$ NP (since P is closed under complement).

**Failure mode: assuming a pseudo-polynomial algorithm always works.** The Subset Sum problem admits an $O(nt)$ dynamic programming solution where $t$ is the target sum. This is *pseudo-polynomial*: polynomial in the numeric value but exponential in the bit-length of $t$. For $t = 2^{100}$, no DP will finish. Practitioners who apply the Subset Sum DP without checking the magnitude of $t$ will mistake feasibility for intractability.

**Misconception: "NP-complete means impossible."** NP-completeness describes worst-case complexity over all instances. Many NP-complete problems have instances that are routinely solved: SAT solvers handle formulas with millions of clauses for industrial verification, and TSP instances with tens of thousands of cities are solved exactly via branch-and-cut. The worst-case hardness does not forbid efficient solution of the instances that actually arise.

## Ladner's Theorem

<MathStatement id="ladner" kind="theorem" title="Ladner's theorem (1975)" proofId="ladner-proof">
If $P \neq NP$, then there exist languages in $NP \setminus P$ that are not NP-complete.
</MathStatement>

<MathProof id="ladner-proof" for="ladner" collapsible>
**Proof idea.** Construct a language $L$ by diagonalization that is not in P (via standard time-hierarchy argument) but is not NP-complete because any reduction from SAT to $L$ would also have to solve SAT. The construction interleaves SAT with a sparse language, ensuring that $L$ is neither too easy (in P) nor too hard (NP-complete) [Ladner 1975]. □
</MathProof>

These languages are called **NP-intermediate**. Candidates include GRAPH ISOMORPHISM (shown to be in quasipolynomial time by Babai [Babai 2016]), FACTORING (in BQP via Shor's algorithm [Shor 1997]), and certain lattice problems.

## The Polynomial Hierarchy

The polynomial hierarchy extends NP to allow alternating quantifiers:

- $\Sigma^p_1 = NP$: $\exists y$ such that $V(x,y)$ accepts.
- $\Pi^p_1 = coNP$: $\forall y$, $V(x,y)$ accepts.
- $\Sigma^p_k$: $\exists y_1 \forall y_2 \exists y_3 \cdots Q_k y_k$ such that $V(x, y_1, \ldots, y_k)$ accepts where $V$ is polynomial-time.

The hierarchy collapses if any level equals the next ($\Sigma^p_k = \Sigma^p_{k+1}$ implies $\Sigma^p_k = PH$). It is believed that the hierarchy is strict (each level properly contains the previous) [Stockmeyer 1976].

## Pseudo-Polynomial Algorithms and Weak NP-Hardness

A problem is **weakly NP-hard** if it remains NP-hard when numeric parameters are encoded in unary; it is **strongly NP-hard** if it is NP-hard even when numbers are bounded by a polynomial in input length [Garey & Johnson 1979].

Subset sum is weakly NP-hard (has a pseudo-polynomial DP $O(nt)$). 3-PARTITION is strongly NP-hard (no pseudo-polynomial algorithm unless P = NP).

## Self-Reducibility

A problem is self-reducible if a decision oracle can be used to find a solution in polynomial time [Schnorr 1976]. SAT is self-reducible: fix variables one by one, querying the oracle after each fixing to check if the remaining formula is still satisfiable.

    '''Use a SAT decision oracle to find a satisfying assignment.'''
    if not oracle(formula):
        return None
    assignment = {}
    for var in sorted(set(abs(l) for c in formula for l in c)):
        test = [[l for l in c if abs(l) != var] for c in formula]

        if oracle(test):

            formula = test
        else:
            assignment[var] = True
            formula = [[l for l in c if abs(l) != var or l > 0] for c in formula]

        if all(any((not a[abs(l)] if l < 0 else a[abs(l)]) for l in c) for c in f):
            return True
    return False

## Practical Responses to NP-Hardness

| Technique | Guarantee | Example |
|-----------|-----------|---------|
| Exact search (branch-and-bound) | Optimal, worst-case exponential | Maximum clique |
| Pseudo-polynomial DP | Optimal, exponential in bit-length | Knapsack for small $W$ |
| Fixed-parameter tractable | $O(f(k) \cdot n^{O(1)})$ | Vertex cover $O(2^k n)$ |
| Approximation | Provable ratio | Vertex cover 2-approx |
| Heuristic | Empirical only | Simulated annealing |
| Solver (SAT/ILP) | Optimal if timeout not hit | MiniSAT, CPLEX [Moskewicz 2001] |

2. **Reduction construction (application).** Reduce 3SAT to SUBSET SUM by constructing numbers whose digits encode clause satisfaction. Specifically, create one number per literal occurrence and two numbers per clause such that a subset summing to a target corresponds to a satisfying assignment.

   <ExerciseSolution hint="Use base-10 digits: assign each variable and each clause its own digit position. Each variable has two numbers (one per truth value) and each clause has two slack numbers.">

   For a 3CNF with $n$ variables and $m$ clauses, create numbers in base 10 with $n+m$ digits. For each variable $x_i$, create two numbers $v_i$ (digit positions: $i$ in variable area = 1, clause digits for clauses where $x_i$ appears) and $\overline{v_i}$ (clause digits where $\overline{x_i}$ appears). For each clause $c_j$, create two slack numbers $s_j, s_j'$ with digit 1 at clause position $j$. Target: variable digits = 1, clause digits = 3. A subset sums to target iff each variable contributes exactly one of $v_i, \overline{v_i}$ (every variable area digit is 1) and each clause's digit sums to exactly 3 (at least one literal satisfies the clause, plus slacks fill to 3).
   </ExerciseSolution>

3. **Strong vs weak hardness (analysis).** Show that 3-PARTITION is strongly NP-hard by reducing 3SAT to 3-PARTITION while keeping all numbers bounded by a polynomial in input size. Contrast with SUBSET SUM.

   <ExerciseSolution>
   3-PARTITION: given $3m$ integers and bound $B$, decide if they can be partitioned into $m$ triples each summing to $B$. This is strongly NP-hard because the numbers can be encoded as $O(m^2)$-bit values (polynomial in input size). SUBSET SUM's DP is pseudo-polynomial ($O(nt)$ where $t$ can be exponential in bits), while 3-PARTITION's DP would be exponential in both $n$ and $\log B$ because $B$ is polynomially related to $n$.
   </ExerciseSolution>

4. **Self-reduction (application).** Prove that VERTEX COVER is self-reducible: given a decision oracle for $\exists$ vertex cover of size $\le k$, construct a minimum vertex cover.

   <ExerciseSolution>
   Initialize $S = \emptyset$. For each vertex $v$, query $\exists$ vertex cover of size $\le k$ in $G[V \setminus \{v\}]$ (the graph after removing $v$ and its incident edges). If yes, remove $v$ (it is not necessary); otherwise add $v$ to $S$ and reduce $k$ by 1. After processing all vertices, $S$ is a minimum vertex cover.
   </ExerciseSolution>

5. **coNP and certificates (recall).** Define coNP. Prove that TAUTOLOGY (given Boolean formula $\phi$, is $\phi$ true for all assignments?) is coNP-complete. Show that TAUTOLOGY is in coNP.

   <ExerciseSolution>
   coNP = $\{L : \overline{L} \in NP\}$. TAUTOLOGY is coNP-complete because its complement (SAT) is NP-complete. TAUTOLOGY $\in$ coNP: a certificate for a no-instance is a falsifying assignment; verifier evaluates the formula.
   </ExerciseSolution>

6. **Transfer: reducing an unfamiliar problem (synthesis).** Consider the problem CLIQUE COVER: given graph $G$ and integer $k$, can the vertices be partitioned into $k$ subsets, each inducing a clique? Prove this problem is NP-complete by reduction from a known NP-complete problem. Explain why a naive reduction from CLIQUE does not work (hint: partitioning vs. existence of a single clique).

   <ExerciseSolution>Reduce from GRAPH COLORING (or K-COLORABILITY). A graph is $k$-colorable iff its complement is $k$-clique-coverable: a color class in $G$ has no edges between its vertices, so in $\overline{G}$ every pair of vertices in the same color class is adjacent — i.e., each color class becomes a clique in $\overline{G}$. Thus $G$ is $k$-colorable $\iff$ $\overline{G}$ has a $k$-clique cover. Since GRAPH COLORING is NP-complete, CLIQUE COVER is NP-complete. A direct reduction from CLIQUE would not work because CLIQUE asks whether a *single* $k$-clique exists, while CLIQUE COVER asks for a *partition* into $k$ cliques covering all vertices — a fundamentally different constraint.</ExerciseSolution>

7. **Transfer: the P vs NP practical consequence (synthesis).** Suppose a researcher proves P = NP by discovering a polynomial-time algorithm for SAT that runs in $O(n^{100})$. Describe two concrete implications for: (a) cryptography, (b) automated theorem proving, and (c) operations research. Explain why an $O(n^{100})$ algorithm might not revolutionize industry overnight, despite being a theoretical breakthrough.

<ExerciseSolution>(a) Most public-key cryptography (RSA, ECC) relies on the hardness of factoring or discrete log; if P = NP, these are solvable in polynomial time, breaking all current public-key infrastructure. (b) Every mathematical conjecture that has a polynomial-verifiable proof could be decided automatically — theorems in NP would be mechanically solvable. (c) Thousands of NP-complete optimization problems (TSP, scheduling, etc.) would become tractable, potentially revolutionizing logistics, manufacturing, and AI. However, $O(n^{100})$ is astronomically large for any practical input size. For $n=100$, $100^{100}$ operations exceeds the number of atoms in the observable universe. The theoretical breakthrough would not yield practical algorithms — but it would open the door to future improvements that bring the exponent down.</ExerciseSolution>

8. **Transfer: decomposition of a reduction (synthesis).** The reduction 3SAT → CLIQUE creates a vertex per literal occurrence and connects vertices from different clauses unless they are complementary. Explain why this construction fails if we replace 3SAT with 2SAT (clauses of exactly 2 literals). What does this tell you about the complexity of 2SAT?

    <ExerciseSolution>
    2SAT is solvable in polynomial time (via strongly connected components in the implication graph). The reduction from 3SAT to CLIQUE relies on each clause having *three* literals to create a non-trivial graph structure. With 2SAT, the reduction would produce a graph where a $k$-clique can be found in polynomial time (since 2SAT ∈ P). The failure of the reduction mirrors the fact that 2SAT is not NP-complete unless P = NP.
    </ExerciseSolution>

9. **Synthesis: coNP and the polynomial hierarchy.** Prove that if NP = coNP, then the polynomial hierarchy collapses to the first level. Hint: show that $\Sigma^p_2 \subseteq NP$ using the assumption that coNP = NP.

    <ExerciseSolution>
    Assume NP = coNP. Take any language $L \in \Sigma^p_2$, so $L = \{x : \exists y \forall z \; V(x,y,z)\}$ with $V$ polynomial-time. The inner statement $\forall z \; V(x,y,z)$ is a coNP predicate. Under NP = coNP, this predicate is also in NP, so there exists a witness $w$ for it. Thus $L = \{x : \exists y \exists w \; V'(x,y,w)\} \in NP$. Therefore $\Sigma^p_2 = NP$ and the hierarchy collapses to the first level.
    </ExerciseSolution>

## Deliberately Omitted

- **Circuit complexity and NC.** The classification of problems by circuit depth (NC, AC) addresses parallel feasibility; this is a separate branch of complexity theory covered in the advanced complexity module [Arora & Barak 2009].

- **Interactive proofs and IP = PSPACE.** The IP class and the Shamir theorem (IP = PSPACE) extend the verification paradigm beyond NP; covered in the computational complexity course [Shamir 1992].

- **Approximation-preserving reductions (AP-reductions).** Reductions that preserve approximability are used to prove inapproximability results beyond the PCP theorem; the approximation algorithms lesson treats hardness via gap reductions [Arora et al. 1998].

- **The PCP theorem.** The Probabilistically Checkable Proof theorem (PCP) characterizes NP as languages verifiable with constant query complexity. This deep result underlies many inapproximability bounds but requires a full course on its own [Arora & Safra 1998].

- **Parameterized complexity and W-hierarchy.** Fixed-parameter tractability (FPT) and the W-hierarchy classify NP-hard problems by their dependence on a parameter $k$, separate from input size $n$. Vertex cover is FPT; CLIQUE is W[1]-complete. This is covered in a dedicated parameterized algorithms lesson [Downey & Fellows 1999].

- **The time hierarchy theorem and EXPTIME.** The time hierarchy theorem guarantees that there exist problems solvable in exponential time but not in polynomial time. EXPTIME-complete problems (e.g., generalized chess with $n \times n$ boards) are provably outside P, unlike NP-complete problems [Sipser 2013].

- **Non-uniform complexity and advice.** The class P/poly captures problems solvable by polynomial-size circuits, which includes all of NP (Karp–Lipton theorem) if NP ⊆ P/poly, and relates to the polynomial hierarchy collapse [Karp & Lipton 1982].
