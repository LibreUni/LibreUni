# Online and Streaming Algorithms

**Prerequisites:** Probability fundamentals (expectation, Markov's inequality, pairwise independence); hash functions; basic amortized analysis.

Ski rental: rent skis costs \$1/day, buying costs \$10. The number of ski days $t$ is unknown. When do you buy?

If you rent every day and the season is long, you overpay. If you buy early and the season is short, you also overpay. No deterministic decision can be optimal for all $t$ — the problem is not about finding the right answer but about bounding the worst-case regret.

This dilemma is the core of **online algorithms**: decisions are irreversible and made without knowledge of the future. **Streaming algorithms** face a different constraint — sublinear memory — but share the need for one-pass processing with bounded resources.

## Competitive Analysis

<MathStatement id="competitive" kind="definition" title="Competitive ratio">
Algorithm $ALG$ is $c$-competitive if there exists a constant $b$ such that for all input sequences $\sigma$:
$$
\text{ALG}(\sigma) \le c \cdot \text{OPT}(\sigma) + b
$$
where $\text{OPT}(\sigma)$ is the optimal offline cost — the cost an algorithm with perfect foresight would incur [Sleator & Tarjan, 1985]. For randomized algorithms, the expectation is over ALG's internal randomness.
</MathStatement>

The additive constant $b$ prevents trivial violations on short inputs. For minimization problems with nonnegative costs, $b$ is often zero.

**Adversary models** determine the power of the input sequence:

- **Oblivious adversary:** fixes $\sigma$ in advance, unaware of ALG's random choices. This is the standard model for randomized online algorithms.
- **Adaptive adversary:** sees ALG's outputs and selects future inputs adversarially. An adaptive adversary can simulate ALG's randomness, making randomization useless.

### Ski Rental

**Problem.** Rent costs \$1/day; buy costs $B$. Season length $t$ is unknown. On each day, either rent (\$1) or buy (\$B, after which future days cost \$0). The goal is to minimize total cost.

**Deterministic algorithm.** Rent for $B-1$ days, buy on day $B$.

- If $t < B$: cost $= t = \text{OPT}$ (optimal offline also rents).
- If $t \ge B$: cost $= (B-1) + B = 2B-1$, while $\text{OPT} = B$ (buy immediately).

Competitive ratio: $(2B-1)/B = 2 - 1/B$.

<MathStatement id="ski-rental-lb" kind="theorem" title="Ski rental lower bound" proofId="ski-rental-lb-proof">
No deterministic online algorithm for ski rental achieves competitive ratio $< 2 - 1/B$. The optimal ratio is $2 - 1/B$.
</MathStatement>

<MathProof id="ski-rental-lb-proof" for="ski-rental-lb">
Any deterministic algorithm buys on some day $k$. If $t = k$ (season ends the day after purchase), cost $= (k-1) + B$. OPT = $\min(k, B)$. The ratio is:
$$
\frac{k-1+B}{\min(k, B)}.
$$
The adversary chooses $t$ to maximize this ratio. For $k \le B$, the worst case is $t = k$, giving ratio $(k-1+B)/k = 1 + (B-1)/k \ge 2 - 1/B$ (achieved at $k = B$). For $k > B$, the worst case is $t = B$, giving ratio $(k-1+B)/B = 1 + (k-1)/B \ge 2 - 1/B$ (worse as $k$ grows). The minimum over $k$ of the worst-case ratio is $2 - 1/B$, attained at $k = B$. □
</MathProof>

**Randomized algorithm (optimal).** Choose buy day $k \in \{1, \ldots, B-1\}$ with probability $p_k$ proportional to $(1 - 1/B)^{k-1}$. Against an oblivious adversary, this achieves competitive ratio $e/(e-1) \approx 1.582$, and no randomized algorithm can do better [Karlin et al., 1994].

### Paging

**Problem.** A cache holds $k$ pages. A request sequence of page numbers arrives. A hit costs \$0; a miss (fault) costs \$1 to fetch the page into cache. Minimize total faults.

**LRU (Least Recently Used):** On a miss, evict the page whose most recent request is farthest in the past.

<MathStatement id="lru-competitive" kind="theorem" title="LRU is $k$-competitive" proofId="lru-competitive-proof">
LRU achieves competitive ratio $k$ for paging with a cache of size $k$. No deterministic algorithm can do better than $k$.
</MathStatement>

<MathProof id="lru-competitive-proof" for="lru-competitive">
Partition the request sequence into *phases*: each phase contains exactly $k$ distinct pages, and the first request of a phase is the $(k+1)$-st distinct page seen since the previous phase started.

**Within a phase:** LRU faults at most $k$ times. Each distinct page inserted may cause a fault, and a phase has at most $k$ distinct pages (it ends when the $(k+1)$-st distinct page appears).

**Between phases:** OPT must fault at least once per phase. The first request of phase $i+1$ is a page $p$ that was not among the $k$ distinct pages of phase $i$. After phase $i$, OPT's cache can hold at most $k$ pages, so at least one of the $k$ distinct pages from phase $i$ is absent. Since $p$ is distinct from all $k$, it must be absent from OPT's cache — OPT faults at least once per phase.

Thus over $P$ phases: LRU faults $\le kP$, OPT faults $\ge P$, giving ratio $k$. □
</MathProof>

**Randomized paging:** The *marking* algorithm (a randomized variant of LRU) achieves $O(\log k)$-competitiveness against an oblivious adversary [Fiat et al., 1991], and this is optimal for any randomized algorithm.

## Counterexample and Failure Modes

**When competitive analysis misleads.** Competitive ratio is a worst-case guarantee. An algorithm may be $c$-competitive yet perform poorly on every realistic input. Example: the "optimal" deterministic paging algorithm (Belady's algorithm, which evicts the page requested farthest in the future) is optimal offline but unimplementable online. LRU is $k$-competitive, but on a workload with working-set size $k+1$, LRU faults on nearly every access — the same as random eviction. The competitive ratio does not distinguish between LRU and random eviction; both are $k$-competitive.

**When an oblivious adversary is too weak.** The randomized ski rental algorithm is optimal only against an oblivious adversary. If the adversary can observe the algorithm's buy day before choosing the season length, randomization provides no advantage — the deterministic lower bound of $2 - 1/B$ reapplies. Choosing the wrong adversary model gives a misleading sense of algorithmic power.

**When streaming goes wrong.** Count-Min Sketch guarantees an overestimate with high probability but never underestimates. For applications that need unbiased estimates (e.g., tracking average frequency), the systematic overestimation bias is harmful. The sketch also cannot answer point queries accurately when frequencies are skewed — a single high-frequency item contamates every bucket it hashes into, raising estimates for all colliding items.

## Streaming Algorithms

A streaming algorithm processes a sequence $a_1, \ldots, a_m$ with $a_i \in [n]$ using $o(\min(m, n))$ memory, ideally $O(\text{poly}(\log m, \log n))$ [Alon et al., 1999].

### Count-Min Sketch

<MathStatement id="count-min" kind="definition" title="Count-Min Sketch">
A sketch with width $w$ and depth $d$ consists of $d$ pairwise-independent hash functions $h_1, \ldots, h_d : [n] \to [w]$ and a $d \times w$ counter array initialized to zero [Cormode & Muthukrishnan, 2005].

- **Update** $(x, +1)$: for each row $r$, $\text{count}[r][h_r(x)] \mathrel{+}= 1$.
- **Query** $\hat{f}(x)$: $\min_{r \in [d]} \text{count}[r][h_r(x)]$.
</MathStatement>

<MathStatement id="count-min-guarantee" kind="theorem" title="Count-Min guarantee" proofId="count-min-proof">
For $\hat{f}(x)$ estimated by Count-Min with $w = \lceil e/\varepsilon\rceil$, $d = \lceil \ln(1/\delta)\rceil$:
$$
\Pr\left[ \hat{f}(x) \le f(x) + \varepsilon \|f\|_1 \right] \ge 1 - \delta.
$$
</MathStatement>

<MathProof id="count-min-proof" for="count-min-guarantee">
Fix item $x$. For a single row $r$, the bucket $\text{count}[r][h_r(x)]$ contains $f(x)$ plus the sum of all $f(y)$ for $y \neq x$ with $h_r(y) = h_r(x)$. By pairwise independence, $\Pr[h_r(y) = h_r(x)] = 1/w$. The expected contribution of other items:
$$
E[\text{count}[r][h_r(x)]] = f(x) + \sum_{y \neq x} \frac{f(y)}{w} \le f(x) + \frac{\|f\|_1}{w}.
$$
Set $w = e/\varepsilon$. Then $E[\text{count}[r][h_r(x)]] \le f(x) + (\varepsilon/e)\|f\|_1$. By Markov's inequality:
$$
\Pr[\text{count}[r][h_r(x)] > f(x) + \varepsilon\|f\|_1] \le \frac{1}{e}.
$$
With $d$ independent rows, the probability that all $d$ rows exceed the bound is $(1/e)^d \le \delta$ when $d \ge \ln(1/\delta)$. The minimum over rows gives the tightest bound. □
</MathProof>

### Reservoir Sampling

**Problem.** Select $k$ uniformly random items from a stream of unknown length $n$ using $O(k)$ memory.

**Algorithm.** Store the first $k$ items. For the $i$-th item ($i > k$), with probability $k/i$, replace a uniformly random stored item with the new item.

**Proof of uniformity (by induction on $i$).** After processing $i$ items, each item has probability $k/i$ of being in the reservoir. Base $i = k$: all $k$ items in reservoir, probability $1 = k/k$. Inductive step: assume true for $i$. For item $i+1$, inclusion probability $= k/(i+1)$. For an old item $x$ present after $i$ steps (probability $k/i$), it survives step $i+1$ if either $i+1$ is not selected ($1 - k/(i+1)$) or selected but replaces a different item ($k/(i+1) \cdot (1 - 1/k)$). Total survival probability: $k/i \cdot ((i+1-k)/(i+1) + 1/(i+1)) = k/(i+1)$. □

### Misra–Gries Heavy Hitters

**Problem.** Find all items with frequency $> N/k$ in a stream of length $N$ [Misra & Gries, 1982].

**Algorithm.** Maintain at most $k-1$ counters and their associated items. For each stream element $x$:
- If a counter for $x$ exists, increment it.
- Else if fewer than $k-1$ counters exist, add $(x, 1)$.
- Else decrement all counters (removing any that reach 0).

**Guarantee.** Let $c(x)$ be the final counter value for item $x$. Then $c(x) \le f(x) \le c(x) + N/k$. Every $N/k$-heavy hitter (item with frequency $> N/k$) appears with $c(x) > 0$. The algorithm is deterministic and uses $O(k)$ memory.

### Linear Sketching and Johnson–Lindenstrauss

A random projection $A \in \mathbb{R}^{k \times d}$ with i.i.d. Gaussian entries approximately preserves pairwise distances: for any $x, y \in \mathbb{R}^d$, with probability $1 - e^{-\Omega(k\varepsilon^2)}$:

$$
(1-\varepsilon)\|x-y\|^2 \le \|Ax - Ay\|^2 \le (1+\varepsilon)\|x-y\|^2,
$$

provided $k = \Omega(\varepsilon^{-2} \log N)$ for $N$ points [Johnson & Lindenstrauss, 1984]. This enables dimensionality reduction in streaming and data mining: $d$-dimensional vectors can be compressed to $k$ dimensions while approximately preserving distances, then processed through a stream.

**Application**

3. **Ski rental randomized.** Derive the optimal randomized competitive ratio $e/(e-1)$ for ski rental by solving for the distribution of buy days.

    <ExerciseSolution hint="Use Yao's minimax principle: find a distribution over season lengths that forces any deterministic algorithm to have expected ratio at least $e/(e-1)$. Then the optimal randomized algorithm against an oblivious adversary achieves the same ratio.">
    Choose buy day $k$ with probability $p_k = \frac{1}{B-1} \cdot \frac{(1-1/B)^{k-1}}{1 - (1-1/B)^{B-1}}$. The expected cost for a given season length $t$ is bounded by $e/(e-1)$ times OPT. This is optimal by Yao's minimax principle: the adversary distribution that sets $t = j$ with probability proportional to $(1 - 1/B)^{j-1}$ forces any deterministic algorithm to have expected ratio at least $e/(e-1)$ [Karlin et al., 1994].
    </ExerciseSolution>

4. **LRU and FIFO.** Prove that FIFO (evict the oldest page) is also $k$-competitive for paging. Show that LIFO (evict the newest) is not competitive.

    <ExerciseSolution hint="For FIFO, adapt the phase-partition argument. For LIFO, construct a request sequence over $k+1$ pages that forces a fault on every access.">
    FIFO partitions phases identically to LRU: when a phase fills the cache with $k$ distinct pages, FIFO faults at least once per page. The same phase-counting argument shows $k$-competitiveness. LIFO fails: consider $k+1$ pages requested in the repeating order $1, 2, \ldots, k+1, 1, 2, \ldots$. LIFO evicts the most recently loaded page, which is the page just requested. Every request is a fault, giving unbounded competitive ratio.
    </ExerciseSolution>

5. **Count-Min width.** Prove that $w = \lceil e/\varepsilon\rceil$ ensures each row's overestimate probability is at most $1/e$.

    <ExerciseSolution>
    For row $r$, $E[\text{count}[r][h_r(x)]] = f(x) + \sum_{y \neq x} f(y) \cdot \Pr[h_r(y) = h_r(x)] \le f(x) + \|f\|_1 / w$. Setting $w = e/\varepsilon$ gives $E[\text{count}[r][h_r(x)]] \le f(x) + (\varepsilon/e)\|f\|_1$. By Markov's inequality, $\Pr[\text{count}[r][h_r(x)] > f(x) + \varepsilon\|f\|_1] \le 1/e$.
    </ExerciseSolution>

**Transfer**

6. **Distinct elements (Flajolet–Martin).** Describe the Flajolet–Martin algorithm for estimating the number of distinct elements in a stream. Prove that the estimate is unbiased with variance bounded by $O(n^2/m)$ where $m$ is the number of hash functions.

    <ExerciseSolution hint="Use a hash function that maps items to bit strings with geometric distribution on the trailing-zero count. The maximum trailing-zero count estimates $\log_2$ of the distinct count.">
    Flajolet–Martin: apply a hash function $h$ that maps items to infinite bit strings with $\Pr[h(x) \text{ has suffix } 0^\rho 1] = 2^{-(\rho+1)}$. Track the maximum $\rho$ (trailing-zero count) observed. The distinct count $n$ is proportional to $2^{\max \rho}$. Use $m$ independent hash functions, average their estimates, and bound variance via the median-of-averages technique. The estimate is asymptotically unbiased; the variance decreases as $O(n^2/m)$ [Flajolet & Martin, 1985].
    </ExerciseSolution>

**Synthesis**

7. **Heavy hitters lower bound.** Prove that any deterministic streaming algorithm for $\varepsilon$-heavy hitters in the insertion-only model requires $\Omega(1/\varepsilon)$ memory.

    <ExerciseSolution>
    Consider $1/(2\varepsilon)$ distinct items each appearing exactly $\varepsilon N$ times, plus $\varepsilon N$ distinct items each appearing once. Distinguishing these two cases requires $\Omega(1/\varepsilon)$ memory by a communication complexity reduction from set disjointness [Alon et al., 1999]. The intuition: the algorithm must distinguish whether the stream contains $1/(2\varepsilon)$ items each with frequency $\varepsilon N$ (all heavy hitters present) or $\varepsilon N$ singletons (no heavy hitters), which requires tracking at least $\Omega(1/\varepsilon)$ distinct identities.
    </ExerciseSolution>

8. **Design a streaming algorithm for range queries.** Given a stream of points $(x_i, y_i) \in [n]^2$, design a data structure using $O(\varepsilon^{-2} \log(1/\delta))$ memory that answers the query "how many points fall in the axis-aligned rectangle $[a,b] \times [c,d]$?" with additive error $\varepsilon N$. (Hint: compose two Count-Min Sketches or use a dyadic range decomposition.)

    <ExerciseSolution hint="Decompose the range into $O(\log^2 n)$ canonical rectangles using a 2D segment tree, then store a CMS in each segment tree node, or use a CMS with a 2D hash function.">
    One approach: build a 2D Count-Min Sketch with independent hash functions for each dimension. For a point $(x, y)$, update counters at $(h_r(x), h'_r(y))$ for each row $r$. Query a rectangle $[a,b] \times [c,d]$ by summing estimates over the rectangle, but this incurs error proportional to the rectangle area. A better approach: partition the $x$-range into $O(\log n)$ dyadic intervals, each with a CMS for the $y$-dimension. A query decomposes into $O(\log^2 n)$ CMS queries, each with additive error $\varepsilon N/(\log^2 n)$, giving total error $\varepsilon N$ with memory $O(\varepsilon^{-2} \log(1/\delta) \log^2 n)$.
    </ExerciseSolution>
