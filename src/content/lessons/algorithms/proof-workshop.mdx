# Proof Workshop: Diagnosis, Repair, and Counterexamples

A colleague claims: "I proved that my scheduling algorithm is optimal." They show you a half-page exchange argument. The algebra checks out. The conclusion seems right. But something feels hollow — the proof assumes all jobs have unit processing time. The algorithm was designed for arbitrary processing times. The "proof" proves a different problem.

This is the workshop reality of algorithm correctness. Published conference papers have contained incorrect proofs of correctness for the greedy algorithm for the Steiner tree problem (the "loss-contracting" proof had a gap that took five years to fix), for the metric TSP 1.5-approximation (the famous Christofides-Serdyukov result required a subtle parity argument), and even for basic data structures like Fibonacci heaps (the initial amortized analysis omitted a critical potential-function term).

This workshop builds the skill of **proof critique** — the ability to find gaps in a correctness argument, fix them, or construct a counterexample that shows the gap is fatal. The four patterns (invariants, exchange, induction, certificates) are assumed known; the focus is on where they break.

> **Prerequisites.** Hoare logic and loop invariants, exchange arguments for greedy algorithms, well-founded induction, and certificate schemes as covered in the Algorithmic Correctness lesson. Familiarity with sorting, graph algorithms, and basic data structures.

<MathStatement id="workshop-goal" kind="remark" title="Workshop objective">
After this workshop, you should be able to: (1) identify the most common gaps in correctness proofs; (2) strengthen an incomplete invariant or exchange argument; (3) produce a counterexample that exposes a proof gap; and (4) adapt a proof technique from one problem to another domain.
</MathStatement>

## The Gap Diagnosis Checklist

Every correctness proof can fail in a small number of recurring ways. When reviewing a proof, check these in order:

### Common Gap Categories

| Gap type | Symptom | Repair strategy |
|---|---|---|
| **Underspecified invariant** | Invariant is too weak to imply postcondition | Strengthen with additional conjuncts |
| **Hidden assumption** | Proof assumes property not guaranteed by precondition | Add precondition clause or prove property separately |
| **Non-well-founded measure** | Termination measure can increase or stall | Redesign measure (lexicographic tuple, different ordering) |
| **Exchange infeasibility** | Swapped solution violates constraints | Prove feasibility structurally, not by enumeration |
| **Residual mismatch** | Subproblem after greedy choice has different structure | Extend problem definition to be self-reducing |
| **Certificate not complete** | Some correct outputs have no valid certificate | Redesign certificate to cover all cases |

## Case Study 1: The Broken Selection Sort Invariant

Consider this attempted correctness proof for selection sort:

> **Attempted invariant $I$:** After $i$ iterations of the outer loop, the subarray $A[0..i]$ contains the $i+1$ smallest elements of the original array, in sorted order.

The algorithm finds the minimum of $A[i..n-1]$ and swaps it into position $i$.

First, after i iterations the sorted prefix is A[0..i-1] (the loop variable i starts at 0 and increments after each iteration; after i=0, 0 elements are sorted). Second, without stating that A[i..n-1] contains the remaining original elements, a correct position for the sorted prefix does not guarantee the full array will be a permutation of the original — elements could have been lost or duplicated."
/>

**Repair.** The correct invariant for selection sort: "After $i$ iterations, $A[0..i-1]$ contains the $i$ smallest original elements in sorted order, and $A[i..n-1]$ contains the remaining $n-i$ original elements in their original relative order (except for the ones that have been swapped into the prefix)."

The second clause is what prevents the algorithm from silently discarding elements. Without it, the algorithm could hypothetically satisfy "sorted prefix" by copying elements out of the array — the invariant is too weak to enforce that the output is a permutation of the input.

## Case Study 2: The Greedy Exchange That Forgets Feasibility

A classic proof mistake in exchange arguments is proving that the objective does not worsen while ignoring that the *exchanged* solution may no longer be feasible.

  explanation="The critical gap: earliest-finish-first makes no claim about weight. **The real exchange for weighted interval scheduling.** The correct approach is dynamic programming, not greedy. The exchange argument for **unweighted** interval scheduling works because all intervals have unit weight, so earliest finish time maximizes count. The moment weights become non-uniform, the exchange foundation crumbles because the objective changes from cardinality to sum.

## Case Study 3: Induction on the Wrong Measure

Recursive algorithms require the induction hypothesis to apply to every recursive call. A common mistake: choosing a measure that does not strictly decrease for all calls.

<MathStatement id="bad-measure" kind="example" title="Failing induction measure">
Consider this recursive function:
$$
f(n) = \begin{cases}
1 & \text{if } n \le 1 \\
f(n-1) + f(\lfloor n/2 \rfloor) & \text{if } n > 1 \text{ and } n \text{ is even} \\
f(n-1) + f(n-2) & \text{if } n > 1 \text{ and } n \text{ is odd}
\end{cases}
$$
</MathStatement>

This is the Collatz function. The measure $M(x) = x$ fails because $3x+1 > x$ for all $x > 0$. Whether this function terminates for all $x$ is an open problem (Collatz conjecture). A well-founded measure requires a **monotone decreasing** mapping — if any call increases the measure, the inductive argument collapses.

**Repair strategy.** For well-founded induction on $\mathbb{N}$, every recursive call must strictly decrease the measure. If some calls increase it, you need either:
1. A lexicographic measure on a tuple where one component decreases even when another increases, or
2. A different well-founded set where the increase cannot continue indefinitely.

## Case Study 4: Certificate Soundness Gap

A certificate scheme must be both complete (every correct answer has a certificate) and sound (no incorrect answer has a certificate). The soundness condition is the one most commonly overlooked.

<MathStatement id="mst-certificate" kind="example" title="MST certificate via cut condition">
For a spanning tree $T$ of graph $G = (V, E, w)$, a certificate of optimality can be: for every edge $e \in T$, the weight $w(e)$ is minimal among all edges crossing the cut defined by removing $e$ from $T$.
</MathStatement>

<CodeExercise client:load
  title="Complete the certificate"
  code={`def verify_mst(T, G):
    # T: list of edges in candidate spanning tree
    # G: graph with edges (u, v, w)
    # Verify T is a spanning tree (covers all vertices, no cycles)
    if not is_spanning_tree(T, G):
        return False
    # Verify certificate: for each tree edge,
    # it is the [!blank!] among edges crossing its cut
    for e in T:
        cut_edges = edges_crossing_cut(G, e, T)
        if any(w < e.weight for _, _, w in cut_edges):
            return False
    return True`}
  answers={["minimum weight", "cheapest", "lightest", "smallest weight"]}
  explanation="The certificate requires that each tree edge be the minimum-weight (cheapest/lightest) edge crossing its cut. This is the 'cut property' of MSTs. Soundness: if all tree edges satisfy this, then by the cut property theorem, T must be a minimum spanning tree — no lighter spanning tree can exist because any alternative would need a cheaper crossing edge somewhere."
/>

**Soundness hole.** Suppose the verifier above does not check that $T$ is a spanning tree (acyclic, covering all vertices). Then an incorrect answer could pass: a set of edges that is not a tree could still satisfy the per-edge cut condition. For example, a cycle of minimum-weight edges could pass the per-edge check but fail to be a tree. The verifier must independently check the structural feasibility of the claimed solution before checking the certificate.

## Counterexample Construction as a Proof Technique

One of the most effective ways to find proof gaps is to attempt to construct a counterexample. If you cannot build one, the gap may be repairable. If you can, the proof is irreparably broken.

### Counterexample Construction Strategy

1. **Identify the weakest link** in the proof. What assumption is least justified?
2. **Isolate that assumption** as a constraint to violate.
3. **Build the smallest instance** that violates the assumption while keeping everything else fixed.
4. **Check the instance against the claim.** If the algorithm fails, you have a counterexample. If not, the assumption may be stronger than you thought.

## Transfer: Proof Techniques Across Problem Domains

The four proof patterns are not tied to specific algorithms — they transfer across domains. Recognizing the underlying structure lets you apply invariants to graph algorithms, exchange arguments to scheduling variants, and induction to combinatorial structures.

<MathStatement id="transfer-table" kind="remark" title="Cross-domain proof transfer">
| Pattern | Origin domain | Transfers to |
|---|---|---|
| Loop invariant | Array sorting | Graph traversal (visited set invariant), caching (LRU invariant), concurrent data structures |
| Exchange argument | Interval scheduling | Scheduling variants (weighted, multiprocessor, release times), packing problems, matroid optimization |
| Well-founded induction | Divide-and-conquer (merge sort) | Game tree analysis (minimax), recursive data structures, graph DP (treewidth decomposition) |
| Certificate | MST, max flow | LP optimality (dual certificate), approximation (gap-preserving reduction), probabilistic verification |
</MathStatement>

### Transfer Example: Invariant for Graph Search

The BFS/DFS visited-set invariant transfers directly from sorting invariants:

> **Invariant for BFS:** After a vertex $v$ is dequeued, all vertices at distance $d$ from the source have been discovered before any vertex at distance $d+1$.

This is structurally identical to the selection sort invariant ("after $i$ iterations, the $i$ smallest elements are in position"), but applied to distance layers instead of sorted prefixes. The proof follows the same three-part structure: initialization (source at distance 0 enqueued), preservation (enqueuing neighbors maintains the layer ordering), and termination (all reachable vertices discovered).

## Deliberately Omitted

**Probabilistic proof techniques (union bound, Chernoff, method of conditional probabilities)** — These require a different correctness framework (randomized algorithms) and are covered in the dedicated Randomized Algorithms lesson. The focus here is on deterministic correctness proofs.

**Formal verification with automated theorem provers (Z3, Dafny, Coq)** — Tool-assisted verification is complementary to hand-construction of proofs. Automated provers can check invariants but do not teach the judgment needed to find gaps in a paper proof. This skill is developed in the software engineering formal methods module.

**Average-case analysis and smoothed analysis** — Worst-case correctness proofs are the foundation; average-case guarantees use a different analytical framework (expected value over input distributions) and are treated in the Advanced Analysis module.

**Amortized analysis as a correctness technique** — Amortization is primarily a resource-bound technique (time/space), not a correctness technique. While amortized proofs also use invariants and potential functions, their goal is proving running time bounds, not functional correctness.

**Proofs of lower bounds (adversary arguments, decision trees, information-theoretic)** — Lower bounds address a different question ("how fast can any algorithm solve this problem?") and require separate combinatorial techniques (adversary construction, Yao's principle). These are covered in the Complexity and Lower Bounds module.

## Exercises

### Recall (R)

1. List three conditions every loop invariant must satisfy. What happens if the invariant is too weak to imply the postcondition?

<ExerciseSolution hint="Review the Hoare while rule's three conditions.">
The three conditions: (1) initialization — invariant holds before the first iteration; (2) preservation — if invariant holds and the loop guard is true, executing the loop body re-establishes the invariant; (3) termination implication — when the loop exits (guard false), the invariant together with the negated guard implies the postcondition. If the invariant is too weak, condition (3) fails — the invariant ∧ ¬guard does not logically imply the postcondition, so the proof is incomplete.
</ExerciseSolution>

2. Name the three properties an exchange argument must establish. Which one is most commonly omitted in flawed proofs?

<ExerciseSolution hint="Exchange, non-worsening, decomposition.">
(1) Exchange: replacing a decision in the optimal solution with the greedy choice yields a feasible solution. (2) Non-worsening: the objective does not degrade. (3) Decomposition: the residual problem is self-similar. The most commonly omitted or hand-waved is non-worsening — proofs often assert it without showing that the greedy choice is at least as good as the optimal's corresponding element (see Case Study 2 where the proof assumed w(g) ≥ w(o₁) without justification).
</ExerciseSolution>

3. What is a well-founded order? Why is $\mathbb{N}$ with the usual $<$ well-founded, but $\mathbb{Z}$ with $<$ is not?

<ExerciseSolution hint="Think about infinite descending chains.">
A well-founded order is a partial order with no infinite strictly descending chains. $\mathbb{N}$ with $<$ is well-founded because starting from any natural number, you can only descend finitely many steps before reaching 0. $\mathbb{Z}$ with $<$ is NOT well-founded because you can descend infinitely: 0, -1, -2, -3, ... has no minimum element.
</ExerciseSolution>

### Application (A)

4. **Repair this invariant.** A student proposes this invariant for insertion sort: "After $i$ iterations, $A[0..i-1]$ is sorted." The proof claims the postcondition "$A$ is sorted" follows when $i = n$. Identify the gap and repair it.

<ExerciseSolution hint="What else must the invariant track besides sortedness?">
Gap: the invariant does not state that $A[0..i-1]$ contains a permutation of the original $A[0..i-1]$, nor what happened to $A[i..n-1]$. A sorted prefix could be achieved by discarding elements. Repair: "After $i$ iterations, $A[0..i-1]$ is sorted AND contains exactly the elements originally in $A[0..i-1]$ (in some order), and $A[i..n-1]$ is unchanged from the original array."
</ExerciseSolution>

5. **Find the exchange gap.** Claim: "For minimizing maximum lateness on a single machine, the 'shortest processing time first' (SPT) rule is optimal." Construct a counterexample with three jobs.

<ExerciseSolution hint="Consider deadlines that are earlier for short jobs.">
Counterexample: jobs (p=1, d=10), (p=2, d=2), (p=3, d=12). SPT order: (1,10), (2,2), (3,12). Completion times: C₁=1, C₂=3, C₃=6. Lateness: L₁=1-10=-9, L₂=3-2=1, L₃=6-12=-6. Max lateness = 1. Optimal: (2,2), (1,10), (3,12). C₁=2, C₂=3, C₃=6. L₁=2-2=0, L₂=3-10=-7, L₃=6-12=-6. Max = 0 < 1. SPT fails because it ignores deadlines. The correct rule for minimizing max lateness is earliest due date (EDD), not SPT.
</ExerciseSolution>

6. **Induction measure design.** The function $g(n)$ calls $g(\lfloor n/2 \rfloor)$ and $g(\lfloor 2n/3 \rfloor)$ for $n > 1$. Define a well-founded measure and prove termination. Bound the maximum recursion depth.

<ExerciseSolution hint="Both calls produce values ≤ cn for some c < 1.">
Measure $M(n) = n$ works because $\lfloor n/2 \rfloor < n$ and $\lfloor 2n/3 \rfloor < n$ for all $n > 1$, both strictly decreasing in $\mathbb{N}$. Recursion depth: the larger argument $\lfloor 2n/3 \rfloor$ shrinks by at most factor $2/3$ each level, so depth is at most $\log_{3/2} n$. The branching factor is 2, but depth bounds the total calls to $O(2^{\log_{3/2} n}) = O(n^{\log_{3/2} 2}) \approx O(n^{1.709})$ in the worst case.
</ExerciseSolution>

### Transfer (T)

7. **Invariant transfer to Dijkstra's algorithm.** Formulate a loop invariant for Dijkstra's shortest-path algorithm that uses the "settled set" analogy from selection sort. Prove that when a vertex $v$ is added to the settled set, its distance is correct.

<ExerciseSolution hint="Model Dijkstra as selecting the 'minimum distance vertex' similar to selecting the 'minimum element' in selection sort.">
Invariant: After $k$ iterations, the set $S$ of settled vertices contains $k$ vertices, each with correct shortest-path distance $d[v] = \delta(s, v)$, and for every unsettled vertex $u$, the current distance estimate $d[u]$ is the length of the shortest path from $s$ to $u$ that uses only $S$ as internal vertices. This is a direct transfer of "the first $i$ positions are correct" to "the first $i$ settled vertices have correct distances." The selection sort parallel: instead of selecting minimum elements, we select minimum-distance vertices. The proof of correctness follows the same initialization-preservation-termination structure.
</ExerciseSolution>

8. **Exchange transfer to Huffman codes.** The exchange argument for Huffman's algorithm (two smallest-frequency symbols as deepest siblings) uses a "swap to transform any optimal tree into one matching the greedy choice." Explain how this exchange differs from the interval scheduling exchange: what property of the objective makes the Huffman exchange more subtle?

<ExerciseSolution hint="The Huffman objective involves depths, not just presence/absence.">
In interval scheduling, swapping intervals only affects feasibility (whether they overlap) and the cardinality objective. In Huffman, swapping symbols changes the depth of the swapped symbols AND the depths of all symbols in the subtrees beneath them (if internal nodes are moved). The Huffman exchange is more subtle because the objective (weighted external path length $\sum f_i \cdot d_i$) depends on depths, and swapping two symbols at different depths changes the depth contributions of all symbols in their respective subtrees. The standard proof handles this by ensuring the swapped symbols are at the maximum depth, so moving a higher-frequency symbol to a shallower depth cannot increase the total cost. This requires the additional lemma that the deepest pair of siblings can be assumed to be the two smallest frequencies — a structural property not needed in interval scheduling.
</ExerciseSolution>

9. **Certificate transfer to bipartite matching.** Design a certificate scheme for maximum cardinality bipartite matching where the verifier runs in $O(V+E)$ time. Prove soundness and completeness.

<ExerciseSolution hint="Use augmenting paths or König's theorem.">
Certificate: a matching $M$ and a vertex cover $C$ of the same size. Verifier: check that $M$ is a valid matching (no shared vertices), that $C$ is a vertex cover (every edge touches $C$), and that $|M| = |C|$. Completeness: by König's theorem, in bipartite graphs the size of a maximum matching equals the size of a minimum vertex cover. Given a maximum matching $M$, a minimum vertex cover of size $|M|$ exists and can be computed by BFS from unmatched vertices in the residual graph. Soundness: if $|M| = |C|$, then $M$ is maximum because any matching has size at most any vertex cover (weak duality: every matching edge must be covered by a distinct vertex in $C$, so $|M'| \le |C| = |M|$ for any matching $M'$).
</ExerciseSolution>

10. **Proof critique.** Below is an attempted proof that the greedy algorithm for the set cover problem achieves a 2-approximation. Find at least two gaps. Is the claim actually true? (Hint: the true approximation ratio of the standard greedy algorithm for set cover is $H_d \approx \ln d$, not 2.)

> *Claim.* The greedy algorithm for set cover (repeatedly pick the set covering the most uncovered elements) achieves a 2-approximation.
>
> *Proof.* Let OPT be the optimal cover of size $k$. Consider any iteration where greedy picks set $S_i$ covering $c_i$ new elements. When OPT covers $n$ total elements with $k$ sets, by pigeonhole principle, some optimal set must cover at least $n/k$ elements. Greedy picks the maximum-coverage set, so $c_i \ge n/k$ in the first iteration. After $k$ iterations, the total covered elements is $\sum_{i=1}^k c_i \ge k \cdot (n/k) = n$, meaning all elements are covered. Thus greedy uses at most $k$ sets, giving a 1-approximation. The 2-approximation bound is therefore trivially satisfied.

<ExerciseSolution hint="The proof assumes the coverage ratio n/k remains constant across iterations, which is false.">
Gap 1: The argument $c_i \ge n/k$ is only guaranteed for the **first** iteration, when $n$ uncovered elements remain. After picking $S_1$, the number of remaining uncovered elements decreases, so the pigeonhole bound becomes $(n - c_1)/k$, not $n/k$. Subsequent iterations have weaker guarantees. The proof incorrectly assumes the same $n/k$ lower bound holds for all $i$.

Gap 2: Even the first-iteration guarantee is wrong. The pigeonhole bound says SOME optimal set covers at least $n/k$ elements, but greedy picks the set covering the MOST uncovered elements. Greedy's coverage $c_1$ is **at least** the average $n/k$, so $c_1 \ge n/k$ is actually correct for the first iteration. The error is in extending this to later iterations.

The claim itself is false. The actual approximation ratio of the standard greedy set cover is $H_d = 1 + 1/2 + \cdots + 1/d \approx \ln d$ (where $d$ is the size of the largest set), not 2. A counterexample exists where the ratio is arbitrarily close to $\ln d$ (the standard tight example: a universe of $d$ elements where greedy picks poorly). The "2-approximation" claim is unsupported — the proof only shows that a tighter bound might work for the first iteration, but the amortization across iterations is not analyzed.
</ExerciseSolution>

11. **Adversarial counterexample design.** The maximum cut problem: given an undirected graph $G = (V, E)$, partition $V$ into two sets $(S, V \setminus S)$ to maximize the number of edges crossing the cut. Claim: "The randomized algorithm that places each vertex independently in $S$ with probability $1/2$ gives a 2-approximation in expectation." Someone attempts to prove this by linearity of expectation and Pr[edge crosses cut] = $1/2$, giving expected cut size $|E|/2$, and since OPT $\le |E|$, the ratio is $\le 2$.

Construct a counterexample graph where this algorithm achieves expected cut size exactly $|E|/2$ but the ratio bound is tight (ratio approaches 2). Is there any graph where the algorithm achieves expected ratio worse than 2?

<ExerciseSolution hint="Consider a complete graph K_n.">
For $K_n$, $|E| = n(n-1)/2$. The maximum cut size is $\lfloor n^2/4 \rfloor$ (by a well-known bound for Max-Cut on complete graphs). The expected cut size is $|E|/2 = n(n-1)/4$. The ratio is:
$$ \text{OPT} / \mathbb{E}[\text{ALG}] \approx \frac{n^2/4}{n^2/4 - n/4} = \frac{n^2}{n^2 - n} = \frac{n}{n-1} \to 1 $$
So the ratio approaches 1 for large $n$, not 2. The bound $|E|/2$ is not tight. Actually the 2-approximation claim is TRUE (the algorithm IS a 2-approximation) but the proof through the ratio $|E|/\text{OPT}$ is loose. The analysis in the claim is incomplete but not wrong — the proof shows ratio $\le 2$ but does not prove it can approach 2. To construct a tight example where ratio approaches 2, consider a complete bipartite graph $K_{n,n}$ (which IS the maximum cut itself — OPT = $n^2$). The expected cut: each of the $n^2$ edges crosses with probability $1/2$ if its endpoints go to different sides, but in $K_{n,n}$ the optimal cut is already the bipartition. Actually $K_{n,n}$ is a poor counterexample. The tight example for this algorithm is a graph consisting of a single edge: OPT = 1, expected = 1/2, ratio = 2. So the bound is tight even though the proof via $|E|$ is loose.
</ExerciseSolution>
