# Randomized Algorithms

Randomized algorithms use random bits to influence execution. Randomness can simplify algorithms, avoid adversarial inputs, reduce expected work, or provide approximate answers. A rigorous analysis states the probability space, whether correctness is always guaranteed, what resource is expected, and how failure probability scales across repeated calls. “Random” is not a substitute for a proof; it changes the quantifiers in the proof.

## Las Vegas and Monte Carlo

A Las Vegas algorithm always returns a correct answer but has random running time. Randomized quicksort is Las Vegas if partitioning is exact: every pivot choice produces a correct sort, while the number of comparisons varies. A Monte Carlo algorithm has a bounded or controlled running time but may return an incorrect answer with probability at most epsilon, or may return an approximate answer with a proved guarantee.

For a fixed input x, expected time averages over the algorithm’s random bits. A stronger statement may hold for every x, while an average-over-inputs statement additionally assumes a distribution on x. Keep these separate. An adversary that chooses x before seeing the random seed is different from an adaptive adversary that observes outputs and learns the seed.

## Expected quicksort

Randomized quicksort chooses a uniformly random pivot, partitions, and recurses. Every input permutation is equally likely to occupy each pivot rank under the random-choice model. Let X indicate whether elements i and j are compared. They are compared at most once, and the probability of comparison is 2 divided by (j-i+1): among the elements between them, the first chosen pivot determines whether they meet, and it must be i or j.

Thus expected comparisons are the sum over all ordered pairs of 2 divided by (j-i+1), which is O(n log n). The correctness proof is deterministic; the expectation proof sums indicator variables using linearity of expectation, which does not require independence. Worst-case comparisons remain O(n squared), but the probability of a highly unbalanced sequence is small under independent random pivots.

## Randomized selection

Randomized quickselect partitions around a random pivot and recurses into only the side containing rank k. Conditional on pivot rank, the cost is linear partition plus the selected subproblem. The expected recurrence or a rank-indicator analysis gives O(n). Unlike quicksort, the algorithm does not sort discarded regions.

Tail bounds can be obtained by proving that with constant probability the pivot lies in a central fraction, then grouping iterations into phases. If each good phase reduces the problem by a constant factor, the number of bad phases before enough good ones has a geometric tail. Cap the work and restart if a rare sequence of bad pivots occurs; this converts expected behavior into a high-probability bound with a small failure/restart probability.

## Randomized incremental algorithms

Random insertion order often keeps expected structures shallow. A randomized treap assigns independent priorities; BST key order and heap priority order determine shape. The expected depth of a key is O(log n), because the first minimum-priority key in each relevant interval becomes an ancestor with probability tied to interval size. The proof uses random priorities, not random input order.

Randomized incremental geometry inserts objects in random order and maintains a solution, repairing only a local conflict region. Backward analysis asks which object was last responsible for the final feature; symmetry can show low expected update cost. The method requires a well-defined conflict graph and a bound on the expected number of features.

## Hashing and universal families

Universal hashing chooses a seed from a family such that any two distinct keys collide with probability at most 1/m. For a fixed set of keys, expected collisions are bounded even if keys are adversarially chosen before the seed. A keyed seed prevents an attacker from predicting the collision pattern, but the seed must be secret and changed safely.

Two-choice hashing computes two candidate buckets and inserts into the less-loaded one. Under suitable independence assumptions, the maximum load drops dramatically compared with one-choice hashing. The implementation and analysis must state whether relocation is allowed, how ties are handled, and what happens when both choices are full.

## Probability tools

Linearity of expectation says (E[sum X_i]=sum E[X_i]) regardless of independence. Markov’s inequality gives (Pr[X at least  a] at most  E[X]/a) for nonnegative X. Chebyshev uses variance; Chernoff bounds give exponentially small tails for sums of independent or suitably related indicator variables. Use the weakest tool that proves the required guarantee, and do not invoke a concentration bound without checking hypotheses.

If one randomized call fails with probability epsilon and calls are union-bounded over k events, total failure is at most kepsilon. To achieve total delta, choose per-event error delta/k or amplify each call. Independent repetition plus majority reduces error exponentially when each trial is correct with probability greater than one half. Correlation can invalidate the amplification.

## Randomness quality and reproducibility

A pseudorandom generator is deterministic from its seed. For algorithmic robustness, record a seed for debugging but do not expose a security seed to an adversary. Low-quality or biased random bits can break expected guarantees. Cryptographic randomness is unnecessary for ordinary randomized quicksort but essential when unpredictability protects against chosen-input attacks.

Reproducible tests should run fixed seeds and also many independent seeds. A single seed is not evidence for an expected bound. Instrument pivot ranks, hash collisions, retries, and failure events so an anomalous run can be explained.

## Amplification and restart

Suppose a Monte Carlo decision procedure returns the correct answer with probability at least 2/3 independently on each run. Repeat r times and take the majority. The expected number of correct trials is above r/2, and Chernoff bounds show the probability that the majority is wrong decreases exponentially in r. For a one-sided algorithm, repeat until a positive witness is found or use a bounded number of trials; the error direction determines whether union or intersection of answers is safe.

A Las Vegas restart strategy runs a randomized algorithm with a budget. If it fails to find a good pivot or certificate within the budget, discard state and restart with fresh randomness. If each attempt succeeds with probability p and costs at most B, expected attempts are 1/p and expected work B/p. To claim a high-probability total bound, bound the tail of the geometric number of attempts and include the cost of detecting failure and resetting memory.

## Adversaries

For a nonadaptive adversary, the input is fixed before the random seed. A randomized algorithm can then average over seeds even when the input is worst-case. An adaptive adversary can observe choices or timing and choose future operations; fresh secret randomness, obliviousness assumptions, or a stronger martingale analysis may be needed. Randomized quicksort with a public deterministic seed is vulnerable to an input generator that learns its pivots.

In online algorithms, randomization may improve competitive ratio against an oblivious adversary but not necessarily an adaptive one. State whether the adversary sees random choices, outputs, memory accesses, or only final answers. Security-sensitive data structures often use keyed hashing because distributional guarantees alone do not protect a public interface from collision attacks.

## Randomized algorithm checklist

Write the deterministic invariant first, identify the random event that yields progress, compute expectation or tail probability with explicit quantifiers, and specify behavior on rare bad sequences. Verify independence or conditional probability assumptions. Then test fixed and fresh seeds, adversarial inputs, and reproducible failure traces. A randomized algorithm is complete only when correctness, resource bound, and failure policy are all stated.

### Diagnostic solutions


1. Las Vegas algorithms are always correct with random cost, such as randomized quicksort; Monte Carlo algorithms have bounded or controlled cost but small error, such as a sampled primality test under its stated guarantee. State the exact error and runtime claims.

2. Let X indicate comparison of ordered elements i and j. They meet if and only if the first pivot chosen from their interval is an endpoint, with probability 2 divided by (j-i+1). Sum these indicators over pairs; harmonic sums give O(n log n).

3. A central pivot event has constant probability and shrinks the active set by a constant factor. The number of failures before a good event has geometric tail; summing linear partition work over phases gives expected O(n), and repetition or capping gives high-probability variants.

4. For every distinct fixed x and y, a uniformly selected hash from the family collides with probability at most 1/m. The keys may be adversarial but must be fixed independently of the secret seed for this statement.

5. If k tests each fail with probability at most epsilon, the union bound makes the probability of any failure at most k times epsilon. Set epsilon to delta divided by k to bound all failures by delta; independence is not needed.

6. Expectation is linear by algebra: the expectation of a sum is the sum of expectations. Dependence changes joint probabilities and variance, but not this sum identity.

7. Use fixed seeds for regression, independent seeds for distribution, pre-generate inputs, record environment and random source, measure output correctness, and report seed-wise variation. Do not infer an expected bound from one favorable run.
