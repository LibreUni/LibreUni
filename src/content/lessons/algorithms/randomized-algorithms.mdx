# Randomized Algorithms

Given three $n \times n$ matrices $A$, $B$, $C$, is $AB = C$? The naive approach multiplies $A$ and $B$ in $O(n^3)$ time and compares entrywise — $\Theta(n^3)$ operations. For $n = 10^4$, that is $10^{12}$ multiplications. Rusins Freivalds (1979) observed that a single matrix-vector multiplication $A(Br)$ costs $O(n^2)$ and can detect a wrong product with probability at least $1/2$ [Freivalds, 1979]. One randomized trial is cheaper than a single pass of the deterministic product, and $k$ trials drive the error below $2^{-k}$. Deterministic verification costs a cubic factor more; randomization buys a quadratic speedup with controllable failure probability.

This is the central trade-off of randomized algorithms: trade bounded error or expected runtime for orders-of-magnitude efficiency gains. Randomness is not a hack — it is a provable computational resource.

## Classification

Randomized algorithms fall into two categories based on what they randomize.

**Las Vegas:** Always returns the correct answer; only the runtime is random. Example: randomized quicksort runs in $O(n \log n)$ expected time on every input.

**Monte Carlo:** Runs in bounded time but may err with controllable probability. Example: Miller–Rabin primality testing declares "prime" or "composite"; a composite slips through with probability $\le 4^{-k}$ after $k$ trials.

## Probability Toolkit

The analysis of randomized algorithms rests on a small set of concentration inequalities.

<MathStatement id="markov" kind="theorem" title="Markov's inequality">
For a nonnegative random variable $X$ and $a > 0$, $\Pr[X \ge a] \le E[X]/a$.
</MathStatement>

<MathStatement id="chebyshev" kind="theorem" title="Chebyshev's inequality">
For $X$ with finite mean $\mu$ and variance $\sigma^2$, $\Pr[|X - \mu| \ge t] \le \sigma^2 / t^2$.
</MathStatement>

<MathStatement id="chernoff" kind="theorem" title="Chernoff bound (multiplicative)" proofId="chernoff-proof">
Let $X = \sum_{i=1}^n X_i$ where $X_i \in \{0,1\}$ are independent, $\mu = E[X]$. For $0 < \delta < 1$:

$$
\Pr[X \le (1-\delta)\mu] \le e^{-\mu\delta^2/2}, \quad
\Pr[X \ge (1+\delta)\mu] \le e^{-\mu\delta^2/3}.
$$
</MathStatement>

<MathProof id="chernoff-proof" for="chernoff" collapsible open>
Apply the moment-generating function $\Pr[X \ge (1+\delta)\mu] = \Pr[e^{tX} \ge e^{t(1+\delta)\mu}]$. By Markov, $\Pr[e^{tX} \ge e^{t(1+\delta)\mu}] \le E[e^{tX}] / e^{t(1+\delta)\mu}$. For independent Bernoulli variables, $E[e^{tX_i}] = 1 - p_i + p_i e^t \le e^{p_i(e^t - 1)}$. Combining: $E[e^{tX}] \le e^{(e^t-1)\mu}$. Choose $t = \ln(1+\delta)$ and simplify to $e^{-\mu\delta^2/3}$. The lower-tail bound follows symmetrically with $t = \ln(1-\delta)$. □
</MathProof>

<MathStatement id="union-bound" kind="theorem" title="Union bound (Boole's inequality)">
$\Pr[\bigcup_i A_i] \le \sum_i \Pr[A_i]$.
</MathStatement>

**Linearity of expectation:** $E[\sum X_i] = \sum E[X_i]$ — no independence required. This is the most frequently used tool in randomized algorithm analysis [Motwani & Raghavan, 1995].

## Randomized Quicksort: Indicator Analysis

**Theorem.** Randomized quicksort makes $2n \ln n + O(n)$ expected comparisons [Motwani & Raghavan, 1995].

**Proof.** Sort the elements to $z_1 < z_2 < \cdots < z_n$. For $i < j$, let $X_{ij} = 1$ if $z_i$ and $z_j$ are compared. They are compared iff the first pivot chosen from the set $\{z_i, \ldots, z_j\}$ is $z_i$ or $z_j$. Since the pivot is uniformly random among these $j-i+1$ elements:

$$
\Pr[X_{ij} = 1] = \frac{2}{j-i+1}.
$$

Expected comparisons: $E[\sum_{i<j} X_{ij}] = \sum_{i<j} \frac{2}{j-i+1} = 2 \sum_{k=1}^{n-1} \frac{n-k}{k+1} = 2n H_n - 4n + O(1) = 2n \ln n + O(n)$. □

**Tail bound.** The probability that quicksort exceeds $c \cdot n \log n$ comparisons is polynomially small for large $c$, provable via Chernoff bounds on bad pivot intervals.

def quicksort(A):
    if len(A) <= 1:
        return A
    pivot = random.choice(A)
    left = [x for x in A if x < pivot]
    mid  = [x for x in A if x == pivot]
    right = [x for x in A if x > pivot]

for seed in range(3):

## Miller–Rabin Primality Test

**Problem.** Determine if $n$ is prime [Rabin, 1980].

Write $n-1 = 2^s \cdot d$ with $d$ odd. Pick random $a \in [1, n-1]$. Compute $x_0 = a^d \bmod n$. For $i = 1, \ldots, s-1$, compute $x_i = x_{i-1}^2 \bmod n$. If $x_0 \equiv 1 \pmod n$ or some $x_i \equiv -1 \pmod n$, declare "prime"; otherwise declare "composite."

  :Declare "composite";

/>

<MathStatement id="miller-rabin" kind="theorem" title="Miller–Rabin correctness">
If $n$ is composite, at most $1/4$ of bases $a$ cause the algorithm to declare "prime" incorrectly. Thus $k$ independent trials give error probability $\le 4^{-k}$.
</MathStatement>

<MathProof id="miller-rabin-proof" for="miller-rabin" collapsible>
For odd composite $n$, the set of *strong liars* (bases that incorrectly pass the test) forms a proper subgroup of $\mathbb{Z}_n^*$. By Lagrange's theorem, $|\text{liars}| \le |\mathbb{Z}_n^*| / 2$. A more refined algebraic analysis shows the bound improves to $1/4$ for all odd composites [Rabin, 1980; Mitzenmacher & Upfal, 2017]. □
</MathProof>

The test runs in $O(k \log^3 n)$ time, polynomial and practical for thousand-digit numbers.

## Karger's Min-Cut Algorithm

**Problem.** Find the minimum cut (set of edges whose removal disconnects the graph) in an undirected unweighted graph [Karger, 1993].

While $> 2$ vertices remain, pick a random edge and contract it (merge its endpoints, keep parallel edges). The two remaining vertices define a cut. Repeat independently many times and return the smallest cut found.

<MathStatement id="karger-mincut" kind="theorem" title="Karger's min-cut success probability" proofId="karger-proof">
Each trial finds a specific minimum cut with probability at least $\binom{n}{2}^{-1}$.
</MathStatement>

<MathProof id="karger-proof" for="karger-mincut" collapsible>
Fix a particular minimum cut $C$ of size $k$. The graph has minimum degree $\ge k$ (otherwise the min-degree vertex's incident edges would form a smaller cut).

In the first step, $|E| \ge nk/2$ (since $\min \text{deg} \ge k$), so $\Pr[\text{contract } e \in C] = k / |E| \le 2/n$. Thus $\Pr[\text{not contract } e \in C] \ge 1 - 2/n$.

Conditional on no $C$-edge contracted in step $j$ (with $n-j+1$ vertices remaining), $\Pr[\text{contract } e \in C \mid \text{success so far}] = k / |E_j| \le 2/(n-j+1)$. Product over $j=1$ to $n-2$:

$$
\Pr[\text{success}] \ge \prod_{j=0}^{n-3} \left(1 - \frac{2}{n-j}\right) = \prod_{i=3}^{n} \frac{i-2}{i} = \frac{2}{n(n-1)} = \binom{n}{2}^{-1}.
$$

Repeating $O(n^2 \log n)$ trials gives high probability of finding the min cut. □
</MathProof>

## Freivalds' Matrix Product Checker

**Problem.** Given $n \times n$ matrices $A$, $B$, $C$, verify $AB = C$ [Freivalds, 1983].

Pick random $r \in \{0,1\}^n$. Compute $x = Br$, $y = Ax = ABr$, $z = Cr$. If $y = z$, accept; else reject.

<MathStatement id="freivalds" kind="theorem" title="Freivalds' algorithm" proofId="freivalds-proof">
If $AB \neq C$, $\Pr[\text{accept}] \le 1/2$. Running $k$ independent trials reduces error to $\le 2^{-k}$.
</MathStatement>

<MathProof id="freivalds-proof" for="freivalds" collapsible>
Let $D = AB - C \neq 0$. Some row $d$ of $D$ is nonzero: $\exists j$ with $d_j \neq 0$. Compute $Dr = ABr - Cr = y - z$. If $y = z$, then $Dr = 0$.

Consider the $j$th entry: $(Dr)_j = \sum_{k=1}^n D_{jk} r_k$. Since $D_{j\ell} \neq 0$ for some $\ell$, treat $r_\ell$ as variable: $(Dr)_j = D_{j\ell} r_\ell + \sum_{k \neq \ell} D_{jk} r_k$. By the principle of deferred decisions, fix all $r_k$ for $k \neq \ell$. Then $(Dr)_j = 0$ iff $r_\ell = -\frac{1}{D_{j\ell}} \sum_{k \neq \ell} D_{jk} r_k$, which holds for at most one of the two values of $r_\ell$. Hence $\Pr[(Dr)_j = 0] \le 1/2$. □
</MathProof>

Time: $O(n^2)$ per trial, much faster than $O(n^{\log_2 7})$ for Strassen or $O(n^3)$ for naive multiplication.

## Random Walks and 2-SAT

Random 2-SAT can be solved in polynomial time by a random walk [Motwani & Raghavan, 1995]. Start with a random assignment. While unsatisfied, pick an unsatisfied clause, flip a random variable in it. For $n$ variables, the expected number of flips is $O(n^2)$. Let $T$ be the number of variables assigned correctly in a fixed satisfying assignment. Each step changes $T$ by $\pm 1$ with probability at least $1/2$. Expected hitting time to $T=n$ is $O(n^2)$.

## Randomized Rounding for Approximation

Given an LP relaxation with fractional solution $x^*$, round each variable $x_i^*$ to 1 with probability $x_i^*$ and 0 otherwise. For set cover, this yields expected cost $= \text{OPT}_\text{LP}$ and covers each element with probability $\ge 1 - 1/e$. Repeating $O(\log n)$ rounds covers all elements with high probability, giving an $O(\log n)$-approximation [Motwani & Raghavan, 1995; Mitzenmacher & Upfal, 2017].

## Counterexample and Failure Modes

Randomized algorithms are not a universal remedy. Several failure modes arise in practice:

**Adversarial randomness.** If the random number generator is predictable (e.g., `rand()` in early C libraries with period $2^{31}$), an adversary can reconstruct the internal state and force worst-case behavior. Miller–Rabin with deterministic witness sets (e.g., the first $k$ primes) stops being randomized and becomes a deterministic test that fails for specific composites.

**High-probability vs. expectation.** An algorithm with $O(n^2)$ expected runtime can, in pathological cases, run forever. For Las Vegas algorithms, this matters in real-time systems where a hard deadline must be met. The expected-case analysis does not guarantee any per-instance bound.

**Chernoff bound failure for dependent variables.** The multiplicative Chernoff bound requires independence. When variables are weakly dependent (e.g., in Karger's algorithm, the events "contract a $C$-edge at step $j$" are conditionally independent only after conditioning on success), a different martingale-based analysis is needed.

**The birthday paradox in Monte Carlo algorithms.** When the error probability of a Monte Carlo algorithm is $2^{-k}$, running $2^{k}$ instances of the algorithm against the same input makes it likely that one instance observes the rare error event — a fact exploited by the Blum–Micali theorem on pseudorandom generators. More concretely, if $10^6$ users each run Miller–Rabin with $k=10$ on a specific composite $n$, the expected number of false primes declared is $10^6 \cdot 4^{-10} \approx 0.95$ — nearly certain that someone sees a mistake.

5. (Transfer) **Miller–Rabin.** For $n = 15$, list the strong liars and compute the probability a random base falsely declares 15 prime. Verify the $1/4$ bound.

   <ExerciseSolution>
   For $n=15$, $n-1 = 14 = 2 \cdot 7$, so $s=1, d=7$. Bases $a$ with $a^7 \equiv \pm 1 \pmod{15}$ are liars. Computing: $2^7=128 \equiv 8$, not liar. $4^7 \equiv 4$, $7^7 \equiv 13$, $11^7 \equiv 11$, $13^7 \equiv 7$. The liars are $a$ where $a^7 \equiv 1$ or $14$: $1^{14}\equiv 1$, $4^{14} \equiv 1$, $11^{14} \equiv 1$, $14^{14}\equiv 1$. Wait check $4^7$: $4^2=16\equiv 1$, so $4^7 = 4\cdot (4^2)^3 = 4\cdot 1^3 = 4 \not\equiv \pm 1$. Actually none except $1, 14$ are liars. So $2/14 = 1/7 < 1/4$, confirmed.
   </ExerciseSolution>

6. (Synthesis) **Freivalds with arbitrary vector entries.** Show that Freivalds' algorithm works with $r$ drawn uniformly from $\{0,1,\ldots,m-1\}^n$ for any integer $m \ge 2$. What is the error probability? How does it change with $m$? *Hint: generalize the deferred-decision argument to $m$ values.*

   <ExerciseSolution>
   Generalizing the proof: $(Dr)_j = D_{j\ell} r_\ell + \sum_{k\neq\ell} D_{jk} r_k$. Fixing all $r_k$ for $k\neq\ell$, $(Dr)_j = 0$ iff $r_\ell = -D_{j\ell}^{-1} \sum_{k\neq\ell} D_{jk} r_k$, which holds for at most $1$ of $m$ values. Thus $\Pr[(Dr)_j=0] \le 1/m$. Error probability per trial improves to $1/m$, giving $\le m^{-k}$ after $k$ trials.
   </ExerciseSolution>

7. (Synthesis) **Derandomizing Freivalds via pairwise independence.** Prove that Freivalds' algorithm still works if the $n$ entries of $r$ are only pairwise independent (not fully independent). Construct a sample space of size $O(n^2)$ supporting pairwise independent $\{0,1\}$-valued random variables, and show how this gives a deterministic verification algorithm with $O(n^2)$ time per test. *Hint: use the fact that Chebyshev's inequality and pairwise independence suffice for the $1/2$ bound. For the sample space, use arithmetic progressions modulo a prime.*

   <ExerciseSolution>
   For pairwise independent $r_i$ with $E[r_i] = 1/2$, $\text{Var}((Dr)_j) = \sum_k D_{jk}^2 \text{Var}(r_k)$ since covariance terms vanish. A Chebyshev bound gives $\Pr[(Dr)_j = 0] \le \text{Var}((Dr)_j) / (E[(Dr)_j])^2$. With $E[(Dr)_j] = \frac{1}{2} \sum_k D_{jk}$ and $\text{Var}((Dr)_j) = \frac{1}{4} \sum_k D_{jk}^2$, and assuming $D$ has no zero rows, one can bound $\Pr[(Dr)_j = 0] \le 1/2$.

   A construction: let $\mathbb{F}_p$ be a prime field with $p > n$. For each $a \in \mathbb{F}_p$, define $r_i = ((a \cdot i) \bmod p) \bmod 2$. These $p$ vectors are pairwise independent. Checking all $p = O(n)$ vectors yields a deterministic $O(n^3)$ verifier — no faster than naive multiplication, but the same idea with higher-order independence can yield a deterministic subcubic verifier.
   </ExerciseSolution>
