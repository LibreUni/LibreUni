# Recurrences and the Master Method

Recursive algorithms reduce a problem to smaller instances and combine their results. A recurrence expresses total cost, but only after the algorithm’s call structure, base cases, partition overhead, and parameter rounding are understood. Solving recurrences is therefore part of algorithm design, not bookkeeping after code is written. This lesson develops several methods and highlights conditions under which a shortcut is invalid.

## Translating code to a recurrence

For a recursive routine, define T(n) as worst-case cost on size n. Count the number and sizes of recursive calls, the nonrecursive work, and the base threshold. If an algorithm makes two calls on n/2 and performs a linear merge, (T(n)=2T(n/2)+cn) for n greater than n0, with T(n)=Theta(1) below n0. If partitions are unequal, preserve those sizes: replacing n/3 and 2n/3 with two n/2 calls can change the analysis.

The chosen unit matters. T(n) may count comparisons, arithmetic operations, bytes moved, or wall-clock work. A recurrence for comparisons can ignore the cost of copying a large record; a recurrence for time cannot. If a subroutine has cost S(n), include S(n) rather than calling it constant without a bound.

**Worked translation.** A procedure scans an array to find a pivot, recursively processes subarrays of sizes a and n-a-1, and performs constant partition bookkeeping. The recurrence is (T(n)=T(a)+T(n-a-1)+Theta(n)). If a is near n/2, the solution is Theta(n log n); if a=0 at every level, it is Theta(n^2). The code is the same; input/pivot behavior determines the branch.

## Expansion and substitution

Expansion repeatedly replaces recursive terms. For (T(n)=T(n/2)+c), after k expansions (T(n)=T(n/2^k)+kc). The recursion stops when n/2^k at most n0, so k=Theta(log n), yielding Theta(log n). For (T(n)=2T(n/2)+cn), level j has 2^j calls of size n/2^j and total cn, producing Theta(n log n).

Substitution proves a guessed bound by induction. To prove (T(n)=O(n log n)) for merge sort, assume the claim for smaller inputs, substitute (2c(n/2)log(n/2)+dn), and choose c large enough to absorb the lower-order term and base cases. A naive guess may fail because the induction needs slack; use (c nlog n-dn) or another strengthened hypothesis when necessary.

For lower bounds, reverse the inequality. If the recurrence has at least two calls and linear combine work, prove Omega(n log n) by showing each recursion level contributes Omega(n) until the leaves. State floor/ceiling effects or bound them by constant factors.

## Recursion trees

A recursion tree groups equal-depth calls. For (T(n)=aT(n/b)+f(n)), level j has a^j nodes of size n/b^j, so level work is (a^j f(n/b^j)). The height is log_b n. Sum all levels and leaves. If level costs increase, leaves dominate; if decrease, the root or upper levels dominate; if equal, multiply by height.

**Three patterns.** For (2T(n/2)+n), every level costs n and there are log n levels, giving n log n. For (2T(n/2)+n^2), level j costs n^2/2^j, giving n^2. For (2T(n/2)+1), internal work sums to n and leaves contribute n, giving n. The tree makes the dominant source visible.

## Master theorem

For (T(n)=aT(n/b)+f(n)), with a at least 1, b greater than 1, compare f(n) to (n^()log_b a()).

1. If (f(n)=O(n^()log_b a-epsilon())) for epsilon greater than 0, then (T(n)=Theta(n^()log_b a())).
2. If (f(n)=Theta(n^()log_b a()log^k n)) for k at least 0, then (T(n)=Theta(n^()log_b a()log^()k+1()n)).
3. If (f(n)=Omega(n^()log_b a+epsilon())) and the regularity condition (a f(n/b) at most  c f(n)) for c less than 1 holds, then (T(n)=Theta(f(n))).

The regularity condition prevents f from oscillating so that a single level unexpectedly dominates. The theorem does not directly apply to unequal subproblem sizes, additive subproblem sizes, or functions falling between the stated cases without an extended theorem. Do not force a recurrence into the template.

## Akra-Bazzi intuition

For unequal fixed fractions, a recurrence (T(x)=sum_i a_iT(b_i x)+g(x)) can be analyzed by finding p satisfying (sum_i a_i b_i^p=1), then integrating the combine cost. This generalizes the Master method. For (T(n)=T(n/3)+T(2n/3)+n), p=1 because 1/3+2/3=1, and the solution is Theta(n log n) under regularity. The recursion tree has unequal shapes but total work per broad level remains linear.

## Recurrences with overlap

Divide-and-conquer recurrence trees assume recursive calls are independent. Naive Fibonacci recursion has (T(n)=T(n-1)+T(n-2)+O(1)) and exponential growth because the same subproblems repeat. Memoization changes the dependency graph from a tree to a DAG and computes each state once, often reducing time to O(n). The recurrence for the memoized algorithm is not the same as the call-count recurrence for the naive one.

Dynamic programming is therefore a recurrence-design transformation: identify state, transitions, dependency order, and base cases; count states and transition work. A table can use bottom-up order or top-down memoization. Space can often be reduced if each state depends only on a bounded frontier, but reconstructing a solution may require storing decisions.

## Base cases and rounding

A recurrence is incomplete without a base threshold and a statement about floors and ceilings. If (T(n)=T(lfloor n/2rfloor)+1), the argument reaches 0 or 1 after Theta(log n) steps; an implementation must ensure that the recursive call is strictly smaller. Using (n/2) in the proof is safe only when floors preserve a constant-factor bound and the base case handles n=0, n=1, and any empty subproblem.

Partition algorithms often create empty subarrays. A recurrence that assumes every recursive call has positive size may not describe a valid implementation. Define T(0) and T(1), and include the cost of partitioning even when one recursive side is empty. For a randomized analysis, condition on the pivot rank and take the expected sum over ranks; for a worst-case analysis, choose the most unbalanced allowed rank.

## Iterative recurrences and amortization

Not every nested computation is recursive. A loop whose work depends on a shrinking variable can be analyzed with a sum or a recurrence over state. For example, repeatedly remove half the remaining items and scan the current set costs (n+n/2+n/4+cdots=O(n)). If each phase doubles a work parameter and performs a scan of that parameter, the total is a geometric sum. Writing the phase recurrence often prevents counting every scan as n.

Recurrences can describe amortized rebuilds as well. A table rebuild at sizes (1,2,4,ldots) has costs that sum geometrically; a sequence recurrence (C(m) at most  C(m-1)+1+text()rebuild()(m)) can be bounded by aggregate accounting. The operation-level worst case remains high, so state whether the desired result is total sequence cost or per-operation latency.

## Recurrence proof checklist

Before applying a theorem, verify: the subproblem parameter is an integer or can be bounded monotonically; every call decreases it; all calls are counted; combine work includes copying and comparisons; base cases cover malformed or empty inputs; and any regularity or distribution assumption is stated. Then solve with a method whose hypotheses match the recurrence. If two methods disagree, inspect the translation before trusting either algebraic manipulation.

When presenting a solution, give both an asymptotic result and a short reason: dominant recursion-tree level, induction inequality, or theorem case. A student should be able to reconstruct why the answer holds and identify what would change if the partition ratio, combine work, or memoization policy changed.

## Exercises

1. Derive the recurrence for merge sort and solve it by a recursion tree.
2. Solve (T(n)=T(n/2)+1) by expansion.
3. Use substitution to prove a Theta(n log n) merge recurrence.
4. Apply the Master theorem to three cases and identify an invalid case.
5. Analyze unequal recurrence (T(n)=T(n/3)+T(2n/3)+n).
6. Explain why memoized Fibonacci has a different cost recurrence.
7. State what a recurrence omits when records have variable length.

### Diagnostic solutions


1. (T(n)=2T(n/2)+Theta(n)). Each level has total Theta(n) combine work and Theta(log n) levels; leaves contribute Theta(n), so T=Theta(n log n).

2. After k expansions, (T(n)=T(n/2^k)+k). Stop at k=Theta(log n), so T=Theta(log n), including constant base cost.

3. Assume T(m) at most c m log m for smaller m. Then (2c(n/2)log(n/2)+dn=cnlog n-cn+dn at most  cnlog n) when c at least d, with base cases absorbed by a constant. A matching lower induction gives Theta.

4. (2T(n/2)+n^2) is case 3 and Theta(n^2); (2T(n/2)+n) is case 2 and Theta(n log n); (2T(n/2)+1) is case 1 and Theta(n). Unequal (T(n/3)+T(2n/3)+n) is not directly the standard Master form.

5. The critical exponent p solves (1/3)^p+(2/3)^p=1, giving p=1. The linear combine term matches n^p, and an unequal recursion tree has Theta(log n) effective levels, so T=Theta(n log n).

6. Naive Fibonacci recomputes the same arguments through exponentially many calls. Memoization stores each integer state once and performs constant transition work, yielding O(n) time and O(n) space, or O(1) rolling space if only the value is needed.

7. It may omit key length, bit complexity, bytes copied, allocation, cache transfers, I/O, or output size. Add parameters or state a unit-cost assumption; otherwise an apparent constant operation may scale with representation length.
