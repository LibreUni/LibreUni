# Recurrences and the Master Method

How many operations does merge sort perform on $10^6$ items? The algorithm divides the array in half, sorts each half recursively, and merges. The cost is

$$
T(n) = 2T(n/2) + n, \qquad T(1) = 1.
$$

Unrolling one level: $T(10^6) = 2T(500,000) + 10^6$. Two levels: $4T(250,000) + 2\cdot 10^6$. After $k$ levels, $T(10^6) = 2^k T(10^6/2^k) + k \cdot 10^6$. The recursion bottoms out when $10^6/2^k \approx 1$, i.e., $k = \log_2 10^6 \approx 20$. The total: $10^6 \cdot 1 + 20 \cdot 10^6 = 21 \cdot 10^6$ operations.

Unrolling level by level works for simple recurrences but is impractical when branching is irregular. For $T(n) = T(n/3) + T(2n/3) + n$, the branching produces a combinatorial explosion of paths. A *closed form* — an expression giving $T(n)$ directly — eliminates iteration. The oldest known recurrence in Western mathematics, Fibonacci's $F_n = F_{n-1} + F_{n-2}$ (1202), admits the closed form $F_n = (\phi^n - \psi^n)/\sqrt{5}$, which reveals exponential growth invisible from the recurrence alone.

Every recursive program generates a recurrence. The major solution methods are substitution with induction, recursion trees, linear recurrences and characteristic equations, generating functions, and the Master Theorem — a single theorem solving $T(n) = aT(n/b) + f(n)$ by comparing $f(n)$ to $n^{\log_b a}$.

## Linear Recurrences: Exact Solutions

Divide-and-conquer recurrences are typically inhomogeneous. But many algorithms — including dynamic programming, combinatorial enumeration, and probabilistic analysis — yield **linear recurrences** with constant coefficients. These can be solved exactly via the characteristic equation.

<MathStatement id="linear-recurrence" kind="definition" title="Linear homogeneous recurrence">
A $k$th-order linear homogeneous recurrence is:
$$
T(n) = c_1 T(n-1) + c_2 T(n-2) + \cdots + c_k T(n-k)
$$
with $c_k \neq 0$. Its characteristic polynomial is $x^k - c_1 x^{k-1} - \cdots - c_k = 0$.
</MathStatement>

**Example: Fibonacci.** $F_n = F_{n-1} + F_{n-2}$ with $F_0 = 0$, $F_1 = 1$. The characteristic polynomial $x^2 - x - 1 = 0$ has roots $\phi = (1 + \sqrt{5})/2$ and $\psi = (1 - \sqrt{5})/2$. The general solution is $F_n = A \phi^n + B \psi^n$. Using initial conditions:

$$
F_0 = A + B = 0, \quad F_1 = A\phi + B\psi = 1.
$$

Solving yields $A = 1/\sqrt{5}$, $B = -1/\sqrt{5}$, giving Binet's formula:

$$
F_n = \frac{1}{\sqrt{5}}\left( \phi^n - \psi^n \right) = \Theta(\phi^n).
$$

**Inhomogeneous recurrences.** For $T(n) = T(n-1) + f(n)$, the homogeneous solution is $T_h(n) = A \cdot 1^n = A$, and a particular solution depends on $f(n)$. If $f(n)$ is a polynomial of degree $d$, try a polynomial of degree $d$; if $f(n) = r^n$, try $C \cdot r^n$ (unless $r$ is a root of the characteristic polynomial, in which case multiply by $n$).

## The Substitution Method

The substitution method proves an asymptotic bound by induction. It requires a guess and careful handling of base cases.

<MathStatement id="substitution" kind="method" title="Substitution method protocol">
1. **Guess** the form of the solution: $T(n) = O(g(n))$.
2. **Assume** the bound holds for all $m < n$.
3. **Substitute** into the recurrence and simplify.
4. **Choose constants** to satisfy the inequality for large $n$.
5. **Verify base cases** — adjust constants or add lower-order terms if needed.
</MathStatement>

**Worked example: $T(n) = 4T(n/2) + n$.** Guess $T(n) = O(n^2)$. Inductive assumption: $T(m) \le c m^2$ for $m < n$.

$$
T(n) \le 4c (n/2)^2 + n = 4c (n^2/4) + n = c n^2 + n.
$$

We need $c n^2 + n \le c n^2$, which is false. The guess $O(n^2)$ is correct asymptotically, but the induction needs slack. Strengthen the hypothesis: $T(n) \le c n^2 - d n$.

$$
T(n) \le 4(c(n/2)^2 - d(n/2)) + n = c n^2 - 2d n + n = c n^2 - (2d - 1)n.
$$

We need $c n^2 - (2d - 1)n \le c n^2 - d n$, i.e., $-(2d - 1) \le -d$, i.e., $d \ge 1$. Choose $d = 1$, $c$ large enough for base cases.

### Dealing with Floors and Ceilings

When the recurrence uses $\lfloor n/2\rfloor$ or $\lceil n/2\rceil$, the substitution must handle the rounding. A common technique: prove the bound for powers of 2, then extend to all integers via monotonicity.

**Lemma (monotonic extension).** If $T(n)$ is nondecreasing and $T(n) = O(g(n))$ for $g$ that is eventually nondecreasing and satisfies $g(2n) = O(g(n))$, then $T(n) = O(g(n))$ for all $n$.

**Proof sketch.** For any $n$, let $m = 2^{\lceil \log_2 n\rceil}$ be the next power of 2. Then $T(n) \le T(m) \le c g(m) \le c g(2n) \le c C g(n) = O(g(n))$. The condition $g(2n) = O(g(n))$ (polynomial growth) is essential; it holds for all common growth rates ($n$, $n \log n$, $n^k$, $2^n$).

## Recursion Trees

The recursion-tree method provides an intuitive picture of cost distribution across levels. It is especially useful when the master theorem does not apply.

<MathStatement id="recursion-tree-cost" kind="theorem" title="Recursion tree cost formula">
For $T(n) = aT(n/b) + f(n)$, the total cost is:

$$
T(n) = \Theta(n^{\log_b a}) + \sum_{j=0}^{\log_b n - 1} a^j f(n/b^j).
$$

The first term accounts for leaf costs ($n^{\log_b a}$ leaves, each $\Theta(1)$). The sum tracks internal work.
</MathStatement>

**Worked example: $T(n) = 2T(n/2) + n/\log n$.** The master theorem does not apply because $n/\log n$ is not $\Theta(n)$ nor $O(n^{1-\epsilon})$. Use the recursion tree.

At depth $j$, there are $2^j$ nodes of size $n/2^j$, each costing $(n/2^j) / \log(n/2^j) = (n/2^j) / (\log n - j \log 2)$. Level total:

$$
2^j \cdot \frac{n/2^j}{\log n - j} = \frac{n}{\log n - j}.
$$

Summing over $j = 0$ to $\log n - 1$:

$$
T(n) = \sum_{j=0}^{\log n - 1} \frac{n}{\log n - j} = n \sum_{k=1}^{\log n} \frac{1}{k} = n H_{\log n} = \Theta(n \log \log n).
$$

## Master Theorem

<MathStatement id="master-theorem" kind="theorem" title="Master theorem (full form)">
Let $a \ge 1$, $b > 1$, $f$ asymptotically positive. Define $T(n) = aT(n/b) + f(n)$ with $n/b$ rounded to floor or ceiling. Then:

1. If $f(n) = O(n^{\log_b a - \varepsilon})$ for $\varepsilon > 0$, then $T(n) = \Theta(n^{\log_b a})$.
2. If $f(n) = \Theta(n^{\log_b a} \log^k n)$ for $k \ge 0$, then $T(n) = \Theta(n^{\log_b a} \log^{k+1} n)$.
3. If $f(n) = \Omega(n^{\log_b a + \varepsilon})$ and $a f(n/b) \le c f(n)$ for some $c < 1$ and all $n \ge n_0$, then $T(n) = \Theta(f(n))$.
</MathStatement>

<MathProof id="master-case2-proof" for="master-theorem" title="Case 2 derivation" collapsible open>
Assume $f(n) = n^{\log_b a}$. The recursion tree has height $h = \log_b n$. At depth $j$, work is $a^j f(n/b^j) = a^j (n/b^j)^{\log_b a} = n^{\log_b a} (a/b^{\log_b a})^j = n^{\log_b a}$. Each level contributes $\Theta(n^{\log_b a})$, and there are $\log_b n$ levels, giving $\Theta(n^{\log_b a} \log n)$.

For $f(n) = n^{\log_b a} \log^k n$, the depth-$j$ work is $n^{\log_b a} \log^k (n/b^j)$. The sum $\sum_{j=0}^{h-1} \log^k (n/b^j) = \Theta(\log^{k+1} n)$ by the integral approximation $\sum_{j=0}^{h-1} (\log n - j \log b)^k \sim \int_0^{\log n} (\log n - x)^k \, dx = \Theta(\log^{k+1} n)$.
</MathProof>

## Counterexample: When the Master Theorem Fails

The Master Theorem is widely applicable but not universal. Applying it without checking preconditions produces erroneous bounds.

> **Counterexample: Naive application of Case 1.**  
> $T(n) = 2T(n/2) + n / \ln n$. Here $\log_b a = \log_2 2 = 1$, and $f(n) = n / \ln n$. Compare $f(n)$ to $n^{1-\varepsilon}$: for any $\varepsilon > 0$, $n / \ln n$ eventually exceeds $n^{1-\varepsilon}$ because $\ln n$ grows slower than $n^\varepsilon$. Thus $f(n)$ is **not** $O(n^{1-\varepsilon})$; Case 1 does not apply. Neither does Case 2 ($f$ is not $\Theta(n)$) nor Case 3 ($f$ is not $\Omega(n^{1+\varepsilon})$). The Master Theorem is silent. The recursion tree resolves it to $\Theta(n \log \log n)$.

> **Counterexample: Regularity condition failure.**  
> $T(n) = 2T(n/2) + n(2 - \cos n)$. Here $\log_b a = 1$, $f(n) = \Theta(n)$, so Case 2 appears applicable, giving $\Theta(n \log n)$. But $f$ oscillates: the hidden constant in $f(n) = \Theta(n)$ masks that $f$ takes values arbitrarily close to $n$ and $3n$ across large spans. The recursion tree's level sums do not converge uniformly. This recurrence still yields $\Theta(n \log n)$ by a more careful bound, but a mechanical Master Theorem application that only checks $f(n) = \Theta(n)$ without verifying the polylog factor domain is not a full justification.

> **Non-example: Non-polynomial $f$.**  
> $T(n) = 2T(n/2) + 2^n$: $f$ grows so fast the root dominates, giving $T(n) = \Theta(2^n)$. Case 3 applies (regularity holds: $2 \cdot 2^{n/2} \le c \cdot 2^n$ for $c = 1/2$, $n \ge 4$). This is a valid application, not a counterexample — included to contrast with the cases above where the theorem fails.

## Akra–Bazzi Theorem

<MathStatement id="akra-bazzi" kind="theorem" title="Akra–Bazzi theorem">
Let $T(n) = \sum_{i=1}^k a_i T(b_i n + h_i(n)) + f(n)$, where:
- $a_i > 0$, $b_i \in (0, 1)$,
- $|h_i(n)| = O(n / \log^2 n)$ (handling floors/ceilings),
- $f$ satisfies $|f'(n)| = O(n^{\delta})$ for some $\delta$ (polynomial growth).
Define $p$ by $\sum_{i=1}^k a_i b_i^p = 1$. Then:

$$
T(n) = \Theta\left( n^p \left(1 + \int_1^n \frac{f(u)}{u^{p+1}} \, du \right) \right).
$$
</MathStatement>

**Proof sketch.** Expand the recurrence to depth where subproblems reach constant size. The number of subproblems of size $u$ at some level is approximately $\Theta(u^{p-1})$ (by the branching equation). The total work is the integral of $f(u)$ weighted by the density of subproblems at size $u$:

$$
\int_1^n (\text{number of subproblems of size } u) \cdot f(u) \, du \approx \int_1^n u^{p-1} f(u) \, du.
$$

But $u^{p-1} = u^{p} / u$, and when $f(u) = u^q$, the integral gives $\Theta(n^p + n^q)$ depending on $q$ vs $p$. More precisely, when $f(n)$ is polynomial, the leading term is $n^{\max(p, q)}$ with log factor when $p = q$.

**Example: $T(n) = T(n/2) + T(n/3) + T(n/6) + n$.** Solve $(1/2)^p + (1/3)^p + (1/6)^p = 1$. Try $p=1$: $1/2 + 1/3 + 1/6 = 1$. Then:

$$
T(n) = \Theta\left( n + n \int_1^n \frac{u}{u^{2}} \, du \right) = \Theta(n + n \log n) = \Theta(n \log n).
$$

## Domain Transformations

Some recurrences are not directly in the Akra–Bazzi or master-theorem form but can be transformed by changing variables.

**Example: $T(n) = 2T(\sqrt{n}) + \log n$.** Let $m = \log n$, so $n = 2^m$ and $\sqrt{n} = 2^{m/2}$. Then:

$$
T(2^m) = 2T(2^{m/2}) + m.
$$

Define $S(m) = T(2^m)$. Then $S(m) = 2S(m/2) + m$. By the master theorem, $S(m) = \Theta(m \log m)$. Substituting back: $T(n) = \Theta(\log n \cdot \log \log n)$.

**Example: $T(n) = \sqrt{n}\, T(\sqrt{n}) + n$.** Let $m = \log n$, $n = 2^m$, $\sqrt{n} = 2^{m/2}$. Define $S(m) = T(2^m) / 2^m$. Then:

$$
S(m) = \frac{T(2^{m/2})}{2^{m/2}} + 1 = S(m/2) + 1.
$$

Solving $S(m) = S(m/2) + 1$ gives $S(m) = \Theta(\log m)$. Therefore $T(n) = n S(\log n) = \Theta(n \log \log n)$.

## Smoothness and Regularity

For the master theorem, $f$ must be *smooth* — $f(bn) = \Theta(f(n))$ for the relevant $b$ — for the bounds to transfer across floors and ceilings.

<MathStatement id="smoothness" kind="definition" title="Smooth function">
$f : \mathbb{N} \to \mathbb{R}^{\ge 0}$ is smooth if $f(\Theta(n)) = \Theta(f(n))$ and $f$ is eventually nondecreasing. For positive nondecreasing $f$, smoothness is equivalent to $f(2n) = \Theta(f(n))$.
</MathStatement>

All common algorithm costs are smooth: $n^k$, $n^k \log^m n$, $2^n$. Functions like $2^{n \bmod k}$ or $n^{1 + \sin n}$ are not smooth and produce irregular costs.

## Generating Functions for Recurrences

Generating functions provide exact closed forms for sequences satisfying linear recurrences.

<MathStatement id="generating-function" kind="definition" title="Ordinary generating function">
For sequence $\{a_n\}$, define $A(x) = \sum_{n=0}^\infty a_n x^n$.
</MathStatement>

**Example: Catalan numbers.** $C_0 = 1$, $C_{n+1} = \sum_{i=0}^n C_i C_{n-i}$. The generating function satisfies $C(x) = 1 + x C(x)^2$, giving $C(x) = (1 - \sqrt{1 - 4x}) / (2x)$. Series expansion gives $C_n = \frac{1}{n+1}\binom{2n}{n} = \Theta(4^n / n^{3/2})$.

**Example: Divide-and-conquer recurrences.** For $T(n) = 2T(n/2) + n$, the generating function approach is less natural because the recurrence operates on $n/b$ rather than $n-1$. The Akra–Bazzi framework handles this more directly.

## Recurrence Application: Matrix Multiplication

Standard: $T(n) = 8T(n/2) + \Theta(n^2) = \Theta(n^3)$. Strassen: $T(n) = 7T(n/2) + \Theta(n^2) = \Theta(n^{\log_2 7})$. The improvement comes from reducing $a$ from 8 to 7. The exponent $\log_b a$ drops from 3 to about 2.807. Further reductions (Coppersmith–Winograd, $O(n^{2.376})$, and subsequent improvements) use tensor algebra rather than divide-and-conquer alone.

## Recurrence Application: Selection

Quickselect's expected-time recurrence $T(n) \le T(3n/4) + \Theta(n)$ gives $T(n) = \Theta(n)$ by the master theorem (Case 3: $a=1$, $b=4/3$, $\log_b a = 0$, $f(n) = n = \Omega(n^{0+\epsilon})$; regularity: $f(3n/4) = 3n/4 \le (3/4) n$ with $c = 3/4$).

Median-of-medians: $T(n) \le T(n/5) + T(7n/10) + \Theta(n)$. By Akra–Bazzi, $p \approx 0.839$ and the integral gives $T(n) = \Theta(n)$.

   <ExerciseSolution>
   Let $A(x) = \sum_{n=0}^\infty a_n x^n$. The recurrence: $\sum_{n\ge 1} a_n x^n = 2 \sum_{n\ge 1} a_{n-1} x^n + \sum_{n\ge 1} x^n$. So $A(x) - a_0 = 2x A(x) + x/(1-x)$. Since $a_0 = 0$, $A(x)(1 - 2x) = x/(1-x)$, so $A(x) = x / ((1-x)(1-2x)) = 1/(1-2x) - 1/(1-x)$. Series: $a_n = 2^n - 1$.
   </ExerciseSolution>

8. **Synthesis: choose the method.** The runtime of a recursive algorithm is described by: given an array of size $n$, partition it into subarrays of sizes $\lfloor n/4\rfloor$ and $\lfloor 3n/4\rfloor$, recurse on each, and combine in $\Theta(n)$ time. Derive the recurrence, decide which method solves it, and give the asymptotic bound.

   <ExerciseSolution>
   $T(n) = T(n/4) + T(3n/4) + \Theta(n)$. Akra–Bazzi: $a_1=1$, $b_1=1/4$, $a_2=1$, $b_2=3/4$. Solve $(1/4)^p + (3/4)^p = 1$. By inspection $p=1$ because $1/4 + 3/4 = 1$. Then $T(n) = \Theta(n + n \int_1^n u / u^2 \, du) = \Theta(n \log n)$.
   </ExerciseSolution>

9. **Synthesis: method comparison.** Two students try to bound $T(n) = 3T(n/3) + \sqrt{n}$.
   - Student A applies the Master Theorem: $a=3$, $b=3$, $\log_b a = 1$, $f(n) = \sqrt{n} = n^{0.5}$. Case 1: $f(n) = O(n^{1-\varepsilon})$ with $\varepsilon = 0.5$, so $T(n) = \Theta(n)$.
   - Student B builds a recursion tree: at depth $j$, $3^j$ nodes each of size $n/3^j$, cost $\sqrt{n/3^j}$ per node.

   Who is correct? If either is wrong, identify the error and give the correct bound.

   <ExerciseSolution>
   Student A is correct. $\sqrt{n} = n^{0.5}$, $\log_3 3 = 1$, so $0.5 < 1$. Case 1 applies with $\varepsilon = 0.5$: $T(n) = \Theta(n)$. Student B's confusion arises from incorrectly summing the geometric series — the level costs form a geometric progression with ratio $r = \sqrt{3} \approx 1.732$, and the geometric sum $\sum_{j=0}^{\log_3 n} (\sqrt{3})^j \sqrt{n} = \Theta(n)$ after simplification.
   </ExerciseSolution>

10. **Transfer: adversarial regularity.** Consider $T(n) = 2T(n/2) + n^{1.5}$. Apply the Master Theorem. Now consider $T'(n) = 2T(n/2) + f(n)$ where $f(n) = n^{1.5}$ when $n$ is a power of 2, and $f(n) = 0$ otherwise. Does the Master Theorem apply to $T'$? Why or why not? What is the asymptotic behavior of $T'$?

    <ExerciseSolution>
    For $T$, Case 3 applies: $\log_2 2 = 1$, $f(n) = n^{1.5} = \Omega(n^{1+\varepsilon})$ with $\varepsilon = 0.5$, regularity $2 (n/2)^{1.5} = 2^{-0.5} n^{1.5} \le c n^{1.5}$ with $c = 2^{-0.5} < 1$, so $T(n) = \Theta(n^{1.5})$.

    For $T'$, $f(n)$ is not smooth. When $n$ is a power of 2, the root cost is $\Theta(n^{1.5})$, and the level costs form a geometric series with ratio $2^{-0.5}$. The total remains $\Theta(n^{1.5})$, but the Master Theorem's proof via the regularity condition requires the inequality for *all sufficiently large n*, which fails here. An ad-hoc recursion tree argument is required.
    </ExerciseSolution>

## Deliberately Omitted

> **Generating functions for divide-and-conquer recurrences.** Applying generating functions to recurrences of the form $T(n) = aT(n/b) + f(n)$ requires advanced analytic combinatorics; the Akra–Bazzi framework covers these cases more directly.
>
> **Adversarial lower bounds for specific recurrences.** Proving that a given recurrence cannot be solved in a certain form (e.g., no closed form exists) is a topic in differential algebra, beyond the scope of algorithm analysis.
>
> **Recurrences with non-constant coefficients and irregular branching.** When the branching factor $a$ or subproblem ratio $b$ depends on $n$ (e.g., $T(n) = \log n \cdot T(n/2) + n$), the standard methods fail and ad-hoc approaches like domain transformation are illustrated only for textbook examples.
