# Shortest Paths

In 1956, Edsger Dijkstra sat down at a cafe in Amsterdam to think about a practical problem: given a road map, what is the shortest route from Rotterdam to Groningen? Within twenty minutes, working without pencil or paper, he designed the algorithm we now know as Dijkstra's algorithm (Dijkstra 1959).

Breadth-first search finds the fewest *edges* between two vertices — but on a road map, a route with fewer road segments may be much longer if those segments are slow. The actual goal is the *minimum-weight* path, where weight represents distance, time, or cost.

**Prerequisites.** Familiarity with directed graphs, adjacency lists, and big-O analysis. The BFS algorithm for unweighted graphs provides a useful comparison point.

**Problem (single-source shortest paths).** Given a directed graph $G = (V,E)$ with edge-weight function $w: E \to \mathbb{R}$ and a source vertex $s$, find $\delta(s,v)$, the minimum weight of any path from $s$ to $v$, for every reachable $v \in V$. If a negative-weight cycle is reachable from $s$, report its existence.

The choice of algorithm depends on the weight structure: Dijkstra's algorithm exploits nonnegative weights to settle vertices greedily in $O((V+E)\log V)$; Bellman–Ford handles arbitrary weights with $O(VE)$ systematic passes and detects negative cycles; on directed acyclic graphs, a single topological pass suffices in $O(V+E)$. This lesson develops each algorithm with complete proofs of correctness.

## Relaxation: The Universal Primitive

<MathStatement id="relaxation" kind="definition" title="Edge relaxation">
For edge $(u,v)$ with weight $w(u,v)$, relaxation is:
$$
\text{if } d[v] > d[u] + w(u,v) \text{ then } d[v] \leftarrow d[u] + w(u,v), \text{ pred}[v] \leftarrow u.
$$
</MathStatement>

<MathStatement id="relaxation-lemma" kind="lemma" title="Relaxation lemma">
1. **Upper bound invariant:** $d[v] \ge \delta(s,v)$ for all $v$, where $\delta$ is the true shortest-path distance (or $-\infty$ for vertices reachable from a negative cycle).
2. **Path property:** If $d[v]$ is finite, it is the length of some $s$-$v$ path.
3. **Convergence:** If $s \rightsquigarrow u \to v$ is a shortest path and $d[u] = \delta(s,u)$ before relaxation, then $d[v] = \delta(s,v)$ after relaxation.
4. **Monotonicity:** $d[v]$ never increases.
</MathStatement>

<MathProof id="relaxation-lemma-proof" for="relaxation-lemma">
(1) Initial values are $0 = \delta(s,s)$ and $\infty \ge \delta$. Each relaxation sets $d[v]$ to a concrete path length, which is at least the shortest distance. (2) By induction: $d[s] = 0$ is the empty path; each update prepends an edge to the path achieving $d[u]$. (3) After $d[u] = \delta(s,u)$, the inequality $\delta(s,v) \le \delta(s,u) + w(u,v) = d[u] + w(u,v)$ holds by the triangle inequality for shortest paths (Cormen et al. 2022, Lemma 24.10). If $d[v] > d[u] + w(u,v)$, relaxation corrects it. (4) $d$ is only decreased. ∎
</MathProof>

The relaxation lemma is the foundation: every shortest-path algorithm is a strategy for choosing *which edges to relax in which order*. The convergence property says that once a vertex's distance is correct, relaxing its outgoing edges propagates correctness to its neighbors.

## Dijkstra's Algorithm

<MathStatement id="dijkstra" kind="theorem" title="Dijkstra's algorithm correctness" proofId="dijkstra-proof">
Let $G = (V,E)$ have nonnegative edge weights. Dijkstra's algorithm maintains a set $S$ of settled vertices whose distance labels are final. It extracts $\min_{v \notin S} d[v]$, settles it, and relaxes its outgoing edges. Upon termination, $d[v] = \delta(s,v)$ for all $v$.
</MathStatement>

<MathProof id="dijkstra-proof" for="dijkstra" collapsible open>
By induction on $|S|$. Base: $S = \{s\}$, $d[s] = 0 = \delta(s,s)$. Inductive step: assume $d[u] = \delta(s,u)$ for all $u \in S$. Let $v$ be the vertex with minimum $d[v]$ among $V \setminus S$. Consider any $s$-$v$ path $P$. Let $x$ be the first vertex on $P$ outside $S$ (possibly $x = v$), and let $y$ be its predecessor in $S$. Since $w(y,x) \ge 0$ and $d[y] = \delta(s,y)$ by induction:

$$
\text{length}(P) \ge d[y] + w(y,x) \ge d[x] \ge d[v].
$$

The first inequality follows from $P$ having prefix $s \rightsquigarrow y \to x$; the second because $d[x]$ is an upper bound on the shortest $s$-$x$ path; the third because $d[v]$ is the minimum among unsettled vertices. Thus $d[v] \le \text{length}(P)$ for every $s$-$v$ path $P$, so $d[v] = \delta(s,v)$. Settle $v$ and relax its outgoing edges; the invariant is preserved for $S \cup \{v\}$. ∎
</MathProof>

**Complexity:** $O((V+E)\log V)$ with binary heap, $O(V^2 + E)$ with array.

    dist[start] = 0

### Failure Mode: Negative Edges

Dijkstra's algorithm fails on graphs with negative edge weights because it settles vertices prematurely. Once a vertex is extracted from the priority queue, its distance is never reconsidered — but a negative edge discovered later could shorten it.

**Example.** Graph $s \to a$ weight 2, $s \to b$ weight 5, $b \to a$ weight $-10$. Dijkstra initializes $d[s]=0$, $d[a]=2$, $d[b]=5$, settles $a$ (smallest estimate), then $b$. The algorithm finalizes $d[a]=2$, but the true shortest path to $a$ is $s \to b \to a$ with total weight $-5$. The negative edge $b \to a$ created a cheaper route to $a$ after $a$ was already settled.

This is not a bug in the implementation — it is a fundamental limitation: Dijkstra's correctness proof relies on the nonnegativity assumption. When negative weights are possible, use Bellman–Ford instead.

## Bellman–Ford Algorithm

<MathStatement id="bellman-ford" kind="theorem" title="Bellman–Ford correctness" proofId="bellman-ford-proof">
Let $G = (V,E)$ have arbitrary edge weights with no negative cycles reachable from $s$. After $|V|-1$ passes of relaxing every edge, $d[v] = \delta(s,v)$ for all $v$.
</MathStatement>

<MathProof id="bellman-ford-proof" for="bellman-ford" collapsible open>
By induction on path length. Let $\delta^{(k)}(s,v)$ be the minimum weight of an $s$-$v$ path using at most $k$ edges. Claim: after $k$ passes, $d[v] \le \delta^{(k)}(s,v)$.

Base $k=0$: $\delta^{(0)}(s,s)=0$, $d[s]=0$; for $v \ne s$, $\delta^{(0)}(s,v)=\infty$, $d[v]=\infty$.

Inductive step: assume after $k$ passes, $d[v] \le \delta^{(k)}(s,v)$. Consider a path achieving $\delta^{(k+1)}(s,v)$. If it has at most $k$ edges, the inductive bound applies. Otherwise it ends with edge $(u,v)$ and its prefix $s \rightsquigarrow u$ has at most $k$ edges and weight $\delta^{(k)}(s,u)$. After pass $k+1$, edge $(u,v)$ is relaxed:

$$
d[v] \le d[u] + w(u,v) \le \delta^{(k)}(s,u) + w(u,v) = \delta^{(k+1)}(s,v).
$$

Thus after $|V|-1$ passes, $d[v] \le \delta^{(|V|-1)}(s,v) = \delta(s,v)$ (since any shortest path is simple and has at most $|V|-1$ edges). Combined with the upper bound invariant ($d[v] \ge \delta(s,v)$), equality holds. This argument follows Bellman (1958) and the exposition in Section 24.1 of Cormen et al. (2022). ∎
</MathProof>

**Negative cycle detection.** After $|V|-1$ passes, if any edge can still be relaxed, a reachable negative cycle exists. Reason: a path with $\ge V$ edges must repeat a vertex, forming a cycle that reduces distance arbitrarily.

<CodeRunner client:load
  title="Bellman–Ford with cycle detection"
  language="python"
  code={`def bellman_ford(edges, V, start):
    dist = [float('inf')] * V
    dist[start] = 0
    for _ in range(V - 1):
        for u, v, w in edges:
            if dist[u] != float('inf') and dist[v] > dist[u] + w:
                dist[v] = dist[u] + w
    for u, v, w in edges:
        if dist[u] != float('inf') and dist[v] > dist[u] + w:
            return None  # negative cycle
    return dist

edges = [(0,1,4),(0,2,1),(2,1,2),(1,3,1),(2,3,5)]
print(bellman_ford(edges, 4, 0))`}
  output="[0, 3, 1, 4]"
/>

### Edge Relaxation Order

Any order works for correctness, but the FIFO (queue-based) variant (SPFA) is faster in practice. Worst-case $O(VE)$ still holds; SPFA can be exponential for adversarial inputs (see Karp 1978 for cycle detection complexity).

## DAG Shortest Paths

When the graph is a directed acyclic graph, the problem simplifies dramatically.

**Algorithm.** Topologically sort vertices; process in order, relaxing outgoing edges.

**Correctness.** In a DAG, every $s$-$v$ path's vertices appear in topological order. When $v$ is processed, all its predecessors have been processed and their distances are final. One pass suffices.

**Complexity.** $O(V+E)$, suitable for arbitrary weights (including negative) when the graph is acyclic — no priority queue needed.

<CodeExercise client:load
  title="Complete DAG shortest paths"
  code={`def dag_shortest_paths(graph, V, start):
    # topological order is given
    topo_order = [0, 1, 2, 3]
    dist = [float('inf')] * V
    dist[start] = 0
    for u in topo_order:
        for v, w in graph[u]:
            if dist[v] > dist[u] + w:
                dist[v] = dist[u] + w  # relax
    return dist

# Complete the graph: edges (u, v, w)
# 0 -> 1 weight 3
# 0 -> 2 weight 6
# 1 -> 3 weight 2
# 2 -> 3 weight 1
graph = [[(1, 3), (2, 6)], [(3, 2)], [(3, 1)], []]
print(dag_shortest_paths(graph, 4, 0))`}

## Johnson's Algorithm for All-Pairs

<MathStatement id="johnson" kind="theorem" title="Johnson's algorithm" proofId="johnson-proof">
Given $G = (V,E)$ with arbitrary edge weights and no negative cycles, Johnson's algorithm computes all-pairs shortest paths in $O(VE + V^2 \log V)$ time.
</MathStatement>

<MathProof id="johnson-proof" for="johnson">
Add a super-source $s'$ with zero-weight edges to all vertices. Run Bellman–Ford from $s'$ to compute potentials $h(v) = \delta(s', v)$. These satisfy $h(v) \le h(u) + w(u,v)$ for all $(u,v)$ — this is the optimality condition for shortest paths, also known as the dual feasibility condition in linear programming (Ahuja, Magnanti, and Orlin 1993). Reweight: $w'(u,v) = w(u,v) + h(u) - h(v) \ge 0$.

For any $s$-$t$ path, the reweighted sum telescopes:

$$
\sum w'(u,v) = \sum (w(u,v) + h(u) - h(v)) = \left(\sum w(u,v)\right) + h(s) - h(t).
$$

Thus shortest paths under $w$ are shortest under $w'$, and vice versa. Since $w' \ge 0$, run Dijkstra from each vertex. The original distances are recovered as $\delta(s,t) = \delta'(s,t) - h(s) + h(t)$. ∎
</MathProof>

## Floyd–Warshall Algorithm

<MathStatement id="floyd-warshall" kind="theorem" title="Floyd–Warshall DP recurrence">
Let $d^{(k)}[i][j]$ be the shortest path from $i$ to $j$ using only intermediate vertices from $\{1,\ldots,k\}$. Then:
$$
d^{(k)}[i][j] = \min\left( d^{(k-1)}[i][j], \; d^{(k-1)}[i][k] + d^{(k-1)}[k][j] \right).
$$
</MathStatement>

**Proof.** A shortest path using vertices $\{1,\ldots,k\}$ either avoids $k$ entirely (first case) or visits $k$ exactly once (second case, since subpaths of shortest paths are shortest). The min of these two covers all possibilities. Base $d^{(0)}[i][j] = 0$ if $i=j$, $w(i,j)$ if $(i,j) \in E$, $\infty$ otherwise. After $n$ iterations, $d^{(n)}[i][j] = \delta(i,j)$. This DP was published by Floyd (1962) as Algorithm 97.

**Complexity:** $O(V^3)$ time, $O(V^2)$ space.

**Transitive closure:** Boolean variant uses logical OR and AND instead of min and addition.

## Algorithm Selection Guide

The choice among shortest-path algorithms depends on three properties: graph structure, weight signs, and whether single-source or all-pairs distances are needed.

## Properties and Extensions

**Shortest-path tree.** Predecessor pointers form a tree rooted at $s$ where the unique path to each vertex is a shortest path. For multiple shortest paths, any tie-breaking rule yields a valid tree.

**Potential theory for shortest paths.** A function $h: V \to \mathbb{R}$ is a *feasible potential* if $h(v) \le h(u) + w(u,v)$ for all edges. This is the dual solution in the linear programming formulation of shortest paths (Papadimitriou and Steiglitz 1982). Feasibility is equivalent to the absence of negative cycles.

**Edge relaxation order for Bellman–Ford.** Any order works, but the FIFO (queue-based) variant (SPFA) is faster in practice. Worst-case $O(VE)$ still holds; SPFA can be exponential for adversarial inputs.

## Counterexample: Floyd–Warshall with Negative Cycles

Consider a three-vertex graph with edges $1 \to 2$ weight 1, $2 \to 3$ weight 1, $3 \to 1$ weight $-3$. Floyd–Warshall computes finite entries for all pairs because the recurrence does not check for negative cycles during intermediate iterations — it only produces the correct result if no negative cycles exist. The entries are not meaningful because repeated traversal of the cycle can make paths arbitrarily short. A detection pass (checking $d[i][i] < 0$ after the main loop) is standard practice.

## Deliberately Omitted

- **SPFA and exponential worst-case behavior.** Queue-based Bellman–Ford variants (Shortest Path Faster Algorithm) are briefly noted above; their adversarial analysis is an implementation detail that does not change the $O(VE)$ bound and is better studied alongside algorithm engineering techniques.
- **Bidirectional Dijkstra and A\* search.** Goal-directed search with admissible heuristics belongs in an AI search lesson rather than in a treatment of provably optimal graph algorithms; the algorithmic core is the same relaxation primitive, but the correctness argument shifts to the heuristic consistency condition.
- **Min-plus matrix squaring for all-pairs.** The $O(V^3 \log V)$ repeated-squaring approach is noted in the matrix multiplication section below but its analysis — especially the base-case dependence and the log-factor improvement over Floyd–Warshall — is deferred to an advanced topics module on algebraic algorithms.
- **Dynamic shortest paths.** Updating distances after edge weight changes (incremental, decremental, and fully dynamic) is a separate research area with different data structures and analysis techniques.
- **Parallel shortest-path algorithms.** \(\Delta\)-stepping, work-efficient parallel BFS, and GPU-based relaxation are beyond the scope of this single-source lesson.

## Matrix Multiplication for Shortest Paths

Define the distance product (min-plus or "tropical") matrix multiplication $(A \otimes B)[i][j] = \min_{k} (A[i][k] + B[k][j])$. Then $d^{(k)}$ (paths with at most $k$ edges) satisfies $d^{(k)} = d^{(k-1)} \otimes W$ where $W$ is the weight matrix. Repeated squaring gives $d^{(2^p)}$ in $O(V^3 \log n)$ time, computing all-pairs shortest paths by repeated squaring.
