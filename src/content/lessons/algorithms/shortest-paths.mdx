# Shortest Paths

Given a weighted graph and source s, the single-source shortest-path problem asks for the minimum path distance from s to every reachable vertex and often a predecessor tree that reconstructs paths. The algorithm depends critically on edge-weight assumptions. Dijkstra requires nonnegative weights; Bellman-Ford tolerates negative edges and detects reachable negative cycles; DAG relaxation works with arbitrary weights when the graph is acyclic. A path-length answer without these assumptions is not a complete algorithm.

## Relaxation

For edge (u,v) with weight w, relaxation tests whether (d[v] greater than d[u]+w). If so, set d[v]=d[u]+w and predecessor[v]=u. Initialize d[s]=0 and all others infinity. The fundamental invariant is that every finite label is the length of some s-to-v path, while labels may be upper bounds on the true optimum. Relaxation preserves the upper-bound property because replacing a label with a concrete path length remains a valid path.

If no edge can relax after a complete pass over all edges, the labels satisfy (d[v] at most  d[u]+w(u,v)) for every edge. Any path’s edge inequalities telescope to (d[t] at most ) length(path), while concrete-path upper bounds give (d[t] at least delta(s,t)) in the appropriate direction; together labels equal shortest distances when no reachable negative cycle invalidates the minimum.

## Dijkstra’s algorithm

Dijkstra maintains a settled set S. It repeatedly extracts the unsettled vertex u with smallest tentative d[u], settles it, and relaxes outgoing edges. With nonnegative weights, the smallest tentative label is final: any alternative path entering u through an unsettled vertex has prefix distance at least d[u], and adding a nonnegative final edge cannot improve it.

**Worked example.** Edges from A: A→B=4, A→C=1; C→B=2, C→D=5; B→D=1. Start d(A)=0, B=4,C=1,D=∞. Settle C and improve B=3,D=6. Settle B and improve D=4. Settle D. Predecessors yield A→C→B→D of cost 4. A negative edge C→B could invalidate the settle proof; a negative edge from a later vertex might improve B after B was settled.

With a binary heap, each edge relaxation can decrease a key and each vertex is extracted, giving O((V+E)log V) time and O(V+E) space. A simple array priority queue costs O(V^2+E), which can be preferable for dense graphs. Lazy heap insertion avoids decrease-key by pushing a new pair; stale pairs are skipped when their key no longer equals d[u]. The skip rule is part of correctness and complexity.

## Bellman-Ford

Bellman-Ford scans every edge repeatedly for V-1 passes. Any simple shortest path has at most V-1 edges, so after k passes all shortest paths using at most k edges have been correctly relaxed by induction. A further pass that relaxes an edge reachable from s proves a reachable negative cycle: paths can keep decreasing by traversing that cycle.

The algorithm handles negative edges but not a finite shortest distance to vertices reachable from a negative cycle; the infimum is minus infinity. Mark vertices affected by a detected cycle and propagate reachability if the application needs to report undefined distances. Early termination is safe when a full pass makes no changes.

## DAG shortest paths

Topologically order vertices and relax outgoing edges once in order. Every path to v ends at a predecessor processed earlier, so when v is processed, all possible predecessor distances are final. Negative edges are safe because cycles are absent. The time is O(V+E), often better than general algorithms.

Topological ordering itself can be obtained by DFS finish times or indegree-based Kahn processing. If not all vertices are ordered, the graph has a cycle and the DAG algorithm’s precondition fails. A graph may have negative edges and still be solvable efficiently if it is acyclic.

## Path reconstruction

A predecessor array stores one edge on a shortest path. Follow predecessors backward from target to source, then reverse. If predecessor pointers form a cycle, the implementation accepted a bad update or is reconstructing a vertex affected by a negative cycle. Unreachable targets have no predecessor path and should return an explicit unreachable result.

Tie-breaking affects reproducibility but not distance. Choose smaller predecessor ID, earlier edge, or first relaxation, and ensure updates on equal distance do not create cycles. For all-pairs or many-target queries, predecessor storage can be O(V) per source; output requirements determine whether it is worth storing.

## Potentials and reweighting

Johnson’s algorithm handles sparse all-pairs shortest paths when there are no negative cycles. Bellman-Ford computes a potential h(v) such that reweighted edge (w'(u,v)=w(u,v)+h(u)-h(v)) is nonnegative. Every path from s to t gains the same offset h(s)-h(t), so shortest paths are preserved while Dijkstra becomes applicable from each source.

The potential is valid only if h(v) at most h(u)+w(u,v) for every edge, precisely the Bellman-Ford feasible-label condition. If a negative cycle exists, no such finite potential can satisfy all inequalities. Reweighting changes numerical distances but not path ordering between a fixed pair.

## Special cases and failure modes

Unweighted shortest paths use BFS: every edge has unit weight, and vertices discovered at distance k are processed before distance k+1. Zero-weight edges are safe for Dijkstra but can create many equal-distance ties. Negative cycles require an explicit application policy; shortest paths may be undefined rather than merely large negative.

Integer nonnegative weights can support bucket queues or radix heaps, changing practical and asymptotic costs under word assumptions. Dense adjacency matrices may make scanning all outgoing edges appropriate. Parallel edges are harmless if relaxation considers each; self-loops matter only when negative or when path definitions permit them.

## Proof boundaries and implementation details

The infinity sentinel must not overflow when adding an edge weight. Guard relaxation with d[u] finite, or use arbitrary-precision/checked arithmetic. If weights can be negative, a fixed-width distance can wrap and falsely appear smaller. Define behavior for unreachable vertices and for distances beyond representable range; numerical correctness is part of algorithm correctness.

Priority queues may contain stale entries under lazy decrease-key. When extracting (key,u), skip it if key differs from the current d[u]. If the implementation settles a vertex on its first extraction without checking staleness, a stale larger key can waste work but a stale smaller key cannot occur if keys are only decreased; state and prove the chosen invariant. A visited flag is safe for Dijkstra only when the nonnegative-weight finality theorem holds.

## Output-sensitive path systems

If only one target is needed, terminate Dijkstra when that target is settled. If many targets share a source, complete the reachable region or stop when all targets settle. If only distances are needed, predecessor storage can be omitted; if paths are required, store enough predecessor information and define ties. For a sequence of changing edge weights, recomputing from scratch may be necessary unless the application uses a dynamic shortest-path structure with a different update contract.

## Testing shortest-path implementations

Compare against Floyd-Warshall or exhaustive simple-path enumeration on tiny graphs, including disconnected graphs, zero edges, parallel edges, negative edges without cycles, and reachable negative cycles. Verify every returned path consists of valid edges, its weight equals the reported distance, and no edge relaxation inequality is violated when the graph has no negative cycle. Test graph representation variants separately; a correct algorithm over a broken adjacency list still yields wrong output.

## Choosing the algorithm

Use BFS for unit weights, DAG relaxation for acyclic graphs, Dijkstra for nonnegative weights, Bellman-Ford when negative edges or cycle detection matter, and Johnson for sparse all-pairs problems without negative cycles. Dense all-pairs instances may favor matrix-based methods. The decision follows graph structure, weight assumptions, output requirement, and density—not the algorithm’s popularity.
