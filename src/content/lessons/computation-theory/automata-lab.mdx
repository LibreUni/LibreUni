# Automata Lab

Choose a finite-state validation rule, construct a DFA, minimize it conceptually, and test positive and negative strings. Record an invariant describing what each state remembers about the prefix.

## Purpose and workflow

This lab treats automata as mathematical objects that can be constructed, simulated, minimized, and audited. The deliverable is not merely a diagram. It is a language specification, a machine definition, a correctness argument, a test suite, and a complexity discussion. Use a fixed alphabet and state naming convention throughout; ambiguous diagrams are difficult to verify and impossible to grade reliably.

Begin with examples. List accepted and rejected words, including the empty word and words one edit away from the boundary. Identify the finite information a machine must retain after reading a prefix. That information becomes the state meaning. For a suffix condition, store the relevant suffix; for parity, store a residue; for a forbidden pattern, store the longest suffix that is also a prefix of the forbidden pattern. Do not start by drawing arbitrary arrows.

## DFA construction task

Construct a DFA for binary strings that contain 101 as a substring. Use states q0, q1, q2, q3 with meanings: no useful suffix, the suffix 1, the suffix 10, and the pattern has occurred. On reading a symbol, transition to the longest suffix of the current history that is a prefix of 101. Thus q0 on 1 goes to q1; q1 on 0 goes to q2 and on 1 stays q1; q2 on 1 goes to q3 and on 0 returns q0; q3 loops on both symbols. Accept q3.

The proof invariant is that the state records the longest suffix of the consumed prefix that is a prefix of 101, unless the pattern has already occurred. Prove the base case for the empty prefix and handle each state-symbol pair. The construction is correct because q3 is reached exactly when a prefix ending at the current position contains 101.

## NFA and subset conversion

Build an NFA for binary strings ending in 01 by looping on either symbol in the start state, nondeterministically guessing that the current position begins the final 01, then consuming 0 and 1 to an accepting state. The NFA accepts if some guess is correct. The subset DFA records all possible NFA states after each prefix. Compute epsilon closure even when the particular machine has no epsilon transitions; that habit prevents errors on Thompson constructions.

For each subset, list transitions on every alphabet symbol and mark it accepting if it contains an accepting NFA state. Remove unreachable subsets before counting. Then compare the DFA’s behavior against an independently written predicate on all words up to a small length. Exhaustive testing is a debugging aid, not a proof; the invariant supplies the proof.

## Minimization experiment

Partition DFA states into accepting and nonaccepting blocks. Repeatedly split a block when two states have transitions into different current blocks on some symbol. When no split remains, choose one representative per block and define transitions between blocks. Prove that states in the same final block have identical future behavior and that states in different blocks have a distinguishing suffix.

Record the original state count, reachable state count, and minimized state count. A large reduction may indicate redundant construction paths; no reduction does not mean the DFA is inefficient in every representation. Compare the minimized DFA with a Myhill-Nerode lower bound when possible.

## PDA trace lab

Represent a configuration as (state, unread input, stack). Fix whether the leftmost or rightmost displayed stack symbol is the top. Trace a balanced-parentheses PDA on (), (()), (()()), and ))(. At each step state the invariant: the stack contains exactly the unmatched opening parentheses in the consumed prefix. The first close with an empty stack rejects; acceptance requires no unread input and an empty stack or bottom marker, according to the chosen convention.

For a^n b^n, trace aabb, aaab, and abab. The phase state prevents a return to pushing after popping begins. This example demonstrates that a stack is not just extra memory; the transition policy determines which memory relationship is recognized.

## Testing and reporting

Include property tests: every accepted word satisfies the specification; every generated word satisfying a bounded specification is accepted; determinism means exactly one outgoing transition for every state-symbol pair; and no transition references an undeclared state. For NFAs, compare acceptance with subset simulation. For regexes, compare a regex engine only after confirming that its syntax and anchors match the formal semantics.

Report the machine as a transition table or machine-readable artifact, not only a picture. State the number of states and transitions, whether epsilon moves exist, acceptance convention, and the asymptotic simulation cost. A DFA simulates a word in linear time; an NFA may require set propagation, and a PDA’s nondeterministic reachability can be substantially more expensive.

## Exercises

1. Construct and prove a DFA for strings with an even number of 1s and ending in 0.
2. Convert an NFA for strings containing 00 into a DFA and remove unreachable subsets.
3. Minimize a four-state DFA of your choice and provide distinguishing suffixes for separated states.
4. Trace a PDA for aabb and identify the configuration invariant.
5. Design five boundary tests for a machine recognizing a fixed substring.

### Diagnostic solutions


1. Track the pair (parity of 1s, last-symbol class), with a start marker for the empty prefix. Update parity on 1 and last-symbol class on every symbol; accept when parity is even and last symbol is 0.
2. The NFA state set after each prefix records whether a suffix has begun matching 00. The DFA uses subsets, marks subsets containing the accepting state, and removes any subset never reached from the start.
3. A valid minimization proof lists the initial partition, every split witness symbol, and a suffix distinguishing representatives in different final blocks.
4. Push A for each a, switch on the first b, pop A for each b, and accept when input ends at the bottom marker. The stack holds exactly the unmatched a obligations.
5. Test empty input, a word shorter than the pattern, the pattern itself, an overlap such as 10101, and a word with the pattern only at the final position.



## Extension: language equivalence

Given two DFAs, construct their product and search from the start pair. A pair is bad when exactly one component is accepting. If a bad pair is reachable, the path labels a distinguishing word; if no bad pair is reachable, the machines agree on every word. This turns an informal test of many strings into a finite proof. Record the predecessor of each visited pair so the lab can output a concrete counterexample.

For an NFA, first determinize or propagate reachable state sets. For a PDA, equivalence is substantially harder because the stack is unbounded; do not assume the DFA product method transfers. The model boundary is itself a learning objective: finite-state equivalence reduces to finite reachability, while pushdown behavior requires grammar or pushdown-specific methods.

**Exercise.** Include a string that differs from a valid one at its final symbol and explain the rejecting state.

## Final audit

The implementation should reject an incomplete transition table rather than silently inventing behavior. For each state, check one outgoing transition per alphabet symbol in a DFA, and for each accepting state check that acceptance is defined only after the complete input has been consumed. A final report should include the machine definition, invariant, traces, and a counterexample-producing equivalence search.

The final submission should also state whether the machine is deterministic, complete, and minimal. These are different properties: a complete DFA may contain redundant states, while a minimized machine is not automatically complete unless its sink behavior is retained. Record these choices explicitly.

## References & Further Reading

## Advanced synthesis: from specification to proof

An automaton lab is strongest when the specification, construction, and test oracle are written independently. First define a predicate on complete words. Then define the state meaning as information about prefixes, rather than as a drawing label. Finally implement a direct predicate for small words and compare it with the machine. Agreement on all words through a chosen length is not the proof; it is evidence that the invariant and implementation are aligned.

For a substring detector, the state should be the longest suffix of the consumed prefix that is also a prefix of the target pattern. This is enough because any future occurrence must extend one of those suffixes. For a modular property, the state is the current residue. For a property involving a fixed finite window, store the relevant suffix, but explain why older symbols can never affect a future decision. This “future-equivalence” explanation is the conceptual bridge to Myhill–Nerode.

When converting an NFA, distinguish a set of possible states from a single guessed path. A subset is accepting if at least one member is accepting, because NFA acceptance is existential. When complementing, determinize and complete first; otherwise swapping marked states changes the semantics of missing runs. When minimizing, give a distinguishing suffix for states placed in different blocks. The suffix is a certificate that no quotient may merge them.

The lab report should include an edge-case table: empty input, shortest accepted input, shortest rejected input, repeated pattern overlap, an input ending exactly at a boundary, and an input that violates the rule only at its final symbol. For each, record the state trace. This catches incomplete transition functions and off-by-one acceptance conditions before the formal proof is read.
