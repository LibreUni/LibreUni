# Complexity Classes and Resource Bounds

Computability asks whether a problem has any terminating algorithm. Complexity asks how much resource a chosen computational model needs as input size grows. A language may be decidable but infeasible for realistic inputs. Complexity theory makes “efficient” precise enough to compare unrelated problems and to prove that a construction preserves resource bounds.

## Measuring computation

Fix an encoding and a machine model. The time of a deterministic Turing machine on input x is the number of transitions before halting; its space is the number of distinct tape cells visited, or a stated work-tape convention. The worst-case time function (T(n)) is the maximum over inputs of length n. Bounds should account for input length, not numerical value: an integer written in binary has length logarithmic in its value.

Asymptotic notation suppresses constants and lower-order terms. (f=O(g)) means f is eventually at most a constant multiple of g; (f=Omega(g)) is the reverse; (f=Theta(g)) means both. A polynomial-time algorithm has time (n^k) for some constant k, while an exponential algorithm has roughly (c^n) or (2^()p(n)()) behavior. Polynomial differences can matter in practice, but polynomial time is robust under reasonable model changes and composition.

Model robustness matters. A multi-tape TM can be simulated by a single-tape TM with polynomial overhead, so the class P does not depend on that choice. A random-access word machine, RAM model, and standard programming language with bounded integer conventions can similarly simulate one another under standard encodings with polynomial changes. State the model when making a lower-bound claim; an unbounded arithmetic operation would hide cost.

## The class P

P is the set of languages decidable by a deterministic Turing machine in polynomial time. It is closed under complement, union, and intersection. For union, run both deciders and accept if either accepts; for intersection, require both. The combined time remains polynomial because a finite sum of polynomials is polynomial. Sequential composition of polynomial-time subroutines is polynomial when output lengths remain polynomially bounded.

Polynomial time is a tractability convention, not a guarantee of fast execution. An (n^()100()) algorithm is in P but likely useless; a (1.01^n) algorithm is not known to be in P but may solve moderate instances. The class is valuable because it abstracts away constant engineering factors while capturing a broad notion of scalable computation.

## NP and certificates

NP is the class of languages for which yes-instances have polynomial-length certificates verifiable in polynomial time. Formally, L is in NP if there is a polynomial-time verifier V and polynomial q such that (xin L) iff there exists y with |y|le q(|x|) and V(x,y)=1. The verifier must reject all certificates for no-instances; merely checking a candidate solution is not enough unless the certificate cannot exist for no cases.

Equivalently, NP is the class decidable by a nondeterministic TM in polynomial time: the nondeterministic branch guesses y, then verifies it. A deterministic simulator can enumerate all certificates, but the number may be exponential. This explains why NP is a resource class, not a claim that problems are impossible or that every NP problem is difficult in practice.

Examples include PATH, where a vertex sequence certifies reachability, and SAT, where an assignment certifies a satisfying formula. A verifier for SAT evaluates every clause in polynomial time. For a no-instance, no assignment passes. The certificate is a proof of existence, not a compressed proof that every possible assignment fails.

## P versus NP

P is contained in NP: a polynomial decider can be used as a verifier that ignores its certificate. Whether P=NP is open. The inclusion does not by itself imply a separation. A statement that “checking is easier than finding” is intuition, not a theorem; if P=NP, a polynomial search procedure can often be obtained by asking a decision solver whether a partial solution can be extended and fixing choices one by one.

A language is NP-complete if it is in NP and every language in NP polynomially reduces to it. If one NP-complete language is in P, then P=NP. If P≠NP, no NP-complete language is in P. The reduction direction is essential: reduce known NP problems to the candidate to prove hardness.

## Polynomial reductions and examples

A polynomial-time many-one reduction f from A to B satisfies (xin A) iff f(x)in B and computes f in polynomial time. It transfers P membership backward and hardness forward. SAT is NP-complete by the Cook-Levin theorem; 3SAT, CLIQUE, VERTEX-COVER, and Hamiltonian Cycle receive reductions from known NP-complete problems.

**Reduction design example.** To reduce 3SAT to CLIQUE, create one vertex for each literal occurrence in each clause. Connect vertices from different clauses when their literals are not contradictory. A clique of size equal to the number of clauses chooses one mutually consistent literal per clause, yielding a satisfying assignment. A satisfying assignment selects one true literal occurrence from each clause, and selected noncontradictory vertices form a clique. The graph has polynomial size and is computable in polynomial time.

The proof has three separate obligations: graph construction cost, yes-direction, and no-direction. An edge within one clause or between contradictory literals would break the correspondence. A proof that constructs a plausible graph but does not show both directions is incomplete.

## Space and hierarchy intuition

PSPACE contains languages decidable using polynomial space, with no polynomial time bound required. Since a machine using polynomial time can visit only polynomially many cells, P is contained in PSPACE. A polynomial-space machine may run exponentially or longer, but configurations can sometimes be reused or searched without storing the entire computation history. Savitch’s theorem gives (NSPACE(f(n))subseteq DSPACE(f(n)^2)) for suitable f, illustrating that nondeterminism behaves differently for space than for time.

Hierarchy theorems establish that more resource permits strictly more languages: for time-constructible bounds, (TIME(o(g(n)))) is properly contained in (TIME(g(n))) under standard conditions; analogous space hierarchies hold. These are existential separations and do not settle P versus NP. They show that complexity classes are not all equal, while many natural pairs remain unresolved.

## Approximation and parameter choice

Worst-case polynomial time is one lens. Approximation algorithms provide efficiently computable solutions with a provable ratio when exact optimization is hard. Parameterized complexity asks whether exponential dependence can be isolated in a structural parameter k, yielding (f(k)n^()O(1)()). Average-case and smoothed analyses make distributional assumptions explicit rather than silently substituting typical inputs for worst cases.

## Exercises

1. Prove P is closed under complement, union, and intersection.
2. Give an NP verifier for CLIQUE and prove its certificate bound.
3. Explain why P⊆NP does not prove P≠NP.
4. Complete both directions of the 3SAT-to-CLIQUE construction.
5. Distinguish computability, polynomial time, and exponential time.
6. Explain why input encoding changes complexity claims.
7. State the resource preservation required for an NP-hardness reduction.

### Diagnostic solutions


1. Run the polynomial decider(s) sequentially. Complement swaps the final bit. Union accepts if either accepts; intersection accepts only if both do. A constant number of polynomial computations and polynomial-size intermediate data remains polynomial.
2. A certificate lists k distinct vertices. Verify length, distinctness, valid endpoints, and that every pair has an edge. The certificate has O(k log n) bits under a vertex-number encoding, polynomial in the input graph size; verification checks O(k^2) pairs.
3. P⊆NP is an inclusion. The equality question asks whether every efficiently verifiable relation can also be decided efficiently. No known proof establishes a strict separation or equality for the natural classes.
4. From a satisfying assignment choose one true literal occurrence per clause; selected literals from different clauses are noncontradictory, so all corresponding vertices form a clause-sized clique. Conversely, a clique of one vertex per clause contains no complementary pair, so assign selected literals true; each clause has a true literal and the formula is satisfied.

5. Computability asks for eventual termination; polynomial time adds an asymptotic resource bound; exponential time is still computable but grows much faster. A decidable problem can lie outside P and have no known practical exact algorithm.
6. A binary number n has input length about log n, while unary n has length n. An algorithm polynomial in numerical n can be exponential in binary input length. Complexity always measures the chosen representation.
7. The reduction must be computable in polynomial time, output polynomial length, and preserve yes/no membership exactly. Then a polynomial solver for the target would yield a polynomial solver for the source.



## Synthesis checkpoint

Complexity claims must name the input encoding and the resource being bounded. A polynomial bound in a numeric capacity can be exponential in its bit length, while a nondeterministic certificate must be polynomial in the encoded input. When using a reduction, preserve the yes/no answer and account for construction time, output length, and the target verifier’s resource bound.
