# Decidability and the Halting Boundary

Computability theory asks which well-defined questions have algorithms that always finish with a correct answer. The answer is not “all questions about finite strings”: some properties require reasoning about an unbounded future computation. The decisive distinction is between a decider, which halts on every input, and a recognizer, which may run forever when the answer is no. This lesson develops diagonalization and reductions as reusable proof methods.

## Deciders and recognizers

A language L is decidable if some Turing machine halts on every input and accepts exactly the strings in L. It is recognizable (recursively enumerable) if some machine accepts every string in L, while strings outside L may be rejected or run forever. Every decidable language is recognizable: use its decider. The converse is false.

The complement of a decidable language is decidable by swapping accept and reject. For a recognizer, swapping outcomes does not work because a no-instance may never halt. If both L and its complement are recognizable, dovetail their machines: simulate one step of each alternately and accept according to whichever machine accepts first. One of them must eventually accept for every input, so L is decidable. This theorem is a practical diagnostic: proving recognizability of both sides proves decidability.

## The acceptance problem

Let (A_()TM()=()langle M,wrangle: Mtext() accepts ()w()). It is recognizable: a universal machine simulates M on w and accepts if the simulation accepts. If M rejects or loops, the recognizer need not halt. The question is whether a decider exists. It does not.

Assume a decider H for (A_()TM()). Construct D that takes an encoded machine (langle Mrangle), runs H on (langle M,langle Mranglerangle), and does the opposite: if H says M accepts its own encoding, D rejects; otherwise D accepts. Now run D on its own encoding. If H says D accepts itself, D rejects by construction; if H says D does not accept itself, D accepts. Both cases contradict H’s claimed correctness. Therefore (A_()TM()) is undecidable.

The proof depends on effective self-reference and complementing the predicted answer. It is not a statement that no machine can ever simulate another; simulation recognizes A_TM. It says no total procedure can always predict acceptance for arbitrary machine-input pairs.

## The halting problem

Let (HALT_()TM()=()langle M,wrangle:Mtext() halts on ()w()). It is recognizable: simulate M and accept when it halts. It is undecidable by a similar diagonal argument or by reduction from A_TM. If a halting decider existed, one could decide acceptance by first asking whether M halts and then simulating it to its final state. Since acceptance is undecidable, halting is too.

The distinction between halting and accepting is important. A machine may halt and reject, or halt in a state that is not designated accept. A reduction must preserve the exact property being asked about. To reduce acceptance to halting, construct a machine that simulates M and halts exactly when M accepts; if M rejects or loops, the new machine loops. A supposed HALT decider would then decide A_TM.

## Mapping reductions

For languages A and B, a computable function f is a many-one reduction (Ale_m B) when (xin A) iff (f(x)in B). If B is decidable, then A is decidable: compute f(x), run the B decider, and transfer the answer. Therefore if A is known undecidable and A≤mB, B is undecidable.

The direction is the common source of mistakes. To prove B hard, reduce a known hard A to B, not B to A. The function must be total and computable, and the equivalence must hold for yes and no instances. A function that merely maps many yes-instances correctly is not a reduction.

**Worked reduction.** Reduce A_TM to (E_()TM()=()langle Mrangle:L(M)=emptyset()). Given (langle M,wrangle), construct N that on input x simulates M on w and accepts x if M accepts w; otherwise N loops. If M accepts w, N accepts every x, so its language is nonempty. If M does not accept w, N accepts nothing. Thus (langle M,wranglein A_()TM()) iff (langle Nranglenotin E_()TM()). This proves the complement-oriented version; adjust the target or use a complement argument carefully rather than silently reversing membership.

## Rice’s theorem

Rice’s theorem says every nontrivial semantic property of the language recognized by a Turing machine is undecidable. “Semantic” means it depends only on (L(M)), not on the machine’s syntax. “Nontrivial” means some recognizable language has the property and some does not. Examples include emptiness, finiteness, containing a particular string, and equality with a fixed nontrivial language.

Syntactic properties such as “the machine has ten states” are decidable by inspecting the encoding and are not covered. Rice’s theorem does not say every property of machines is hard; it targets meaningful properties of their behavior. The proof generally embeds an arbitrary acceptance question into a machine whose language has the desired property exactly when the original computation accepts.

## Closure and decidability constructions

Decidable languages are closed under union, intersection, complement, concatenation, and star. For union or intersection, run both deciders and combine answers. For concatenation, enumerate every split of the input and decide membership of each pair; accept if some split works. For star, enumerate all finite segmentations and decide each, but a direct bounded dynamic program is cleaner: compute reachable positions where a substring belongs to the base language. Because the input has finite length, the computation halts.

Recognizable languages are closed under union and intersection by dovetailing or product simulation, but complement need not preserve recognizability. If a language and complement are both recognizable, the earlier theorem upgrades it to decidable. This hierarchy explains why a recognizer for a problem is useful evidence but not a complete algorithm when no-instances can run forever.

## Proof discipline

A diagonal proof should name the hypothetical decider, define the constructed machine completely enough to be effective, and evaluate the self-input in both cases. A reduction proof should state source, target, function, computability, and iff condition. A theorem application should verify hypotheses rather than rely on a slogan. These details prevent common errors such as reducing in the wrong direction, confusing acceptance with halting, or using an uncomputable construction.

## Semi-decision as useful computation

Recognizability is not merely a failed form of decidability. A recognizer can enumerate all members of a language by dovetailing its computation on every input: simulate one step of input 0, then one step of inputs 0 and 1, then one step of inputs 0, 1, and 2, and print a string whenever its computation accepts. This eventually prints every member, though it may never announce that the enumeration is complete. The asymmetry between positive evidence and negative evidence is exactly what makes many verification and search procedures useful without being total decision procedures.

When reading an undecidability proof, ask what information is being predicted. A halting oracle would answer a global question about an unbounded computation; a language recognizer supplies only a finite witness of acceptance. The impossibility result is about total, correct prediction for every encoded machine and input, not about the practical success of simulation on instances that happen to halt quickly.

## Exercises

1. Prove that every decidable language has a decidable complement.
2. Prove that if L and its complement are recognizable, L is decidable.
3. Give a diagonal proof that (A_()TM()) is undecidable.
4. Reduce (A_()TM()) to (HALT_()TM()) with an explicit machine transformation.
5. Explain the direction needed to prove B undecidable using a reduction from known-undecidable A.
6. Apply Rice’s theorem to “the machine recognizes a language containing epsilon.”
7. Give a recognizable but undecidable language and explain why its recognizer may not halt on no-instances.

### Diagnostic solutions


1. Run the L decider. If it accepts, reject; if it rejects, accept. Since the original halts on every input, the complement machine does too.
2. Dovetail one step of an L recognizer and one step of a complement recognizer. Exactly one accepts for each input, so the combined machine halts and returns the correct answer.
3. Assume H decides A_TM. Define D(x) to run H on (langle x,xrangle), rejecting if H predicts acceptance and accepting otherwise. D on its own encoding contradicts H in both possible answers.
4. Given (langle M,wrangle), construct N that simulates M on w and halts exactly when M accepts; on reject or nontermination it continues forever. Then M accepts w iff N halts on a fixed input, giving a computable reduction.
5. Reduce A to B: if B were decidable, compute the reduction and decide A. Reducing B to A would show only that B is no harder than an undecidable problem and does not establish B’s undecidability.
6. The property is semantic and nontrivial: some machines recognize languages containing epsilon and some recognize empty language. Rice’s theorem says it is undecidable.
7. (A_()TM()) is recognizable by universal simulation. It is undecidable by diagonalization. On a pair where M rejects or loops, the recognizer may never reach an accepting configuration, so it is not a decider.
