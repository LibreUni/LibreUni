# Finite Automata and Regular Languages

Finite automata are mathematical machines whose memory is one element of a finite set of states. They are deliberately weak: a machine cannot remember an unbounded counter or an arbitrary prefix. That weakness makes them useful. Regular languages can be recognized, searched, and compiled efficiently, and their limitations give the first rigorous boundary between what a computational model can and cannot express.

## Alphabets, strings, and languages

An alphabet Σ is a finite nonempty set of symbols. A string over Σ is a finite sequence of symbols. The empty string is ε, its length is 0, and Σ* is the set of all finite strings over Σ. Concatenation joins strings; if (x=ab), then (a) is a prefix and (b) is a suffix. A language is any subset of Σ*. A decision problem can often be represented as a language: encodings with answer “yes” belong to the language.

Encoding matters. If a problem speaks about integers, graphs, or programs, the alphabet-level machine receives a string encoding. A theorem about a language is meaningful only after the encoding is fixed or shown to preserve the intended property. For elementary examples, use Σ=()0,1(); the language of binary strings with an even number of 1s is a set of strings, not a set of numerical values.

## Deterministic finite automata

A deterministic finite automaton (DFA) is a 5-tuple (M=(Q,Σ,δ,q_0,F)), where Q is a finite set of states, Σ is the input alphabet, δ:Q×Σ→Q is a total transition function, q0 is the start state, and F⊆Q is the set of accepting states. On each input symbol, the machine moves to exactly one next state. It accepts w if the state reached after the entire string is in F.

The transition function extends to strings: δ*(q,ε)=q and δ*(q,xa)=δ(δ*(q,x),a). The language recognized by M is (L(M)=()win𝒳^*: δ^*(q_0,w)in F()). The extended function is not optional notation; it is what turns a one-symbol transition rule into a statement about complete inputs.

**Worked construction.** To recognize strings with an even number of 1s, use Q=()Even,Odd(), start Even, accepting set ()Even(). On 0, remain in the same state; on 1, toggle. Induction on input length proves that the current state records the parity of the number of 1s read. The empty string is accepted, `0` is accepted, `1` is rejected, and `10110` is accepted because it has four 1s.

The same construction can remember a finite suffix. To recognize binary strings ending in `01`, use states representing the longest suffix of the input that is also a prefix of `01`: ε, `0`, and `01`. Transitions preserve this invariant and the `01` state is accepting. Designing states as information summaries is more reliable than guessing a diagram.

## Correctness proofs by induction

The standard proof pattern is an invariant about the state after a prefix. Let (P(q,x)) say that after reading x, state q has a specified semantic meaning. Prove the base case for ε, then prove that if the meaning holds for x, every next symbol preserves the meaning according to δ. The conclusion follows for complete strings.

For the parity DFA, the induction hypothesis is: δ*(Even,x)=Even exactly when x contains an even number of 1s, and similarly for Odd. Appending 0 preserves parity and appending 1 toggles it. Since the start state represents zero 1s and the accepting state represents even parity, the machine is correct.

This style distinguishes machine correctness from language membership. A trace of three examples is evidence that a construction may work; it is not a proof for infinitely many strings. The induction invariant supplies the general argument and often suggests the right state representation.

When a construction fails, inspect the claimed state meaning before adding arbitrary states. A state should summarize exactly the past information that can affect every possible continuation; this continuation-based viewpoint anticipates distinguishability and minimization.

## Nondeterministic finite automata

An NFA permits zero, one, or many possible next states for a state and symbol, often with ε-transitions that consume no input. It accepts if at least one computation branch consumes the whole input and ends in an accepting state. Nondeterminism is existential: a branch is a certificate of acceptance, while all branches must fail for rejection.

For an NFA (N=(Q,Σ,Δ,q_0,F)), define the extended reachable set after a string. Begin with the ε-closure of ()q0(); after symbol a, take all a-successors and close under ε again. The subset construction creates a DFA whose states are subsets of Q. Its start state is the initial closure, and a subset is accepting if it intersects F. The construction proves that the DFA and NFA recognize the same language.

The subset DFA may have up to (2^()|Q|()) states, and this exponential bound is sometimes necessary. Nondeterminism does not make finite automata more expressive, but it can make a description exponentially more compact. This distinction between expressiveness and representation size recurs throughout theoretical computer science.

## Closure constructions

Regular languages are closed under union, intersection, and complement. If DFAs A and B recognize languages (L_A) and (L_B), the product machine has states (Q_A×Q_B), transitions componentwise, and accepting states selected according to a Boolean operation. For intersection, accept pairs in (F_A×F_B); for union, accept pairs with at least one accepting component.

For complement, a complete DFA can swap accepting and nonaccepting states. Completeness matters: if a transition is missing, the machine has not specified what happens on that symbol. Add a rejecting sink state before complementing. The proof follows from the fact that a deterministic complete machine has exactly one final state for every input.

Regular languages are also closed under concatenation and Kleene star, but the simplest proof often uses NFAs. Build an ε-transition from a new start to the first machine, from accepting states of the first to the second, and from accepting states of the second back to the first for star. Convert the resulting NFA to a DFA if a deterministic representation is required.

## Regular expressions and equivalence

Regular expressions are built from ∅, ε, symbols, union, concatenation, and star. Their semantics is defined recursively. The equivalence theorem states that a language is regular exactly when it is described by a regular expression, recognized by an NFA, or recognized by a DFA. Conversions can change representation size dramatically, so choose the form that makes the proof or implementation clear.

Two DFAs are equivalent if they accept the same strings. Build a product machine for symmetric difference and ask whether any reachable pair has exactly one accepting component. If no such pair is reachable, the machines are equivalent. This reduces equivalence to graph reachability and gives a concrete counterexample when they differ: the path label to a mismatching pair is a distinguishing string.

## Exercises

1. Construct a DFA over ()0,1() for strings containing `101` as a substring. State the invariant for each state.
2. Prove by induction that your parity DFA recognizes exactly the even-1 language.
3. Convert an NFA for strings ending in `ab` or containing `aa` into a DFA using reachable subsets.
4. Give a product construction for (L_1cap L_2), and explain why complement requires complete transitions.
5. Find a shortest string distinguishing two proposed DFAs and show the reachable product pair.
6. Explain why a DFA cannot remember an arbitrary integer counter even though it may have many finite states.

### Diagnostic solutions


1. Use states for the longest suffix matching a prefix of `101`: ε, `1`, `10`, and `101`, plus transitions that fall back to the longest viable suffix. The accepting state is sticky under all input because once the substring has appeared it remains present. The invariant is the longest relevant suffix, or “already found” after acceptance.
2. Base: ε has zero 1s and the machine is in Even. Step: appending 0 preserves both state and parity; appending 1 changes parity and toggles state. Therefore the state after every prefix equals its parity, so the final accepting condition is exactly even parity.
3. Track subsets of NFA states after each input and ε-closure. Mark a subset accepting if it contains a state that has completed either pattern. Do not list unreachable subsets as necessary DFA states; reachable-subset construction is both a proof and a minimization aid.
4. Use pair state (p,q), componentwise transition, and accepting pairs where both components accept. Complement swaps final status only after adding a sink for undefined transitions; otherwise some strings have no run and swapping does not represent set complement.
5. Explore the product from the pair of starts breadth-first. A pair with one accepting and one rejecting component yields a shortest distinguishing string via its predecessor labels. This is more reliable than comparing diagrams by eye.
6. A DFA has finitely many states, so by the pigeonhole principle two different counter values eventually share a state. From that point, identical future input produces identical behavior, so the machine cannot distinguish all counter values required by an unbounded equality or counting condition.
