# NP-Completeness

A problem is NP-complete when it is in NP and every problem in NP reduces to it in polynomial time. Cook–Levin establishes Boolean satisfiability as a foundational complete problem; later proofs reduce known complete problems to new ones.

To prove completeness, membership and hardness are separate obligations. A reduction from an easy problem proves little; direction matters because an efficient solver for the target must solve the source.

## Decision problems and certificates

NP is a class of decision languages. An instance is a yes-instance when a polynomial-length certificate can be checked by a deterministic polynomial-time verifier. The certificate is not an algorithm and need not be unique. For SAT it is an assignment; for CLIQUE it is a set of vertices; for Hamiltonian Cycle it is an ordered list of vertices. The verifier must check both syntactic validity and the semantic constraint.

The certificate bound is measured in input length. If numbers are binary encoded, a certificate listing a value proportional to the numeric magnitude may be exponentially too long. State encodings before claiming membership. A problem can be in NP even if finding a certificate appears difficult, because verification is the defining requirement.

## Polynomial reductions

A polynomial many-one reduction A  at most p B is a polynomial-time function f with x in A exactly when f(x) in B. If B is in P, then A is in P by computing f and solving B. Consequently, reducing a known hard problem A to a new problem B proves B is at least as hard as A. The reverse direction gives a different conclusion and is the most common exam error.

For NP-hardness, choose a source whose structure resembles the target. A 3-SAT-to-graph reduction uses variable and clause gadgets; a graph-to-set-system reduction turns vertices or edges into sets and constraints. Every gadget must have local completeness and soundness, and the composition must preserve global solutions. Drawing a gadget is not a proof; enumerate its possible local solutions and connect them to the source semantics.

## SAT as a source problem

CNF-SAT asks whether a conjunction of clauses has a satisfying assignment. 3-SAT restricts each clause to three literals and remains NP-complete. It is convenient because a target construction can use one gadget per variable and clause, with connections enforcing consistency.

In a reduction from 3-SAT, a variable gadget usually offers two mutually exclusive choices, representing true and false. Clause gadgets must be satisfiable exactly when at least one attached literal is selected. Wires or equality gadgets propagate a choice without allowing it to flip. Count target elements and show the construction is polynomial; a reduction that creates exponentially many assignments is not valid.

## Canonical complete problems

VERTEX COVER asks whether a graph has a vertex set of size at most k meeting every edge. CLIQUE asks whether a graph has a complete subgraph of size at least k. INDEPENDENT SET asks for k pairwise nonadjacent vertices. Complementing edges maps clique to independent set, and a vertex cover is the complement of an independent set, giving reductions among these problems.

HAMILTONIAN CYCLE asks whether a graph contains a cycle visiting each vertex exactly once. 3-COLORING asks whether vertices can be assigned three colors so adjacent vertices differ. SET COVER asks whether k sets cover a universe. These problems differ in surface form but share the proof pattern: polynomial certificate and a reduction from a known complete source.

## Complements and coNP

If L is NP-complete, its complement is coNP-complete under corresponding reductions, but it is not known in general that the complement is in NP. TAUT is a classic coNP problem: a formula is accepted when every assignment satisfies it. A short counterexample certifies that a formula is not a tautology, but a yes certificate for tautology may require reasoning over all assignments.

Do not say “no certificate exists” when explaining coNP. The class concerns certificates for yes instances of the complement language, equivalently certificates for no instances of the original language. Whether NP equals coNP is open.

## Self-reducibility and search

For many problems, a decision oracle can construct a witness by fixing variables one at a time. Ask whether a satisfying completion exists with x1=true; if yes retain it, otherwise set x1=false. Repeat. The oracle is called polynomially many times, and each query remains polynomial in input size. This is self-reducibility, not a general theorem that every optimization problem is easy once its decision version is known.

Optimization can use binary search when objective values have a polynomial-bit representation and the decision predicate is monotone. If weights are binary encoded, count bit complexity rather than numeric magnitude. Reconstruction still requires a proof that queried bounds and predicates are correct.

## Hardness versus practical solvability

NP-complete is a worst-case classification. It does not say every instance is hard, that no useful exact solver exists, or that approximation is impossible. Structured instances, small parameters, preprocessing, branch-and-bound, and SAT/ILP solvers can handle large practical cases. A theorem says that a polynomial-time algorithm for all instances would imply P=NP.

Parameterized algorithms refine this picture. If k is a solution-size or structural parameter, an algorithm with running time f(k)n^c may be excellent for small k even though the problem is NP-hard when k is unrestricted. Kernelization reduces an instance to a size bounded by a function of k. State parameter dependence explicitly.

## Reduction checklist

1. Define source and target decision problems precisely.
2. Give the transformation on arbitrary instances.
3. Prove source yes implies target yes.
4. Prove target yes implies source yes.
5. Bound output size and construction time.
6. Prove target membership in NP if claiming completeness.
7. Check empty clauses, isolated vertices, k=0, and repeated elements.

## Exercises

1. Prove that CLIQUE reduces to INDEPENDENT SET by complementing edges.
2. Give the certificate and verifier for VERTEX COVER.
3. Explain why reducing 3-SAT to a problem proves NP-hardness rather than membership in NP.
4. Describe a self-reduction that extracts a satisfying assignment from a SAT decision oracle.
5. Distinguish an NP-complete problem from an instance that happens to be difficult.
6. Identify the missing step in a gadget reduction that only proves yes-to-yes.

### Diagnostic solutions


1. Map G to its complement graph. A set is a clique in G exactly when it is an independent set in the complement, so the same k is preserved and the map is polynomial.
2. The certificate lists at most k vertices. Check every graph edge has at least one endpoint in the list and count distinct listed vertices; this is polynomial.
3. It shows a polynomial target solver would solve 3-SAT, so the target is NP-hard. NP membership requires an independent polynomial certificate verifier.
4. Query whether a satisfying assignment remains with each variable tentatively true; keep true when possible and otherwise set false. The invariant is that the partial assignment has a satisfying completion.
5. NP-completeness concerns a language’s worst-case family under polynomial reductions. One instance may be easy, and a solver may be fast on a structured distribution without changing the classification.
6. It lacks soundness: a target yes solution might exist when the source formula is unsatisfiable. Decode every target solution and prove it yields a source solution.



## Exam standard

A reduction diagram should have arrows labelled by the actual transformation and an accompanying size bound. For each gadget, state what assignments or local choices it permits. Then compose the local lemmas and handle degenerate parameters. A polished proof makes it possible for a reader to reconstruct an algorithm for the source problem from a hypothetical target solver; if that algorithm cannot be described, the reduction direction or decoding argument is probably incomplete.

**Exercise.** Outline a reduction from 3-SAT to a constraint-selection problem. Identify variable consistency, clause satisfaction, and polynomial construction size.

## Advanced synthesis: constructing a reduction

Start with the source problem’s witness structure. If the source witness is a truth assignment, the target must offer two consistent choices per variable. If the source witness is a path, the target must prevent skipping vertices and enforce adjacency. If the source witness is a set, the target must represent membership and cardinality without allowing duplicated or fractional choices unless the target explicitly permits them.

For every gadget, write two lemmas. Completeness says a valid source choice can be extended to a target solution. Soundness says every target solution decodes to a valid source choice. Then show that gadgets compose: shared connector vertices or variables cannot be assigned inconsistently. Count vertices, edges, clauses, or weights and show the output is polynomial in the source encoding length, including the bit length of numeric parameters.

NP membership is not implied by hardness. Give a certificate and verifier independently. Verify syntactic constraints, semantic constraints, and the bound on certificate length. If a target problem is an optimization problem, formulate a decision version before discussing NP-completeness. If the objective values are binary encoded, be careful that binary search and certificate bounds are polynomial in bit length.

Finally distinguish worst-case classification from practical algorithms. Branch-and-bound, preprocessing, small parameters, approximation, and structured inputs can make complete problems useful in practice. The theorem says that a polynomial algorithm for every instance would imply P=NP; it does not say a particular solver cannot finish a large instance.
