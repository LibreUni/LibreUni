# Proof Techniques

Formal language theory depends on precise quantifiers. To prove $forall xin S:P(x)$, establish an arbitrary case; to refute it, one counterexample suffices. Induction proves a base case and an inductive step from $n$ to $n+1$.

An invariant proof for a machine states a property true initially and preserved by every transition. Contradiction assumes the negation and derives an impossibility; reductions use the same discipline to transfer impossibility.

## Quantifiers before intuition

Most errors in theory proofs are quantifier errors. “For every DFA there is a word” is different from “there is one word for every DFA.” Write the order explicitly. To prove a language is regular, one must exhibit one finite machine that works for every input. To prove that no finite automaton recognizes a language, one must show that every proposed finite machine fails, often by choosing a distinguishing input after seeing the machine’s state bound.

Negation is a practical tool. The negation of for every x, P(x) is there exists x with not P(x); the negation of there exists x, P(x) is for every x, not P(x). Thus “every decomposition can be pumped” is refuted by finding one decomposition that cannot be pumped, while a pumping-lemma non-regularity proof must show that every legal decomposition fails for some pumping exponent.

## Induction patterns

Weak induction proves P(0), then P(n) implies P(n+1). Strong induction assumes P(0) through P(n) and proves P(n+1). Strong induction is natural for parse trees, recursive algorithms, and decompositions where an object depends on several smaller objects. Structural induction generalizes the same idea to syntax: prove a property for atomic constructors and prove it is preserved by every grammar constructor.

For a DFA, a useful induction statement is: after reading a word w from state q, the machine reaches delta-star(q,w). The empty-word case follows from the definition of delta-star. For a word xa, the induction hypothesis gives the state after x, and one transition on a gives delta(delta-star(q,x),a), exactly the recursive definition. Notice that the statement is stronger than the target “the machine accepts w”; the stronger lemma makes later proofs compositional.

Induction can fail when the step changes the object class or when the base cases are incomplete. For a grammar, include every production form. For a machine, prove the invariant for every transition type, including epsilon transitions. For a recursive program, the measure must decrease on every recursive call; otherwise an induction argument may prove a property of terminating calls while saying nothing about termination.

## Invariants and variants

An invariant is preserved; a variant or ranking function decreases and proves termination. For a PDA recognizing balanced parentheses, the stack-height nonnegative invariant holds on every legal prefix. For a terminating algorithm, a natural-valued variant decreases under each loop iteration. If the variant can stay equal, it is not yet a termination proof. If it can become negative, the claimed well-founded order is wrong.

To design an invariant, describe what information the machine must remember and express it as a relation between configuration and input prefix. A DFA configuration can encode a residue class, a parity condition, or a finite suffix. A PDA configuration can encode the unmatched portion of a nesting obligation. An invariant that mentions only the current state may be too weak; one that restates the entire desired theorem may be impossible to verify locally.

## Contradiction and minimal counterexamples

Contradiction is especially effective for finite-state limitations. Assume a machine with p states recognizes the target language. Choose a word or family of prefixes that forces more than p distinguishable futures. By the pigeonhole principle, two prefixes share a state; append the same suffix to both; one resulting word should be accepted and the other rejected, contradicting identical continuation behavior.

Minimal-counterexample arguments assume a smallest failing object and derive a smaller one. They are useful for parse ambiguity and closure proofs. The measure must be well-founded, such as length or number of nodes. “Smaller” must be specified: fewer symbols is not necessarily fewer grammar nodes, and a shorter derivation may have more steps.

## Equivalence proofs

To prove two machines or definitions equivalent, prove both inclusions. For languages, show every word accepted by A is accepted by B and vice versa. For automata constructions, induction on the input word commonly establishes a simulation relation. For grammars, show each production step can be simulated and then show every generated string has a corresponding derivation in the other direction.

When proving a construction, state the invariant relating source and target configurations. For subset construction, the target DFA state is the set of NFA states reachable after the consumed prefix, including epsilon closure. The induction step must account for both symbol transitions and newly reachable epsilon states. Omitting epsilon closure is a common reason an apparently plausible equivalence proof fails.

## Reductions as proof architecture

A reduction maps instances of problem A to instances of problem B while preserving yes/no answers. To prove B is hard, assume an algorithm for B, transform an arbitrary A instance, run the B algorithm, and decode its answer. The map must be computable, its output size must be controlled, and correctness must hold in both directions. A one-way implication is not enough for many undecidability reductions.

Reduction direction is a frequent trap. If A reduces to B and A is known impossible, then B cannot be easier in the relevant sense. Showing B reduces to A does not establish that B is hard merely because A is hard; it may show instead that B is no harder than A. Draw the assumed solver and the transformed instance before writing the proof.

## Proof debugging checklist

Ask: What is the universe of objects? Which variables are arbitrary? Which are chosen by the proof? Is the machine deterministic or nondeterministic? Is the word fixed before the machine or chosen after its state bound? Does every transition case appear? Does the construction terminate? Is the claimed conclusion stronger than the premises? Can a tiny counterexample be enumerated?

A polished proof should identify definitions at the point of use, state the invariant or induction hypothesis, justify the transition or case split, and close the quantifiers. Replace phrases such as “clearly” with the exact reason when that reason carries the theorem. Formal clarity is not decoration: it is how a reader checks that no case or quantifier has been silently dropped.

## Exercises

1. Prove by structural induction that every full binary tree with n internal nodes has n+1 leaves.
2. State and prove a DFA simulation lemma using an induction on input length.
3. Design an invariant and ranking function for a procedure that repeatedly removes adjacent equal symbols.
4. Give a contradiction proof that two distinct prefixes with different future behavior cannot reach the same DFA state.
5. Diagnose the direction error in the claim “A reduces to B, therefore an algorithm for A solves B.”
6. Write the negation of: for every grammar G there exists a word w that has two distinct parse trees.

### Diagnostic solutions


1. A one-node tree has one leaf. Combining trees with n1 and n2 internal nodes creates one new internal node and joins their roots, giving (n1+1)+(n2+1)=n1+n2+2 leaves, which is one more than n1+n2+1 internal nodes.
2. Induct on w. The empty word reaches q by delta-star’s base case. For wa, use the hypothesis for w and then the single transition on a; this is exactly the recursive definition of delta-star.
3. An invariant is that the current string is obtainable by deletions from the original, so every remaining symbol has an origin. A ranking function is the current length, which decreases by two per deletion; it is a nonnegative integer, so the process terminates.
4. If prefixes x and y reached the same state, every common suffix z would produce the same accept/reject result because the deterministic continuation is identical. A suffix distinguishing x and y therefore contradicts the shared state.
5. A solver for B would solve A after mapping A instances to B, not the other way around. The reduction supplies a direction for transferring an assumed algorithm.
6. There exists a grammar G such that every word w has at most one parse tree. Quantifier negation changes both scopes and the predicate.



A final proof check is to replace every pronoun in the central argument with a named object. If “it” might mean a state, prefix, stack, or language, the proof is not yet precise enough for a theorem whose quantifiers range over all of them. A second check is to write the proof’s dependency graph: definitions support lemmas, lemmas support the construction, and the construction supports the final theorem. If the conclusion uses a fact not present in that graph, either state the missing lemma or remove the claim.

## Oral-exam standard

When presenting a proof, begin with the definition that makes the target precise. Announce the quantifier order, choose the induction measure or reduction map, and separate the invariant from the conclusion. A grader should be able to identify the base case, preservation step, and termination or contradiction without reconstructing them from prose. If a proof depends on determinism, totality, finiteness, or effective computability, name that assumption at the exact step where it is used.

**Exercise.** Prove by induction that a deterministic automaton processing a word has exactly one state after every prefix. Identify where determinism is used.
