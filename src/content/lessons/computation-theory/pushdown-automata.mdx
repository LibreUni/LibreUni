# Pushdown Automata

A pushdown automaton combines finite control with a stack, allowing it to remember nested obligations. It can recognize balanced parentheses by pushing on `(` and popping on `)`, rejecting a pop from an empty stack and accepting only with no pending obligations.

The stack is LIFO, so it remembers nesting but not arbitrary multiple counters. CFG derivations and pushdown runs are equivalent descriptions of context-free languages.

## Formal model

A nondeterministic pushdown automaton has a finite state set Q, input alphabet Sigma, stack alphabet Gamma, transition relation, start state, and accepting states or an acceptance condition based on empty stack. A transition can inspect the next input symbol or take an epsilon step, inspect the top stack symbol, replace it by a finite string, and change state. The stack top convention must be fixed; otherwise traces are ambiguous.

Acceptance by final state and acceptance by empty stack describe the same class of languages, although the conversions add control states or a bottom marker. For a machine accepting by final state, the input must be consumed and the current state must be accepting. For empty-stack acceptance, the stack must be empty after all input is consumed. Epsilon transitions make it possible to change stack state without reading input, but they also introduce nondeterministic runs that need careful reasoning.

## Designing a stack invariant

For the language of balanced parentheses, the stack represents unmatched opening parentheses in order. Reading an opening parenthesis pushes it. Reading a closing parenthesis requires a nonempty stack and pops one symbol. After a prefix, the stack height equals the number of unmatched opens exactly when no prefix has more closes than opens. The accepting configuration has consumed all input and has height zero.

For ()a^n b^n(), push one marker per a, then switch to a state that pops one marker per b. A b before the switch, an a after the switch, or a pop from an empty stack rejects. The invariant after a^i b^j is that the stack contains n-i? More precisely, after i a symbols and j b symbols in the second phase, it contains max(i-j,0) markers, and the machine has not accepted a prefix with j greater than i. The phase state carries order information; the stack alone does not.

The PDA cannot generally compare two independently growing counts after both have been forgotten. It can compare a later block with an earlier block because it stores the earlier count. It cannot store arbitrary information about two blocks and then retrieve both in LIFO order. This explains why some three-block equality languages are not context-free.

## Nondeterminism and grammar simulation

A PDA may guess where a useful decomposition begins. For a grammar S - greater than  a S b | epsilon, the machine can push a marker while reading a’s, nondeterministically switch, and pop while reading b’s. For a general CFG, a PDA can simulate a leftmost derivation: initialize the stack with the start variable; replace a variable by the right side of a production using an epsilon transition; match a terminal on the stack against the next input symbol.

The construction is nondeterministic because a variable can have several productions and because a derivation may choose among them. A word is accepted if at least one complete run succeeds. The existence of a rejecting branch is irrelevant. This existential semantics is different from universal verification and must be stated in correctness proofs.

To prove grammar-to-PDA correctness, establish that the stack contents describe a sentential-form suffix whose terminals match the already consumed prefix. Every production transition preserves the represented derivation; every terminal-matching transition consumes the next expected symbol. Conversely, read an accepting run as a sequence of production choices and terminal matches to construct a derivation.

## Grammar simulation by a PDA

Given a PDA, one can construct a CFG whose variables encode state pairs and a stack obligation. A variable such as A_pq represents strings that take the PDA from state p with a designated stack symbol exposed to state q after that symbol is removed, while all intermediate stack activity is properly nested. Productions describe a direct pop, an epsilon move, or a push followed by a matching pop with an intermediate state.

The construction is larger than the intuitive machine but proves expressive equivalence. The key is balanced stack behavior: a pushed symbol must eventually be matched by its pop, and nested obligations must complete before the outer one can complete. This is analogous to matching parentheses in a derivation tree.

## Determinism and ambiguity

Deterministic PDAs recognize a proper subset of context-free languages. Nondeterminism can guess a midpoint, a production, or a future matching relationship that cannot be known from the current input and stack alone. The language of palindromes is a standard illustration: a nondeterministic PDA guesses the center, while a deterministic one cannot generally know when to switch from pushing to popping without a delimiter.

A grammar can be ambiguous when one word has two parse trees. PDA nondeterminism and grammar ambiguity are related but not identical: different runs can exist for an unambiguous grammar because of operational choices, and an ambiguous grammar can have multiple derivations for reasons that a transformed PDA encodes differently. State what is being counted: accepting runs, derivations, or parse trees.

## Limits and closure

Context-free languages are closed under union, concatenation, and star, and under intersection with a regular language. They are not closed under arbitrary intersection or complement. PDA constructions make the positive closure results intuitive: branch for union, sequence machines for concatenation, and repeat a component for star. Complement would require deterministic behavior, and nondeterministic PDAs cannot simply swap accepting and rejecting runs.

The intersection-with-regular result is useful for impossibility proofs. If L were context-free, L intersect R would be context-free for every regular R. Choose R to eliminate irrelevant structure, then use the context-free pumping lemma or a known non-context-free core. This technique often turns a complicated language into a clean block-count language.

## Traces and implementation discipline

A trace should record state, unread input, stack top or full stack, and whether the next step consumes input. Epsilon loops can create infinitely many runs, so a simulator needs a visited-configuration policy or bounded exploration when debugging. Do not identify configurations solely by state and input position; different stacks can lead to different futures.

For parser implementations, distinguish recognition from construction. A recognizer returns yes/no; a parser returns a derivation, parse forest, or error position. Ambiguous grammars may require a shared packed parse forest rather than one arbitrary tree. Complexity also depends on grammar form and input length; a PDA model establishes expressiveness, not automatically an efficient parser.

## Exercises

1. Give a formal PDA strategy for ()a^n b^n:n at least 0() and state its phase invariant.
2. Explain why a delimiter makes palindrome recognition easier for a deterministic PDA.
3. Trace a PDA for balanced parentheses on (()()) and on ())(.
4. Describe the grammar-to-PDA simulation and prove one direction of correctness.
5. Explain why complement is not obtained by swapping accept and reject states in an NPDA.
6. Use intersection with a regular language to isolate a non-context-free sublanguage.

### Diagnostic solutions


1. In the a-phase push A; on the first b move to the b-phase and pop A per b; reject illegal symbols or empty pops; accept after input and stack bottom are both reached. After a^i b^j in the b-phase, the stack has i-j A markers when j at most i.
2. The delimiter explicitly identifies the midpoint, so the machine can stop pushing exactly when it reads the delimiter and then pop while comparing the reverse half.
3. On (()()), push for the first two opens, pop at the first close, push at the next open, then pop twice; the final stack is empty. On the input starting with ))(, the first close attempts to pop an empty stack and rejects immediately.
4. Put the start variable on the stack. Epsilon transitions replace variables by production right sides; terminal transitions match and consume input. The stack invariant says it is the unexpanded suffix of a sentential form, proving that an accepting run yields a derivation.
5. Nondeterministic acceptance is existential: one accepting run suffices. Swapping states would reject only when all runs reject, but a run that fails does not witness nonmembership. Determinization is not available for all CFLs.
6. Choose R to force a fixed block order, such as a^*b^*c^*. If the intersection has equal counts across three blocks and is known not context-free, closure under regular intersection contradicts context-freeness of the original language.



## Exam practice

For every PDA problem, first state the acceptance convention and the stack-top convention. Then write a configuration invariant before drawing transitions. Test the empty word, a shortest accepted word, a shortest rejected prefix, and a word that crosses each phase boundary. These tests do not replace the proof, but they expose missing epsilon moves, accidental acceptance with a nonempty stack, and transitions that read the wrong symbol.

**Exercise.** Trace a PDA on `(()())`. At every symbol record state, unread input, and stack contents; identify the first configuration that would reject `())`.
