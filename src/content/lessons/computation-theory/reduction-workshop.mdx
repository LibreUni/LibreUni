# Reduction Workshop

## What a reduction must preserve

A reduction is a computable translation between problems. For language reductions, a map f from source strings to target strings must satisfy x belongs to A exactly when f(x) belongs to B. The two directions are completeness and soundness. A reduction that maps only yes instances correctly is not enough, because a target solver could return yes on a malformed image of a source no instance.

Before constructing a gadget, write the assumed solver and the algorithm that uses it. If A reduces to B, the algorithm receives x for A, computes f(x), invokes the B solver, and returns the same answer. This picture immediately fixes the direction and exposes whether f must preserve a witness, a computation history, or only a yes/no property.

## Many-one and Turing reductions

In a many-one reduction the target solver is called once on one translated instance. A polynomial-time Turing reduction may call an oracle multiple times, adapt later queries to earlier answers, and sometimes reconstruct a search solution. Many-one reductions are stronger for completeness proofs because they preserve a single-instance structure. State which kind is being used.

For undecidability, computable reductions need not be polynomial. To show problem B undecidable, reduce a known undecidable problem A to B: if B had a decider, composing the computable translation with it would decide A. Termination of the translation is still required. A partial map that fails to output an instance on some inputs does not support the standard proof.

## A worked graph reduction pattern

To reduce CLIQUE to INDEPENDENT SET, given G=(V,E) construct the complement graph with the same vertices and edges exactly where G has no edge. A set S is a clique in G iff no pair in S is a complement edge, which is exactly independent set. The construction examines each vertex pair, so it is polynomial. It preserves k unchanged and handles k=0 and k=1 without special cases.

This example illustrates a direct encoding. Gadget reductions are needed when one target constraint cannot directly express a source choice. A variable gadget provides consistent truth states; a clause gadget provides a target witness if at least one literal state is satisfying; connector gadgets ensure a choice is shared. Prove local behavior first, then prove that local choices can be assembled globally.

## Undecidability reduction pattern

Suppose a target problem asks whether a program eventually exhibits property P. To reduce HALT, take an input program M and word w, construct a program N that simulates M(w), then exhibits P only after the simulation halts. If a target decider existed, run it on N to decide whether M halts. The construction must be effective and must not itself need to know whether M halts.

Rice-style reasoning says every nontrivial semantic property of the partial function computed by a program is undecidable. “Semantic” means depending on behavior rather than syntax; “nontrivial” means some programs have the property and some do not. A proof still needs a reduction or a theorem application with the hypotheses checked.

## Common failure modes

Reducing the target to the source proves the wrong hardness claim. Mapping a source solution to a target solution but not decoding target solutions leaves soundness unproved. An exponential-size truth table invalidates a polynomial reduction. A gadget that permits extra local states may create false target solutions. A reduction to a recognizer does not automatically give a decider when the target may run forever on no instances.

## Exercises

1. State the direction needed to prove B is NP-hard from known NP-hard A.
2. Prove the CLIQUE complement-graph reduction in both directions.
3. Design a computable reduction from a halting question to a program-property question.
4. Explain why an exponential output construction is not polynomial-time.
5. Identify whether an adaptive oracle algorithm is many-one or Turing reduction.

### Diagnostic solutions


1. Reduce A to B. Then a B solver would solve A by translation, so B is at least as hard as A.
2. If S is a G-clique, every pair is absent from the complement, so S is independent. If S is independent in the complement, every pair is an edge in G, so S is a clique.
3. Given M,w construct N that simulates M(w) and prints a marker or accepts exactly after simulation halts. A decider for the marker property would decide HALT.
4. Writing the output already takes time proportional to output length. If output is exponential in source length, no algorithm producing it explicitly can run in polynomial time.
5. It is a Turing reduction because it may make multiple adaptive oracle calls. A many-one reduction computes one target instance before the oracle answer is known.



## Extended reduction checklist

When a reduction is proposed, first identify the source witness and the target witness. Does the target solution contain one source choice, many repeated choices, or a complete computation history? Write a decoding function before writing the gadget. If decoding is impossible because the target solution can use an unintended state, the gadget is not sound.

For graph gadgets, list the degree and adjacency of every new vertex and test the smallest parameter values. For formula gadgets, write local satisfying assignments in a table. For machine reductions, specify the constructed machine’s behavior on every branch, including malformed inputs and nonhalting simulations. Polynomiality includes representation size, not only the number of conceptual gadgets.

In an oral examination, draw the composition: source input, transformation, target oracle, decoded answer. Then say what contradiction follows if the target oracle existed. This diagram is often more informative than several paragraphs and prevents reversing the arrow.

Write a polynomial transformation from a known hard problem into a domain problem. State encoding size, yes direction, no direction, and why the transformation terminates. Reject arguments that merely map similar vocabulary.

**Exercise.** Annotate every line of your construction with the property it preserves.

## Worked reduction protocol

Suppose the source is a constraint problem with variables x1 through xn and clauses C1 through Cm. A target gadget construction should begin by defining a target representation for a truth choice. Use two distinguished configurations, one for true and one for false, and prove that every target solution selects exactly one. A consistency gadget connects all occurrences of a variable and proves that they share the same choice. A clause gadget has a target completion exactly when at least one attached literal is selected. The global target instance is the disjoint union plus connector constraints.

The proof is written in two stages. Given a satisfying source assignment, select the matching state in every variable gadget and use the satisfied literal to complete each clause gadget. Conversely, take an arbitrary target solution, decode each variable choice, and use the clause-gadget soundness lemma to show that each source clause has a true literal. This is the point where extra target solutions are excluded. Then count gadgets: if there are O(n+m) gadgets of constant size and each connector has polynomial description, the output is polynomial.

For an undecidability reduction, use the same discipline but replace finite witnesses with program behavior. Given M and w, construct N whose first action is to simulate M on w. Only after that simulation halts does N perform the target property. If M halts, N has the property; if M does not halt, N never reaches it. A hypothetical total decider for the property therefore decides halting. The construction itself is finite and computable because it embeds the fixed code of M and w as data; it does not wait for the simulation to finish while constructing N.

## Exercises

1. For a proposed gadget, list the local states and prove that no third state is possible.
2. Write both implications for a reduction from a clause problem to a target selection problem.
3. Explain why a target recognizer is insufficient when the source no case requires detecting nonhalting.
4. Give an example where a reverse reduction establishes membership or easiness rather than hardness.
5. Design a bounded exhaustive test for all source instances of size at most three and compare translated answers.

### Diagnostic solutions


1. Enumerate every target constraint involving the gadget and show that any assignment violating the intended two states violates one constraint. Then exhibit the two intended assignments to prove existence.
2. A source yes assignment extends to a target solution by choosing matching variable and clause states. Any target solution decodes to one source assignment, and each satisfied target clause gadget identifies a true source literal.
3. A recognizer can loop on a target no instance, so it cannot necessarily return the source no answer. A decider reduction requires a target procedure that halts on both cases.
4. If B reduces to an efficiently solvable A, the reduction shows B is no harder than A; it does not show A is hard. Hardness requires a known hard source reducing into the target.
5. Enumerate assignments or graphs, compute the source predicate, construct the target, and solve the target by brute force. Any disagreement yields a smallest counterexample to inspect.



The final proof should let a reader run the hypothetical target solver as a subroutine and recover the source answer. If that algorithm cannot be written in a few precise lines, the claimed reduction is probably missing its direction, decoding, or termination argument.
