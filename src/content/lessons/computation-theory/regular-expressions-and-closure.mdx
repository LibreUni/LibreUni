# Regular Expressions and Closure

Regular languages are closed under union, concatenation, and Kleene star by automaton constructions. They are also closed under complement for complete DFAs and intersection through a product automaton. Closure lets a proof compose known machines instead of inventing a new one.

If $L_1$ and $L_2$ are regular, the product state $(q_1,q_2)$ tracks both machines; accepting pairs define intersection. Complement swaps accepting and rejecting states only after missing transitions are sent to a sink.

## Regex semantics and precedence

A regular expression denotes a language, not a particular matching procedure. The basic expressions are empty language, epsilon, and a symbol a; union denotes choice, concatenation denotes juxtaposition, and star permits zero or more repetitions. A conventional precedence is star highest, concatenation next, and union lowest. Parentheses should make the intended parse explicit, especially when translating between notation and automata.

For example, (0+1)*01 denotes all binary strings ending in 01, while 0+1* denotes either the one-symbol word 0 or any number of 1s. The empty language is not the same as epsilon: one contains no words, while the other contains the zero-length word. Confusing them changes concatenation and star behavior.

## From expressions to automata

Thompson’s construction builds an epsilon-NFA recursively. A symbol creates one edge between a start and accept state. Union creates a fresh start and accept with epsilon branches into the alternatives and epsilon exits from them. Concatenation joins the first accept to the second start. Star adds epsilon paths that allow zero repetitions or return from the repeated fragment to its start.

The construction proves correctness by induction on expression structure. For a union, a path must choose one branch; for concatenation, it must complete the first fragment before entering the second; for star, it may take zero iterations or a sequence of complete iterations. Epsilon transitions do not consume input, so the path’s labeled edges spell exactly a word in the expression’s language.

Subset construction converts an epsilon-NFA to a DFA whose state is a set of NFA states. The initial DFA state is the epsilon closure of the NFA start. On symbol a, move from every member and take epsilon closure again. A subset is accepting when it contains an accepting NFA state. The number of possible subsets is exponential, although many may be unreachable and minimization may later merge equivalent states.

## Closure constructions as products

For union and intersection, run two DFAs in lockstep. The product state records both current states. For intersection, accept exactly when both components accept; for union, accept when at least one accepts. The transition on a is the pair of component transitions. Induction on input length proves that each component of the product state equals the state reached by its original DFA.

Difference is intersection with complement: L1 minus L2 equals L1 intersect complement(L2). Symmetric difference accepts when exactly one component accepts and is useful for equivalence testing. Two DFAs are equivalent if their symmetric difference language is empty; a breadth-first search of the product can find a distinguishing word if one exists.

Complement requires a complete deterministic automaton. If a DFA lacks a transition for a symbol, add a sink state with self-loops on every symbol. Only then does every input have one run, and swapping accepting with nonaccepting states complement the language. Swapping states in an NFA is invalid because the original language uses existential paths.

## Closure under homomorphisms

A homomorphism maps each alphabet symbol to a string and extends by concatenation. The image of a regular language under a homomorphism is regular: replace every transition labeled a by a path labeled h(a), introducing intermediate states as needed. Epsilon images are allowed and simply remove the label. Inverse homomorphism is also regularity-preserving: run a DFA for the target while, on source symbol a, advancing through the string h(a).

These operations support proofs. If a language’s image under a homomorphism is known non-regular, the source cannot be regular because regular languages are closed under images. Inverse homomorphism can pull a target property back to a simpler source alphabet. Always specify whether the map is applied to words or to individual transitions.

## Algebraic laws and identities

Regular-expression equivalence follows from language equality. Union is commutative, associative, and idempotent; concatenation is associative but not generally commutative; epsilon is the identity for concatenation; the empty language is absorbing for concatenation; and star satisfies L* = epsilon union L L*. These identities help simplify expressions but do not justify distributing star carelessly. In particular, (L1 union L2)* permits arbitrary alternation of both languages, whereas L1* union L2* permits only one kind of repetition at a time.

The derivative of a language with respect to a symbol a is the set of suffixes x such that ax belongs to the language. Brzozowski derivatives provide a conceptual DFA construction: each derivative is a DFA state, and a derivative is accepting exactly when epsilon belongs to it. Repeated derivatives can be represented as regular expressions and then deduplicated modulo language equivalence. This gives a clean connection between symbolic matching and automata.

## Reachability and minimization

After constructing an automaton, remove unreachable states before measuring it. For a DFA, partition refinement separates accepting and nonaccepting states and then repeatedly splits states whose transitions lead to different blocks. The final blocks are the equivalence classes of indistinguishable states, and the quotient DFA is minimal up to renaming.

A minimal DFA has a semantic lower bound: it needs at least as many states as there are pairwise distinguishable prefixes. This is the Myhill-Nerode perspective. Regex size and DFA size can differ exponentially, so minimizing one representation may worsen another. Choose the representation according to the operation: regex is concise for specification, DFA is predictable for repeated matching, and an NFA may be compact for construction.

2. Describe Thompson construction for (0+1)*01 and count states up to constant choices.
3. Build the product DFA for even length intersected with an even number of 1s.
4. Explain why complementing an incomplete DFA without a sink is wrong.
5. Prove closure under homomorphic image using transition replacement.
6. Give a pair of expressions showing that star does not distribute over union in the naïve way.
7. Explain why product reachability is an equivalence test for two DFAs.

### Diagnostic solutions


1. One expression is (0+10)*11(0+01)* with care: a cleaner specification uses a DFA state construction; the expression must prevent a second overlapping 11. A proof should test strings such as 111 and 11011, which expose overlap mistakes.
2. Build fragments for 0, 1, union, star, and concatenation recursively, connecting with epsilon edges. Thompson’s construction uses a constant number of states per operator and symbol, so the total is linear in expression length.
3. Track (length parity, number-of-1s parity), giving at most four reachable states. On every symbol, length parity flips; on 1, the second parity flips; on 0, it stays.
4. An omitted transition means the machine has no run on that word, but after swapping final states the absence is not represented as a rejecting run. A sink makes the transition function total and preserves exactly one run per word.
5. Replace an edge labeled a by a path spelling h(a), using fresh intermediate states and epsilon for h(a)=epsilon. Induction on paths shows the new automaton accepts exactly h(L).
6. (0+1)* contains alternating words such as 0101, while 0*+1* contains only all-zero or all-one words. Thus (0+1)* is not equal to 0*+1*.
7. The product explores pairs of states reached by the same prefix. A reachable pair with exactly one accepting component supplies a distinguishing word; no such pair means every word has equal acceptance results.



When a construction is graded, the transition table is often more reliable than a picture. List every state-symbol pair, mark the start and accepting states, and state which invariant proves the table. Then give one accepted and one rejected trace. The combination tests syntax, semantics, and proof rather than relying on visual intuition. For a closure theorem, also identify the exact operation on machines and explain why the resulting accepting condition matches the set operation. This prevents a product construction for union from being accidentally presented as intersection, and prevents an incomplete DFA from being complemented as though it were total.

## Construction audit

For a closure construction, write the invariant before simplifying the diagram. In a product automaton, each component must equal the state reached by the same prefix; in a complement construction, every word must have exactly one run; in a regex-to-NFA construction, epsilon paths must correspond to the expression operator rather than consume symbols. After building the machine, test epsilon, a shortest positive word, a shortest negative word, and words that exercise every operator.
