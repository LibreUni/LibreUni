# Turing Machines and Effective Computation

Finite automata and pushdown automata constrain memory in useful ways. A Turing machine (TM) provides a more general model of effective computation: a finite control reads and writes a two-way unbounded tape and can move in either direction. The model is simple enough for proofs and expressive enough to simulate ordinary programs, interpreters, and other reasonable machine models. This lesson develops the formal semantics carefully, because later decidability and complexity theorems depend on distinguishing halting, acceptance, rejection, and nontermination.

## Formal definition

A deterministic single-tape Turing machine is a tuple (M=(Q,Sigma,Gamma,delta,q_0,q_()accept(),q_()reject())). Q is a finite state set, (Sigma) is the input alphabet, (Gamma) is the tape alphabet containing (Sigma) and a blank symbol (sqcup), and (delta) maps a nonhalting state and scanned tape symbol to a new state, symbol to write, and direction L or R. The start state is q0; accept and reject are halting states. Some definitions omit a transition into either halting state and treat the partial function as undefined there.

The input is written on consecutive tape cells beginning at the head position, with blanks elsewhere. One transition changes at most the scanned cell, moves the head one cell, and changes state. The tape is unbounded in the mathematical model, but every finite computation visits only finitely many cells. A machine may overwrite its input, use delimiters, and store work symbols not in the input alphabet.

A configuration records the complete instantaneous situation: tape contents, head position, and state. It is often written (u q v), meaning the tape has u to the left of the head, state q is active, and v begins at the scanned cell. The one-step relation (vdash_M) is defined by (delta). A computation is a sequence of configurations linked by this relation. A machine accepts w if the initial configuration reaches qaccept; it rejects if it reaches qreject; it loops if it never reaches either.

## Recognition and decision

The language recognized by M is (L(M)=()w: Mtext() eventually enters ()q_()accept()text() on ()w()). Recognition permits nontermination on strings outside the language. A decider is a machine that halts on every input and accepts exactly the language; it must reject every no-instance rather than loop. This distinction is the basis for computability theory.

An algorithmic description must be compiled into transitions or justified by a simulation. Phrases such as “scan to the next separator” are abbreviations for a finite sequence of moves and state changes. The finite control can remember a constant amount of information, while the tape stores unbounded structured data. When proving a machine correct, state the tape invariant after each major phase and identify the halting condition.

**Worked machine sketch.** Decide (L=()0^n1^n:nge0()). Repeatedly find the leftmost unmarked 0, mark it X, scan right to find the leftmost unmarked 1, mark it Y, and return left. When no unmarked 0 remains, scan the remainder to ensure no unmarked 1 exists; accept if none does. Reject if a required 1 is absent or a 1 appears before the process has paired all 0s. The invariant is that marked X and Y symbols form equal pairs and all unmarked symbols retain the order needed for the next search.

The machine halts because every successful round marks one previously unmarked 0, and a failed round rejects. At the final scan, equal counts imply every symbol is marked; unequal counts leave an unmatched symbol and cause rejection. The algorithm may be inefficient, but it is a decider and establishes computability independently of running time.

## Multi-tape and nondeterministic variants

A k-tape TM has k heads and tapes. It can copy, compare, and organize work more conveniently, but it recognizes exactly the same class of languages as a single-tape TM. Simulate the k tapes on one tape using tracks or delimiters; a simulated step scans the encoded tape to locate each head, updates each symbol, and returns to a standard position. The simulation may be slower, but it preserves halting and acceptance.

A nondeterministic TM has several possible transitions. It accepts if some branch accepts, rejects if all branches reject or the chosen definition requires a halting rejecting computation, and may have infinite branches. For decidability equivalence, a deterministic simulator can explore the computation tree breadth-first. Depth-first search can get trapped on an infinite branch before finding a later accepting branch, so fair enumeration is essential.

Nondeterminism does not change computability, but it changes complexity measures. A breadth-first simulator may use much more time than a direct nondeterministic computation. This separation anticipates the P versus NP question: both models recognize the same decidable languages, but polynomial resource bounds may differ.

## Subroutines and composition

Turing-machine proofs routinely use subroutines such as “copy the word,” “compare two blocks,” or “restore the head to the delimiter.” A subroutine specification should state precondition, postcondition, tape region used, head location on return, and whether it halts. Composition is valid only when the postcondition of one satisfies the next precondition.

To simulate a machine N inside M, encode N’s states and tape symbols on M’s tape. A universal machine can receive an encoding of a machine and input, then simulate its transition function. Encoding must be effective: finite states, symbols, and transitions have finite strings, and a decoder can locate the relevant rule. The universal-machine idea explains why programs can be treated as data and underlies diagonalization.

## Configurations as a proof language

Configuration arguments are useful for both correctness and impossibility. For a machine that scans a block, prove that before each iteration the head is at a delimiter, the processed prefix satisfies a predicate, and the remaining suffix is untouched. The transition analysis then checks each symbol case. At halting, the invariant plus the halt condition implies the desired answer.

For simulation, define a relation between source configurations and encoded simulator configurations. Prove a step-simulation lemma: if source C yields C', the simulator transforms an encoding of C into an encoding of C'. Induction gives preservation of whole computations. If the source halts, the simulator halts with the same result; if the simulator halts prematurely, the invariant identifies a simulation bug.

## Encodings and malformed inputs

When a machine accepts encodings of objects, it must specify what happens to malformed strings. A decider for “does encoded machine M accept encoded input w?” should reject or otherwise handle strings that do not decode into a valid pair. Ambiguity in encodings can invalidate a reduction or make a claimed language depend on parsing conventions.

Use self-delimiting encodings or explicit length/separator conventions. A pairing function can encode two strings into one while remaining computably decodable. The exact coding affects time complexity, but computability results are robust under standard effective encodings. It is not legitimate to assume an oracle that recognizes valid encodings unless that recognizer is itself computable or its cost is accounted for.

## Exercises

1. Give a complete transition-level description for a TM that decides strings with equal numbers of 0s and 1s, in any order.
2. State a loop invariant for the (0^n1^n) marking machine and prove termination.
3. Explain how to simulate a two-tape TM on one tape and why the simulation preserves acceptance.
4. Why is breadth-first exploration required to simulate a nondeterministic machine for language recognition?
5. Design a TM subroutine that copies a delimited input block and state its pre/postconditions.
6. Explain why malformed encodings must be specified in a language-of-machines problem.
7. Distinguish recognition, co-recognition, and decision with examples of machine behavior.

### Diagnostic solutions


1. Mark one unprocessed 0 or 1, scan to find an opposite unprocessed symbol, mark it, and return; reject if no opposite exists. When no unprocessed symbols remain, accept. The invariant is that marked symbols form opposite pairs and the unmarked multiset is the remaining input.
2. At the start of each round, all X marks equal all Y marks, the marked pairs correspond to removed 0/1 pairs, and the head is at the left boundary. A successful round marks one new 0 and one new 1, preserving equality. Each round reduces unmarked 0s, so finitely many rounds occur; a final scan halts.
3. Encode both tapes with separators and head markers on one tape. To simulate one source step, scan the encoding to read both marked cells, apply the transition, write symbols and move markers, then return to the canonical scan position. Each source step has a finite simulator sequence, so an accepting source computation yields an accepting simulator computation.
4. An accepting branch may occur at arbitrary finite depth while another branch is infinite. Breadth-first enumeration eventually reaches every finite-depth branch; depth-first search can follow the infinite branch forever and miss acceptance.
5. Preconditions: the head is at the left delimiter and the block contains no work-marker collision. Scan the block, writing a marked copy in a fresh delimited region, then restore or retain the source and return the head to the specified delimiter. The postcondition must include exact copied order and head position.
6. Otherwise the machine may be undefined on strings that are not machines, and a reduction can accidentally rely on a parser that is not part of the model. A total decider must reject or consistently classify malformed encodings.
7. A recognizer halts and accepts yes-instances but may loop on no-instances. A co-recognizer accepts no-instances under an equivalent convention. A decider halts on both and gives both answers. The language of machines that accept their input is recognizable but its complement is not recognizable, as later diagonal arguments show.
