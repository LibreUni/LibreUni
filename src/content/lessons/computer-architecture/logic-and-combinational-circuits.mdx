# Logic and Combinational Circuits

Digital design is the refinement of a mathematical function into a physical network whose outputs become valid after a bounded propagation delay. A *combinational circuit* has no intentional storage: once its inputs have remained stable long enough, its outputs depend only on those inputs. This property distinguishes it from sequential logic, where the circuit’s state records history. The distinction is architectural, not merely terminological: an ALU, decoder, address calculator, and cache tag comparator are combinational components that must produce values within the timing budget of a clocked system.

## Boolean functions as specifications

The simplest specification is a Boolean function $f:\{0,1\}^n\to\{0,1\}^m$. A truth table lists all $2^n$ input combinations and the required output. A circuit is correct when, for every input row, its stable output equals the specified output. This exhaustive definition is practical for small circuits and becomes the semantic foundation for symbolic simplification, formal equivalence checking, and test generation for larger ones.

The primitive operations are NOT ($\lnot x$), AND ($x\land y$), and OR ($x\lor y$). NAND and NOR are *functionally complete*: every Boolean function can be built from either one alone. For example, $\lnot x=x\operatorname{NAND}x$, $x\land y=\lnot(x\operatorname{NAND}y)$, and $x\lor y=(\lnot x)\operatorname{NAND}(\lnot y)$. Functional completeness is an expressiveness result; it does not imply that a NAND-only implementation is fastest, smallest, or easiest to verify.

Boolean algebra supports local transformations that preserve a function. Important identities include commutativity, associativity, distributivity, identity ($x\land1=x$, $x\lor0=x$), domination ($x\land0=0$, $x\lor1=1$), idempotence ($x\land x=x$), complement ($x\land\lnot x=0$), absorption ($x\lor(x\land y)=x$), and De Morgan’s laws:

$$
\lnot(x\land y)=\lnot x\lor\lnot y,
\qquad
\lnot(x\lor y)=\lnot x\land\lnot y.
$$

These are equations over truth values, so each transformation can be justified for all inputs. A designer should prefer a proof-preserving transformation to an intuition such as “this gate seems redundant.”

**Worked derivation.** Simplify $F=xy+x\bar y+\bar x y$, where juxtaposition denotes AND and `+` denotes OR. First factor the first two terms: $xy+x\bar y=x(y+\bar y)=x$. Therefore $F=x+\bar x y$. Apply absorption in its dual form, $x+\bar x y=x+y$, which can be verified by distributivity: $(x+\bar x)(x+y)=1\cdot(x+y)=x+y$. The original three-product circuit can therefore be implemented by one OR gate after the appropriate inputs are available. The simplification is a theorem about the function, not an approximation.

## Canonical forms and minimization

Every Boolean function can be written as a sum of products (SOP). For each truth-table row where $F=1$, form a *minterm* that is 1 only on that row, then OR the minterms. Likewise, a product of sums (POS) uses maxterms for rows where $F=0$. Canonical forms are systematic and easy to derive, but they may contain many literals and gates.

Consider a three-input function that is 1 for inputs `001`, `010`, and `111`. With variables $a,b,c$ in that order, its SOP is $\bar a\bar b c+\bar a b\bar c+abc$. If the function has don't-care input combinations—states that never occur or whose output does not matter—those rows may be assigned whichever value produces a cheaper implementation. A don't-care is not permission to ignore a reachable safety-critical state; it is a specification statement that must be defended by the surrounding design.

Karnaugh maps expose adjacent minterms that differ in one variable. Grouping powers of two allows the differing variable to disappear. For larger functions, Quine–McCluskey or binary decision diagrams provide systematic alternatives, while synthesis tools use cost models and technology libraries. “Minimal number of terms” and “minimal delay” are different objectives: factoring may reduce area but add a logic level; duplicating a term may reduce fanout and improve timing.

## Multiplexers and decoders

A 2-to-1 multiplexer is the conditional operator of hardware:

$$
M(s,a,b)=\bar s a\lor sb.
$$

When $s=0$, the first product survives and $M=a$; when $s=1$, the second survives and $M=b$. A $2^k$-to-1 multiplexer can select one of $2^k$ inputs using $k$ select bits. Multiplexers are powerful because a data path can be described as “choose this value when the control predicate holds” rather than drawn as a separate circuit for every case.

A decoder performs the inverse-shaped task: $k$ input bits activate one of $2^k$ outputs. A 2-to-4 decoder with inputs $a,b$ generates minterms $\bar a\bar b$, $\bar a b$, $a\bar b$, and $ab$. Decoders construct register-file enables, memory chip selects, instruction subfields, and one-hot state encodings. A one-hot representation uses one state bit per state, increasing storage but often simplifying next-state logic and making illegal-state detection explicit.

**Worked construction.** Implement a 4-to-1 multiplexer from three 2-to-1 multiplexers. First select between $d_0$ and $d_1$ using $s_0$, and between $d_2$ and $d_3$ using $s_0$. Select those two intermediate results using $s_1$. The first layer has two parallel gates and the second layer one gate, so the data path has two multiplexer delays. A balanced tree is preferable to chaining three selections because its depth is logarithmic in the number of inputs.

## Adders and arithmetic structure

The half-adder adds two bits. Its sum is $s=a\oplus b$ and carry is $c=ab$. A full adder adds $a$, $b$, and carry-in $c_{in}$:

$$
s=a\oplus b\oplus c_{in},
\qquad
c_{out}=ab+ac_{in}+bc_{in}.
$$

The carry expression says at least two of the three inputs are 1. It can also be written $ab+c_{in}(a\oplus b)$, separating the generate case from the propagate case. Chaining $w$ full adders creates a ripple-carry adder. Its area is linear, but the carry may traverse every bit, so worst-case delay is $O(w)$ gate stages.

The carry-propagate structure motivates a carry-lookahead adder. Define per bit $g_i=a_ib_i$ and $p_i=a_i\oplus b_i$. Then $c_{i+1}=g_i+p_ic_i$. Expanding gives $c_2=g_1+p_1g_0+p_1p_0c_0$, and so on. Parallel prefix structures compute these equations with lower depth at greater wiring and area cost. This is a recurring architecture trade-off: asymptotic latency improves by investing physical resources and design complexity.

Subtraction uses the identity $a-b=a+\lnot b+1$ in two’s complement. A single adder can therefore add either $b$ or $\lnot b$ selected by a control signal, with the same signal supplied as the initial carry-in. The ALU must define flag semantics: zero, negative, carry/borrow, and signed overflow are not interchangeable. A branch instruction that tests unsigned less-than uses a different interpretation from one that tests signed less-than even when both consume the same subtractor outputs.

## Comparators and predicates

Equality of two $w$-bit words is the AND of bitwise equivalences:

$$
eq(a,b)=\bigwedge_{i=0}^{w-1}\lnot(a_i\oplus b_i).
$$

An equality tree reduces delay compared with a serial chain. Unsigned less-than can be determined by the most significant bit at which operands differ. A lexicographic comparator therefore scans from high bits conceptually, although hardware uses a tree or prefix formulation. Signed comparison treats the sign bit specially: a negative operand is less than a nonnegative operand; if signs match, the remaining bits can be compared as an unsigned magnitude under the two’s-complement ordering.

Predicates are often more valuable than numeric results. The ALU’s output may be discarded while its zero or sign condition controls a branch, a memory-protection check, or an instruction exception. The control contract must say whether a comparison is signed, unsigned, exact, masked, or tolerant of invalid encodings.

## Timing, hazards, and physical reality

Boolean equivalence describes the stable function, but a physical circuit also has propagation delay. If an output feeds another gate before it has settled, the receiving logic can observe a transient value. The longest input-to-output path is the critical path and bounds the maximum clock frequency when the circuit lies between registers. Gate count is only a rough area proxy; fanout, wire length, loading, and technology-specific delays matter.

Unequal path delays can produce a *glitch*: an output briefly changes even though the logically correct value before and after the transition is the same. For example, in $F=ab+\bar ac$, changing $a$ while $b=c=1$ should leave $F=1$, but the direct and inverted paths may not switch simultaneously, producing a temporary zero. Adding the consensus term $bc$ yields a hazard-resistant two-level form for this transition, at the cost of an extra product. Whether a glitch matters depends on the receiver: a level-sensitive combinational consumer may ignore it after settling, while an asynchronous control input or clock edge can capture it.

Static timing analysis uses arrival and required times to verify that paths meet setup constraints. The logic designer cannot safely assume zero delay merely because a truth table is correct. Conversely, adding arbitrary “delay gates” is not a robust fix; synthesis may remove them, process variation may change their delay, and the real solution may require registering the signal or changing the protocol.

## CMOS and implementation cost

At the transistor level, complementary CMOS uses networks of p-type and n-type devices to implement logic with low static power in steady states. A NAND gate is naturally efficient: the pull-down network places nMOS devices in series for the AND condition, while the pull-up network is the dual parallel structure. A NOR gate has the opposite topology and often different delay characteristics because series and parallel transistor arrangements load differently.

The same Boolean expression can map to different physical costs. Factoring reduces repeated logic but may increase fanout; a NAND-NAND implementation can exploit the technology’s native gate; a balanced tree reduces depth but increases wiring. Modern synthesis tools optimize against timing, area, power, and testability objectives. Architectural reasoning remains necessary because the tool only optimizes within the constraints and library the designer supplies.

## Verification strategy

For a small combinational block, exhaustive simulation covers all inputs. For a 32-bit ALU, exhaustive testing is impossible, so verification combines equivalence checking, constrained random tests, directed corner cases, and compositional proofs. Directed cases should include all-zero and all-one operands, one-hot patterns, sign transitions, carry chains, alternating bits, and illegal control encodings.

An assertion expresses a property such as “if operation is ADD, then result plus a widened carry equals the mathematical sum” or “if select is 0, output equals input A regardless of input B.” Assertions turn the specification into an executable proof obligation. A test that observes only the common path can miss a carry that propagates through every bit or a mode field that selects the wrong input.

## Independent practice

1. Derive a full-adder circuit from two half-adders. Give the equations for the intermediate carry and prove that the final carry equals $ab+ac_{in}+bc_{in}$.
2. Minimize $F(a,b,c,d)=\Sigma m(0,2,5,7,8,10,13,15)$ and identify any symmetry. Verify the expression on at least one minterm and one zero row.
3. Build an 8-to-1 multiplexer from 2-to-1 multiplexers. Give the number of gates and logic depth for a balanced tree and a serial chain.
4. Explain why a ripple-carry adder has a data-dependent delay even though its topology is fixed. Give an input pattern that propagates carry through every bit.
5. A comparator receives signed two’s-complement operands. Prove that comparing sign bits first and magnitudes second yields the correct order, including the minimum negative value.
6. Construct a transition that can glitch in $ab+\bar ac$. Why does adding $bc$ cover the static-1 hazard? What timing assumption makes the term useful?
7. Propose an assertion set for a 4-bit ALU with ADD, SUB, AND, OR, and XOR. Include one property about unused control codes.

A correct solution may use XOR/XNOR as a primitive or expand it into SOP. Check `0000` (1), `0001` (0), and a paired edge such as `1010` (1).
3. A balanced tree uses seven 2-to-1 multiplexers and depth three. A serial chain also uses seven gates but has depth seven and would require a different select-routing scheme; the balanced tree meets a shorter critical path.
4. Each full adder waits for the incoming carry. An operand pair such as `01111111` plus `00000001` generates a carry at the low bit and propagates it through all higher positions, exposing the linear chain delay.
5. If signs differ, every negative two’s-complement value is less than every nonnegative value, so the sign bit decides. If signs match, subtracting the common sign weight leaves an ordering by the lower bits; for two negative operands, the lower-bit ordering is reversed relative to unsigned magnitude, which the signed comparator’s subtract/overflow logic accounts for. The minimum value is not negatable in range, so a proof must not assume symmetric magnitude bounds.
6. Set $b=c=1$ and change $a$ from 0 to 1. The function should stay 1, but $ab$ may fall before $\bar ac$ rises. The consensus term $bc$ remains 1 throughout, covering the static-1 gap. It helps only when the input transition is single-variable or the timing environment makes the consensus reasoning applicable.
7. For each opcode, assert the output equation. For ADD and SUB, use widened arithmetic to relate result, carry, and operands; for bitwise operations assert bitwise equality. For unused codes, specify a safe result and side-effect behavior—such as no write enable—rather than leaving the output unconstrained.
