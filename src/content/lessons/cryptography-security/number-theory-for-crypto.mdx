# Number Theory for Cryptography

Public-key constructions rely on operations that are easy to compute and hard to invert under a stated assumption. Prime fields provide modular arithmetic with inverses; cyclic groups provide repeated exponentiation and discrete-logarithm assumptions.

Security claims are parameterized: a small modulus can make factoring easy, and an implementation may leak a key even when the mathematical problem is hard.

## Primes and factorization

An integer greater than one is prime when its only positive divisors are 1 and itself. The fundamental theorem of arithmetic gives a unique factorization into primes. Trial division up to the square root proves small examples but is not a practical general factorization algorithm at cryptographic sizes. Probabilistic primality tests such as Miller–Rabin identify composites with controllable error; production generation combines tested rounds, parameter selection, and sometimes a proof or certificate appropriate to the standard.

For an odd candidate $n$, write $n-1=2^s d$ with $d$ odd. A Miller–Rabin round chooses a base $a$ and checks whether $a^d$ is $1$ or one of the subsequent squares is $-1$ modulo $n$. If neither occurs, $n$ is composite. The proof that primes pass follows from the structure of the multiplicative group modulo a prime. A composite can pass a round, so state the number and independence of bases or use a deterministic bound for the permitted range.

Prime generation must use a cryptographically secure random source. Testing predictable candidates or using a weak seed can make the factorization of an apparently large modulus trivial. Reject repeated or structurally related primes according to the scheme’s requirements. Store enough metadata to audit generation without storing secret prime material in logs.

## Groups and discrete logarithms

A group has an associative operation, identity, and inverse for every element. A cyclic group has a generator $g$ such that every element is $g^x$ for some exponent $x$. The discrete-log problem asks for $x$ given $g$ and $g^x$. Diffie–Hellman schemes depend on the difficulty of computing a shared value or distinguishing it from random under specified group assumptions.

The group order and subgroup structure are security parameters. In a group with composite order, an element may generate a small subgroup. A malicious public value can force an honest party to expose information about a secret exponent modulo that subgroup order. Use a documented prime-order group or validate membership and cofactor behavior. The identity element should generally be rejected as a peer public value unless the protocol proof explicitly handles it.

The generic baby-step giant-step algorithm solves a discrete log in roughly $\sqrt q$ time and memory. Pollard’s rho reduces memory while retaining $\sqrt q$ expected work and can be parallelized. These attacks motivate group sizes: a $256$-bit prime-order elliptic-curve group offers a different security level from a $256$-bit finite-field modulus because the best known generic attacks differ.

## Quadratic residues and RSA

An integer $a$ is a quadratic residue modulo an odd prime $p$ if $x^2\equiv a\pmod p$ for some $x$. The Legendre symbol records whether $a$ is a nonzero residue, nonresidue, or zero. Euler’s criterion says $a^{(p-1)/2}\equiv \left(\frac{a}{p}\right)\pmod p$. These facts support reasoning about square roots, primality tests, and some cryptographic assumptions.

For a composite $N=pq$, the Jacobi symbol can be computed without knowing $p$ and $q$, but a value with Jacobi symbol $1$ need not be a quadratic residue. This gap is useful in some protocols but dangerous if an implementation assumes the symbol proves existence of a root. CRT lets an entity with $p$ and $q$ compute roots modulo each prime and recombine them; an entity without the factors may face the quadratic residuosity problem.

RSA correctness uses the exponent relation $ed\equiv 1$ modulo the appropriate group orders. For messages coprime to $N$, Euler’s theorem is enough. For all message representatives in a properly formed RSA setting, reason modulo $p$ and $q$ and recombine with CRT. Security requires randomized and domain-separated encodings; the raw equation $c\equiv m^e\pmod N$ is malleable and deterministic.

## Elliptic-curve groups

For a short Weierstrass curve $y^2=x^3+ax+b$ over a finite field, points together with the point at infinity form a group under a geometric chord-and-tangent law when the discriminant is nonzero. The affine formulas use field inverses and have exceptional cases; projective coordinates represent the same point with scale equivalence and avoid inversions during repeated additions.

Security depends on the subgroup generated by the base point, its order, the cofactor, and the absence of exploitable structure. The curve equation alone is not a complete parameter set. Point encodings must be canonical or explicitly accepted according to the protocol. Invalid-curve and small-subgroup attacks exploit implementations that perform scalar multiplication on values outside the intended group.

Scalar multiplication uses double-and-add, a ladder, or a window method. The mathematical operation count is not enough: branch patterns and memory access can reveal scalar bits. Constant-pattern algorithms, complete formulas where available, and input validation reduce risk. Public scalar multiplication may tolerate different performance tradeoffs from secret scalar multiplication, but the distinction must be documented.

## Complexity assumptions and reductions

A cryptosystem’s security statement should say: under assumption A, in group or encoding model M, no adversary with resource bound R wins game G with probability greater than epsilon. If an algorithm reduces breaking the scheme to factoring, the reduction may lose a factor in time or probability; report the loss. A heuristic claim based on no known attack is weaker than a reduction, and a reduction to an assumption not understood at the selected parameters is not a guarantee.

## Worked complexity comparison

Suppose a group has prime order $q$. Brute-force discrete-log search tests exponents one by one and takes $O(q)$ group operations. Baby-step giant-step chooses $m\approx\sqrt q$, stores $g^0$ through $g^{m-1}$, and searches for a collision between a target times $g^{-jm}$ and a table entry. The equality gives the exponent $i+jm$. Memory is $O(\sqrt q)$, and time is $O(\sqrt q)$. Pollard rho instead walks through a pseudorandom partition of the group and uses a collision to obtain an equation in the unknown exponent. Its expected time is also $\sqrt q$ but memory is small; parallel walks require distinguished-point coordination.

This comparison illustrates parameter selection. A group order of $2^k$ gives only about $k/2$ bits against generic discrete-log attacks. Increasing the field representation length does not automatically increase the group order or security. Protocol documentation must identify the subgroup order, the best known generic and specialized attacks, and the intended security level.

## Algebraic proof practice

When proving a cryptographic identity, state the domain of every variable and check every cancellation. Fermat’s theorem requires a prime modulus and a nonzero base; Euler’s theorem requires coprimality; cancellation modulo a composite may be invalid when the cancelled element is not invertible. A computational hardness claim is a separate statement quantified over efficient adversaries. Keeping these categories separate prevents a correct algebraic derivation from being mistaken for a security proof.
