# Balanced Search Trees

A binary search tree (BST) supports ordered dictionaries by maintaining a local ordering invariant: every key in a node’s left subtree is smaller and every key in its right subtree is larger, under a stated duplicate policy. Search follows one root-to-leaf path, so its cost is proportional to height. An unbalanced insertion sequence can make height n; balancing restores logarithmic worst-case behavior while preserving sorted order.

## BST invariant and operations

For a node x with key k, define lower and upper bounds inherited from ancestors. A valid BST has every descendant key within those bounds. Search compares the target with the current key and descends left or right. Insert follows the same path to an allowed empty child. Inorder traversal yields keys in sorted order; prove this by induction on subtree structure.

Delete has three cases. A leaf is removed directly. A node with one child is replaced by that child. A node with two children is replaced by its inorder successor (minimum of the right subtree) or predecessor, then that simpler node is deleted. Copying the successor’s key/value instead of moving the node preserves the ordering invariant, but payload identity and external references must be considered.

The height recurrence explains performance. A path-shaped BST has h=n-1, making search and update O(n). If every subtree splits evenly, (h=O(log n)). Balancing algorithms do not require perfect balance; they maintain a local condition strong enough to guarantee a logarithmic global height.

## Rotations

A rotation changes local pointers while preserving inorder sequence. A right rotation around y with left child x makes x the new root of the local subtree, y its right child, and x’s former right subtree y’s left subtree. The keys remain in sorted inorder because the middle subtree lies between x and y. A left rotation is symmetric.

**Worked rotation.** Suppose y has key 30, left child x=20, x’s right subtree contains keys 25, and y’s right subtree contains 40. A right rotation yields 20 as local root, 30 as its right child, 25 as 30’s left child, and 40 remains to the right. Inorder is 20,25,30,40 before and after. A rotation changes height and parent links but not dictionary order.

Parent pointers simplify upward rebalancing but add an RI: child.parent must equal the containing node, and the root.parent must be null or a sentinel. Without parent pointers, recursive return values must provide the new subtree root. Mixing conventions leads to lost subtrees or cycles.

## AVL trees

An AVL tree maintains balance factor (bf(x)=height(left(x))-height(right(x))) in ()-1,0,1() for every node. Store height or derive it recursively; deriving repeatedly can turn an update into O(n), so cached height is part of the representation. After insertion, walk from the changed leaf toward the root, update heights, and rotate the first violated node using one of four cases: left-left, right-right, left-right, right-left.

The minimum number (N(h)) of nodes in an AVL tree of height h satisfies (N(h)ge1+N(h-1)+N(h-2)), with base cases. This Fibonacci-like recurrence implies (N(h)) grows exponentially in h, so h=O(log n). The proof is a global consequence of the local balance invariant. Deletion may cause multiple ancestors to require rebalancing, but still performs O(log n) rotations and updates.

## Red-black balance

A red-black tree colors nodes red or black and maintains conditions such as: the root is black; red nodes have black children; every path from a node to a descendant null leaf has the same black height. These rules permit a little more imbalance than AVL but ensure logarithmic height. A standard proof shows the shortest root-to-leaf path has at least half the black nodes of the longest, and no root-to-leaf path contains more red nodes than black nodes; therefore (hle2log_2(n+1)) up to convention.

Insertion and deletion use recoloring and rotations to restore invariants. The exact cases are easier to verify with a state diagram, but the proof obligations remain local: preserve BST order, restore color conditions, and preserve black height. Red-black trees often offer fewer rotations under updates, while AVL trees can offer tighter search height. Both provide worst-case O(log n) dictionary operations.

## Ordered operations

Balanced BSTs support more than exact lookup. Minimum and maximum follow extreme paths; predecessor and successor use child subtrees or ancestor relationships; range reporting finds a split point and traverses only relevant subtrees plus a logarithmic search overhead. Augment a node with subtree size to support order statistics: select the kth smallest by comparing k with the left subtree size, and rank a key by accumulating sizes while descending.

Augmentation requires an invariant for the extra field and update rules under rotations. For subtree size, (size(x)=1+size(left)+size(right)). A rotation changes only a constant number of nodes, so recompute those sizes bottom-up. An augmented field that is not updated during every mutation can silently return plausible but incorrect answers.

## Deletion and rebalancing discipline

Deletion is where a proof must account for more than insertion intuition. In an AVL tree, replacing a two-child node by its successor preserves the BST order, but the physical deletion occurs in the successor’s subtree and can reduce its height. Walk upward recomputing height and rebalance every affected ancestor. Stopping after the first rotation is not generally justified for deletion because a height reduction can propagate to the root.

In a red-black tree, deletion can remove a black node and create a black-height deficit. The repair cases move, recolor, or eliminate the deficit while preserving the inorder sequence. A sentinel null leaf simplifies the case analysis, but it is a real participant in the color and black-height invariant. Implementations that treat null as “not a node” in one case and as a black leaf in another often produce subtle violations.

For any balanced tree implementation, use a validation routine that checks ordering bounds, parent links, cached heights or colors, subtree sizes, and root conditions. Run it after every mutation in tests. Validation is not the proof, but it localizes which operation first violated the invariant and makes rotation debugging tractable.

## Choice among ordered structures

AVL trees keep tighter height and can favor read-heavy workloads. Red-black trees tolerate more structural slack and often bound update rotations well. Treaps assign random priorities and maintain heap order by rotations, giving expected logarithmic height without deterministic balance cases. B-trees widen nodes and reduce height and I/O operations, making them appropriate for external memory. The abstract ordered-map operations are similar, but memory hierarchy and failure model determine the right representation.

When comparing structures, state whether the bound is worst-case or expected, whether keys are adversarial, and whether the cost of comparison itself is constant. If keys are strings of length (ell), a comparison can cost O((ell)), so a tree’s O(log n) comparisons do not imply O(log n) character work. This distinction is important in real dictionaries and indexing systems.

Always state the input model explicitly in the specification.

4. List AVL rotation cases for an insertion and explain balance restoration.

### Diagnostic solutions


1. By induction, inorder(left) is sorted and less than the root key, inorder(right) is sorted and greater, so concatenating left, root, right is sorted. The empty tree is the base case.

2. Each new key is larger than all existing keys and descends right until a null child. The result is one node per level, height n-1, so search and updates can be linear.

3. Let A be x’s left subtree, B x’s right subtree, and C y’s right subtree with keys A less than x less than B less than y less than C. Before inorder is A,x,B,y,C; after rotation it remains A,x,B,y,C. Only local pointers change, and parent/root links must be repaired.

4. LL uses one right rotation, RR one left rotation, LR left rotation at child then right at grandparent, and RL the mirror. Recompute child heights after rotations; the resulting local subtree has balance factors within the invariant.

5. Let N(h) be minimum nodes. (N(h)ge1+N(h-1)+N(h-2)), so N grows at least as fast as Fibonacci numbers, exponentially in h. Thus nu2265cu03c6^h for constants c,u03c6 greater than 1, giving h=O(log n).

6. If black height is b, a shortest path has b black nodes. The red-child rule means a longest path has at most 2b nodes. Since a tree of black height b contains at least (2^b-1) internal nodes under standard conventions, b=O(log n), and hu22642b.
7. Store size(x)=1+size(left)+size(right). After a right rotation, recompute the old root first from its new children, then the new root from its children. The formula is preserved and rank/select follows by subtree sizes.



## Synthesis checkpoint

An implementation review should separate the representation invariant from the rebalancing case analysis. After every rotation, verify ordering, parent links, subtree metadata, and the height or color condition before discussing asymptotic cost. A proof that only names the rotation is incomplete because stale metadata can silently invalidate later queries.
