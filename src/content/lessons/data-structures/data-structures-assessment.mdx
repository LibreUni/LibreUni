# Data Structures Comprehensive Assessment

This assessment treats a data structure as an abstract contract plus a concrete representation. For every answer, state the model, assumptions, duplicate and error policy, and whether a bound is worst-case, amortized, or expected. A correct complexity label without the invariant that justifies it is incomplete. A correct invariant without an abstraction function does not establish user-visible behavior.

## Part I: abstraction and sequences

1. Define an abstract sequence ADT with indexed read, update, insertion, deletion, iteration, and size. Specify invalid indices and mutation during iteration.
2. Give a representation invariant and abstraction function for a dynamic array with capacity C and logical size n. Prove append in the spare-capacity case and resize case.
3. Compare arrays, singly linked lists, and doubly linked lists for random access, front insertion, known-node insertion, memory overhead, and locality.
4. A circular queue stores head, tail, count, and capacity. State the modular invariant and explain how it distinguishes empty and full.
5. Prove that a two-stack queue has O(1) amortized operations using a potential function.
6. A client receives a mutable slice of an internal array. Give a counterexample to representation independence and a safe API alternative.

## Part II: dictionaries and order

7. State a chaining hash-table invariant and explain lookup correctness under collisions.
8. Why are tombstones needed in open addressing? Trace a false negative caused by clearing a slot.
9. Explain load factor, resizing, and the probability assumption behind expected O(1) hashing.
10. Define a BST bound invariant and prove insertion preserves it.
11. Trace deletion of a two-child BST node and prove successor replacement preserves order.
12. Compare unbalanced BST, AVL, red-black, treap, and B-tree guarantees and memory models.
13. Design subtree-size augmentation for rank/select and show how a rotation updates it.
14. A trie stores prefixes and terminal keys. Define exact lookup, prefix enumeration, and safe pruning.

## Part III: priority and connectivity

15. Define the binary-heap shape/order invariant and prove sift-down after delete-min.
16. Prove bottom-up heap construction is O(n), not O(n log n).
17. Explain indexed priority queues and the map/array bijection invariant.
18. State union-find partition semantics, union-by-size proof, and path-compression guarantee.
19. Why cannot ordinary union-find support arbitrary edge deletion? Describe rollback union-find for offline dynamic connectivity.
20. Compare adjacency matrix, adjacency list, CSR, and edge list by space and primitive costs.

## Part IV: complexity and memory

21. Distinguish worst-case, amortized, expected, and average-case analysis with one data-structure example each.
22. Explain cache locality and why two O(n) traversals can differ substantially.
23. Design a benchmark that measures a data-structure operation without timing input generation or allowing dead-code elimination.
24. Explain stable sorting, comparison lower bounds, quickselect, and a domain-specific non-comparison sort.
25. A persistent tree update changes an old version. Identify the violated principle and path-copy repair.
26. Give a test strategy that combines a reference model, invariant validator, property tests, and adversarial workloads.

## Part V: integrated debugging cases

27. A hash map passes uniform random tests but times out in production. What evidence and repairs do you seek?
28. A balanced tree returns sorted inorder output but rank is wrong after rotations. Which invariant and update rule are missing?
29. A queue fails only when capacity is a power of two. What modular-index and full/empty bugs are plausible?
30. A persistent map retains excessive memory after old snapshots are released. Analyze roots, reclamation, and measurement.
31. A graph traversal visits a vertex twice after a repeated run. Explain stale metadata and a generation-based fix.
32. A rollback dynamic-connectivity solver reports connections in sibling time ranges. Identify the temporal leak.

## Diagnostic solutions and grading guidance

1. Define sequence value as ordered elements; get/set require 0u2264i less than n; insertion allows 0u2264iu2264n; removal requires valid i; state iterator invalidation or snapshot behavior. All operations must specify ownership and errors.
2. RI: 0u2264nu2264C and elements occupy prefix [0,n); AF is that prefix. Spare append writes at n and increments. Resize allocates C' greater than n, copies the prefix in order, publishes it, writes x, and sets n+1; old state remains valid until commit.
3. Arrays give O(1) indexing and locality but shifts for middle insertion; singly linked lists give O(1) front and known-node splicing but O(n) indexing; doubly linked lists support O(1) known-node deletion with two pointers at extra space. Bounds condition on a known node versus an index.
4. Logical element j is at (head+j) mod C for 0u2264j less than count; tail=(head+count) mod C; 0u2264countu2264C. Empty iff count=0, full iff count=C. Enqueue writes at tail and increments; dequeue reads head and decrements.
5. Set potential to twice the number of elements in input stack. Enqueue has constant actual cost and raises potential by two. Transfer of k costs 2k and lowers potential by 2k; dequeue is constant. Nonnegative potential proves constant amortized cost.
6. The client can mutate an element without set or write beyond logical size, violating the RI. Return a copy, immutable view, or controlled iterator; document ownership if sharing is intentional.
7. Every live pair is in exactly one bucket h(k), equality has no duplicate ambiguity, and size counts pairs. Lookup computes the same bucket and scans it; collision changes cost, not correctness.
8. A key inserted after a collision may lie beyond an earlier slot. Clearing the earlier slot makes lookup stop and falsely report absence. Tombstones mean “continue searching, but reusable for insertion.”
9. u03b1=n/m is average occupancy. Under uniform or universal hashing and bounded u03b1, expected chain/probe cost is constant. The bound is not worst-case and can fail under concentration or adversarial keys.
10. Every subtree obeys inherited lower/upper key bounds. The search path narrows the interval; the new leaf is placed in the final valid interval, preserving all ancestor constraints.
11. Replace a two-child node by the minimum of its right subtree, then delete that successor, which has no left child. All left keys remain smaller and all remaining right keys larger.
12. Unbalanced BST is O(h), worst-case linear; AVL and red-black are worst-case logarithmic with different balance/update tradeoffs; treaps are expected logarithmic under random priorities; B-trees reduce block I/O with wide nodes.
13. Store size(x)=1+size(left)+size(right). Select compares rank with left size. After rotation recompute the old root then new root; all unaffected subtrees retain their sizes.
14. Follow symbols and test terminal for exact lookup. For a prefix, traverse then enumerate descendants; output complexity includes results. Prune only nonterminal childless nodes while moving upward.
15. Complete-tree array layout plus parentu2264children. After root replacement, child subtrees are heaps and only the root path can violate order; swap with the smaller child until fixed.
16. At height h there are at most n/2^()h+1() nodes and each costs O(h); u03a3h/2^h is constant, giving O(n).
17. The item-to-index map and array contents are inverse bijections. Every swap updates both entries. Decrease-key locates an item, changes priority, and sifts up.
18. Parent forests represent partition classes. Union-by-size doubles the root size whenever a node’s depth rises, so height O(log n). Path compression preserves roots and together gives O(u03b1(n)) amortized operations.
19. Parent pointers discard enough edge history to know whether a deleted edge was a bridge. Offline segment-tree intervals plus rollback union-find add active edges on recursion paths and undo them at exit.
20. Matrix uses u0398(V^2), O(1) edge lookup, O(V) neighbor scan; lists use O(V+E), O(degree) scan; CSR is compact and scan-friendly but update-heavy; edge lists use O(E) and favor global scans.
21. Worst-case bounds every call; amortized bounds every sequence; expected bounds average over specified randomness; average-case may assume an input distribution. Dynamic-array append is amortized, hash lookup is conditional expected, and BST search has worst-case height.
22. Arrays fetch adjacent values in cache lines and can be prefetched; pointer chasing may miss per node. Both perform n logical steps, but transfer and latency costs differ.
23. Pre-generate inputs, consume outputs via checksum, warm up separately, randomize repetitions, record environment, report distributions, and validate against a model outside the timed core or in a controlled phase.
24. Stable sorting preserves equal-key order; comparison sorts require u03a9(n log n) comparisons; quickselect finds kth element in expected linear time; counting/radix sort exploit bounded representations and avoid the comparison lower bound.
25. A shared node was mutated. Copy every node on the update path, reuse untouched subtrees, and publish a new root; old roots must never reach a write.
26. Differential reference model, per-operation invariant checks, property/metamorphic relations, boundary cases, adversarial distributions, reproducible seeds, and performance tests separated from correctness.
27. Measure bucket/probe distributions, key patterns, load/resizes, and hash seed. Use keyed/universal hashing, bucket trees, lower thresholds, rate limits, or a fallback; expected random tests did not model production.
28. Recompute subtree sizes after every rotation and update path. Inorder can remain correct while cached augmentation is stale; validate size equations recursively.
29. Overflow/masking may confuse capacity with index range; head=tail can mean both empty and full if count is not used consistently; modulo updates may skip the final slot. Test empty, full, wrap, and resize transitions.
30. Old roots or retained handles still reach shared nodes; garbage collection cannot reclaim reachable data. Track root ownership, release snapshots, compact versions, and measure retained graph size rather than allocation alone.
31. Visited marks from the previous run were not cleared or scoped. Store a generation number and compare against the current traversal token, or reset all marks before starting.
32. A union made in one segment-tree branch was not rolled back before entering its sibling. Snapshot the history length on entry and pop exactly to it on exit; test nested branches against a brute-force model.
