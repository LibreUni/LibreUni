# Stacks, Queues, and Deques

Stacks and queues are restricted sequence ADTs. A stack exposes last-in-first-out (LIFO) behavior; a queue exposes first-in-first-out (FIFO); a deque permits insertion and removal at both ends. Restricting operations removes irrelevant states and enables simpler proofs and efficient implementations. The structures appear in parsing, recursion, breadth-first search, buffering, scheduling, and undo systems.

## Stack specification

A stack stores a finite sequence with operations push(x), pop(), peek(), and empty or size. Push places x at the top; pop removes and returns the most recently pushed element that has not been popped. If the stack is empty, pop and peek must have a specified result such as an option, exception, or blocking wait. The LIFO law can be stated algebraically: after pushing x, peek returns x and a subsequent pop restores the previous stack.

An array stack uses a logical size n and stores elements in positions 0 through n-1, with the top at n-1. Push writes at n and increments; pop decrements and returns the former last element. Both are O(1) amortized with a dynamic array. A linked stack pushes and pops at the head in O(1) worst case, but allocates nodes and has pointer locality costs. Neither implementation changes the abstract sequence.

**Invariant proof.** For an array stack, RI is (0le nle C) and the abstract stack equals the array prefix in bottom-to-top order. Push when n less than C writes x at position n; the new prefix is the old prefix followed by x. Pop when n greater than 0 returns position n-1 and decrements; the new prefix is the old prefix without its last element. Resize preserves the prefix by copying it unchanged.

Stacks support nested structure. A delimiter checker scans characters, pushes opening delimiters, and on a closing delimiter checks that the top has the matching kind before popping. At end, the stack must be empty. The invariant is that the stack is exactly the sequence of unmatched opening delimiters in input order. This proves correctness for balanced delimiters with properly nested types.

## Queue specification

A queue supports enqueue(x), dequeue(), front(), empty, and possibly size. Enqueue appends at the rear; dequeue removes the oldest element. The FIFO law says that after enqueuing x1,...,xk into an empty queue, repeated dequeue returns x1,...,xk. A queue may be bounded; then enqueue on full must reject or block, and the capacity is part of semantics.

A naïve array queue that dequeues by shifting every element left has O(n) dequeue. A circular buffer stores head, tail, count, and capacity. The q logical elements occupy modular positions from head through q-1 positions; tail is the next write location. Enqueue and dequeue change indices modulo capacity, giving O(1) worst-case when full/empty behavior is handled.

The count field distinguishes full from empty when head equals tail. An alternative leaves one slot unused, making head=tail empty and the next position equal to head full. Both designs are correct; mixing their invariants causes off-by-one errors. Resizing a circular buffer must copy logical order into a fresh linear layout, set head=0, and set tail=q, not copy physical indices blindly.

## Two-stack queue

A queue can be implemented with an input stack and an output stack. Enqueue pushes onto input. Dequeue pops output; if output is empty, move every element from input to output, reversing order. The oldest element then lies on top of output. A single operation can cost O(n), but each element moves from input to output at most once before being dequeued, so a sequence of m operations costs O(m) and dequeue is O(1) amortized.

The potential (Phi=2|input|) pays for transfers. An enqueue costs one push and increases potential by 2; a transfer of k elements costs O(k) and drops potential by 2k, covering the movement. A dequeue from nonempty output has constant actual cost and no harmful potential increase. This is a concrete example of amortization through stored work rather than an average-case probability assumption.

## Deques

A deque supports push-front, push-back, pop-front, and pop-back. A circular array supports all four in O(1) amortized or worst-case depending on resize policy. A doubly linked list with sentinels supports all four in O(1) worst case, given endpoint pointers. The tradeoff is locality and metadata: arrays compact elements, links avoid shifting and can splice nodes.

Deque semantics are useful for work-stealing and sliding-window algorithms. A monotonic deque can maintain the maximum of a moving window: remove dominated values from the back, append the new value, and remove the front when it leaves the window. Each value enters and leaves at most once, so total time is linear even though one update may remove many values. The structure’s invariant is that values decrease from front to back and all retained indices lie in the current window.

## Traversal and scheduling

Depth-first search uses a stack, explicit or implicit through recursion. Breadth-first search uses a queue and explores vertices in nondecreasing edge distance in an unweighted graph. The BFS invariant is that dequeued vertices have finalized shortest distance, and enqueued but not dequeued vertices have distance at most one greater than the current frontier. Marking visited on enqueue prevents duplicate work and preserves O(V+E) traversal.

Schedulers can use a queue for fair arrival order, a stack for depth-first task exploration, or a deque for local work plus stealing. The ADT does not decide fairness; the policy and concurrency model do. A queue that allows arbitrary removal is no longer a simple FIFO abstraction, and proofs must account for the additional operation.

## Bounded buffers and backpressure

A bounded queue is a communication contract between a producer and consumer. When full, the producer can block, reject, drop the newest item, drop the oldest item, or apply a priority policy. These choices have different correctness consequences. A video stream may prefer dropping stale frames; a transaction log must not silently drop records. State the policy rather than treating “queue full” as an implementation accident.

For a single producer and consumer, a circular buffer can use head and tail counters with a capacity invariant. In a concurrent implementation, visibility and atomicity matter: a producer must publish an element only after writing it, and a consumer must not read a slot before publication. Lock-free designs require memory-order arguments beyond the sequential RI; a sequentially correct ring buffer is not automatically thread-safe.

Backpressure prevents unbounded memory growth. If arrival rate (lambda) exceeds service rate (mu) for a sustained interval, any unbounded queue grows without limit. A bounded queue forces the system to expose overload through blocking or loss. Monitoring queue length and age helps distinguish a burst that will drain from a stable overload that needs more service capacity or admission control.

## Recursive depth and explicit stacks

Replacing recursion with an explicit stack preserves a depth-first order only if frames store the same continuation information as the call stack. A frame may contain a vertex, the next neighbor index, local variables, and a parent reference. Pushing only the vertex and then restarting its neighbor scan can produce the same reachability result but worse time, because edges may be rescanned. Representation choices affect both proof and complexity.

## Exercises

1. Prove the stack LIFO law from the array-stack invariant.
2. Give the complete RI for a circular queue with head, tail, count, and capacity.
3. Trace a two-stack queue through an alternating sequence of enqueue and dequeue.
4. Prove two-stack queue operations are O(1) amortized.
5. Explain why marking BFS vertices on dequeue can cause duplicate enqueues.
6. Design a monotonic deque algorithm for sliding-window maximum and prove linear total time.
7. Compare linked and circular-array deques under locality, allocation, and worst-case guarantees.

### Diagnostic solutions


1. Push appends x to the abstract bottom-to-top sequence, so the last position is x. Pop returns that position and removes it, restoring the prior prefix. The invariant ensures the concrete operations implement these abstract transformations.
2. Require (0le countle capacity), all indices in range, logical elements at ((head+j)bmod capacity), tail=((head+count)bmod capacity), and no significance for unused cells. Empty iff count=0; full iff count=capacity.

3. Enqueue a,b pushes both to input. Dequeue transfers b,a to output, then pops a. Enqueue c pushes input. Dequeue pops b from output. A later dequeue transfers c and pops c. The oldest unserved item always remains on output’s top.

4. Charge each enqueue enough for its push and a credit attached to the element. When moved, the credit pays for one input pop and one output push; when dequeued, use its remaining constant charge. Each element pays for O(1) work once, so m operations cost O(m).

5. Two neighboring vertices can both discover the same unmarked vertex before either is dequeued. Both enqueue it, and without a separate in-queue mark the queue contains duplicates. Marking on enqueue establishes ownership of the first discovery and bounds enqueues by V.

6. Store candidate indices with decreasing values. Before appending i, pop from the back while value is no larger; remove front indices outside i-k+1. The front is the maximum. Each index is appended once and removed once, so total deque operations are O(n); the invariant preserves dominance and window membership.

7. A circular array has compact storage and strong sequential locality but resizing can be expensive; a linked deque has O(1) endpoint updates without resizing but allocates nodes and follows pointers. Both can give O(1) endpoint operations; worst-case versus amortized behavior depends on array growth and whether a bounded capacity is used.
