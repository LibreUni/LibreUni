# Hierarchical Logic

A Decision Tree is a non-parametric supervised learning method used for classification and regression. Unlike linear models that find a global hyperplane, decision trees partition the feature space into a set of rectangular regions. The goal is to create a model that predicts the target variable by learning simple "if-then-else" decision rules.

The structure is intuitive:
- **Internal Nodes**: Represent a test on an attribute (e.g., "Is Age > 30?").
- **Branches**: Represent the outcome of the test.
- **Leaf Nodes**: Represent a class label (classification) or a continuous value (regression).

## Measuring Impurity: The Math of Splits

To build an optimal tree, we need a mathematical criterion to decide which feature to split on. We want to choose the split that results in the "purest" child nodes.

### 1. Entropy
Entropy measures the amount of uncertainty or "disorder" in a dataset. For a classification problem with $C$ classes, where $p_i$ is the proportion of examples belonging to class $i$:

$$ H(S) = -\sum_{i=1}^{C} p_i \log_2(p_i) $$

- **Purity**: If all examples in node $S$ belong to one class, $H(S) = 0$.
- **Chaos**: If examples are evenly distributed across classes, $H(S)$ is maximized.

### 2. Gini Impurity
Gini impurity is the default metric for the popular CART (Classification and Regression Trees) algorithm. It calculates the probability of a randomly chosen element being incorrectly labeled.

$$ G(S) = 1 - \sum_{i=1}^{C} p_i^2 $$

Gini is often preferred in large-scale implementations because it avoids the computationally expensive $\log_2$ calculation required by Entropy.

## Selecting the Best Split: Information Gain

Information Gain (IG) measures the reduction in entropy achieved by partitioning the data according to a specific feature $A$.

$$ IG(S, A) = H(S) - \sum_{v \in Values(A)} \frac{|S_v|}{|S|} H(S_v) $$

Here, $|S_v|/|S|$ is the weight of the $v$-th child node. The algorithm (ID3/C4.5) iterates through all possible features and all possible split points for each feature, choosing the one that maximizes Information Gain.

## Recursive Induction: The CART Algorithm

The Classification and Regression Trees (CART) algorithm builds binary trees. The process is recursive:
1. Start at the root node.
2. For every feature, find the split point that minimizes impurity (Gini/Entropy).
3. Split the dataset into two subsets.
4. Repeat 2-3 for each subset until a "Stopping Criterion" is met.

### Stopping Criteria
Without limits, a tree will continue splitting until every leaf contains only one sample. This is perfect on the training data but disastrous on new data (Overfitting). We stop when:
- Maximum depth is reached.
- The number of samples in a node falls below a threshold.
- The impurity reduction is below a certain value.

## The Problem of Overfitting and Pruning

Decision trees are notorious for high variance. Because they can adapt to every tiny wiggle in the data, they often "memorize" the noise.

### Pruning Strategies
1. **Pre-pruning (Early Stopping)**: Use hyperparameters like `max_depth`, `min_samples_split`, or `min_samples_leaf` to limit tree growth.
2. **Post-pruning (Cost Complexity Pruning)**: Let the tree grow to its maximum length and then remove subtrees that do not contribute significantly to predictive accuracy (based on a validation set).

## Regression Trees: Predicting Numbers

Contrary to classification trees, **Regression Trees** predict a continuous value at the leaf nodes. 
- **Splitting Criterion**: Instead of Entropy/Gini, it uses **Mean Squared Error (MSE)**. The algorithm finds a split that minimizes the sum of squared residuals in the child nodes.
- **Leaf Value**: The prediction at a leaf node is the **average** of the target values of all training samples in that leaf.

## Logic and Interpretabilty

Decision trees are "#FFFFFF Box" models. You can trace the exact logic used to reach a prediction. This makes them invaluable in regulated industries like finance or healthcare.

### Feature Importance
The total reduction in Gini/Entropy contributed by a feature across all nodes where it was used tells us how important that feature is to the model.

## Pros and Cons

| Advantages | Disadvantages |
| :--- | :--- |
| **No Data Scaling**: Doesn't require normalization as splits are based on ordering. | **Instability**: Small changes in data can lead to a completely different tree. |
| **Handles Non-linearity**: Can capture complex interaction between features. | **Linearity**: Decision trees struggled to capture purely linear relationships (they use 'stairstep' approximations). |
| **Interpretability**: Easy to explain to non-technical stakeholders. | **Greedy**: They make locally optimal splits at each node, which doesn't guarantee a globally optimal tree. |

## Conclusion

Decision trees are the building blocks of most modern winning algorithms. While a single tree is prone to high variance, combining hundreds of trees together leads to **Random Forests** and **Gradient Boosted Machines (XGBoost/LightGBM)**, which currently dominate tabular data competitions. Understanding the core logic of a single tree is the prerequisite to mastering these ensemble methods.
