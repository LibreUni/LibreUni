# Logistic Regression and Probability Estimation

Logistic Regression is a classifier that computes a weighted sum of inputs and maps the output to a probability between $0$ and $1$.

## Probabilistic Formulation: The Sigmoid Function

The model feeds its linear output into the **Sigmoid Function** $\sigma(t)$:

$$ \hat{p} = \sigma(\theta^T \mathbf{x}) = \frac{1}{1 + e^{-\theta^T \mathbf{x}}} $$

The prediction rule for binary classes is:

$$ \hat{y} = \begin{cases} 1 & \text{if } \hat{p} \ge 0.5 \\ 0 & \text{if } \hat{p} < 0.5 \end{cases} $$

The prediction decision boundary is linear where $\theta^T \mathbf{x} = 0$.

## Parameter Optimization: Log Loss

To optimize parameters $\theta$, we minimize the convex **Log Loss** (binary cross-entropy):

$$ J(\theta) = -\frac{1}{m} \sum_{i=1}^{m} \left[ y^{(i)} \log(\hat{p}^{(i)}) + (1 - y^{(i)}) \log(1 - \hat{p}^{(i)}) \right] $$

It penalizes confident incorrect predictions with infinite cost (if $y=1$ and $\hat{p} \to 0$, then $-\log(\hat{p}) \to \infty$).

### Multiclass Classification Strategies

While Logistic Regression is inherently a binary classifier, it can be extended to handle multiple classes (e.g., classifying images of digits 0-9) using specific strategies:

- **One-vs-Rest (OVR)** (also called One-vs-All): Trains $N$ separate binary classifiers for an $N$-class problem. Each classifier is trained to distinguish one specific class from all the rest combined. During prediction, the classifier that outputs the highest probability wins. This is the default approach in most ML libraries for logistic regression.
- **One-vs-One (OVO)**: Trains a binary classifier for every possible pair of classes, resulting in $\frac{N \times (N-1)}{2}$ classifiers. For a new data point, all classifiers are run, and the class that wins the most duels is selected. OVO is particularly useful for algorithms that scale poorly with dataset size (like SVMs), because each classifier is only trained on the subset of data belonging to the two classes.

Alternatively, **Softmax Regression** (Multinomial Logistic Regression) generalizes Logistic Regression to natively support multiple classes by normalizing raw logits into a direct probability distribution across all classes without training multiple binary models.
