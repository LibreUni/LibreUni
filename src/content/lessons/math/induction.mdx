# Mathematical Induction and Well-Ordering

Mathematical Induction is a method of proof used to establish the truth of an infinite set of statements indexed by natural numbers. Formal grounding lies in the structure of the natural numbers $\mathbb{N}$ and the **Well-Ordering Principle**.

### The Principle of Mathematical Induction

To prove that $P(n)$ is true for all $n \in \mathbb{N}$:
1.  **Base Case**: Demonstrate $P(n_0)$ is true.
2.  **Inductive Step**: Prove $P(k) \implies P(k+1)$ for all $k \ge n_0$.

### Strong Induction

In **Strong Induction**, we assume $P(m)$ is true for *all* $n_0 \le m \le k$ to prove $P(k+1)$. This is essential for structures like the Fibonacci sequence.

<CodeRunner 
  client:load 
  code={`# The Inductive Base: stored in a persistent structure
cache = {0: 0, 1: 1}

def fibonacci(n):
    # Strong induction: we use multiple previous values (n-1 and n-2)
    if n not in cache:
        cache[n] = fibonacci(n - 1) + fibonacci(n - 2)
    return cache[n]

print(f"F(10) = {fibonacci(10)}")
print(f"Current Inductive Cache: {[(k, cache[k]) for k in sorted(cache.keys())[:7]]}...")`} 
/>

### The Well-Ordering Principle

The Well-Ordering Principle states that *every non-empty subset of the natural numbers has a least element*. This is logically equivalent to the principle of induction.

### Structural Induction

Induction extends to recursively defined structures like Trees.
- **Base Case**: Property holds for leaves.
- **Recursive Step**: If holds for children, it holds for parent.

<CodeRunner 
  client:load 
  code={`class Node:
    def __init__(self, val, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right

def count_nodes(node):
    if not node: return 0
    # Structural induction: parent = 1 + left + right
    return 1 + count_nodes(node.left) + count_nodes(node.right)

tree = Node(1, Node(2), Node(3, Node(4)))
print(f"Total nodes in tree: {count_nodes(tree)}")`} 
/>
