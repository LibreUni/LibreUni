# References and Borrowing

The ownership model would be crippling without a way to let functions inspect or modify data without taking ownership. **References** solve this.

{/* Source tracking: References and borrowing synthesized from "The Rust Programming Language" by Klabnik & Nichols, Ch. 4.2 (MIT/Apache-2.0) */}

## Borrowing with &

A reference is a non-owning pointer. Creating a reference is called **borrowing**:

```rust
fn calculate_length(s: &String) -> usize {
    s.len()
}   // s goes out of scope, but the referent is NOT dropped

fn main() {
    let s = String::from("hello");
    let len = calculate_length(&s);  // borrow, not move
    println!("{} length: {}", s, len); // s still valid
}
```

## Mutable References

To modify a borrowed value, use `&mut`:

```rust
fn add_world(s: &mut String) {
    s.push_str(" world");
}

let mut s = String::from("hello");
add_world(&mut s);
println!("{}", s); // "hello world"
```

## The Borrowing Rules

The borrow checker enforces two rules at compile time:

1. **At any time, you may have either one mutable reference or any number of immutable references.**
2. **References must always be valid.**

```rust
let mut s = String::from("hello");
let r1 = &s;        // ok
let r2 = &s;        // ok (multiple immutable refs)
// let r3 = &mut s; // error: cannot borrow as mutable because immutable refs exist
println!("{}, {}", r1, r2); // r1 and r2 go out of scope here

let r3 = &mut s;    // ok: no immutable refs active
println!("{}", r3);
```

This rule eliminates **data races** at compile time.

## Dangling References

Rust guarantees that references never dangle. The borrow checker verifies that any reference outlives the data it points to:

```rust
fn dangle() -> &String {
    let s = String::from("hello");
    &s
}   // error: returns reference to local variable
```
