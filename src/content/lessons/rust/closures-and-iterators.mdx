# Closures and Iterators

Closures and iterators are two of Rust's most powerful abstractions. They enable expressive, functional-style programming with zero or minimal overhead.

{/* Source tracking: Closures and iterators synthesized from "The Rust Programming Language" by Klabnik & Nichols, Ch. 13 (MIT/Apache-2.0) */}

## Closures

Closures are anonymous functions that can capture variables from their environment:

```rust
let x = 42;
let add_x = |y| x + y;  // captures x
println!("{}", add_x(8)); // 50
```

Closures infer their parameter and return types from usage. Type annotations are optional but can be added:

```rust
let expensive = |num: u64| -> u64 {
    println!("computing...");
    num
};
```

### Capturing Modes

Closures capture variables in three ways, matching the function traits:

| Trait       | Captures By      | When?                               |
|-------------|------------------|-------------------------------------|
| `FnOnce`    | By move (consumes) | Once, on first call                 |
| `FnMut`     | &mut T           | Can mutate captured variables       |
| `Fn`        | &T               | Only reads, can call multiple times |

```rust
let mut count = 0;
let mut increment = || { count += 1; };  // FnMut
increment(); // count = 1

let readonly = || println!("{}", count); // Fn
readonly();

let consume = || drop(count);  // FnOnce
```

## Iterators

The `Iterator` trait requires one method: `next()`.

```rust
pub trait Iterator {
    type Item;
    fn next(&mut self) -> Option<Self::Item>;
}
```

### Iterator Adaptors

Iterator adaptors transform an iterator into another kind of iterator. They are **lazy**—nothing happens until you call a consuming adaptor:

```rust
let numbers = vec![1, 2, 3, 4, 5, 6];

let even_squares: Vec<i32> = numbers
    .iter()
    .filter(|x| *x % 2 == 0)     // lazy: 2, 4, 6
    .map(|x| x * x)               // lazy: 4, 16, 36
    .collect();                   // eager: drives iteration

println!("{:?}", even_squares); // [4, 16, 36]
```

Common adaptors: `map`, `filter`, `fold`, `take`, `skip`, `zip`, `chain`, `enumerate`.

Common consumers: `collect`, `sum`, `count`, `for_each`, `reduce`, `any`, `all`.
