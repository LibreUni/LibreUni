# Collections: Vec, String, and HashMap

Unlike the compound types (tuples and arrays) which are stack-allocated with fixed size, **collections** are heap-allocated and can grow or shrink at runtime.

{/* Source tracking: Collections synthesized from "The Rust Programming Language" by Klabnik & Nichols, Ch. 8 (MIT/Apache-2.0) */}

## Vectors

`Vec<T>` is a contiguous growable array. It stores elements of the same type on the heap.

```rust
let mut v: Vec<i32> = Vec::new();
v.push(1);
v.push(2);

let v2 = vec![1, 2, 3];  // vec! macro for convenience

let third = &v2[2];       // indexing, panics if out of bounds
let third = v2.get(2);    // returns Option<&T>, None if out of bounds

for i in &v2 {
    println!("{}", i);
}
```

Borrowing rules apply: you cannot push to a vector while holding an immutable reference to an element.

## Strings

`String` is a growable, UTF-8 encoded string type. It is not a sequence of characters but a sequence of UTF-8 bytes.

```rust
let mut s = String::new();
s.push_str("hello");
s.push(' ');
s.push_str("world");

let s2 = String::from("hello");
let s3 = s2 + " world";  // concatenation moves s2

let s4 = format!("{}-{}", "hello", "world");  // doesn't take ownership
```

Strings do not support indexing by character position because UTF-8 is variable-width. Instead, iterate over bytes, chars, or grapheme clusters:

```rust
let hello = "\u{0048}\u{0065}\u{006C}\u{006C}\u{006F}";
for b in hello.bytes() { /* ... */ }
for c in hello.chars() { /* ... */ }
```

## Hash Maps

`HashMap<K, V>` stores key-value pairs. It uses a hash function to map keys to entries.

```rust
use std::collections::HashMap;

let mut scores = HashMap::new();
scores.insert(String::from("Blue"), 10);
scores.insert(String::from("Yellow"), 50);

let team = String::from("Blue");
let score = scores.get(&team);  // Option<&i32>

for (key, value) in &scores {
    println!("{}: {}", key, value);
}
```

### Entry API

The entry API handles the "insert if absent" pattern cleanly:

```rust
scores.entry(String::from("Blue")).or_insert(50);  // only inserts if no entry exists
```
