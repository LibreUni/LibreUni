# Error Handling with Result and Option

Rust separates errors into two categories without exceptions: **recoverable errors** represented by `Result<T, E>` and **unrecoverable errors** that trigger `panic!`.

{/* Source tracking: Error handling synthesized from "The Rust Programming Language" by Klabnik & Nichols, Ch. 9 (MIT/Apache-2.0) */}

## Unrecoverable Errors with panic!

When something goes so wrong that the program cannot continue, call the `panic!` macro. It prints an error message, unwinds the stack, and exits:

```rust
fn main() {
    panic!("something went horribly wrong");
}
```

`panic!` is appropriate for tests, prototyping, and truly unrecoverable states (like array index out of bounds).

## Recoverable Errors with Result

`Result<T, E>` is an enum defined in the standard library:

```rust
enum Result<T, E> {
    Ok(T),
    Err(E),
}
```

```rust
use std::fs::File;
use std::io::Read;

fn read_file(path: &str) -> Result<String, std::io::Error> {
    let mut file = File::open(path)?;  // ? propagates the error
    let mut contents = String::new();
    file.read_to_string(&mut contents)?;
    Ok(contents)
}
```

## The ? Operator

The `?` operator is syntactic sugar for matching against `Result` (or `Option`):

```rust
// This:
let f = File::open("hello.txt")?;

// Desugars approximately to:
let f = match File::open("hello.txt") {
    Ok(file) => file,
    Err(e) => return Err(e.into()),
};
```

The `?` operator can only be used in functions that return `Result` (or `Option` or another type implementing `FromResidual`).

## Combining Result and Option

The standard library provides combinators for chaining operations on `Result` and `Option`:

```rust
let result: Result<i32, &str> = Ok(10);
let doubled = result.map(|x| x * 2);          // Ok(20)
let checked = result.and_then(|x| {
    if x > 5 { Ok(x) } else { Err("too small") }
});
```

## Custom Error Types

For library code, define custom error types implementing `std::error::Error`:

```rust
#[derive(Debug)]
pub enum MyError {
    Io(std::io::Error),
    Parse(std::num::ParseIntError),
    InvalidInput(String),
}

impl std::fmt::Display for MyError {
    fn fmt(&self, f: &mut std::fmt::Formatter) -> std::fmt::Result {
        match self {
            MyError::Io(e) => write!(f, "IO error: {}", e),
            MyError::Parse(e) => write!(f, "Parse error: {}", e),
            MyError::InvalidInput(s) => write!(f, "Invalid input: {}", s),
        }
    }
}

impl std::error::Error for MyError {
    fn source(&self) -> Option<&(dyn std::error::Error + 'static)> {
        match self {
            MyError::Io(e) => Some(e),
            MyError::Parse(e) => Some(e),
            MyError::InvalidInput(_) => None,
        }
    }
}
```
