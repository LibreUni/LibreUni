# Functions and Control Flow

Rust uses `fn` to declare functions. Function and variable names follow `snake_case` convention. Every function body contains a series of statements, optionally ending with an expression whose value becomes the return value.

{/* Source tracking: Functions and control flow synthesized from "The Rust Programming Language" by Klabnik & Nichols, Ch. 3.3 & 3.5 (MIT/Apache-2.0) */}

## Expressions vs. Statements

- A **statement** performs an action but does not return a value.
- An **expression** evaluates to a resulting value.

```rust
fn main() {
    let y = {
        let x = 3;
        x + 1   // no semicolon: this is an expression
    };
    println!("{}", y); // 4
}
```

The absence of a semicolon at the end of a block makes it an expression. This distinction is central to Rust's expression-oriented design.

## Functions with Return Values

```rust
fn add(x: i32, y: i32) -> i32 {
    x + y   // expression, no semicolon = return value
}

fn is_even(x: i32) -> bool {
    return x % 2 == 0;  // explicit return also works
}
```

## if Expressions

`if` is an expression, not just a statement. This enables concise conditional assignment:

```rust
let condition = true;
let number = if condition { 5 } else { 6 };
println!("{}", number); // 5
```

Both arms must produce the same type.

## Loops

Rust provides three loop forms:

```rust
// infinite loop with break
loop {
    break;
}

// while with condition
let mut n = 3;
while n != 0 {
    n -= 1;
}

// for over iterator (most idiomatic)
let a = [10, 20, 30];
for element in a.iter() {
    println!("{}", element);
}
```

The `loop` construct can return a value with `break`:

```rust
let counter = loop {
    break 42;
};
println!("{}", counter); // 42
```
