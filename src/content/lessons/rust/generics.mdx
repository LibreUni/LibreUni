# Generics

**Generics** allow writing code that operates on abstract types, with concrete types substituted at compile time. They eliminate code duplication without sacrificing type safety.

{/* Source tracking: Generics synthesized from "The Rust Programming Language" by Klabnik & Nichols, Ch. 10 (MIT/Apache-2.0) */}

## Generic Functions

```rust
fn largest<T: PartialOrd>(list: &[T]) -> &T {
    let mut largest = &list[0];
    for item in list {
        if item > largest {
            largest = item;
        }
    }
    largest
}

fn main() {
    let numbers = vec![34, 50, 25, 100, 65];
    println!("largest: {}", largest(&numbers)); // 100

    let chars = vec!['y', 'm', 'a', 'q'];
    println!("largest: {}", largest(&chars)); // 'y'
}
```

The generic `T: PartialOrd` means "any type that supports comparison." The function works for integers, floats, characters, strings—any ordered type.

## Generic Structs and Enums

```rust
struct Point<T> {
    x: T,
    y: T,
}

let integer = Point { x: 5, y: 10 };        // Point<i32>
let float = Point { x: 1.0, y: 4.0 };       // Point<f64>

// Multiple type parameters:
struct Pair<X, Y> {
    first: X,
    second: Y,
}
```

Generic enums are everywhere in Rust—`Option<T>` and `Result<T, E>` are the primary examples.

## Monomorphization

Rust compiles generics via **monomorphization**: it generates concrete code for each type the generic is instantiated with. This means generic code has zero runtime overhead—it performs identically to handwritten concrete code.

```rust
let integer = Some(5);   // generates Option<i32> code
let float = Some(5.0);   // generates Option<f64> code (separate copy)
```

The tradeoff is increased binary size, as the same generic logic is duplicated for each concrete type.

## impl Blocks for Generic Types

```rust
impl<T> Point<T> {
    fn x(&self) -> &T {
        &self.x
    }
}

// Methods only available for specific concrete types:
impl Point<f64> {
    fn distance_from_origin(&self) -> f64 {
        (self.x.powi(2) + self.y.powi(2)).sqrt()
    }
}
```
