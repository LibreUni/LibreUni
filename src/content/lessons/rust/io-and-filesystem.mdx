# I/O Operations and the File System

Rust's standard library provides comprehensive I/O primitives through `std::io`, `std::fs`, and `std::path`. The design emphasizes explicit error handling and zero-cost abstractions.

{/* Source tracking: I/O synthesized from "The Rust Programming Language" by Klabnik & Nichols, Ch. 12, and std::fs docs (MIT/Apache-2.0) */}

## Path and PathBuf

`Path` (borrowed, unsized) and `PathBuf` (owned) represent file system paths. They are platform-independent:

```rust
use std::path::Path;

let path = Path::new("/tmp/example.txt");
println!("exists: {}, extension: {:?}", path.exists(), path.extension());

// PathBuf is the owned, growable version
let mut buf = std::path::PathBuf::from("/tmp");
buf.push("example.txt");
```

## Reading Files

```rust
use std::fs;

// Read entire file to String
let contents = fs::read_to_string("hello.txt")?;

// Read file to byte vector
let data = fs::read("image.png")?;
```

For line-by-line processing, `BufRead` provides efficient buffered access:

```rust
use std::fs::File;
use std::io::{BufRead, BufReader};

let file = File::open("lines.txt")?;
let reader = BufReader::new(file);

for line in reader.lines() {
    println!("{}", line?);
}
```

## Writing Files

```rust
use std::fs;

// Write a string (overwrites existing content)
fs::write("output.txt", "Hello, world!")?;

// Append with OpenOptions
use std::fs::OpenOptions;
use std::io::Write;

let mut file = OpenOptions::new()
    .append(true)
    .open("log.txt")?;
writeln!(file, "Log entry at {:?}", std::time::Instant::now())?;
```

## Working with Directories

```rust
use std::fs;

// Create directory (and parents)
fs::create_dir_all("/tmp/rust/example")?;

// List directory contents
for entry in fs::read_dir(".")? {
    let entry = entry?;
    println!("{} (len: {})", entry.path().display(), entry.metadata()?.len());
}

// Copy and remove
fs::copy("source.txt", "target.txt")?;
fs::remove_file("temp.txt")?;
fs::remove_dir_all("/tmp/old_dir")?;
```
