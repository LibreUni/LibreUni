# Macros

Macros are a form of metaprogramming—code that writes code. Rust has two kinds: **declarative macros** (`macro_rules!`) and **procedural macros** (custom derives, attribute-like, function-like).

{/* Source tracking: Macros synthesized from "The Rust Programming Language" by Klabnik & Nichols, Ch. 19.5 (MIT/Apache-2.0) */}

## Declarative Macros with macro_rules!

Declarative macros match patterns and generate code at compile time. The `vec!` macro is a familiar example:

```rust
let v: Vec<u32> = vec![1, 2, 3];
```

Defining a simple macro:

```rust
macro_rules! say_hello {
    () => {
        println!("Hello!")
    };
    ($name:expr) => {
        println!("Hello, {}!", $name)
    };
}

say_hello!();            // "Hello!"
say_hello!("World");     // "Hello, World!"
```

## Macro Metavariables

Macro patterns capture different kinds of Rust syntax using metavariables:

| Fragment Specifier | Example         | Matches                     |
|--------------------|-----------------|-----------------------------|
| `:expr`            | `$x:expr`       | An expression               |
| `:ident`           | `$name:ident`   | An identifier               |
| `:ty`              | `$t:ty`         | A type                      |
| `:stmt`            | `$s:stmt`       | A statement                 |
| `:pat`             | `$p:pat`        | A pattern                   |
| `:tt`              | `$t:tt`         | A token tree (generic)      |
| `:block`           | `$b:block`      | A block expression          |

## Repetition

Macros can handle variable numbers of arguments using `$()*` (zero or more) or `$()+` (one or more):

```rust
macro_rules! create_function {
    ($func_name:ident, $val:expr) => {
        fn $func_name() {
            println!("The value is {}", $val);
        }
    };
}

// Build a HashMap from key-value pairs
macro_rules! map {
    ($($key:expr => $value:expr),* $(,)?) => {{
        let mut m = ::std::collections::HashMap::new();
        $(m.insert($key, $value);)*
        m
    }};
}
```

## Hygiene

Rust macros are **hygienic**: they cannot accidentally capture or shadow variables from the calling scope. This prevents the bugs common to C preprocessor macros.

```rust
macro_rules! set_x {
    ($val:expr) => {
        let x = $val;  // creates a new x, doesn't shadow outer x
    };
}

let x = 10;
set_x!(20);
println!("{}", x);  // still 10
```

## Procedural Macros (Overview)

Procedural macros operate on the AST of Rust code. They are more powerful but defined in separate crates:

```rust
// Custom derive
#[derive(Debug, Clone, PartialEq)]
struct Point { x: i32, y: i32 }

// Attribute-like macro
#[route(GET, "/api/users")]
fn get_users() {}

// Function-like macro
let sql = sql!("SELECT * FROM users WHERE id = $1");
```
