# Ownership

Ownership is Rust's most distinctive feature. It enables memory safety without a garbage collector by encoding a **contract about who is responsible for a value** into the type system.

{/* Source tracking: Ownership synthesized from "The Rust Programming Language" by Klabnik & Nichols, Ch. 4.1 (MIT/Apache-2.0) */}

## The Three Rules

1. Each value has exactly one **owner** at any time.
2. When the owner goes out of scope, the value is **dropped**.
3. Ownership can be **transferred** (moved), but there is never more than one owner.

## Move Semantics

When a value is assigned to another variable, the ownership **moves**:

```rust
let s1 = String::from("hello");
let s2 = s1;                   // ownership moves to s2
// println!("{}", s1);        // compile error: s1 was moved
```

Rust does **not** perform a shallow copy. Moving ownership means the original variable is invalidated. The compiler tracks this and prevents access to moved values.

## The Copy Trait

Types stored entirely on the stack implement the `Copy` trait. For `Copy` types, assignment creates an independent copy instead of a move:

```rust
let x = 42;        // i32 implements Copy
let y = x;         // x is still valid
println!("{}", x); // works: 42
```

`Copy` types include all scalar types, tuples of `Copy` types, and arrays.

## Ownership in Functions

Passing a value to a function moves ownership:

```rust
fn take_ownership(s: String) {
    println!("{}", s);
}   // s dropped here

fn main() {
    let s = String::from("hello");
    take_ownership(s);
    // println!("{}", s); // error: s was moved
}
```

Returning a value transfers ownership back:

```rust
fn give_and_take(s: String) -> String {
    s   // ownership returned
}
```
