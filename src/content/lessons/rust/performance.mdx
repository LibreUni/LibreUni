# Performance and Zero-Cost Abstractions

Rust's philosophy is built on **zero-cost abstractions**: what you don't use, you don't pay for. And what you do use, you couldn't hand-code better. This means high-level Rust code compiles to machine code as efficient as carefully written C.

{/* Source tracking: Performance concepts synthesized from "The Rust Programming Language" by Klabnik & Nichols, and rustc perf docs (MIT/Apache-2.0) */}

## What Zero-Cost Means

```rust
// This high-level iterator chain:
let result: Vec<i32> = (0..1000)
    .filter(|x| x % 2 == 0)
    .map(|x| x * 3)
    .collect();

// Compiles to the same machine code as this manual loop:
let mut result = Vec::new();
for i in 0..1000 {
    if i % 2 == 0 {
        result.push(i * 3);
    }
}
```

The Rust compiler monomorphizes generics and inlines closures, eliminating all abstraction overhead.

## Optimizing Rust Code

### Profile First

Use the built-in benchmarking or external profilers:

```bash
cargo bench              # stable benchmark harness
# or use perf, flamegraph, etc. on nightly
```

### Common Optimization Patterns

**Pre-allocate when size is known:**

```rust
// Avoid repeated reallocations
let mut v = Vec::with_capacity(1000);
for i in 0..1000 { v.push(i); }
```

**Avoid bounds checks on hot paths:**

```rust
// The compiler often eliminates bounds checks for iterators
// Use iterators instead of index-based loops
for item in &slice[..n] { /* safe, no bounds check */ }
```

**Use references instead of cloning:**

```rust
// Cloning a large String is expensive
fn process(s: &str) { /* take reference */ }
// vs
fn process(s: String) { /* takes ownership, caller must clone */ }
```

**Prefer stack allocation:**

```rust
// Fixed-size arrays on stack are faster than Vec on heap
let data: [u32; 1024] = [0; 1024];  // stack
// vs
let data: Vec<u32> = vec![0; 1024]; // heap
```

### Compiler Optimization Flags

```toml
[profile.release]
opt-level = 3                  # max optimization
lto = "fat"                    # link-time optimization
codegen-units = 1              # single codegen unit for more inlining
```

## Benchmarking

```rust
// Using std::time for simple measurements
use std::time::Instant;

let start = Instant::now();
// ... computation ...
let duration = start.elapsed();
println!("took: {:?}", duration);
```

Rust nightly provides `#[bench]`; stable uses `criterion` crate for robust benchmarks.
