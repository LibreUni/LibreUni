# Scalar and Compound Types

Every value in Rust has a fixed type, determined either through explicit annotation or type inference. The type system is **static** and **strong**: type checking happens entirely at compile time, and implicit coercions are rare.

## Scalar Types

Rust provides four scalar types representing single values:

| Category  | Types                                     | Size Range          |
|-----------|-------------------------------------------|---------------------|
| Integer   | `i8`, `i16`, `i32`, `i64`, `i128`, `isize` | signed (two's complement) |
| Integer   | `u8`, `u16`, `u32`, `u64`, `u128`, `usize` | unsigned |
| Float     | `f32`, `f64`                              | IEEE-754 |
| Boolean   | `bool`                                    | `true` or `false` |
| Character | `char`                                    | 4-byte Unicode scalar |

Integer literals can use underscores as visual separators: `1_000_000u32`. The default integer type is `i32`; the default float is `f64`.

```rust
let decimal = 98_222;        // i32 by default
let hex = 0xff;              // 255
let octal = 0o77;            // 63
let binary = 0b1111_0000;    // 240
let byte = b'A';             // 65 (u8 only)
```

## Compound Types

### Tuples

Tuples group values of possibly different types into a single compound value. They have a fixed length determined at compile time.

```rust
let tup: (i32, f64, u8) = (500, 6.4, 1);
let (x, y, z) = tup;        // destructuring
let first = tup.0;           // dot-index access: 500
```

### Arrays

Arrays hold multiple values of the **same type** with a **fixed length** known at compile time.

```rust
let a: [i32; 5] = [1, 2, 3, 4, 5];
let first = a[0];
let months = ["Jan", "Feb", "Mar", "Apr"];
```

Attempting to access an array out of bounds causes a **runtime panic**, which Rust performs bounds checking on.

## Type Inference

Rust can often infer types from context:

```rust
let guess = "42".parse().expect("Not a number!");
// error: type annotation needed
let guess: u32 = "42".parse().expect("Not a number!");
// explicit annotation resolves
```
