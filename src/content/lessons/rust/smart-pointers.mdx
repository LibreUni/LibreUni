# Smart Pointers

A **smart pointer** is a data structure that acts like a pointer but provides additional capabilities such as reference counting or interior mutability. Rust's smart pointers implement the `Deref` and `Drop` traits.

{/* Source tracking: Smart pointers synthesized from "The Rust Programming Language" by Klabnik & Nichols, Ch. 15 (MIT/Apache-2.0) */}

## Box<T>

`Box<T>` allocates a value on the heap. It is the simplest smart pointer, useful when:
- You need a type whose size is unknown at compile time (recursive types).
- You want to transfer ownership of large data without copying.

```rust
// Recursive type requires Box for known size
enum List {
    Cons(i32, Box<List>),
    Nil,
}

let list = List::Cons(1, Box::new(List::Cons(2, Box::new(List::Nil))));
```

## Deref Trait

The `Deref` trait allows a smart pointer to be used with dereference operator `*`:

```rust
let x = 5;
let y = Box::new(x);

assert_eq!(5, x);
assert_eq!(5, *y);  // deref through Box
```

## Drop Trait

The `Drop` trait customizes what happens when a value goes out of scope. The compiler calls `drop()` automatically.

```rust
struct Custom {
    data: String,
}

impl Drop for Custom {
    fn drop(&mut self) {
        println!("Dropping: {}", self.data);
    }
}
```

## Rc<T>

`Rc<T>` (Reference Counted) enables **multiple ownership**. It keeps a reference count and drops the value when the count reaches zero. Only for single-threaded use.

```rust
use std::rc::Rc;

let a = Rc::new(5);
let b = Rc::clone(&a);  // increments reference count
let c = Rc::clone(&a);  // count is now 3
```

## RefCell<T>

`RefCell<T>` enforces borrowing rules at **runtime** instead of compile time, enabling **interior mutability**—mutating data through an immutable reference.

```rust
use std::cell::RefCell;

let data = RefCell::new(vec![1, 2, 3]);
{
    let mut mut_ref = data.borrow_mut();  // panics if already borrowed
    mut_ref.push(4);
}
println!("{:?}", data.borrow());  // [1, 2, 3, 4]
```
