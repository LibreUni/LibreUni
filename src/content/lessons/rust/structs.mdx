# Structs: Composing Related Data

A **struct** (short for structure) is a custom data type that groups related values together. Structs are Rust's primary way to create named composite types.

{/* Source tracking: Structs synthesized from "The Rust Programming Language" by Klabnik & Nichols, Ch. 5 (MIT/Apache-2.0) */}

## Defining and Instantiating

```rust
struct User {
    username: String,
    email: String,
    sign_in_count: u64,
    active: bool,
}

let user = User {
    email: String::from("alice@example.com"),
    username: String::from("alice"),
    active: true,
    sign_in_count: 1,
};
```

Fields are accessed with dot notation: `user.email`. Mutable structs allow field mutation:

```rust
let mut user = User { email: String::from("bob@example.com"), .. };
user.email = String::from("bob@newdomain.com");
```

### Field Init Shorthand

When the parameter name matches the field name, you can omit the field name:

```rust
fn build_user(email: String, username: String) -> User {
    User {
        email,      // shorthand for email: email
        username,   // shorthand for username: username
        active: true,
        sign_in_count: 1,
    }
}
```

### Struct Update Syntax

Copying most fields from an existing instance:

```rust
let user2 = User {
    email: String::from("carol@example.com"),
    ..user1
};
```

## Tuple Structs

Named tuples without field identifiers:

```rust
struct Color(i32, i32, i32);
struct Point(i32, i32, i32);

let black = Color(0, 0, 0);
let origin = Point(0, 0, 0);
// Color and Point are distinct types even though same field types
```

## Methods

Methods are functions defined within the context of a struct (or enum). They take `&self`, `&mut self`, or `self` as their first parameter:

```rust
struct Rectangle {
    width: u32,
    height: u32,
}

impl Rectangle {
    fn area(&self) -> u32 {
        self.width * self.height
    }

    fn can_hold(&self, other: &Rectangle) -> bool {
        self.width > other.width && self.height > other.height
    }
}
```

### Associated Functions

Functions within `impl` blocks that do not take `self` are called associated functions. They are called with the `::` syntax. The constructor pattern is common:

```rust
impl Rectangle {
    fn square(size: u32) -> Rectangle {
        Rectangle { width: size, height: size }
    }
}

let sq = Rectangle::square(3);
```
