# Testing

Rust provides first-class testing support through the `#[test]` attribute and `cargo test`. The test framework is built into the language, not a separate tool.

{/* Source tracking: Testing synthesized from "The Rust Programming Language" by Klabnik & Nichols, Ch. 11 (MIT/Apache-2.0) */}

## Writing Tests

A test in Rust is a function annotated with `#[test]`:

```rust
#[cfg(test)]
mod tests {
    #[test]
    fn it_works() {
        let result = 2 + 2;
        assert_eq!(result, 4);
    }

    #[test]
    fn failing_test() {
        panic!("This test fails");
    }
}
```

Run tests with `cargo test`. By convention, test functions go in a `tests` module annotated with `#[cfg(test)]`, which ensures the test code is compiled only during testing.

## Common Assertions

```rust
assert!(condition);                          // panics if false
assert_eq!(a, b);                            // panics if a != b
assert_ne!(a, b);                            // panics if a == b
assert!(result.is_ok(), "Custom message: {:?}", result);
```

## Controlling Test Execution

```rust
#[test]
#[should_panic(expected = "out of bounds")]    // test passes if code panics
fn index_out_of_bounds() {
    let v = vec![1, 2, 3];
    v[100];
}

#[test]
#[ignore]                                        // skipped unless --ignored
fn expensive_test() {
    // takes a long time
}
```

## Test Organization: Unit vs Integration

### Unit Tests
Unit tests live in the source files alongside the code, in a `tests` module:

```rust
// src/lib.rs
pub fn add_two(a: i32) -> i32 {
    a + 2
}

#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn test_add_two() {
        assert_eq!(4, add_two(2));
    }

    #[test]
    fn test_private_function() {
        // can test internal helpers
    }
}
```

### Integration Tests
Integration tests live in a `tests/` directory at the crate root. Each file is compiled as a separate crate:

```rust
// tests/integration_test.rs
use my_crate;

#[test]
fn test_integration() {
    assert!(my_crate::some_function());
}
```

## Doc Tests

Rust runs code examples in documentation as tests:

```rust
/// Adds two numbers.
///
/// ```
/// let result = my_crate::add(2, 3);
/// assert_eq!(result, 5);
/// ```
pub fn add(a: i32, b: i32) -> i32 {
    a + b
}
```
