# Traits: Defining Shared Behavior

A **trait** defines a set of methods that types can implement. Traits are Rust's approach to polymorphism: any type implementing a trait can be used where that trait is expected.

{/* Source tracking: Traits synthesized from "The Rust Programming Language" by Klabnik & Nichols, Ch. 10.2 (MIT/Apache-2.0) */}

## Defining and Implementing Traits

```rust
pub trait Summary {
    fn summarize(&self) -> String;
}

pub struct Article {
    pub headline: String,
    pub content: String,
}

impl Summary for Article {
    fn summarize(&self) -> String {
        format!("{}", self.headline)
    }
}

pub struct Tweet {
    pub text: String,
}

impl Summary for Tweet {
    fn summarize(&self) -> String {
        format!("Tweet: {}", self.text)
    }
}
```

## Default Implementations

Traits can provide default method bodies:

```rust
trait Summary {
    fn summarize(&self) -> String {
        String::from("(Read more...)")
    }
}

// Types can use the default or override it
impl Summary for Article {}  // uses default
impl Summary for Tweet {
    fn summarize(&self) -> String {  // overrides
        format!("Tweet: {}", self.text)
    }
}
```

## Trait Bounds

Traits constrain generic type parameters:

```rust
fn notify(item: &impl Summary) {
    println!("Breaking news! {}", item.summarize());
}

// Equivalent desugaring with trait bound:
fn notify<T: Summary>(item: &T) {
    println!("Breaking news! {}", item.summarize());
}

// Multiple bounds:
fn notify(item: &(impl Summary + Display)) {}
fn notify<T: Summary + Display>(item: &T) {}
```

### Where Clauses

For complex bounds, `where` clauses improve readability:

```rust
fn some_function<T, U>(t: &T, u: &U) -> i32
where
    T: Display + Clone,
    U: Clone + Debug,
{}
```

## Trait Objects

Trait objects enable dynamic dispatch. Unlike generics (static dispatch), trait objects resolve methods at runtime:

```rust
fn make_summarizable() -> Box<dyn Summary> {
    if true {
        Box::new(Article { headline: String::from("News"), content: String::from("...") })
    } else {
        Box::new(Tweet { text: String::from("hello") })
    }
}
```

Use `dyn Trait` for trait objects. They are unsized and must be behind a pointer (`&dyn Trait`, `Box<dyn Trait>`).
