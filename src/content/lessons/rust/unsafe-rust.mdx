# Unsafe Rust

Rust's safety guarantees are enforced at compile time, but some low-level operations cannot be checked by the compiler. **Unsafe Rust** provides five superpowers that bypass the borrow checker. With these powers comes the responsibility of maintaining the invariants yourself.

{/* Source tracking: Unsafe Rust synthesized from "The Rust Programming Language" by Klabnik & Nichols, Ch. 19.1 (MIT/Apache-2.0) */}

## The Five Superpowers

Unsafe code is marked with the `unsafe` keyword. Inside a block, you can:

1. Dereference raw pointers
2. Call unsafe functions (including FFI)
3. Access or modify mutable statics
4. Implement unsafe traits
5. Access fields of unions

```rust
unsafe {
    // All five superpowers available here
}
```

## Raw Pointers

Raw pointers (`*const T` and `*mut T`) are unchecked counterparts to references. They can be null, dangling, or aliased freely:

```rust
let mut num = 5;

let r1 = &num as *const i32;
let r2 = &mut num as *mut i32;

unsafe {
    println!("r1: {}", *r1);  // dereference is unsafe
    *r2 = 10;
    println!("r2: {}", *r2);
}
```

## Foreign Function Interface (FFI)

Calling C functions requires `unsafe` because the compiler cannot verify that the external code upholds Rust's safety guarantees:

```rust
extern "C" {
    fn abs(input: i32) -> i32;
}

unsafe {
    println!("Absolute value of -3: {}", abs(-3));
}
```

Calling Rust from other languages:

```rust
#[no_mangle]
pub extern "C" fn call_from_c() {
    println!("Called from C!");
}
```

## Unsafe Traits

A trait is unsafe if implementing it could violate memory safety. The classic example is `Send` and `Sync`:

```rust
unsafe trait MyUnsafeTrait {}

unsafe impl MyUnsafeTrait for i32 {}
```

## When to Use Unsafe

Unsafe should be rare. Justification scenarios:
- Interfacing with C code (FFI)
- Implementing performance-critical data structures
- Writing abstractions the compiler cannot verify (e.g., `Vec`, `HashMap` internals)
- Embedded or kernel programming

Always minimize the scope: keep unsafe code in small, auditable blocks with safe wrappers.
