# Variables, Mutability, and Bindings

Rust makes variables **immutable by default**. This is not a restriction but a deliberate design choice: immutability eliminates entire categories of bugs where a value changes unexpectedly.

{/* Source tracking: Variables and mutability synthesized from "The Rust Programming Language" by Klabnik & Nichols, Ch. 3.1 (MIT/Apache-2.0) */}

## Immutability by Default

```rust
let x = 5;
x = 6;  // compile error: cannot assign twice to immutable variable
```

To allow mutation, add the `mut` keyword:

```rust
let mut y = 5;
y = 6;  // allowed
```

## Shadowing

Rust allows declaring a new variable with the same name as a previous one, effectively **shadowing** the earlier variable:

```rust
let x = 5;
let x = x + 1;    // shadows previous x with 6
let x = x * 2;    // shadows with 12
println!("{}", x); // 12
```

Shadowing differs from mutation because it creates a fresh binding. The old variable ceases to exist at that point. Shadowing also lets us change the type:

```rust
let spaces = "   ";      // string slice
let spaces = spaces.len(); // now a number (usize)
```

## Constants

Constants are always immutable, always have a known type annotation, and can be declared in any scope:

```rust
const MAX_POINTS: u32 = 100_000;
```

Unlike `let`, constants are inlined at every use site and cannot be marked `mut`.
