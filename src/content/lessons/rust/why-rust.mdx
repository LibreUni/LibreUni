# Memory Safety Without a Garbage Collector

Systems programming languages have historically forced a choice between two undesirable options: fine-grained control over memory (C, C++) with the risk of buffer overflows, use-after-free, and data races, or memory safety via garbage collection (Java, Go) at the cost of unpredictable pauses and reduced control over hardware resources.

Rust resolves this tension through a compile-time ownership system that guarantees memory safety without a garbage collector.

## The Problem with Manual Memory Management

In C, memory management is entirely manual:

```c
void leak_example() {
    int *data = malloc(10 * sizeof(int));
    // if we forget to free, the memory leaks
    // if we free and then use, we have a use-after-free
    // if we free twice, we have a double-free
}
```

These bugs are not edge cases—they account for roughly 70% of critical security vulnerabilities in large C and C++ codebases according to Microsoft and Google security research.

## The Cost of Garbage Collection

Garbage collectors provide safety by automatically reclaiming memory, but they introduce:
- Unpredictable pause times (stop-the-world GC cycles)
- Increased memory pressure (objects survive longer)
- Difficulty integrating with real-time systems

## Rust's Solution: Ownership

Rust introduces three simple rules enforced at compile time:

1. Each value in Rust has exactly one **owner** at any time.
2. When the owner goes out of scope, the value is dropped.
3. References to a value must follow borrowing rules (explored in the next lesson).

```rust
fn ownership_example() {
    let s = String::from("hello");  // s owns the string
    let t = s;                      // ownership moves to t
    // println!("{}", s);           // compile error: s no longer valid
}                                   // t goes out of scope, memory freed
```

This single rule eliminates the entire class of use-after-free and double-free bugs without needing a garbage collector.

## Zero-Cost Abstractions

Rust's abstractions compile down to the same efficient machine code as hand-written C. The ownership checks exist only at compile time; they have zero runtime overhead.
